# ADR-0001 实现计划：Path 曲线 step kind（curve / cubic / bend）

> 写于 2026-05-09。配套 [ADR-0001](../adr/0001-path-curve-step-kind.md)。

**Goal：** 给 IRStep 加 `curve` / `cubic` / `bend` 三个 kind，支持 SVG `Q` / `C` 输出 + bend 几何拟合，React DSL 对齐，IR↔JSX 双向 round-trip 通过。

**Architecture：**
- `step.ts` 的 `discriminatedUnion('kind', …)` 加 3 分支
- `geometry/bend.ts` 新增：`bendControlPoints(from, to, direction, angle)` 返回 `[c1, c2]`
- `compile/path.ts` 主循环加 3 个 branch：`curve` → `Q`，`cubic` → `C`，`bend` → 调 `bendControlPoints` 退到 `C`
- 用首/末控制点当 `clipForTarget` 的 `toward` 方向（首段从 c1 反推，末段从最末控制点反推）
- React `Step.tsx` `StepProps` union 同步加 3 种；`_builder.ts` / `_unbuilder.ts` 透传新字段

**字段命名（确认 ADR + 代码 camelCase 惯例）：**
- `curve`：`control: Position`
- `cubic`：`control1: Position` / `control2: Position`
- `bend`：`bendDirection: 'left' | 'right'`，`bendAngle?: number`（默认 30）

**控制点类型决策（已确认）：** 当前实现用 `Position` 元组，但 schema 留扩展位——抽出 `ControlPointSchema` 别名，今天等价于 `PositionSchema`，未来要支持节点 ref / 极坐标时只改这一处，不动 curve / cubic schema。React `StepProps` 的 control 字段类型也对齐为命名类型 `IRControlPoint`（导出自 core）。

**bend 几何精度（已确认）：** 按 ADR 字面 — apex offset = chord × tan(angle/2)，控制点位于 chord 1/3 / 2/3 处沿法向偏移 (4/3)×apex_offset，使 cubic 在 t=0.5 穿过 apex。小角度（≤90°）视觉接近圆弧，大角度可接受。

 * 控制点的 schema 别名。
 *
 * alpha.3：仅支持笛卡尔 `[x, y]`（与 ADR-0001 字面一致）。
 * 未来扩展点：要支持节点 ref / 极坐标作为控制点（如 TikZ `controls (B)`）时，

 * curve / cubic schema 与下游消费侧不变。
 */
export const ControlPointSchema = PositionSchema.describe(
  'Bezier control point. Currently Cartesian [x, y]; reserved for node ref / polar in future versions.',
);

/** 控制点类型（曲线 step 用）；alpha.3 = `[number, number]`，未来可能扩展 */
export type IRControlPoint = z.infer<typeof ControlPointSchema>;

export const CurveStepSchema = z
  .object({
    type: z.literal('step').describe('Discriminator marking this as a path step node'),
    kind: z
      .literal('curve')
      .describe('Quadratic Bezier curve from cursor to target with one control point (TikZ `.. controls (B) ..`, SVG path "Q")'),
    to: TargetSchema.describe('Destination point of the curve'),
    control: ControlPointSchema.describe('Single control point for the quadratic Bezier'),
  })
  .describe('Curve action: quadratic Bezier; one control point shapes the bend');

export const CubicStepSchema = z
  .object({
    type: z.literal('step').describe('Discriminator marking this as a path step node'),
    kind: z
      .literal('cubic')
      .describe('Cubic Bezier curve from cursor to target with two control points (TikZ `.. controls (B) and (C) ..`, SVG path "C")'),
    to: TargetSchema.describe('Destination point of the cubic curve'),
    control1: ControlPointSchema.describe('First control point (influences the start tangent)'),
    control2: ControlPointSchema.describe('Second control point (influences the end tangent)'),
  })
  .describe('Cubic action: cubic Bezier; two control points give precise tangent control at both ends');

export const BendStepSchema = z
  .object({
    type: z.literal('step').describe('Discriminator marking this as a path step node'),
    kind: z
      .literal('bend')
      .describe('Arc-like bend from cursor to target by direction + angle (TikZ `to[bend left=N]` / `to[bend right=N]`); compiles to a cubic Bezier approximation'),
    to: TargetSchema.describe('Destination point of the bend'),
    bendDirection: z
      .enum(['left', 'right'])
      .describe('Bend side relative to from→to direction (visual left vs right of the chord)'),
    bendAngle: z
      .number()
      .optional()
      .describe('Bend angle in degrees; default 30 (matches TikZ `bend left` without explicit angle)'),
  })
  .describe('Bend action: shorthand for an arc-like cubic; control points computed at compile time');

```ts
import type { Position } from './point';

/**
 * 用 cubic Bezier 拟合 from→to 的弧形 bend。
 *
 * 算法（ADR-0001）：
 * 1. apex offset = chord × tan(bendAngle / 2)（沿 path 法向 left / right 偏移量）
 * 2. 控制点取 chord 1/3 / 2/3 处沿法向偏移；为让 cubic 在 t=0.5 处穿过 apex，
 *    控制点 offset = (4/3) × apex offset（推导：B(0.5) = midpoint + (3/4)·s·n）
 *
 * 法向定义（SVG y 向下，"left/right" 按视觉，不按数学）：
 *   chord direction = (dx, dy) / |chord|
 *   visual-left normal  =  ( dy, -dx) / |chord|   （从 from 看向 to，向左偏）
 *   visual-right normal =  (-dy,  dx) / |chord|
 *
 * chord 长度为 0 时无方向可推，两个控制点都返回 from。
 */
export const bendControlPoints = (
  from: Position,
  to: Position,
  direction: 'left' | 'right',
  bendAngle: number,
): [Position, Position] => {
  const dx = to[0] - from[0];
  const dy = to[1] - from[1];
  const chord = Math.hypot(dx, dy);
  if (chord === 0) return [[from[0], from[1]], [from[0], from[1]]];

  const sign = direction === 'left' ? 1 : -1;
  const nx = (dy / chord) * sign;
  const ny = (-dx / chord) * sign;

  const apexOffset = chord * Math.tan((bendAngle * Math.PI) / 180 / 2);
  const ctlOffset = (4 / 3) * apexOffset;

  const c1: Position = [from[0] + dx / 3 + ctlOffset * nx, from[1] + dy / 3 + ctlOffset * ny];
  const c2: Position = [from[0] + (2 * dx) / 3 + ctlOffset * nx, from[1] + (2 * dy) / 3 + ctlOffset * ny];
  return [c1, c2];
};
```

## 风险 & 待澄清

1. **控制点不接受节点 ref / 极坐标**——ADR 文本说 `control: Position`，但举例用 `(B)` 节点。计划按 `Position` 走，需用户确认。
2. **bend 的几何**：ADR 描述模糊（"chord × tan(bendAngle/2)" + "1/3 + 2/3 处控制点"）；本计划解读为：apex offset = chord×tan(angle/2)，控制点位于 chord 1/3 / 2/3 + (4/3)×apex_offset×normal——使 cubic 在 t=0.5 穿过 apex。**对小角度（≤90°）视觉接近圆弧**；大角度（>120°）会越来越偏离圆，但 alpha.3 可接受。
3. **shrink + 曲线的相互作用**未细查——hollow 箭头与曲线组合是 corner case。如果存在 bug，留 alpha.4 处理。
4. **法向方向**：SVG y 向下，"left"/"right" 按视觉而非数学；`bend left` 在水平 chord 上会让 y < 0（屏幕上方）。
