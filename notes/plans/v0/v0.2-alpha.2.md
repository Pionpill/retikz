## 背景与定位

alpha.1 落了 `<Scope>` IR 容器——分组 + 局部 transform + 跨 scope anchor/nodeIndex 解析，但**只做容器本身**，样式继承显式留给 alpha.2。

alpha.2 把 Scope 从"纯几何分组"升级为"样式默认值挂点"，承接 TikZ 三个用户语义：

- `\begin{scope}[draw=red, line width=2pt]` —— scope 直接挂样式，作为内部子节点的默认值
- `every node/.style={...}` / `every path/.style={...}` —— scope 上挂 `nodeDefault` / `pathDefault` 样式 map，按元素类型分发默认值
- `every label/.style={...}` + 彩色线段标注同色 —— StepLabel 加 `textColor` / `opacity` / `font`，配 scope 级 `labelDefault` 继承

**依赖关系**：依赖 alpha.1（Scope 容器是样式默认值的唯一挂点；没有容器就没有"作用域"概念）。**可与 alpha.3（ShapeRegistry）并行**——样式继承不依赖 shape 注册机制，两段文件 scope 不交叉（alpha.2 动 `scope.ts` / `step.ts` / `compile`，alpha.3 动 shape 注册面）。

**衡量标准**：alpha.2 完工后，retikz 能等价表达 TikZ 的 `every <X>/.style` 与 scope-level option 样式继承；给彩色函数线配标注能让标签与线同色，不再被 `currentColor` 锁死。

## 范围（Scope 接受的样式子集）

Scope 接受 Node 的**样式相关 props 子集**，文本 / 位置 / 缩放旋转相关 props 不接受。下表沿用 [`v0.2.md §alpha.2`](./v0.2.md#alpha2-设计预想scope-样式子集)：

| 类别 | 字段 | Scope 是否接受 |
|---|---|---|
| 描边 | `stroke` / `strokeWidth` / `dashed` / `dotted` / `dashArray` / `drawOpacity` | ✅ |
| 填充 | `fill` / `fillOpacity` | ✅ |
| 整体 | `opacity` | ✅ |
| 形状 | `shape` / `roundedCorners` / `minimumWidth` / `minimumHeight` / `minimumSize` | ✅（作为默认值） |
| 间距 | `innerXSep` / `innerYSep` / `outerSep` / `padding` / `margin` | ✅ |
| 缩放 | `scale` / `xScale` / `yScale` | ❌（与 `transforms` 的 scale 变体重复，避免双概念） |
| 旋转 | `rotate` | ❌（同上，与 `transforms` 的 rotate 重复） |
| 位置 | `position` | ❌（Scope 用 `transforms` 表达位置，Node 的 `position` 不适用） |
| 文本 | `text` / `align` / `lineHeight` / `font` / `textColor` / `label` | ❌ |

设计取舍：

- **文本相关一律不接受**：scope 是分组容器不是文字载体；允许 `<Scope text="...">` 会让 IR / DSL 语义双重化（容器又是文字节点？）。要在 scope 上挂说明性文字应当用 `<Coordinate>` + 显式 `<Node>`。
- **位置相关不接受**：scope 视觉位置完全由 `transforms` 表达，引入 `position` 会有"位置由哪个字段决定"的歧义。
- **缩放 / 旋转不接受**：与 `transforms` 的 scale / rotate 变体语义重复——一个概念两个入口必生歧义。
- **形状 / 间距 / 描边 / 填充接受**：TikZ scope option 最常用的样式继承场景。

注意上表是 **Scope 直挂**（窄子集）的取舍。`nodeDefault` 是另一回事——作为 `every node/.style` 它要更宽（含文本 / 字体 / scale / rotate，仅排除身份与逐节点定位），否则 `every node/.style={font, text}` 无法表达、达不到衡量标准。两者关系见下节「三层派生」。

实施时 `ScopeSchema` 字段集 = 当前字段（`type` / `id` / `localNamespace` / `transforms` / `children`）+ Scope 直挂样式子集（上表 ✅，19 props）+ `nodeDefault` / `pathDefault` / `labelDefault`（样式 map）+ `resetStyle`（继承屏障，见下节）。

## 样式继承机制（every-X/.style 全套）

alpha.2 做**完整的 every-X/.style 机制**，不止 scope 直挂。两条挂法配合：

1. **Scope 直挂样式子集**（§范围 ✅ 字段，窄子集）—— scope 本身的样式默认值，作用于 scope 内子节点。对应 `\begin{scope}[draw=red]`。
2. **Scope `nodeDefault` / `pathDefault` / `labelDefault`** —— 按元素类型分发的样式 map。对应 TikZ `every node/.style` / `every path/.style` / `every label/.style`。

### 三层派生（单一真源，禁手抄）

三个通道的字段集都从已有 schema `.omit()` 派生，NodeSchema 加字段时自动跟随，杜绝三处漂移：

```
NodeSchema
  └─ omit{ type, id, position, text, label }  ⇒  NodeDefaultSchema  （27 字段：含文本样式 + scale/rotate；= nodeDefault 值类型）
        └─ omit{ scale, xScale, yScale, rotate, font, textColor, align, lineHeight }
                                              ⇒  ScopeStyleSchema   （19 字段；= Scope 直挂用）
```

- **`nodeDefault` 宽（27）**：它是"子节点默认值 map"，不是 Scope 自身 prop，`nodeDefault.scale` 明确指"子节点默认缩放"、不与 `transforms` 歧义；TikZ `every node/.style={font, text, rotate}` 都合法。正是这宽度让 `every node/.style` 达标。
- **Scope 直挂窄（19）**：挂在 Scope 元素本身，`scale`/`rotate` 会与 Scope 的 `transforms` 撞概念、文本类不该让容器当文字载体——故在 NodeDefault 基础上再砍 8 个。
- **三通道都排除**：`type`（鉴别常量）/ `id`（唯一身份，默认必冲突）/ `position`（逐节点各异，且是 NodeSchema 唯一必填）/ `text`（默认 = 所有节点同文字）/ `label`（内容；label 的*样式*默认走 `labelDefault`）。
- `pathDefault` 从 Path 视觉 props 派、`labelDefault` 从 `StepLabelSchema` ∪ `NodeLabelSchema` 的样式交集派。

字段全 `.optional()`；**禁挂 schema `.default()`**——否则 Zod 把缺省字段填上值使其"永远存在"、永远覆盖外层，继承直接失效。内置默认在解析 fold 的**最后一步**才补（见下）。

### 优先级链

Node 某个样式字段的最终取值，按以下优先级**就近覆盖**：

```
子节点显式 props  >  scope 直挂同字段  >  scope.nodeDefault 同字段  >  内置默认
```

Path / StepLabel 同构（Path 走 `pathDefault`、StepLabel 走 `labelDefault`，无"直挂"档——scope 直挂只面向 Node 样式子集）：

```
StepLabel:  label 显式  >  scope.labelDefault  >  段级 stroke 推导  >  currentColor 兜底
Path:       path 显式    >  scope.pathDefault    >  内置默认
```

### 嵌套解析（compile 维护 style frame 栈）

嵌套不发明新机制，收敛成一次 **inside-out per-field 解析**（与 alpha.1 ADR-02 anchor lookup 同方向）。compile 下行维护栈，enter push / exit pop，leaf 处自外向内 fold（后写覆盖 ≡ inside-out 取首个有定义）：

```
for S in stack 外→内（S1, S2, ...）:
    if S.resetStyle(channel): eff[channel] = {}     # 屏障：丢弃外层累积（见下）
    merge(eff, S.nodeDefault)   # 同层低优先
    merge(eff, S.direct)        # 同层高优先（覆盖本层 nodeDefault）
merge(eff, node.explicit)       # 最高
补内置默认（仍 undefined 的字段）  # ← 唯一兜底，不在 schema 层
```

展开（S1 ⊃ S2 ⊃ S3，node 在 S3）：

```
node 显式 > S3.direct > S3.nodeDefault > S2.direct > S2.nodeDefault > S1.direct > S1.nodeDefault > 内置默认
```

即**就近优先（模型 A）**：内层整体压外层，同层内 direct 压 nodeDefault。**关键是 per-field 合并、不是整对象替换**——字段 A 可来自 S3、字段 B 来自 S1（≈ TikZ `.append style`，非 `every node/.style=` 的整体替换；文档须明示与 TikZ 字面行为的差异）。

### 「缺省」vs「显式 none」（merge 按存在性判定）

合并必须按"字段是否存在"（`value !== undefined` / `Object.hasOwn`），**绝不按真假值**——否则 `0` / `false` / `""` / `"none"` 这些合法显式值会被误当未设置：

- **缺省（这层没写）** = 让位，穿过本层继续向外查；全链皆缺才落内置默认。
- **显式值（含 `fill="none"` / `0` / `false`）** = 在本层拍死、截断向外查。

例：外层 `stroke="white"` ⊃ 内层无 stroke → 继承 **white**（缺省让位）；内层 `stroke="none"` → **none**（显式截断）。

### resetStyle —— 朝外的继承屏障（形态 B）

`resetStyle` 切断**外层对某通道的全部贡献**（外层 direct + 外层 nodeDefault 都算——节点样式两条来源都要切干净），回到内置基线；本 scope 自己的值 + 内层照常生效。**只朝外切**，不影响本 scope 内部优先级、也不挡更内层从本 scope 继承。

```ts
resetStyle?: boolean | Array<'node' | 'path' | 'label'>
// true / 省略数组 = 全通道切；数组 = 仅列出的通道切
```

走一遍 `<Scope stroke="red" nodeDefault={{strokeWidth:4}}>`（S1）⊃ `<Scope resetStyle nodeDefault={{fill:"white"}}>`（S2）⊃ `<Node/>`：

| 字段 | 来源 | 值 |
|---|---|---|
| `fill` | S2.nodeDefault | white（本层值仍生效） |
| `stroke` | 外层 red 被屏障切断 → 无人接管 | currentColor（内置，非 red） |
| `strokeWidth` | 外层 4 被切断 | 1（内置，非 4） |
| `opacity` | 链上乘积归零重启 | 1 |

- **正交于几何 / 命名空间**：`resetStyle` 只动样式继承，不影响 `transforms` 复合 / `localNamespace`（id 可见性）/ `scope.id` bbox。
- **与 opacity 复合联动**：若 opacity 沿链相乘（见下），屏障须把累积乘积也归 1。
- **与 per-field `'initial'` 互补**：`resetStyle` = scope 级整通道切；`'initial'`（单字段单元素回默认）**alpha.2 不做，留未来扩展**。

### opacity 例外：相乘 vs 覆盖

`opacity` 在 SVG `<g>` / TikZ transparency 里是**相乘复合**（S1 0.5 × S2 0.5 = 0.25），其余样式走覆盖。`NodeLabelSchema` 已声明 label opacity"与 node opacity 相乘"。所以 opacity 不能简单走上面的覆盖 fold——需 ADR 定：沿栈 product，且 node 显式 opacity 是"替换整链"还是"再乘进去"，以及 `resetStyle` 如何归零累积乘积。

## 待定（ADR 阶段敲定）

> **已定**（本轮收敛，ADR 复核即可）：三层派生方案、`nodeDefault` 宽 / Scope 直挂窄、按存在性 merge + 禁 `.default()`、就近优先（模型 A）、per-field 合并、`resetStyle` 形态 B 入 alpha.2。下面是仍开放的：

- **跨层 × 跨通道顺序**：模型 A（就近优先，内层 nodeDefault 可压外层 direct，倾向）vs 模型 B（所有 direct 压所有 nodeDefault）。ADR 须明写并配测试。

- **`labelDefault` 挂法**：scope 上独立 `labelDefault` 字段 vs `nodeDefault.label` / `pathDefault.label` 嵌套。倾向独立（node label 与 step label 共享一套继承）。
- **label 继承顺序细则**：`label 显式 > scope.labelDefault > 段级 stroke 推导 > currentColor 兜底`——"段级 stroke 推导"是否真介于 labelDefault 与 currentColor 之间。
- **style-map 未知 key 校验严格度**：`nodeDefault` 收到非样式 key 时 `.strict()` 严拒（倾向）vs 静默忽略；错误信息列合法字段集。
- **scope 直挂 `shape` 作为默认值的边界**：直挂 `shape='circle'` 时子节点未给 shape 是否真继承 circle。

## 设计 ADR

v0.2 开工前另起 ADR（位置 `notes/adr/v0/v0.2-alpha.2/`，编号到时定），固化：

### ADR-01 — Scope 样式继承

**三层派生**：`NodeSchema.omit{type,id,position,text,label}` ⇒ `NodeDefaultSchema`（27）；再 `.omit{scale,xScale,yScale,rotate,font,textColor,align,lineHeight}` ⇒ `ScopeStyleSchema`（19，直挂用）。`pathDefault` / `labelDefault` 同法从 Path 视觉 props / `StepLabelSchema`∪`NodeLabelSchema` 派生。全 `.optional()`、禁 `.default()`。

**继承解析契约**：

- 单层优先级：`子节点显式 > scope 直挂 > scope.nodeDefault > 内置默认`
- 嵌套：compile 维护 style frame 栈，per-field 自外向内 fold；就近优先（模型 A，待 ADR 拍）；per-field 合并而非整对象替换
- merge 按 `!== undefined` 判存在性（缺省让位、显式 `none` / `0` / `false` 截断）；内置默认 fold 末尾补、不进 schema
- `resetStyle: boolean | ('node'|'path'|'label')[]` 屏障语义：切外层 direct + default、只朝外、正交于 transforms / localNamespace
- `opacity` 相乘 vs 覆盖 + `resetStyle` 对累积乘积的归零
- style-map 未知 key 校验严格度 + 错误信息
- compile 期解析落点（renderer-neutral：解析进具体 primitive，不靠 SVG `<g>` 继承）

**测试象限**：优先级链 4 档 × 嵌套就近优先 × per-field × 缺省/显式 none × resetStyle（全切 / 通道切 / 本层生效 / 内层继承）× opacity 复合 × 边界排除 × adversarial
