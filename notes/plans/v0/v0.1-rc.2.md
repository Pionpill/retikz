## TODO-1 — 新增 Examples / Recipes section

### 问题

现有示例数量不少，但分散在组件页中。用户常见问题通常不是“某个 prop 怎么写”，而是“怎么画一个流程图 / 箭头关系图 / 带 label 的路径 / 虚线样式图”。

### 建议页面

- `core/examples/basic-diagram`
- `core/examples/flowchart`
- `core/examples/arrow-relations`
- `core/examples/labels`
- `core/examples/styling`

每页结构：

1. 一句话说明适用场景。
2. 一个可复制的完整 `<TikZ>` 示例。
3. 简短拆解：用了哪些组件、关键 props 是什么。
4. 指向相关 API 页面。

### 验收

- 每页中英文并行。
- 每页至少一个 `<ComponentPreview>`。
- 示例使用 rc API，不出现旧字段。
- 示例能作为用户复制粘贴入口，不依赖阅读内部 ADR / plan。

- 从 mdx raw 提取 frontmatter title / description。
- 提取 `##` / `###` 标题。
- 提取代码标识符或 inline code。
- 搜索结果保留当前页面跳转粒度，先不做 heading 深链。

### 验收

- 搜 `dashPattern` 能命中 Draw / Path / Node stroke style 相关页。
- 搜 `layout` 能命中 TikZ / Scene schema / changelog 或 migration 相关页。
- 搜 `anchor` 能命中 Anchors / Coordinate / Node 相关页。

## TODO-3 — AI 对话输入框重构：Copilot 风格 + 多 Provider + Add Context + 自定义 Provider

### 问题

当前 `AiChatSettings` 把两类性质不同的参数堆在同一表单：

- **一次性配置**（写一次几乎不再动）：Provider 选择、API Key、Base URL
- **每请求参数**（按对话切换）：Model、Context Mode

Model 是按请求传给 provider 的运行时字段，同一 API Key 对该 provider 旗下所有模型通用；用户想"快问用便宜模型、难题切高端"时被迫返回 Settings 视图，与 ChatGPT / Claude / Cursor / VSCode Copilot 等主流 chat UI 心智不符。Context Mode 同理。

同时，当前 provider 列表硬编码 DeepSeek / OpenAI / Anthropic 三家，OpenAI 兼容生态里大量厂商（Together / Mistral / Groq / 本地 vLLM 等）用户无法接入。

### 设计（整体参考 VSCode Copilot Chat）

`AiChatInput` 重构为带 chips 行的三段式 prompt 框，外加 meta 行；整个 prompt 框 `bg-muted` 灰底以区分。

**prompt 框（bg-muted、border 圆角）**：

1. **顶部 chips 行**：左侧 `+ Add Context` 按钮 + 右侧已选 context chips
   - **Add Context 按钮与 chip 视觉一致**：同 padding（`px-1.5 py-0.5`）、同字号（`text-[11px]`）、同 border、无背景填充，仅左侧 `+` icon 区分
   - 每个 chip：File icon + 文件名 + × 移除按钮
   - chip 样式：**仅 border，无背景填充**（外层 bg-muted 直接透出）
   - 当前页自动作为首个 chip，带 `·当前页` 副文本，仍可手动移除
   - **chips 多到放不下时自动换行**（`flex-wrap`），不出现横向滚动条；prompt 框高度随之增长
2. **中部 textarea**：**默认 1 行高度**（`rows="1"`），随内容自适应展开**最多到 3 行**（`max-h ≈ 72px`），超出后内部 y 轴滚动 —— 不让 prompt 框无限上长把整个会话区挤掉。**滚动条按项目约定隐藏**（与 shadcn 文档站一致），靠鼠标滚轮 / 触控板 / 键盘自然滚动。其余行为不变：Enter 提交、Shift+Enter 换行
3. **底部 picker + 发送**：
   - 左侧：Model picker + Context Mode picker，**仅文字 + ChevronDown**（无 icon），ghost 样式，点击弹 popover
   - 右侧：Send button **ghost 样式**（Lucide `Send` 飞机图标），生成中或输入为空时 `disabled`（不再切 Square abort —— Esc 沿用面板级监听）

**prompt 框外的 meta 行**（贴在 prompt 框下方、与外层 bg-background 同色）：

- 左侧：`[Esc]` kbd + "取消生成" 提示（取代原本面板级隐藏的 Esc abort —— 给用户可见线索）
- 右侧：**上下文用量圆环 + 百分比**（donut SVG，已用部分高亮、剩余 30% opacity；hover 时 title 显示 `<used>K / <max>K`）。砍掉之前的 `↑/↓/≈$` 三段式 token 用量，避免数字过密
- 字号 `text-[10px] text-muted-foreground`，左右两端对齐

需要一份 `MODEL_CONTEXT_LIMIT: Record<string, number>` 映射拿到当前模型的窗口大小；未知模型 fallback 一个保守值（如 32K）。已用 token 取 `usage.input + usage.output + system 大小` 估算。

### Add Context 命令面板

复用 docs-search Ctrl+K 的 UI / 键盘交互（shadcn `Command` 组件）：

- 顶部 search input + 底部 hint 行（↑↓ 切换 / ↵ 选择 / Esc 关闭）
- 结果按 section 分组（Components / Concepts / Examples / Reference / About …）
- 每行：checkbox（反映已选态）+ File icon + title + 右侧 slug 路径（font-mono 小字）
- 多选：点击 / Enter 切换该项已选态，不关闭面板；Esc 关闭

### Model picker（按 Provider 分组）

popover 列表按 provider 分组：

- **只展示已填 Key 的 provider 组**（含已添加的自定义 provider）
- 每组内：内置 `MODEL_CHOICES[providerId]` + 该 provider 下用户曾自定义过的模型（带"自定义"小字标识）
- **未填 Key 的内置 provider 整组隐藏**——不在 picker 露出，避免无意义的灰显条目；用户要新增走 Settings 或底部"+ 添加 Provider…"入口
- popover 底部两个并列入口：
  - `+ 自定义模型名…` —— 行内展开 Input，回车把名字写进当前 provider 的自定义列表
  - `+ 添加 Provider…` —— 跳 Settings 的 Custom Providers section（或弹出同款行内表单）

切换不同 provider 的模型时，**同步把 `providerId` 切到那家**，相当于一次操作完成 provider + model 切换。

### Settings 改造

Settings 表单结构（自上而下）：

1. **Provider segmented**：内置三家（DeepSeek / OpenAI / Anthropic）
2. **API Key** / **Base URL**（对应当前选中的内置 provider）
3. **Custom Providers** section（新增）：
   - 卡片列表：每个自定义 provider 一行，显示 label / baseUrl / apiFormat / 模型数；右侧编辑 / 删除按钮
   - `+ Add Provider` 按钮：展开行内表单
     - `label`（如 "Together AI"）
     - `baseUrl`（如 `https://api.together.xyz/v1`）
     - `apiFormat` segmented：`openai-compat` ｜ `anthropic`（v1 只支持这两种）
     - `apiKey`
     - `defaultModels`（可选，逗号分隔，作为该 provider 的 model 预置列表）
4. **网络提示** / Anthropic 警告（保留）

**删除**：原 Model section、原 Context Mode section。Context Mode 的 i18n label key 保留给 picker popover 复用。

### Store 扩展

`useAiChatStore` 新增 / 修改字段：

```ts
type CustomProvider = {
  id: string;            // 持久化用 slug；用户填 label 后系统生成
  label: string;
  baseUrl: string;
  apiFormat: 'openai-compat' | 'anthropic';
  apiKey: string;
  models: ReadonlyArray<string>;  // 用户预置 + 自定义新增合并
};

type State = {
  // 既有：providerId / models / apiKeys / baseUrls / contextMode
  customProviders: Record<string, CustomProvider>;  // 持久化
  contextSelection: Array<string>;                  // 已选 context page id 列表；ephemeral 或持久化待定
  // ...
};
```

发送时根据 `providerId` 是内置还是自定义，从对应字段取 config 调 provider 流。providers/ 目录需要把 openai-compat 包装成可配置 baseUrl 的通用 client。

### 实现拆分

1. **mockup 已确认** → 拆 PR
2. `AiChatInput` 重构：拆出 `ContextChips` / `ContextPicker`（Add Context popover）/ `ModelPicker` / `ContextModePicker` / `InputMetaRow`（Esc + token）五个子组件
3. `AiChatSettings` 改造：删 Model / Context Mode section，新增 CustomProviders section + 表单
4. Store：加 `customProviders` / `contextSelection`，迁移 send 逻辑支持自定义 provider
5. Provider 层：抽象一个 `getProviderConfig(state, providerId)` 把内置和自定义统一拿出来，避免 send 里大量 if 分支
6. 依赖：`pnpm dlx shadcn@latest add command popover`（如未引入）

### 验收

- Settings 表单不再出现 Model / Context Mode 字段；新增 Custom Providers section 可增删改
- input 框底部两个 picker 可切换并写回 store；Model picker 按 provider 分组、能切换到自定义 provider 下的模型
- Add Context 按钮可打开命令面板、多选 docs 页面，chips 行同步更新
- 当前页自动成为首个 chip；导航到新页时（用户未手动移除前）chip 自动跟随
- chips 多到放不下时自动换行，不出现横向滚动条
- textarea 默认 1 行高度；输入展开到第 4 行时框总高不再增长、内部可滚动（滚动条隐藏，行为符合项目约定）
- 自定义 Provider 可填 label / baseUrl / apiFormat / apiKey 后保存，Model picker 立即出现对应分组并可用
- 生成中点 Send 无反应（disabled），Esc 仍可 abort（meta 行有可见 hint）
- meta 行右侧上下文圆环随 usage 实时更新；百分比基于当前模型的窗口大小
- light / dark 主题下整框 bg-muted 与外层 bg-background 拉开层次
