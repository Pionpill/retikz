## v0.1.0-alpha.4

节点关系层一次到位：节点间相对定位、Coordinate 占位节点、Node 边挂标签。配 3 篇 ADR 与 +33 个新测试用例。

### `@retikz/core`

- **节点间相对定位**：`Node.position` 接受 `{ direction, of, distance? }` 形态，对应 TikZ `[<direction>=<distance> of <id>]`；direction 走 8 方向枚举（4 主向 + 4 对角，对角分量 1/√2 保斜向距离一致）；`of` 字符串引用要求被引节点先在 IR 中定义（同 polar `origin` / Step `to`） —— ADR-0001
- **Tikz 容器 `nodeDistance` prop**：注入 `CompileOptions.nodeDistance`，作为相对定位 distance 缺省值；自带 `distance` 优先，未配回退到 1（对应 TikZ `node distance=1cm`）
- **`<Coordinate>` 占位节点**：新增 `IRChild` discriminator `'coordinate'`（仅 id + position 两字段），与 Node / Path 在顶层平级；编译期 Pass 1 单独分支——不发任何 primitive、不参与 viewBox 扩展，但注册到 nodeIndex（0×0 矩形 layout，boundaryPoint 恒返中心），让 path target / `at.of` 字符串引用都能命中 —— ADR-0002
- **Node 边挂标签**：`Node.label?` 接 `NodeLabel` 单对象或数组；每条 label 接 `text` / `position?`（8 方向枚举或数字角度，对应 TikZ `label=above:foo` / `label=30:foo`）/ `distance?`（缺省 4）/ `textColor?` / `opacity?` / `font?`；`font` / `textColor` 缺省自动继承 Node 块级值；emit 阶段每条 label 一个 TextPrim 放在 inner group 内，节点 rotate 时 label 跟着转 —— ADR-0003

### `@retikz/react`

- `<Tikz>` 加 `nodeDistance` prop；`<Node position={{ direction, of, distance? }}>` 接受相对定位形态
- 新增 kernel 第三件 `<Coordinate id="..." position={...} />`（与 `<Node>` / `<Path>` 平级，IR 一对一）；builder / unbuilder 全链路 round-trip
- `<Node label={...}>` prop 接 NodeLabel 单对象或数组，与 IR 字段一致

## v0.1.0-alpha.3

Path 增强一波到位：曲线三件套、路径级形状、相对坐标、边标注、视觉属性补齐。配 4 篇 ADR 与 +120 个新测试用例。

### `@retikz/core`

- **Path 曲线三件套**：`Step.kind` 加 `'curve'`（quadratic）/ `'cubic'`（cubic）/ `'bend'`（弧形简记，方向 + 角度自动算控制点退到 cubic）—— ADR-0001
- **Path-level 形状**：`Step.kind` 加 `'arc'` / `'circlePath'` / `'ellipsePath'`，以游标为圆心；arc 留笔在弧终点，circle / ellipse 留笔回圆心 —— ADR-0002
- **相对坐标**：`IRTarget` 加 `{ rel: [dx, dy] }` / `{ relAccumulate: [dx, dy] }` 两个变体——`rel` 不推进 prevEnd（TikZ `(+x, +y)`），`relAccumulate` 累积（TikZ `(++x, ++y)`）；compile 入口 `normalizeRelativeTargets` 一次性解析 —— ADR-0003
- **边标注**：除 `'move'` / `'cycle'` 外八种 step kind 加 `label?: { text, position?, side? }`；`position` ∈ `'midway' | 'near-start' | 'near-end'`，`side` ∈ `'above' | 'below' | 'left' | 'right' | 'sloped'`；compile 出 TextPrim（sloped 裹 group rotate）—— ADR-0004
- **新增 geometry/segment.ts**：line / quad / cubic / fold / arc / circle / ellipse 七种段几何采样工具，给 label 算几何点 / 切线
- **Path 描边端点 / 拐点**：`lineCap` / `lineJoin`（SVG `stroke-linecap` / `stroke-linejoin`）
- **语义 stroke 档位**：`thickness` ∈ `'ultraThin'` … `'ultraThick'`（7 档），`thin = 1` 与默认 `strokeWidth` 对齐；显式 `strokeWidth` 始终优先
- **Path 透明度**：`opacity` 整体 + `fillOpacity` 仅 fill + `drawOpacity` 仅 stroke（与 Node 命名一致）
- **Sugar way DSL 扩展**：`{ curve | cubic | bend }` infix 算子（与下一项合并）；`{ arc | circle | ellipse }` infix 算子（不消耗下一项）；`WayRelItem` 对象形态 `{ position, type: DrawWay.Relative | DrawWay.Accumulate }` + sugar 字符串 `'+1,0'` / `'++1,0'`；`{ label }` infix 算子修饰下一段

### `@retikz/react`

- 新增 sugar `<EdgeLabel>`：作为 `<Step>` child 等价于 `<Step label={...}>`，prop 优先；不可逆（unbuilder 只产 prop 形态）
- Step / Path / Draw 透传所有新字段；builder / unbuilder 全链路 round-trip
- Draw way DSL 一并支持 label / 曲线 / 形状 / 相对坐标 sugar，与 Kernel 字面等价

## v0.1.0-alpha.2

Node 美化层 P1 全集落地：多行文本、字体对象、分轴边距、颜色 / 不透明度 / 描边样式 / 尺寸约束 / 缩放全套。配 3 篇 ADR 与 +47 个新测试用例。

> **BREAKING**：`<Node fontSize={n}>` 改为 `<Node font={{ size: n }}>`，IR `NodeSchema.fontSize` 字段直接删除；alpha 期不留 deprecate 共存窗口（见 ADR-0001）。

### `@retikz/core`

- **Node font 嵌套对象**：`font: { family?, size?, weight?, style? }` 取代标量 `fontSize` — ADR-0001
- **Node 多行文本**：`text: string | Array<string | LineSpec>`，行级可覆盖 `fill` / `opacity` / `font`；配 `align`（left / center / right）+ `lineHeight`（user units，缺省 `font.size × 1.2`） — ADR-0002
- **Node 分轴边距**：`innerXSep` / `innerYSep` / `outerSep` 三字段对齐 TikZ `inner xsep` / `inner ysep` / `outer sep`；`padding` / `margin` 保留为对称别名（轴特化字段优先） — ADR-0003
- **Node 颜色与不透明度**：`textColor` / `opacity`（整节点）/ `fillOpacity` / `drawOpacity`
- **Node 描边样式**：`dashed`（预设 `4 2`）/ `dotted`（预设 `1 2`）/ `dashArray`（显式 SVG `stroke-dasharray`）；优先级 `dashArray` > `dashed` > `dotted`
- **Node 尺寸约束**：`roundedCorners`（rectangle 圆角）/ `minimumWidth` / `minimumHeight` + 对称别名 `minimumSize`
- **Node 缩放**：`scale` / `xScale` / `yScale`；等比时字号同步缩放，非等比时字号取 `min(sx, sy)` 保形
- **TextPrim 升级**：`lines: Array<TextLine>` 取代 `content: string`；行级字段 `fontSize` / `fontFamily` / `fontWeight` / `fontStyle` / `fill` / `opacity`

### `@retikz/react`

- 新增 sugar `<Text>` 组件——`<Node>` 内行级样式覆盖：`<Node><Text fill="red" font={{ weight: 'bold' }}>L1</Text>L2</Node>`
- `<Node>` children 直接支持多行：字符串内嵌 `\n`、数组、混 `<Text>` 三种写法均可
- builder / unbuilder 透传所有新字段；派生类型 `IRFont` / `IRLineSpec` / `NodeTextAlign` 从 core re-export
- renderer `<text>` 改为多 `<tspan>` 渲染，按 `baseline` 自动反推首行 `dy` 实现块级垂直对齐

## v0.1.0-alpha.1

P0 + P1 流程图 / UML / 状态机刚需能力一次到位：节点 4 形状、path 折角 / 闭合 / 多端箭头 / 区域填色、显式锚点引用。配 4 篇 ADR 与 ~180 个测试用例。

### `@retikz/core`

- **Step kind 扩到 4 种**：`'move'` / `'line'` / `'step'`（折角，TikZ `(A) -| (B)` 风格）/ `'cycle'`（闭合到起点）—— ADR-0001
- **Path 路径级视觉**：`arrow`（4 方向）+ `arrowShape`（7 种：normal / open / stealth / diamond / openDiamond / circle / openCircle，含 hollow shape 自动 shrink）+ `fill` / `fillRule` —— ADR-0002
- **Node shape 多态**：`rectangle` / `circle` / `ellipse` / `diamond` 4 种几何 + boundaryPoint 多态；外接形状严丝合缝包住"text + padding" —— ADR-0003
- **Target 字符串扩展**：`'A'`（auto-clip）/ `'A.<anchor>'`（命名 anchor）/ `'A.<deg>'`（角度边界）—— ADR-0004
- **Sugar way DSL 升级**：`DrawWay` 关键字常量（cycle / hv / vh）+ infix 折角算子 `'-|'` / `'|-'` + cycle 关键字
- **路径 clip 算法重构**：每段独立用节点中心算两端 boundary——避免线穿过中间节点本体（与 TikZ 语义一致）
- **新工具类型**：`ValueOf<T extends object> = T[keyof T]`，统一 `as const` 对象 + 派生枚举模式
- **新 geometry 模块**：`circle` / `ellipse` / `diamond` 各自的 `boundaryPoint` / `anchor` / `contains`

### `@retikz/react`

- Path / Draw / Step / Node 透传所有新 IR 字段；`DrawWay` 与 IR 字面量类型完全互通
- Tikz 容器按需注入 SVG `<defs><marker>`（每种用到的 arrow shape 一份，`useId()` 防多实例冲突）
- Renderer 新增 `'ellipse'` primitive case（circle / ellipse 节点共用）
- 渲染管线现支持 GroupPrim 把多 sub-path 包起来——多节点 path + 箭头时只首段 / 末段挂 marker

## v0.1.0-alpha.0

首版 npm 发布。retikz v0.1 重构基线，IR 居中——为后续 alpha 迭代打底。

### `@retikz/core`

- **IR + Scene 编译器**：framework-agnostic 的中间表示与编译器，零 React、零 DOM
- **持久化形式**：内部 React state → 可序列化 IR JSON，跨平台、可被 LLM 直接读写

### `@retikz/react`

- **架构拆分**：从单包 React 组件库改为薄 React adapter，core 与 React 解耦
- **Kernel + Sugar 双层 JSX**：Kernel 直对应 IR 节点，Sugar 编译期展开为 Kernel
- peerDep React 18+
