---
title: Kernel 与 Sugar
description: retikz DSL 的两层组件：Kernel 直接进入 IR，Sugar 在 adapter 中展开为等价 Kernel。
---

retikz 的 React DSL 分成 Kernel 与 Sugar。区分它们有助于判断一段写法是底层契约，还是更短的书写便利。

## 定义

| 层 | 例子 | 是否直接进 IR | 说明 |
| --- | --- | --- | --- |
| Kernel | `<TikZ>` / `<Node>` / `<Coordinate>` / `<Path>` / `<Step>` | 是 | 与 IR 节点一一对应 |
| Sugar | `<Draw way={...}>`、`DrawWay.Cycle`、相对坐标字符串 | 否 | adapter 同步展开为 Kernel |

Sugar 不引入新能力。能用 `<Draw>` 画出来的图，也能用 `<Path>` + `<Step>` 手写出来。

## 选择建议

| 场景 | 推荐 |
| --- | --- |
| 普通连线、流程图边、带箭头路径 | `<Draw>` |
| 需要逐段控制 move / line / cycle / curve | `<Path>` + `<Step>` |
| 需要最接近 IR 的测试或转换器输出 | Kernel |
| 面向日常作者的文档示例 | 优先 Sugar，必要时解释对应 Kernel |

## 约束

Sugar 组件由 builder 同步调用来收集 JSX，不在 React render 调用栈上运行。因此 Sugar 的展开逻辑不能依赖 hooks 或异步状态。

当 Sugar 展开后丢掉构造名字仍能 1:1 反推原意图，它才适合做 Sugar。涉及数据布局、自动刻度、图算法这类高层结构时，应放到后续独立包，而不是塞进 core。
