---

retikz 使用一组统一的位置表达来描述实体放置与路径目标解析。`<Node>.position` / `<Coordinate>.position` 用于确定图元中心或引用点位置；`<Step>.to` / `<Draw>.way` 用于解析路径要连接或经过的目标。

| 形态 | 类型 | 含义 | 用在哪 |
| --- | --- | --- | --- |
| 笛卡尔 | `[x, y]` | 直接给出画布坐标 | 节点位置、路径目标 |
| 极坐标 | `{ origin?, angle, radius }` | 从极点按角度和半径推导位置 | 节点位置、路径目标 |
| 节点 id | `'node-id'` | 引用已命名节点或 Coordinate，并由路径自动贴边界 | **仅**路径目标（`Step.to` / way item） |
| 相对定位 | `{ direction, of, distance? }` | 按方向和距离放在另一个节点 / Coordinate 周围 | 节点位置、Coordinate 位置 |
| 偏移定位 | `{ of, offset }` | 基于节点、坐标或极坐标做二维偏移 | 节点位置、Coordinate 位置、路径目标 |
| 相对 target | `{ relative }` / `{ relativeAccumulate }` 或 `'+x,y'` / `'++x,y'` | 基于上一段终点做相对偏移 | **仅**路径目标 |

> 坐标系：x 向右，y 向下（与 SVG / Canvas 一致，与数学课本相反）。单位是 user units，不是像素，最终物理尺寸由 `<TikZ>` 的 `width` / `height` 与编译得到的 layout bounds 决定。

## 极坐标

`{ origin?, angle, radius }`：

| 字段 | 含义 |
| --- | --- |
| `origin` | 极点。可选，省略时为 `[0, 0]`；可传节点 id 字符串、笛卡尔坐标，**或嵌套的极坐标**（链式偏移） |
| `angle` | 角度（度），逆时针为正 |
| `radius` | 半径（user units） |

<Comparison target="tikz" title="极坐标角度">

`angle` 的正方向沿用 TikZ 习惯：度数制，逆时针为正。retikz 的屏幕坐标仍是 x 向右、y 向下，因此示例里的视觉方向会跟浏览器坐标系一致。

`origin` 还可以**嵌套**——例如 `{ origin: { origin: 'a', angle: 0, radius: 30 }, angle: 90, radius: 20 }` 表示"从 a 向右 30 再向上 20"。这种链式表达在画径向 / 局部偏移时很顺手。

## 节点 id 引用（path target）

`<Step>` / `<Draw>` 的 target 还接受**字符串**，表示引用同一 `<TikZ>` 内的节点 id：

<ComponentPreview name="positioning-id-ref" size="sm" />

线段的实际起止点由 retikz 自动算——见 [锚点](/core/concepts/anchors)。

`<Node>` / `<Coordinate>` 不能直接把 `position` 写成节点 id，因为"把一个实体放在另一个实体中心"通常不是最有用的默认。要表达节点之间的关系，用 `{ direction, of, distance? }`；要表达任意二维偏移，用 `{ of, offset }`。这两种写法把布局意图留在文档里，比提前算好每个绝对坐标更容易维护。

## 前向引用要求

字符串引用、`origin: 'node-id'` 都要求**被引用节点先于引用方定义**（按 IR children 源码顺序）。引用未定义节点时编译失败、path 整体跳过。

```tsx
// ✅ b 引用 a，a 在前
<Node id="a" position={[0, 0]}>A</Node>
<Node id="b" position={{ origin: 'a', angle: 0, radius: 50 }}>B</Node>

// ❌ a 引用 b，b 在 a 之后
<Node id="a" position={{ origin: 'b', angle: 0, radius: 50 }}>A</Node>
<Node id="b" position={[0, 0]}>B</Node>
```
