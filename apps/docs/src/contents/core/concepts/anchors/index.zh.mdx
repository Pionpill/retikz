每个 `<Node>` 在几何上有 9 个具名锚点。Path / Draw 引用节点时既能让 retikz 自动选边界点（最常见），也能用对象形态 `{ id, anchor?, offset? }` 锁死具体位置（写 JSX 时可用 `'A.north'` / `'A.30'` 字符串简写）。v0.2 起，带 `id` 的 `<Scope>` 也会注册一个矩形边界，可用同样的 anchor 语法引用整组图元。

## 9 个 anchor

| 名 | 位置 |
| --- | --- |
| `center` | 节点几何中心 |
| `north` / `south` / `east` / `west` | 上 / 下 / 右 / 左 4 个方位的边界点 |
| `north-east` / `north-west` / `south-east` / `south-west` | 4 个对角方位的边界点 |

内置四种 [`shape`](/core/components/node) 都遵守这套命名（按各自几何取对应位置）：

| | rectangle | circle | ellipse | diamond |
| --- | --- | --- | --- | --- |
| `north` / 等 4 方位 | 边中点 | 圆周 4 方位 | 椭圆轴端 | 4 顶点 |
| `north-east` / 等 4 对角 | 4 角 | 圆周 45° 处 | 椭圆参数 t=π/4 处 | 4 边中点 |

节点 `rotate` 时 9 个锚点跟着旋转。

自定义 shape 的 anchor 行为由它的 `ShapeDefinition` 提供：推荐支持这 9 个几何 anchor，方便与内置形状互换；如果某个注册形状不支持某个 anchor，编译期会给出对应错误。见 [自定义形状](/core/reference/extending/shape-registry)。

## 自动贴边界

最常用：path 用纯 id 字符串引用节点时（`<Step to="a">` / `<Draw way={['a', ...]}>`），retikz 在 compile 阶段调 `boundaryPoint`——从节点中心向"邻居 step 方向"画射线，取与 border 的交点作为线段端点。

选择原则：流程图、DAG 边、普通关系线默认只写节点 id，让 retikz 自动贴边；只有需要固定从某一侧出线、做排版对齐或复刻已有图时，才写显式 anchor / 角度。要让线离边框远一点，用 `margin`，不要为了留空隙把目标改成手算坐标。

`<Scope id="cluster">` 会把子树的视觉外接框注册成一个可引用边界，外部 path 可以写 `'cluster'` / `'cluster.north'` / `'cluster.30'`。这适合给一组节点画外部连线；更细的命名空间和 bbox 行为见 [Scope](/core/components/layout/scope)。

## 显式 anchor：对象形态（主契约）

不想自动选边、要锁死某个具体位置？path target 用**对象形态** `{ id, anchor?, offset? }`——这是 core IR 的主契约（序列化 / LLM 生成 / JSON patch 都用对象）：

| anchor 形态 | 含义 |
| --- | --- |
| 省略 anchor | `{ id: 'A' }`：自动贴边（同纯 id，**最常用**） |
| 命名 anchor | `{ id: 'A', anchor: 'north' }`：9 个具名锚点之一 |
| 角度 anchor | `{ id: 'A', anchor: 30 }`：deg 角度方向的视觉边界点 |
| 边上比例点 | `{ id: 'A', anchor: { side: 'north', t: 0.25 } }`：见下「边上比例点」 |
| anchor + 偏移 | `{ id: 'A', anchor: 'west', offset: [-4, 0] }`：解析到 anchor / 边点后再加世界系 `[dx, dy]`（不随节点 rotate 旋转） |

命名 / 角度 / 边上比例点 **位置定死**——不受邻居 step 方向影响（auto 模式才动态调整）。下面 8 条 Draw 用 8 个不同的命名 anchor 作端点：

### 字符串 shorthand（React DSL 便捷写法）

写 JSX / Draw way 时可用字符串简写，React 层会 eager 解析成上面的对象（core IR 仍是对象形态）：

| shorthand | 等价对象 |
| --- | --- |
| `'A'` | `{ id: 'A' }` |
| `'A.north'` | `{ id: 'A', anchor: 'north' }` |
| `'A.30'` | `{ id: 'A', anchor: 30 }` |

> **dotted-id 限制**：字符串 shorthand 按**第一个 `.`** 切分，所以 **id 含 `.` 时不能用 shorthand**——改写对象形态 `{ id: 'a.b', anchor: 'north' }`。`{ side, t }` 边上比例点也**只有对象形态**能表达。
>
> **支持的 anchor 名**：上述 9 个几何 anchor。`'text'` / `'base'` / `'mid'` 等文字基线 anchor 暂不属于公开 API。

## 边上比例点 `{ side, t }`

`{ side, t }` 取 shape **真实边界**某条边上从约定起点起 `t ∈ [0, 1]` 比例处——方向 north/south 西→东、east/west 北→南：rectangle 沿直边、circle / ellipse 沿该侧可见周长弧段、diamond 沿可见斜边。`{ side: 'west', t: 1/3 }` = 左边三等分点，`{ side: 'north', t: 0.5 }` = 上边中点（等于命名 `'north'`）。

## margin 让线"留呼吸"

`<Node margin={n}>`：border 不变，但 **auto 模式** 的 path 端点向外推 `n` user units——多节点流程图里 `margin={4}` 就够让连线不顶死 border。

<ComponentPreview name="anchors-margin" size="sm" />

注意：显式 anchor / 角度（`'A.east'` / `'A.30'`）取的是**视觉 border**，不应用 margin。如要把端点推得更远，加额外 padding 或显式坐标偏移。
