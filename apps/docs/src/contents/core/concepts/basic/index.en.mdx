Basic Concepts is the first group under Concepts. It answers "where is a point, how does retikz interpret that statement, and how does a figure change over time?" Read this before moving into the primitive model and primitive relations; for the DSL, IR, Scene, and renderer split, see Design Philosophy.

## What each page covers

| Page | Topic | Reach for it when |
| --- | --- | --- |
| Coordinate System | Cartesian as the first-class form, polar coordinates, user units, viewBox, and render size | You want to know where `[x, y]` lands, or how other coordinate forms eventually become Cartesian points |
| Positioning | Cartesian, polar, relative placement, offsets, proportional points, and path-relative targets | You want to place nodes / helper points or send a path to a target |
| Animation | Time-based changes on primitive properties, paths, and camera state | You want primitives to enter, blink, rotate, draw on, or control playback |
