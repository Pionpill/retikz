title: Core Concepts
description: Understand retikz through primitives themselves and the relationships between them: start with the primitive model, then move to primitive relations.
---

Core concepts answer two questions: what a primitive is made of, and how primitives establish stable geometric references between each other. The primitive model focuses on visible objects such as `Node`; primitive relations focus on anchors, boundaries, and how paths connect objects.

## Responsibilities

| Chapter | Topic | Read it when |
| --- | --- | --- |
| Primitive Model | Content box, padding, shape, bounding box, boundary, and anchors | You want to know why a node is this large, why a line attaches there, or how shapes participate in layout |
| Primitive Relations | Node boundaries, auto-attach, explicit anchors, edge-proportional points, and margin | You want to control which side a connection leaves from, where it lands, or how much space it keeps |
