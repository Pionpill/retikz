图元（primitive）是 retikz 里可被绘制和引用的最小单位。**图元模型**描述一个图元在几何上如何被搭起来——从内容、内框、shape，到外接框、boundary 与 anchor——使它能被测量、布局、连接和引用。它是一层抽象概念，不绑定某个具体组件。

Node 是这套图元模型最常用的具体实现。你可以把它想成一个“会被连线识别的盒模型”：`position` 给出节点的语义参考点，文本测量、`innerXSep` / `innerYSep` 和最小尺寸先形成内框，shape 再把内框包成视觉外接框，boundary 和 anchor 决定路径从哪里接上来。

图里从内到外是这套 Node 的分层：最内是内容盒，往外依次是内框、shape 视觉轮廓、外接框、boundary，最外是 anchor。实际渲染时你通常只会看到橙色 shape；其余几层是编译与连线阶段使用的几何信息。

| 层 | 由什么决定 | 影响什么 |
| --- | --- | --- |
| 内容盒<br />content box | `text` / children / font | 文本测量的基础尺寸 |
| 内框<br />inner frame | 文本测量 + `innerXSep` / `innerYSep` + 最小尺寸 | 传给 shape 的基础半轴 |
| 形状<br />shape | `shape.circumscribe` 与 `shape.emit` | 节点真正画出来的轮廓 |
| 外接框<br />bounding box | shape 对内框的包围结果，不包含 `margin` | viewBox、布局占位、命中范围的基础 |
| 连接面<br />boundary | `boundary`，默认跟随 shape | 自动连线贴到哪里 |
| 锚点<br />anchor | compass / angle / shape-specific / edge point | 精确引用节点上的点 |

## 盒模型

一个 Node 像 CSS 盒模型一样，由三层从内到外嵌套的框组成。

- **内容盒**：由 `text` / children 和 `font` 决定，是文本测量出的基础尺寸。
- **内框**：在内容盒外加上 `innerXSep` / `innerYSep`（对称别名 `padding`），再与 `minimumWidth` / `minimumHeight`（或 `minimumSize`）取较大者，得到传给 shape 的基础半轴——对矩形节点来说，它就是画出来的 shape。
- **外框**：在内框外再加 `outerSep`（对称别名 `margin`）。它不改变 shape，只把自动连线的端点往外推——也就是连线最终贴靠的边界。

<ComponentPreview name="node-frames" size="md" hideCode />

最内灰块是内容盒，中间虚线是内框（矩形节点的 shape），最外虚线是外框（连线贴靠的边界）。两道间距分别是 `innerSep`（即 `padding`，撑在内容盒与 shape 之间、把 shape 顶大）和 `outerSep`（即 `margin`，留在 shape 与边界之间，shape 不变、只让连线端点提前停）。

内框决定“这个节点至少要多大”，是后续所有几何的起点：shape 拿它算外接框，连线与 anchor 最终都落在由它派生的边界上。空文本节点也有内框——inner sep 与最小尺寸保证它不会塌成一个点。

| 属性 | 改的是 | 结果 |
| --- | --- | --- |
| `padding`（innerSep） | 内容盒到 shape 的距离 | shape 变大，外接框也变大 |
| `margin`（outerSep） | shape 到自动连线端点的额外距离 | shape 不变，路径提前停下 |

padding 是 Node 模型内部的一部分；margin 更像路径接入时的安全距离。

## 形状 Shape

shape 不是简单的 CSS border-radius。它是一套几何定义：给定内框半轴和参数后，shape 会回答“我要多大的外接框才能装下自己”，并负责画出最终的视觉轮廓。

| shape 类型 | 尺寸来源 | 例子 |
| --- | --- | --- |
| 文本容器形状 | 文字 + padding + 最小尺寸 | rectangle、circle、ellipse、polygon、diamond |
| 参数驱动形状 | shape params 自己决定几何 | sector、arc、star |

<ComponentPreview name="shapes" size="md" hideCode />

灰块是内容盒、虚线是内框，这五个文本容器形状共享同一个内框，shape 以内框为准画出不同轮廓：rectangle 即内框，其余把它包得更大（circle / diamond 分别是 ellipse / polygon 的 preset）。参数驱动形状（sector / arc / star）的几何来自 params，不走这套内框逻辑。

文本容器形状更像按钮或 DOM 盒子：文字变长，节点会变大。参数驱动形状更像一个几何 glyph：半径、角度、尖角数量已经定义了大小，文本只是附着在它上面。

用 `defineShape` 扩展 shape 时，要补齐 shape 的几何回答——外接框怎么包住自己、射线打到哪、具名 anchor 在哪、如何 emit 成基础图元——让 Node 在可测量、可连接、可引用、可渲染上和内置 shape 一样。实现细节见 [自定义形状](/core/reference/extending/shape-registry)。

## 外层边界 Boundary

外接框（bounding box）和连接面（boundary）都是 shape 之外的一层包围面，但分工不同：外接框管布局与渲染占位，连接面管自动连线贴到哪。默认两者都跟随 shape，只有显式改 `boundary` 时才分开。

<ComponentPreview name="bbox-boundary" size="sm" hideCode />

两个节点是同一个 polygon shape（六边形）、同一个内框（最内虚线）和外接框（灰色虚线矩形），两侧完全相同。左边 `boundary="shape"`（默认），自动连线贴到多边形边缘；右边 `boundary="circle"`，连线改贴到蓝色虚线圆——注意这个圆可以伸出外接框。改 boundary 只挪连线落点，不动 shape、内框、外接框。

**外接框**不是视觉轮廓本身，而是能包住视觉轮廓的矩形。复杂形状也会先被归约到这个矩形里，后续的 viewBox、布局和渲染裁剪都可以用统一模型处理。少数形状还会提供 `circumscribeOffset`，让 `position` 这个语义参考点和外接框中心分离，例如扇形以圆心定位，但外接框中心偏在扇形一侧。

**连接面**只回答一个问题：路径自动连接到这个 Node 时，端点应该贴到哪一层表面。默认 `boundary="shape"`，自动路径端点会贴到视觉 shape 的边上。你也可以把五角星的连接面改成圆或矩形，让连线看起来更规整，但这不会改变星形本身，也不会改变它的外接框。

借用其它 boundary 时，retikz 会把它套在当前 Node 的视觉外接框上，不会重新调用那个 boundary shape 的 `circumscribe`。例如 `boundary="circle"` 的半径来自当前外接框的长半轴；它可能伸出外接框，但仍然不参与布局占位。

| 设置 | 改变什么 | 不改变什么 |
| --- | --- | --- |
| `shape` | 视觉轮廓、外接框、shape anchor | 节点中心语义 |
| `boundary` | 自动连线端点、compass anchor 的连接面 | 节点视觉、外接框、布局占位 |
| 单条边覆盖 boundary | 当前路径端点 | 节点默认设置 |

因此，外接框是“布局和渲染用的占位矩形”，连接面更接近“连接用的 hit surface”——都不是“画出来的外壳”。

## 锚点 Anchor

Anchor 是 Node 暴露给其它图元引用的点。路径、定位和标签都可以通过 anchor 找到节点上的稳定位置。

| anchor 类型 | 例子 | 适合用途 |
| --- | --- | --- |
| 中心点 | `center` | 对齐、旋转、放射关系 |
| 方位点 | `north`、`east`、`south west` | 常规连线和布局 |
| 角度点 | `35` / `{ angle: 35 }` | 沿射线找边界交点 |
| shape 专属点 | star 的尖角、sector 的圆弧端点 | 复杂形状的语义连接 |
| 边比例点 | `{ side: 'north', t: 0.25 }` | 表格、泳道、框图边缘定位 |

<ComponentPreview name="anchors" size="lg" hideCode />

上排都用最常见的矩形：左边是固定的 8 个方位 anchor（外加 `center`）；右边是角度 anchor（`A.30`，沿 30° 射线打到边界）和边比例 anchor（`{ side: 'north', t: 0.25 }`，north 边西→东取 1/4 处）。下排是几种自带 shape 专属点的非矩形形状：star 的 `tip-N` 尖角（另有 `notch-N` 凹角）、sector 的 `apex` 与 `outer-arc` 端点 / 中点、arc 的 `start` / `arc-mid` / `end`。这些专属点用对象形态引用，如 `{ id: 'A', anchor: 'tip-0' }`。

方位锚点有两套名字：`CompassAnchor` 是 canonical 名称，沿用 TikZ 风格；`WebAnchor` 是面向 Web / CSS 心智的输入别名，编译时归一到对应的 `CompassAnchor`。

| CompassAnchor | 位置 |
| --- | --- |
| `center` | 节点几何中心 |
| `north` / `south` / `east` / `west` | 上 / 下 / 右 / 左 4 个方位的边界点 |
| `north-east` / `north-west` / `south-east` / `south-west` | 4 个对角方位的边界点 |

内置四种 [`shape`](/core/components/node) 都遵守这套命名（按各自几何取对应位置）：

| | rectangle | circle | ellipse | diamond |
| --- | --- | --- | --- | --- |
| `north` / 等 4 方位 | 边中点 | 圆周 4 方位 | 椭圆轴端 | 4 顶点 |
| `north-east` / 等 4 对角 | 4 角 | 圆周 45° 处 | 椭圆参数 t=π/4 处 | 4 边中点 |

节点 `rotate` 时 9 个锚点跟着旋转。`WebAnchor` 别名归一如下：

| WebAnchor | 等价 CompassAnchor |
| --- | --- |
| `top` / `bottom` / `right` / `left` | `north` / `south` / `east` / `west` |
| `top-right` / `top-left` | `north-east` / `north-west` |
| `bottom-right` / `bottom-left` | `south-east` / `south-west` |

<Comparison target="tikz" title="锚点命名">

`center` / `north` / `south` / `east` / `west` / `north-east` / `north-west` / `south-east` / `south-west` 沿用 TikZ 的常用 anchor 词汇，方便把节点关系从 LaTeX 图迁到 JSX 写法。

</Comparison>

自定义 shape 的 anchor 行为由它的 `ShapeDefinition` 提供：推荐支持这 9 个标准方位 anchor，方便与内置形状互换；如果某个注册形状不支持某个 anchor，编译期会给出对应错误。见 [自定义形状](/core/reference/extending/shape-registry)。

边上比例点 `{ side, t }` 取 shape **真实边界**某条边上从约定起点起 `t ∈ [0, 1]` 比例处——方向 north/south 西→东、east/west 北→南：rectangle 沿直边、circle / ellipse 沿该侧可见周长弧段、diamond 沿可见斜边。`{ side: 'west', t: 1/3 }` = 左边三等分点，`{ side: 'north', t: 0.5 }` = 上边中点（等于命名 `'north'`）。

解析时，方位点会经过 boundary 的连接面计算；当 `boundary="shape"` 时，方位点按视觉外接框的矩形方位解析。数字角度点走 `boundaryPoint`，会沿射线打到选定 boundary 上。shape 专属点和边上比例点始终走视觉 shape，不受 `boundary` 改写。

这些点怎么被另一个图元引用、连接端点如何在 auto / 锁定两种模式间解析，见 [图元关系](/core/concepts/core/primitive-relations)。
