动画让固定图元**按时间从一个样子过渡到另一个样子**：淡入、画线、放大、旋转、变色、镜头推拉。你不用手写关键帧——调一个**preset 工厂**（`fadeIn()` / `spin()` …）得到一条动画描述，挂到元素的 `animations` 数组即可。

```tsx
import { Layout, Node, fadeIn, scaleIn } from '@retikz/react';

<Node animations={[fadeIn(), scaleIn()]}>hi</Node>
```

`animations` 可挂在 `<Node>` / `<Path>` / `<Scope>`（元素动画）与 `<Layout>`（镜头）。一个元素能挂多条，互不干扰。

<ComponentAlert title="末帧 = 静止态，所以能降级" description="入场类动画的最后一帧就是元素本来的样子（fadeIn 的末帧 = 不透明）。于是不支持动画的环境、用户开了「减少动态效果」、或你传 animate={false} 时，直接渲染这个静止态——看到的是完整最终图，不是动画起点那个可能全透明的样子。" />

## 全部动画一览

| 动画 | 家族 | 底层通道 | 说明 |
| --- | --- | --- | --- |
| `fadeIn` | 入场 | opacity | 淡入（0→1） |
| `drawOn` | 入场 | pathDraw | 描边沿路径画出 |
| `scaleIn` | 入场 | scale | 从略小弹入（0.8→1） |
| `grow` | 入场 | scale | 从无到有放大（0→1） |
| `growUp` | 入场 | scaleY | 从基线长出（柱状图） |
| `slideIn` | 入场 | translateX/Y | 从偏移滑入 |
| `colorShift` | 强调 | fill / stroke | 颜色过渡（oklch 插值） |
| `pulse` | 循环 | scale | 脉冲缩放（无限） |
| `spin` | 循环 | rotate | 匀速旋转（无限，loader） |
| `loop` | 循环 | 任意 | 把任意 track 变成无限循环 |
| `cameraTo` | 镜头 | viewBox | 镜头推拉 / 平移（挂 `<Layout>`） |
| `stagger` | 编排 | — | 给一组动画依次叠加延迟 |

每个 preset 都接受公共项 `{ duration, delay, easing, trigger }`，部分另有专有项（`scaleIn.from` / `slideIn.axis,offset` / `growUp.origin` / `pulse.peak` …）。

## 强调与循环

<ComponentPreview name="color-shift" size="xs" />

`colorShift`：`fill` / `stroke` 在 **oklch** 空间过渡（感知均匀、无灰带）。起止色都要显式给。作用于**形状自身的填充 / 描边**，用在纯形状或路径上；带文字 label 的节点其形状已自带 fill，不随之变。

<ComponentPreview name="pulse" size="xs" />

`pulse`：1→peak→1 无限脉冲（`peak` 默认 1.1）。

<ComponentPreview name="spin" size="xs" />

`spin`：rotate 0→360 无限匀速，绕几何中心——典型 loader。

<ComponentPreview name="loop" size="sm" />

`loop(track, { direction })`：把任意一次性 track 包成无限循环；配 `direction: 'alternate'` 往返播。

## 镜头

<ComponentPreview name="camera-to" size="md" />

`cameraTo({ from, to })`：动 scene 的取景框 `[x, y, w, h]`，实现推拉 / 平移。挂在 `<Layout animations>`（镜头属 scene 根，不是某个元素）；起止取景都要显式给。

## 编排

<ComponentPreview name="stagger" size="sm" />

`stagger(tracks, step)`：给一组 track 依次叠加 `delay`（0、step、2×step…），实现「依次入场」。

## 触发与降级

**何时播**由 `trigger` 决定，默认 `load`（渲染即播）：

| trigger | 行为 | 后端支持 |
| --- | --- | --- |
| `load`（默认） | 渲染即播 | SVG（内联 CSS 自播，SSR 零 JS）/ Canvas（起 rAF） |
| `visible` | 进视口才播 | SVG（IntersectionObserver）；Canvas 暂渲染静止态 |
| `manual` | 不自动播，由运行时句柄控制 | SVG；Canvas 暂渲染静止态 |
| `{ onEvent: 'click' }` | 命中事件才播 | SVG；Canvas 暂渲染静止态 |

**何时不播**（三者殊途同归到静止态，见上方提示）：传 `animate={false}`、用户系统开了「减少动态效果」（`prefers-reduced-motion`）、或后端不支持动画。降级时**不报错、不丢图**，渲染完整静止图并给一条可诊断告警。

各后端能力：**SVG** 支持全部 trigger；**Canvas** 本阶段自动播放 `load` 动画（`visible` / `manual` / `onEvent` 渲染静止态，后续补齐）。
