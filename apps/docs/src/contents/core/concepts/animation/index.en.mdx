Animation transitions a static figure **from one look to another over time**: fade in, draw a line, scale up, spin, shift color, pan the camera. You don't hand-write keyframes â€” call a **preset factory** (`fadeIn()` / `spin()` â€¦) to get an animation descriptor and attach it to an element's `animations` array.

```tsx
import { Layout, Node, fadeIn, scaleIn } from '@retikz/react';

<Node animations={[fadeIn(), scaleIn()]}>hi</Node>
```

`animations` attaches to `<Node>` / `<Path>` / `<Scope>` (element animation) and `<Layout>` (camera). An element can carry several; they don't interfere.

## Triggers & degradation

**When it plays** is set by `trigger`, defaulting to `load` (plays on render):

| trigger | Behavior | Backend support |
| --- | --- | --- |
| `load` (default) | plays on render | SVG (inline CSS, zero-JS SSR) / Canvas (starts rAF) |
| `visible` | plays when in viewport | SVG (IntersectionObserver); Canvas renders resting state for now |
| `manual` | no autoplay, driven by a runtime handle | SVG; Canvas renders resting state for now |
| `{ onEvent: 'click' }` | plays when the event fires | SVG; Canvas renders resting state for now |

**When it doesn't play** (all three converge to the resting state, see the note above): you pass `animate={false}`, the user's system prefers reduced motion (`prefers-reduced-motion`), or the backend can't animate. Degradation never throws and never drops the figure â€” it renders the complete resting figure and emits a diagnosable warning.

Backend capability: **SVG** supports every trigger; **Canvas** autoplays `load` animations at this stage (`visible` / `manual` / `onEvent` render the resting state, to be completed later).
