Animation transitions a static figure **from one look to another over time**: fade in, draw a line, scale up, spin, shift color, pan the camera. You don't hand-write keyframes ‚Äî call a **preset factory** (`fadeIn()` / `spin()` ‚Ä¶) to get an animation descriptor and attach it to an element's `animations` array.

```tsx
import { Layout, Node, fadeIn, scaleIn } from '@retikz/react';

<Node animations={[fadeIn(), scaleIn()]}>hi</Node>
```

`animations` attaches to `<Node>` / `<Path>` / `<Scope>` (element animation) and `<Layout>` (camera). An element can carry several; they don't interfere.

<ComponentAlert title="Last frame = resting state, so it degrades" description="An intro animation's final frame is the element's normal look (fadeIn's last frame = fully opaque). So environments without animation support, users who prefer reduced motion, or animate={false} simply render that resting state ‚Äî you see the complete final figure, not the possibly-transparent starting point." />

## All animations

| Animation | Family | Channel | What it does |
| --- | --- | --- | --- |
| `fadeIn` | intro | opacity | fade in (0‚Üí1) |
| `drawOn` | intro | pathDraw | stroke draws along the path |
| `scaleIn` | intro | scale | pop in from slightly small (0.8‚Üí1) |
| `grow` | intro | scale | grow from nothing (0‚Üí1) |
| `growUp` | intro | scaleY | grow from baseline (bars) |
| `slideIn` | intro | translateX/Y | slide in from an offset |
| `colorShift` | emphasis | fill / stroke | color transition (oklch lerp) |
| `flash` | emphasis | opacity | flash once (2√ó by default) |
| `blink` | emphasis | opacity | continuous blink (infinite) |
| `wiggle` | emphasis | rotate | rock left/right (3√ó by default) |
| `pulse` | loop | scale | pulsing scale (infinite) |
| `spin` | loop | rotate | steady rotation (infinite, loader) |
| `loop` | loop | any | turn any track into an infinite loop |
| `cameraTo` | camera | viewBox | pan / zoom the camera (on `<Layout>`) |
| `stagger` | choreography | ‚Äî | add increasing delay across a group |

Every preset accepts the common options `{ duration, delay, easing, trigger }`; some add their own (`scaleIn.from` / `slideIn.axis,offset` / `growUp.origin` / `pulse.peak` ‚Ä¶).

## Triggers & degradation

**When it plays** is set by `trigger`, defaulting to `load` (plays on render):

| trigger | Behavior | Backend support |
| --- | --- | --- |
| `load` (default) | plays on render | SVG (inline CSS, zero-JS SSR) / Canvas (starts rAF) |
| `visible` | plays when in viewport | SVG (IntersectionObserver); Canvas renders resting state for now |
| `manual` | no autoplay, driven by a runtime handle | SVG; Canvas renders resting state for now |
| `{ onEvent: 'click' }` | plays when the event fires | SVG; Canvas renders resting state for now |

**Imperative control**: pass a ref to `<Layout animationRef={ref}>` and after mount you get `AnimationControls` (`play` / `pause` / `seek`) to drive playback from your own buttons outside the component (most useful with `trigger: 'manual'`):

```tsx
const ref = useRef<AnimationControls | null>(null);

<Layout animationRef={ref}>
  <Node animations={[{ ...spin(), trigger: 'manual' }]} />
</Layout>
<button onClick={() => ref.current?.play()}>Play</button>
```

SVG controls the WAAPI handle for interactive (`manual` / `visible` / `onEvent`) tracks; Canvas controls the rAF clock ‚Äî the peer of vanilla's `view.animation`. `ref.current` is `null` when there's no animation or it degraded. (Inside a handler, use hydration's [`context.animation`](/core/reference/renderer/hydration#the-handler-context) instead.)

**When it doesn't play** (all three converge to the resting state, see the note above): you pass `animate={false}`, the user's system prefers reduced motion (`prefers-reduced-motion`), or the backend can't animate. Degradation never throws and never drops the figure ‚Äî it renders the complete resting figure and emits a diagnosable warning.

Backend capability: **SVG** supports every trigger; **Canvas** autoplays `load` animations at this stage (`visible` / `manual` / `onEvent` render the resting state, to be completed later).

## Static snapshot (`snapshotAt`)

Pass `<Layout snapshotAt={ms}>` (or `renderToSvgString(ir, { snapshotAt })` / `mountSvg(‚Ä¶, { snapshotAt })`) to render **a single frame frozen at that time**: no playback, no emitted animation ‚Äî each track's value at that instant is baked into static attributes / transforms. Use it for SSR poster frames, thumbnails, social-card previews.

<ComponentPreview name="snapshot" size="xs" />

```tsx
// grow (0‚Üí1, 300ms) frozen at 150ms ‚âà half size
<Layout snapshotAt={150}>
  <Node animations={[grow({ duration: 300 })]}>150ms</Node>
</Layout>
```

`snapshotAt ‚â• duration` ‚Üí the settled state (= the full base figure); `snapshotAt` overrides `animate`. SVG reuses the same `evaluateTrack` engine, so a snapshot matches the same instant during playback frame-for-frame; Canvas paints one frame at that time without starting a rAF loop.
