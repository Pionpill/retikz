title: 设计理念
description: 理解 retikz 为什么把书写、声明、编译和渲染拆成稳定边界。
---

设计理念解释 retikz 的整体取舍：用户写 DSL，系统保留可持久化的 IR，再编译成 renderer 可以直接消费的 Scene。这里适合在已经能画出基础图形后阅读，用来判断一段能力应该落在 Kernel、Sugar、IR、Scene 还是 renderer。

## 职责一览

| 章节 | 主题 | 适合什么时候读 |
| --- | --- | --- |
| 分层 | DSL、IR、Scene 与渲染器之间的关系 | 想知道 JSX 怎么变成 SVG / Canvas，或想理解 Kernel / Sugar 的边界 |
| 复合 | domain 包如何在 core 之上注册高层节点，并在 compile 期下沉 | 想理解 composite、plot 这类高层能力为什么不直接塞进 core |
