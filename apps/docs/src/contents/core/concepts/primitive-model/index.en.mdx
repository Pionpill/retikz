The primitive model answers "what objects exist in a figure, and how are they rendered?" Start here when you want to understand how `<Node>`, `<Path>`, `<Coordinate>`, and `<Scope>` relate to the rendering pipeline.

## What each page covers

| Page | Topic | Reach for it when |
| --- | --- | --- |
| Layers | The relationship between DSL, IR, Scene, and renderers | You want to know how JSX becomes SVG / Canvas, or where Kernel and Sugar split |
| Animation | Time-based changes on primitive properties, paths, and camera state | You want primitives to enter, blink, rotate, draw on, or control playback |
