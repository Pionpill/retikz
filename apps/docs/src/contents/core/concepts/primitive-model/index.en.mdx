The primitive model answers "what is a primitive made of, and what behavior does it carry?" Start with the Node model, then continue to how primitive properties change over time. For how primitives enter the rendering pipeline, read Layers under Design Philosophy.

## What each page covers

| Page | Topic | Reach for it when |
| --- | --- | --- |
| Node Model | Content box, padding, shape, bounding box, boundary, and anchors | You want to know why a node is this large, why a line attaches there, or how shapes participate in layout |
| Animation | Time-based changes on primitive properties, paths, and camera state | You want primitives to enter, blink, rotate, draw on, or control playback |
