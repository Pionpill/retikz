---

这一分组是图的「容器」层。`<Layout>` 是每张图必需的最外层包装：接收组件树或 IR，编译出 Scene，再交给当前渲染器输出。`<Scope>` 则在内部把一组 children 聚成一个单元，对它们整体施加平移 / 旋转 / 缩放，并可凭 `id` 被外部当作一个整体引用 / 定位。两者都在「确立坐标语境」上分工：`<Layout>` 给出根坐标系，`<Scope>` 在其中开一层局部坐标系。

## 职责一览

| 组件 | 类型 | 职责 | 适合场景 |
| --- | --- | --- | --- |
| Layout | Kernel | 唯一必需的顶层容器：children / IR 入口，编译 Scene 并输出；`width` / `height` 控制显示尺寸 | 每张图的最外层包装 |
| Scope | Kernel | 局部 transform 容器：把一组 children 当作整体平移 / 旋转 / 缩放，可凭 `id` 被整体引用 | 子图整体变换、局部坐标系、id 命名空间隔离 |
