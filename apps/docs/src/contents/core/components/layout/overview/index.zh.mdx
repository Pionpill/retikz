`<Layout>` 是 retikz 唯一必需的顶层包装，所有 Kernel / Sugar 组件都必须放在它里面。

`<Layout>` 是 DSL 与渲染器之间的入口：使用 children 时，它会把内部组件树转换为 IR；使用 `ir` 时，则直接采用这份已构造好的 IR。随后它把 IR 编译为 Scene，并交给当前渲染器输出。`width` / `height` 控制的是最终结果在页面上的显示尺寸；内部坐标仍然使用 retikz 用户单位，编译阶段会根据布局边界计算可见范围。

## 用法

```tsx
import { Layout } from '@retikz/react';
```

```tsx
<Layout width={300} height={80}>
  {/* Kernel / Sugar children */}
</Layout>
```

## 组合

`<Layout>` 是容器；children 是任意 Kernel / Sugar 组件树：

```tsx
<Layout>
  <Node id="..." position={...}>label</Node>
  <Path>
    <Step kind="move" to="..." />
    <Step kind="line" to="..." />
  </Path>
  <Draw way={[...]} />
</Layout>
```

### 全图默认样式

`<Layout>` 直接接受 [`<Scope>`](/core/components/layout/scope) 的级联样式子集（`color` / `stroke` / `fill` / `strokeWidth` / `opacity` / `fillOpacity` / `drawOpacity` + `nodeDefault` / `pathDefault` / `labelDefault` / `arrowDefault`）。设了任一样式 prop，Layout 会把 children 包进一个**隐式根 `<Scope>`**——等价于你手写一层根 `<Scope>`，省掉每张图都套一层的样板。内层 `<Scope>` 与图元显式属性照常级联覆盖。

<ComponentPreview name="layout-scope-style" size="sm" />

```tsx
// 给整张图设默认样式：不用再手写嵌套 <Scope>
<Layout stroke="gray" nodeDefault={{ shape: 'circle', fill: 'white' }} pathDefault={{ strokeWidth: 3 }}>
  <Node id="A" position={[0, 0]}>A</Node>
  <Draw way={['A', 'B']} arrow="->" />
</Layout>

// 等价于旧写法（手写根 <Scope>）
<Layout>
  <Scope stroke="gray" nodeDefault={{ shape: 'circle', fill: 'white' }} pathDefault={{ strokeWidth: 3 }}>
    {/* ... */}
  </Scope>
</Layout>
```

> 仅当至少一个样式 prop 真正携带样式指令时才包隐式 scope；无样式 prop（或只传空对象 default 如 `nodeDefault={{}}`）时 IR 形态保持不变、不引入空 scope。与 `ir` prop 并用时样式 props 被忽略（dev 模式发 console 警告）——`ir` 已是完整 IR，根样式请写进 IR 里的根 `<Scope>` 节点。

### 从 IR 渲染

`<Layout>` 接受 `ir` prop——直接把一段已经持久化的 IR JSON 渲染成图，**不写一行 JSX children**。来源不限：LLM 生成、JSON 编辑器、URL 反序列化、数据库存档都可以。

### 自定义 viewBox

默认 `viewBox` 由内容自动算（AABB + padding）。传 `viewBox={{ x, y, width, height }}` 显式定死视框——固定尺寸、故意裁剪、多图对齐、排除溢出都靠它。有 viewBox 则**覆盖**自动算、**忽略 padding**；`width` / `height` 必须 > 0。

<ComponentPreview name="layout-viewbox" size="md" />

> viewBox 注入构造出的 IR 根（`ir.viewBox`）；直接传 `ir` prop 且 IR 自带 viewBox 时，prop 优先、prop 缺省则用 IR 内置值。与 `width` / `height`（渲染尺寸）正交。

## 渲染目标（SVG / Canvas）

`renderer` 选择输出后端：默认 `'svg'`，设 `'canvas'` 则用同一份 Scene 画到 `<canvas>`。缺省 `renderer` prop 时，`<Layout>` 回退读取最近的 `RendererModeProvider` 注入值——优先级 **显式 `renderer` prop > context > `'svg'`**。

这适合「外层想统一切换一棵无法逐个改 `renderer` 的子树」的场景，比如把一个交互式（含 hooks / 异步）子树整体在 svg / canvas 间切——你改不到它内部的 `<Layout renderer>`，但用 Provider 一包就行：

```tsx
import { Layout, RendererModeProvider } from '@retikz/react';

// 子树内未写 renderer 的 <Layout> 随 Provider 切换；写了 renderer 的不受影响
<RendererModeProvider mode="canvas">
  <SomeInteractiveChart />
</RendererModeProvider>;
```

## API 参考

| 属性 | 类型 | 默认值 | 描述 |
| --- | --- | --- | --- |
| `width` | `number \| string` | — | 渲染结果宽度 |
| `height` | `number \| string` | — | 渲染结果高度 |
| `viewBox` | `{ x, y, width, height }` | — | 显式视框，覆盖自动算的范围（忽略 padding）；注入 IR 根、prop 优先于 IR 内置 |
| `children` | `ReactNode` | — | Kernel / Sugar 组件树（与 `ir` 二选一） |
| `ir` | `IR` | — | 已构造好的 IR JSON（与 `children` 二选一） |
| `className` | `string` | — | 透传到渲染根元素 |
| `style` | `CSSProperties` | — | 透传到渲染根元素 |
| `idPrefix` | `string` | — | 覆盖 `<defs>` 资源 id 前缀；SSR 输出和客户端 `<Layout>` 传同一值时可保持引用 id 一致 |
| `renderer` | `'svg' \| 'canvas'` | `'svg'` | 渲染后端；缺省时回退最近 `RendererModeProvider` 的 context 值（prop > context > svg） |
| `color` / `stroke` / `fill` / `strokeWidth` / `opacity` / `fillOpacity` / `drawOpacity` | 同 [`<Scope>`](/core/components/layout/scope) | — | 全图级联 graphic state；设值则把 children 包进隐式根 `<Scope>` |
| `nodeDefault` / `pathDefault` / `labelDefault` / `arrowDefault` | 同 [`<Scope>`](/core/components/layout/scope) | — | 全图 every-X 默认样式（四通道）；同上挂隐式根 scope |
| `animate` | `boolean` | `true` | 是否播放动画；`false` 渲染 base 静态图（不 emit CSS/WAAPI、canvas 不起 rAF）。见 [动画](/core/concepts/animation) |
| `snapshotAt` | `number` | — | 静态截帧时刻（毫秒）：定格在该时刻的一帧、不播放，覆盖 `animate`；SSR 海报帧 / 缩略图 |
| `animations` | `Array<AnimationTrack>` | — | scene 根（镜头）动画 tracks（`viewBox` 通道），配 [`cameraTo()`](/core/concepts/animation) |
| `animationRef` | `Ref<AnimationControls \| null>` | — | 命令式动画句柄出口：挂载后写入 `play` / `pause` / `seek`（svg=WAAPI、canvas=rAF 时钟），无动画为 `null` |
| `easings` | `EasingRegistry` | — | 自定义缓动注册表（名 → cubic-bezier 四元组 / 函数）。见 [自定义动画](/core/reference/extending/custom-animation) |
| `animationProperties` | `AnimationPropertyRegistry` | — | 自定义动画属性通道插值器（如 `blur`）；当前仅 Canvas。见 [自定义动画](/core/reference/extending/custom-animation) |
| `handlers` | `HydrationHandlers` | — | `ir` 模式按 id 提供的水合 handler 注册表（JSX 模式用组件 `on<Event>` props）。见 [水合](/core/reference/renderer/hydration) |

### 直接喂 IR

`ir` prop 适合"已经有 IR JSON"的场景——AI 生成、JSON 编辑器、URL 反序列化。配合 `convertReactNodeToIR` / `convertIRToReactNode` 做 DSL ↔ IR 双向桥：

```tsx
import { Layout, Node, convertReactNodeToIR } from '@retikz/react';

const ir = convertReactNodeToIR(
  <Node id="a" position={[0, 0]}>A</Node>,
);
// 序列化、传 LLM、存数据库……
<Layout ir={ir} width={200} height={80} />;
```

`ir` 与 `children` 互斥；同时传时优先 `ir`。

如果图形看起来被裁掉或留白过多，先看 **View Code → IR** 里的节点坐标和 path bounds：通常是内部坐标离可见图元太远、只定义了不可见 coordinate，或用了显式坐标但没有对应的可见边界。
