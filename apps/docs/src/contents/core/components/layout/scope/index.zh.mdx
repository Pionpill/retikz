description: 局部 transform 容器——把一组 children 包起来批量平移 / 旋转 / 缩放。
---

`<Scope>` 在 IR 层把一组子元素聚成容器，对它们应用局部 transform 链。

## 用法

```tsx
import { Scope } from '@retikz/react';
```

```tsx
<Scope transforms={[{ kind: 'translate', x: 50, y: 0 }]}>
  {/* Node / Path / Coordinate / 嵌套 Scope */}
</Scope>
```

没有属性与子元素的 `<Scope></Scope>` → compile 不发任何 Scene primitive；任一非空 → 发 `GroupPrim`，不会留下空容器开销。

### 何时用 Scope

设计上有"分组"概念时——一组节点 / path 在意图上属于同一个子图，需要作为整体应用 transform，或需要让外层通过 `scope.id` 把它们当作一个单元引用 / 定位。

第一次读 Scope，先抓住三件事：

| 目标 | 用法 |
| --- | --- |
| 整体移动 / 旋转 / 缩放一组图元 | `transforms` |
| 给一组图元统一默认样式 | `color` / `nodeDefault` / `pathDefault` / `labelDefault` / `arrowDefault` |
| 让外部连到整组图元边界 | `id` |

## 组合

`<Scope>` 的 children 可以是 `<Node>`、`<Coordinate>`、`<Path>` 或嵌套 `<Scope>`。Scope 自己不画形状；只有加了 transform / clip / 样式默认 / `id` 等属性时，才在 Scene 里形成一个 group。

把 Scope 当成“子图容器”而不是“普通 div”：它适合承载几何关系、局部坐标和整组引用；如果只是想让 JSX 更好看，优先用普通 React 组件拆分代码。

### transforms 6 变体

4 个 translate 变体一一镜像 `<Node position>` 的 4 种形态（笛卡尔 / 极坐标 / 方向相对 / 偏移）。

| kind | 字段 | 语义 |
|---|---|---|
| `translate` | `{ x, y }` | 笛卡尔平移，user units |
| `polar-translate` | `{ origin?, angle, radius }` | 极坐标平移，`origin` 为 `id` / `[x,y]` / `PolarPosition`，缺省 = 原点 |
| `at-translate` | `{ direction, of, distance? }` | 落到 referent 节点的某方向 distance 处，`distance` 缺省走 `<Layout nodeDistance>` |
| `offset-translate` | `{ of, offset? }` | scope 原点 = referent 全局坐标 + `[dx, dy]`，`offset` 缺省 = `[0, 0]` |
| `rotate` | `{ degrees, cx?, cy? }` | 旋转 degrees 度，`cx/cy` 缺省 = scope 局部原点 |
| `scale` | `{ x, y? }` | 缩放，`y` 缺省 = `x`（等比） |

数组顺序应用——第 0 个最先生效（最内层）。`transforms: [translate, rotate]` 表示先 translate 再 rotate；compile 展平 / 下沉细节详见 § 技术原理 §「transforms 展平到 Scene」。

<Comparison target="tikz" title="Scope 对照">

`<Scope transforms={[{ kind: 'translate', x: 50, y: 0 }]}>...</Scope>` 对应 TikZ 的 `\begin{scope}[shift={(50,0)}] ... \end{scope}`：在子图开始处压一层局部 transform，结束时弹出。TikZ 多个 option 写一起按出现顺序叠加 CM；retikz `transforms` 数组按 index 顺序应用，复用 SVG / Scene 约定。

</Comparison>

### 样式继承

Scope 兼作样式默认值挂点，给内部元素提供三组样式表面：**主色 `color`** + 跨类共享分项级联到全部元素；**四类默认** `nodeDefault` / `pathDefault` / `labelDefault` / `arrowDefault` 按元素类型分发、扁平独立；**`resetStyle`** 切外层继承。

#### 主色级联

`<Scope color>` 把当前色下传给内部全部元素——节点边 / 文字、path stroke / 箭头 / 边标注都默认随它；分项（`stroke` / `fill` / `textColor`）显式给则覆盖主色：

<ComponentPreview name="scope-style-cascade" size="xs" />

#### 分类默认

`nodeDefault` / `pathDefault` / `labelDefault` / `arrowDefault` 各给一类元素挂默认样式，扁平独立、互不干涉。`nodeDefault` 含形状 / 间距等节点专属字段（直接写 `color` 染不到 `shape`，得走 `nodeDefault`）；箭头走独立 `arrowDefault`，`pathDefault` 不含箭头：

## 技术原理

### 命名空间栈与 lookup

scope 默认**不引入命名空间边界**——子 node / coordinate / 嵌套 scope.id 全部注册到当前栈顶 frame；多层 scope 串起来 id 仍汇到 `<Layout>` 根 frame，外部 path 可直接引用。`localNamespace: true` 才让 scope 推一个子 frame、隔离子树 id。

<ComponentPreview name="scope-namespace-stack" hideCode />

<Comparison target="tikz" title="命名空间默认行为">

TikZ pgf 的 `\begin{scope}` 不引入局部命名空间——scope 内 `\node (foo) {...}` 注册的 `foo` 全局可见、scope 外可直接 `\draw (foo) -- ...`。retikz 沿用此约定，让 v0.1 已写好的 IR 嵌入 scope 后无需改 id 引用即可继续工作；需要"子图复用 / 模板化"再 opt-in `localNamespace`。

</Comparison>

| 机制 | 行为 |
|---|---|
| **inside-out lookup** | 内层 path 引用 id 时从当前 frame 向栈底逐层查找——内部**可见**外部 id；外部**不可见**内部 id |
| **shadowing** | 内层 frame 注册同名 id 不算冲突、不发 warn；lookup 命中内层；frame 退出后外层 id 重新可见 |
| **scope.id 始终落父 frame** | scope.id 是外部句柄，不受自身 `localNamespace` 影响——外部 path 永远可以引用 `<Scope id="X">` 的 `X`，无论其内部是否启用本地命名空间 |
| **栈式 frame** | 每层 `<Scope localNamespace>` 都 push 一个子 frame；frame 在 scope 子树 compile 结束时 pop——超出层级的 id 引用自动断 |

### 重复 id 与冲突处理

同一 frame 内重复出现的 id（node × node、node × coordinate、node × scope.id 任意组合）触发 `DUPLICATE_NODE_ID` warning，**后定义覆盖前定义**（last-wins）。

| 规则 | 行为 |
|---|---|
| 触发 | 同 frame 内同 id 第 2 次及之后的 register |
| 处理 | warn 通过 `CompileOptions.onWarn` 收集；compile **不抛错**；nodeIndex 最终保留最后一次定义 |
| Warn 单元 | 每次重复都单独发一条 warn（不合并），含两条 IR locator（前 / 现）+ 当前 frame 深度 |
| 跨 frame 同 id | 算 shadowing 不是 duplicate，**不发 warn** |
| scope.id 自引用 | bbox 在 scope 子树完成后才能算出——`at.of='self'` / `offset.of='self'` 等自引用前向解析失败，抛错（不走 last-wins） |

### scope 下相对定位投影

scope 内 `<Node>` / `<Coordinate>` / `<Path>` `Step.to` 使用 `PolarPosition` / `AtPosition` / `OffsetPosition` 时按以下规则解析：

1. **referent 取全局**：`polar.origin` / `at.of` / `offset.of` 引用的 referent（节点 id / scope.id / 坐标字面量）取**全局**坐标
2. **relative 部分在当前 scope 局部度量**：polar 的 `(angle, radius)` / at 的 `(direction, distance)` / offset 的 `[dx, dy]` 都在**当前 node 所属 scope 局部坐标系**内表达
3. **末端 apply 当前 scope chain 投回全局**：referent 全局点先反向投影到当前 scope 局部，加 relative，再正向投影回全局

数学：`final = scopeTransform(inverseScopeTransform(referent_global) + relative_part)`

<Comparison target="tikz" title="scope 内相对定位">

对应 TikZ `\begin{scope}[rotate=90] \node at ($(hub) + (50:0)$) {orbit}; \end{scope}`——`(hub)` 引用解析为全局坐标（不受 scope rotate 影响），`+ (50:0)` 在当前 scope 局部坐标系内度量——前者全局、后者局部。retikz 复用此约定：referent 全局可见，relative 跟着 scope rotate / scale 一起旋转 / 缩放。

</Comparison>

边界规则：

| case | 行为 |
|---|---|
| **嵌套 polar / 嵌套 offset** | 整条嵌套链都在**当前 node 所属 scope 局部**度量——不切换基准 scope |
| **`Position` 笛卡尔字面量** | scope 内 `position={[10, 0]}` 与 v0.1 一致——`[10, 0]` 在当前 scope 局部度量，经 scope chain 投影到全局 |
| **scope translate 不改 relative 矢量** | translate 不旋转 / 不缩放——`relative=(10, 0)` 在 translate(50, 0) 内视觉仍是 (10, 0) 方向 / 长度 |
| **scope `scale=0`** | 反向投影未定义——retikz 退化为局部 (0, 0) 防 NaN；实践中避免 0 scale |
| **path `step.to` 内的相对定位** | 与 node position 同规则——polar / offset / `{ relative: [dx, dy] }` 字面量时，relative 部分在当前 path 所属 scope 局部度量 |

### scope.id synthetic bbox

`<Scope id="X">` 设值后，compile 在 Pass 1 子树结束时收集子树**全部** layout（node / coordinate / 嵌套 scope.id bbox）的轴对齐**全局 bounding box**，注册为一条 `rectangle` synthetic `NodeLayout`（id = `X`）到**父 namespace frame**。

| 项 | 行为 |
|---|---|
| **计算时机** | Pass 1 递归 scope 子树结束时（所有内层 node / coordinate / 嵌套 scope.id 都已 register） |
| **bbox 输入** | 子树全部 `NodeLayout` 的 4 角点（含累积 rotate / scale 后的全局坐标）求轴对齐 AABB |
| **不计入 bbox** | path 走线端点（move / line `to`）—— 与 TikZ `local bounding box` 默认一致 |
| **注册位置** | **父 frame**（不受自身 `localNamespace` 影响）——scope.id 是外部句柄 |
| **形态** | synthetic rectangle `NodeLayout`，`rect.rotate=0`（轴对齐） |
| **作为 anchor referent** | 9 个命名 anchor + 数字角度都走共享的 rect anchor 路径——`'X.north'` / `'X.30'` / 无后缀走 `boundaryPoint` clip |
| **作为 polar / at / offset referent** | 取 bbox 中心，与普通 rectangle 节点一致 |
| **嵌套** | 每层 scope.id 各算各的 bbox——外层 bbox 含全部内层 layout；内层 bbox 仅含自己子树 |
| **空 scope.id** | 子树无任何 layout 时 bbox 退化为 0×0 占位点（落在 scope 局部原点经 chain 投影到的全局点） |

<Comparison target="tikz" title="scope.id 对照">

`<Scope id="cluster">` 对应 TikZ 的 `\begin{scope}[local bounding box=cluster] ... \end{scope}`：pgf 收集 scope 内所有元素的轴对齐 bbox 并注册一个名为 `cluster` 的矩形节点。retikz 复用已有的 scope `id` 字段作为唯一入口——不另加 `boundingBox` flag——保持字段单一职责（设 id ⇒ 拿到可引用的 bbox 句柄）。

</Comparison>

### transforms 展平到 Scene

IR 6 变体在 compile 阶段统一压成 Scene 3 变体（Cartesian translate / rotate / scale），再下沉到 `GroupPrim.transforms`：

| IR 形态 | compile 处理 | Scene 形态 |
|---|---|---|
| `translate` | 直接透传 | `translate(x, y)` |
| `polar-translate` | `resolvePosition` 解出 origin 全局点 + (angle, radius) → Cartesian (x, y) | `translate(x, y)` |
| `at-translate` | `resolvePosition` 解出 referent 全局点 + direction + distance → Cartesian (x, y) | `translate(x, y)` |
| `offset-translate` | `resolvePosition` 解出 referent 全局点 + (dx, dy) → Cartesian (x, y) | `translate(x, y)` |
| `rotate` | 直接透传 | `rotate(degrees, cx, cy)` |
| `scale` | 直接透传 | `scale(x, y)` |

`transforms` 数组顺序应用——第 0 个最先生效（最内层），与 SVG `transform="t1 t2 t3"` / Scene `GroupPrim.transforms` 一致。`transforms: [translate, rotate]` 等价 SVG `transform="translate(...) rotate(...)"`：先 translate 再 rotate。

外部 nodeIndex 看到的节点 layout 已是**全局坐标**（compile 累积 chain 投影完成）；scope 内 path primitive 的落点取决于该 scope 的 transform 链：**无 transform 的 scope**，path 回填到该 scope 的 `GroupPrim.children`、保住与同层 node 的 IR 声明顺序；**带 transform 的 scope**，path 端点已是全局坐标，仍 hoist 到顶层 `primitives`（不进 GroupPrim 避免 transform 二次 apply），其跨 frame z-order 不完全等于 IR 顺序（已知限制）。

### 样式继承解析

compile 维护一个 style frame 栈，进入每层 scope 压一帧，每个元素按 **inside-out per-field** 解析最终样式。优先级链（每个分项，就近优先）：

```
元素显式分项 > 元素 color > 分类默认分项 > 分类默认 color > scope 级联分项 > scope color > 内置
```

内层整体压外层；同层分类默认压级联。

| 机制 | 行为 |
|---|---|
| **主色展开** | 每个样式来源内，分项色（stroke / fill / textColor、arrow color）未单设则取该源 `color`——同源内分项覆盖主色 |
| **颜色级联** | 容器已解析 color 下传子元素色默认：node.color → 文字 / 边 label；path.color → stroke / 箭头 / step label |
| **缺省 vs 显式** | 按存在性判定（非真假值）：缺省让位向外查；显式 `none` / `0` / `false` 本层截断 |
| **opacity 替换不复合** | 嵌套 scope opacity 各自替换（0.5 ⊃ 0.5 → 各 0.5，非 0.25），与 TikZ 默认一致；仅元素内 label × 元素 opacity 相乘 |
| **resetStyle host 轴不切** | 只切 scope 继承轴；label / arrow 仍跟宿主 path / node 已解析色，不成孤岛 |

### clip 资源物化

> **渲染无关**：裁剪区编译成 Scene 资源表里的 `ClipResource`（去重 + 稳定 id），`<clipPath>` 物化只在 SVG adapter——任何 adapter 都能复刻。
>
> **限制**：clip 只作用于 scope 的 GroupPrim 子元素；**带 transforms 的 scope 内 path** 会被 hoist 到顶层、不被该 scope 裁（既有架构）；要裁单个 Node / Path 用 `<Scope clip>` 包一层。任意贝塞尔曲线裁剪路径暂不支持。
