This group is about *drawing lines*: node to node, folds, curves, arcs, and closed fills. Reach for the Sugar `<Draw>` for everyday connections; drop to the Kernel `<Path>` + `<Step>` when you need step-by-step control or the closest match to the IR. Arrow specs get their own page and are shared by both Draw and Path.

<ComponentPreview name="draw-anatomy" size="md" hideCode />

## What each part does

| Component | Type | Responsibility | Use it when |
| --- | --- | --- | --- |
| Draw | Sugar | A way array threads targets / folds / curves / cycle / edge labels / styling, expanding to Path + Step | Everyday connections, flowchart edges, arrowed links (the default) |
| Way | Sugar | Draw's way item forms, DrawWay constants, folds, cycle, relative coordinates | Looking up what a way array can contain |
| Path | Kernel | Path container, a sequence of Step children defining a polyline / curve / arc | Multiple moves, step-by-step control, closest to the IR |
| Step | Kernel | One step = one action (move / line / curve / arc / cycle…) | Controlling each segment's path action semantics |
| Arrow | Shared | Arrows shared by Path / Draw: `arrow` sets direction, `arrowDetail` covers visual specs | Fine-grained arrow shape / size / color / per-end config |
