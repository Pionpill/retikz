`<Path>` 是底层路径容器：children 必须是若干 `<Step>`（至少 2 个），定义路径的 move / line / step（折角）/ cycle（闭合）/ curve / cubic / bend（曲线）/ arc / circlePath / ellipsePath（圆 / 椭圆）等动作序列。日常连线用 `<Draw>` 更简洁，需要折角、闭合、曲线、自定义 stroke、不同 step kind 时回到 `<Path>`。

它最适合三类场景：一条 path 里有多个子路径（多次 `move`）、不同段要混用不同 `kind` / label / 控制点、或者你希望 JSX 结构和 IR `path.children` 几乎一一对应，方便调试和生成。

## 用法

```tsx
import { Path, Step } from '@retikz/react';
```

```tsx
<Path stroke="currentColor">
  <Step kind="move" to="a" />
  <Step kind="line" to="b" />
</Path>
```

## 组合

`<Path>` 必须搭配 `<Step>` 子节点（至少 2 个）；首个非 `move` 时编译器会强制改成 `move`：

```tsx
<Path>
  <Step kind="move" to={...} />     {/* 第 1 个：必为 move */}
  <Step kind="line" to={...} />     {/* 之后任意多个 line / move / step */}
  ...
</Path>
```

`<Step>` 支持 10 种 kind——`move` / `line` 处理直线，`step` 走直角折角，`cycle` 闭合回起点，`curve` / `cubic` 画贝塞尔曲线，`bend` 是弧形简记，`arc` / `circlePath` / `ellipsePath` 以游标为圆心画弧 / 整圆 / 整椭圆，详见 [Step](/core/components/draw/step)。

## 曲线与变换

### 出 / 入射角与自环

`<Step kind="bend">` 除对称的 `bendDirection` + `bendAngle` 外，还可用 `outAngle` / `inAngle` 指定起点出射角与终点入射角，画非对称曲线（对应 TikZ `out=` / `in=`）；`looseness` 调控制点距离（松紧，缺省 ≈1）。**out/in 给定时优先于 `bendDirection`**。当 `from == to`（同节点 / 同坐标）时退化为**自环**——这是 out/in 最大价值场景（状态机自跳转），对称 bend 画不出。

<ComponentPreview name="path-outin-loop" size="md" />

### 整体变换（rotate / scale）

`rotate`（度）/ `scale`（数字等比，或 `{ x, y }` 非等比）把整条 path 绕**包围盒中心**变换，免去为单条 path 包一层 `<Scope transforms>`。变换顺序有硬契约：path 端点先 resolve、箭头 shrink 在**未变换的几何**上算，最后整体包进 group——故箭头方向随变换正确、不错位。

<ComponentPreview name="path-transform" size="md" />

### 中段标记（marks）

`marks` 沿路径放方向标记：每项 `{ pos, mark }`，`pos ∈ [0, 1]`（越界 schema 拒），`mark` 首批为 `{ kind: 'arrow', shape, ... }`（复用已注册箭头名 + 视觉子集，见 [自定义箭头](/core/reference/extending/custom-arrow)），**朝向由该点切线决定**（不是固定方向）。直线 / 贝塞尔 / 弧段都按切线定向。

## API 参考

| 属性 | 类型 | 默认值 | 描述 |
| --- | --- | --- | --- |
| `color` | `string` | — | 主色（TikZ `color=`）；`stroke` / 箭头 / step label 未单设则随它（跟主色不跟 `stroke`） |
| `stroke` | `string` | `currentColor` | 描边色 |
| `strokeWidth` | `number` | `1` | 描边宽度 |
| `dashPattern` | `Array<number>` | — | dash/gap 长度序列（如 `[4, 2]`） |
| `lineCap` | `'butt' \| 'round' \| 'square'` | `'butt'` | 描边端点形状 |
| `lineJoin` | `'miter' \| 'round' \| 'bevel'` | `'miter'` | 描边拐点形状 |
| `thickness` | `'ultraThin' \| 'veryThin' \| 'thin' \| 'semithick' \| 'thick' \| 'veryThick' \| 'ultraThick'` | — | 语义 stroke 档位（`thin = 1`）；显式 `strokeWidth` 始终优先 |
| `arrow` | `'none' \| '->' \| '<-' \| '<->'` | — | 路径级箭头方向（详见 [Arrow](/core/components/draw/arrow)） |
| `arrowDetail` | `ArrowDetail` 对象 | — | 箭头视觉详细配置；字段详见 [Arrow](/core/components/draw/arrow) |
| `fill` | `string` | `'none'` | 闭合区域填充色（任意 CSS 颜色） |
| `fillRule` | `'nonzero' \| 'evenodd'` | `'nonzero'` | 填充规则；环形 / 孔洞场景用 `'evenodd'` |
| `opacity` | `number` | — | 整 path 透明度 0~1（同时作用于 stroke 与 fill） |
| `fillOpacity` | `number` | — | 仅 fill 透明度 0~1 |
| `drawOpacity` | `number` | — | 仅 stroke 透明度 0~1 |
| `zIndex` | `number`（整数） | `0` | 显式栈序：大者在上；同值保持声明顺序；只在同层子节点间生效 |
| `rotate` | `number` | — | 整条 path 旋转角（度，绕包围盒中心） |
| `scale` | `number \| { x, y }` | — | 整条 path 缩放（数字等比 / `{ x, y }` 非等比，绕包围盒中心） |
| `marks` | `Array<{ pos, mark }>` | — | 沿路径标记：`pos∈[0,1]`、`mark.kind:'arrow'` + 已注册箭头名，朝向随该点切线 |
| `children` | `Array<<Step>>` | — | 至少 2 个；首个非 `move` 时编译器会强制改成 `move`。`<Step>` 支持 `move` / `line` / `step`（折角）/ `cycle`（闭合）/ `curve` / `cubic`（贝塞尔曲线）/ `bend`（弧形简记，可带 out/in 角与自环）/ `arc` / `circlePath` / `ellipsePath`（圆 / 椭圆） |

## 相关

### 与 Draw 的取舍

| | `<Draw>` | `<Path>` + `<Step>` |
| --- | --- | --- |
| 写法 | 一行 way 数组 | 容器 + 子节点 |
| 表达力 | 与 Path 等价（编译展开） | 全部 Kernel 能力 |
| 适用 | 90% 连线场景 | 显式 Step 序列、未来扩展 |

`<Draw>` 由 `<Path>` 编译而成；写文档 / 教程时一律先讲 `<Draw>`，`<Path>` 是给"想看底层"的用户。
