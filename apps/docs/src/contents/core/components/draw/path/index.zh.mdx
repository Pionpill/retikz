title: Path
description: Kernel 路径容器——`<Step>` 子节点序列定义折线 / 曲线 / 弧形，最少 2 个 step；Sugar 的 `<Draw>` 编译后就是它。
---

`<Path>` 是底层路径容器：children 必须是若干 `<Step>`（至少 2 个），定义路径的 move / line / step（折角）/ cycle（闭合）/ curve / cubic / bend（曲线）/ arc / circlePath / ellipsePath（圆 / 椭圆）等动作序列。日常连线用 `<Draw>` 更简洁，需要折角、闭合、曲线、自定义 stroke、不同 step kind 时回到 `<Path>`。

## 用法

```tsx
import { Path, Step } from '@retikz/react';
```

```tsx
<Path stroke="currentColor">
  <Step kind="move" to="a" />
  <Step kind="line" to="b" />
</Path>
```

## 组合

`<Path>` 必须搭配 `<Step>` 子节点（至少 2 个）；首个非 `move` 时编译器会强制改成 `move`：

```tsx
<Path>
  <Step kind="move" to={...} />     {/* 第 1 个：必为 move */}
  <Step kind="line" to={...} />     {/* 之后任意多个 line / move / step */}
  ...
</Path>
```

`<Step>` 支持 10 种 kind——`move` / `line` 处理直线，`step` 走直角折角，`cycle` 闭合回起点，`curve` / `cubic` 画贝塞尔曲线，`bend` 是弧形简记，`arc` / `circlePath` / `ellipsePath` 以游标为圆心画弧 / 整圆 / 整椭圆，详见 [Step](/core/components/draw/step)。

#### 7 种 shape

`arrowDetail.shape` 选形状，缺省 `'normal'`：

| 形状 | 描述 | 典型用途 |
| --- | --- | --- |
| `'normal'` | 实心三角 | 流程图默认 |
| `'open'` | 空心三角 | UML 泛化 / 继承 |
| `'stealth'` | 倒钩三角 | 现代流图、视觉锐利 |
| `'diamond'` | 实心菱形 | UML 组合 |
| `'openDiamond'` | 空心菱形 | UML 聚合 |
| `'circle'` | 实心圆点 | 连接点 / dot end |
| `'openCircle'` | 空心圆点 | 白圈端 / o-end |

<ComponentPreview name="path-arrow" />

> 多节点路径（如 A → B → C）按段独立 clip，但箭头**只**贴在整体起点 / 终点——中间节点不会出现额外箭头。

#### 起末分别配置

`arrowDetail.start` / `arrowDetail.end` 子对象与顶层默认**逐字段 merge**——缺省字段继承顶层，已填字段 override。常见 UML "组合 + 继承"用 `start.shape` + `end.shape` 表达：

<ComponentPreview name="path-arrow-uneven" size="xs" />

#### 颜色 override

`color` 把 marker 描边 / 实心填充改为指定颜色（脱离 `context-stroke` 继承）；`fill` 仅实心 shape 生效。`start.color` / `end.color` 单端 override 同样支持——常用于"线浅色、箭头深色"或起末异色：

<ComponentPreview name="path-arrow-color" size="sm" />

> **空心 shape silent ignore `fill`**：`shape ∈ { open / openDiamond / openCircle }` 时，`fill` 字段被静默丢弃——空心形状由 `color`（描边色） + `lineWidth`（描边粗细）共同决定视觉，写 `fill='red'` 不会渲染红色填充。

#### 缩放：scale / length / width

- `scale` 是 marker 等比缩放因子（默认 1），乘到 `length` / `width` 之后——`length=10, scale=1.5` → 实际 `markerWidth=15`
- `length` 控 marker 沿 path 方向的长度（默认 6）
- `width` 控 marker 垂直 path 方向的宽度（默认 6）

### 描边端点 / 拐点

`lineCap` 控制描边端点形状（SVG `stroke-linecap`）：`'butt'` 默认（直角端）/ `'round'` 半圆 / `'square'` 方头外扩半个 stroke。`lineJoin` 控制拐点形状（SVG `stroke-linejoin`）：`'miter'` 默认（尖角）/ `'round'` 圆角 / `'bevel'` 切角。

<ComponentPreview name="path-line-style" />

### 语义 stroke 档位

`thickness` 是 TikZ 风格的语义档位——`'ultraThin'` / `'veryThin'` / `'thin'` / `'semithick'` / `'thick'` / `'veryThick'` / `'ultraThick'`，编译期映射到具体数值（`thin = 1` 与默认 strokeWidth 对齐，两端按 TikZ 比例外推）。**显式 `strokeWidth` 始终优先**——`thickness` 仅在 `strokeWidth` 缺省时生效：

### 透明度

三个独立通道：`opacity` 整体（同时作用于 stroke 与 fill），`fillOpacity` 仅 fill，`drawOpacity` 仅 stroke（TikZ `draw opacity` 命名，与 `<Node>` 一致）。三者可叠加：

## API 参考

| 属性 | 类型 | 默认值 | 描述 |
| --- | --- | --- | --- |
| `stroke` | `string` | `currentColor` | 描边色 |
| `strokeWidth` | `number` | `1` | 描边宽度 |
| `strokeDasharray` | `string` | — | SVG `stroke-dasharray` 模式 |
| `lineCap` | `'butt' \| 'round' \| 'square'` | `'butt'` | 描边端点形状（SVG `stroke-linecap`） |
| `lineJoin` | `'miter' \| 'round' \| 'bevel'` | `'miter'` | 描边拐点形状（SVG `stroke-linejoin`） |
| `thickness` | `'ultraThin' \| 'veryThin' \| 'thin' \| 'semithick' \| 'thick' \| 'veryThick' \| 'ultraThick'` | — | 语义 stroke 档位（`thin = 1`）；显式 `strokeWidth` 始终优先 |
| `arrow` | `'none' \| '->' \| '<-' \| '<->'` | — | 路径级箭头方向 |
| `arrowDetail` | `{ shape?, scale?, length?, width?, color?, fill?, opacity?, lineWidth?, start?, end? }` | — | 箭头详细配置；顶层 8 视觉字段全 optional + 可选 `start` / `end` 子对象逐字段 merge override |
| `fill` | `string` | `'none'` | 闭合区域填充色（任意 CSS 颜色） |
| `fillRule` | `'nonzero' \| 'evenodd'` | `'nonzero'` | 填充规则；环形 / 孔洞场景用 `'evenodd'` |
| `opacity` | `number` | — | 整 path 透明度 0~1（同时作用于 stroke 与 fill） |
| `fillOpacity` | `number` | — | 仅 fill 透明度 0~1 |
| `drawOpacity` | `number` | — | 仅 stroke 透明度 0~1（TikZ `draw opacity`） |
| `children` | `Array<<Step>>` | — | 至少 2 个；首个非 `move` 时编译器会强制改成 `move`。`<Step>` 支持 `move` / `line` / `step`（折角）/ `cycle`（闭合）/ `curve` / `cubic`（贝塞尔曲线）/ `bend`（弧形简记）/ `arc` / `circlePath` / `ellipsePath`（圆 / 椭圆） |

### arrowDetail 字段

`arrowDetail` 对象 8 个视觉字段 + 可选 `start` / `end` 子对象。`start` / `end` 子对象用相同字段集（不递归），与顶层默认逐字段 merge：

| 字段 | 类型 | 默认值 | 描述 |
| --- | --- | --- | --- |
| `shape` | `'normal' \| 'open' \| 'stealth' \| 'diamond' \| 'openDiamond' \| 'circle' \| 'openCircle'` | `'normal'` | 箭头形状 |
| `scale` | `number > 0` | `1` | 等比缩放因子；乘到 `length` / `width` 之后（`length=10, scale=1.5` → 实际 15） |
| `length` | `number ≥ 0` | `6` | 尖长（user units，沿 path 方向） |
| `width` | `number ≥ 0` | `6` | 尖宽（user units，垂直 path 方向） |
| `color` | `string` | — | 描边色 override；缺省走 `context-stroke` 继承 path stroke |
| `fill` | `string` | `color` 或 `context-stroke` | 填充色 override；**空心 shape 上 silent no-op**（写了被静默丢弃） |
| `opacity` | `number ∈ [0, 1]` | — | 箭头独立透明度；缺省继承 path opacity |
| `lineWidth` | `number ≥ 0` | `1.5` | 空心 shape 描边粗细；实心 shape 忽略 |
| `start` | `ArrowEndDetail`（同字段集，无 `start` / `end`） | — | 起端 override（仅 `arrow` 含起端时生效：`'<-'` / `'<->'`） |
| `end` | `ArrowEndDetail`（同字段集，无 `start` / `end`） | — | 末端 override（仅 `arrow` 含末端时生效：`'->'` / `'<->'`） |

> **merge 语义**：`start` / `end` 子对象的字段优先；子对象未填的字段 fallback 到顶层默认；顶层未填的字段 fallback 到内置默认。例：`arrowDetail={{ shape: 'stealth', start: { color: 'red' } }}` → 起末都用 stealth、起端用红色、末端继承 path stroke。

## 与 Draw 的取舍

| | `<Draw>` | `<Path>` + `<Step>` |
| --- | --- | --- |
| 写法 | 一行 way 数组 | 容器 + 子节点 |
| 表达力 | 与 Path 等价（编译展开） | 全部 Kernel 能力 |
| 适用 | 90% 连线场景 | 显式 Step 序列、未来扩展 |

`<Draw>` 由 `<Path>` 编译而成；写文档 / 教程时一律先讲 `<Draw>`，`<Path>` 是给"想看底层"的用户。
