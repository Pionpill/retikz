`<Path>` is the low-level path container: children must be `<Step>` nodes (at least 2) defining a sequence of move / line / step (folded) / cycle (close) / curve / cubic / bend (curves) / arc / circlePath / ellipsePath (circle / ellipse) actions. For everyday connections `<Draw>` is shorter; reach for `<Path>` when you need folds, closes, curves, custom stroke, or different step kinds.

## Usage

```tsx
import { Path, Step } from '@retikz/react';
```

```tsx
<Path stroke="currentColor">
  <Step kind="move" to="a" />
  <Step kind="line" to="b" />
</Path>
```

## Composition

`<Path>` requires `<Step>` children (at least 2); if the first isn't `move`, the compiler coerces it to `move`:

```tsx
<Path>
  <Step kind="move" to={...} />     {/* First: must be move */}
  <Step kind="line" to={...} />     {/* Then any number of line / move / step */}
  ...
</Path>
```

`<Step>` supports ten kinds — `move` / `line` for straight segments, `step` for the right-angle fold, `cycle` to close back, `curve` / `cubic` for Bezier curves, `bend` as the arc shorthand, and `arc` / `circlePath` / `ellipsePath` for arcs / full circles / full ellipses centered on the cursor. See [Step](/core/components/draw/step) for details.

#### 7 shapes

`arrowDetail.shape` picks the tip shape (default `'normal'`):

| Shape | Description | Typical use |
| --- | --- | --- |
| `'normal'` | Filled triangle | Flow chart default |
| `'open'` | Hollow triangle | UML generalization / inheritance |
| `'stealth'` | Sharp barbed triangle | Modern flow charts |
| `'diamond'` | Filled diamond | UML composition |
| `'openDiamond'` | Hollow diamond | UML aggregation |
| `'circle'` | Filled circle | Connection / dot end |
| `'openCircle'` | Hollow circle | White circle / o-end |

<ComponentPreview name="path-arrow" />

> Multi-node paths (e.g. A → B → C) clip per-segment, but the arrow only attaches to the overall **start** / **end** — never to intermediate nodes.

#### Per-end overrides

`arrowDetail.start` / `arrowDetail.end` sub-objects **merge field-by-field** into the top-level defaults — omitted fields inherit, present fields override. The classic UML "composition + inheritance" shows up as different `start.shape` + `end.shape`:

<ComponentPreview name="path-arrow-uneven" size="xs" />

#### Color override

`color` overrides the marker stroke / solid fill (escaping the default `context-stroke` inheritance); `fill` is only effective on solid shapes. `start.color` / `end.color` give per-end overrides — handy for "light line, dark arrow" or two-color endpoints:

<ComponentPreview name="path-arrow-color" size="sm" />

> **Hollow shapes silently ignore `fill`**: when `shape ∈ { open / openDiamond / openCircle }`, the `fill` field is silently dropped. Hollow shapes are driven entirely by `color` (stroke) + `lineWidth` (stroke width); writing `fill='red'` will not render a red fill.

#### Sizing: scale / length / width

- `scale` is the uniform marker scale factor (default 1), multiplied **after** `length` / `width` — `length=10, scale=1.5` → effective `markerWidth=15`
- `length` controls the marker dimension along the path direction (default 6)
- `width` controls the marker dimension perpendicular to the path (default 6)

<ComponentPreview name="path-arrow-scale" size="md" />

### Stroke endpoints / corners

`lineCap` controls the stroke endpoint shape (SVG `stroke-linecap`): `'butt'` default (square end at endpoint) / `'round'` half-circle cap / `'square'` square cap extended past the endpoint by half a stroke. `lineJoin` controls the corner shape (SVG `stroke-linejoin`): `'miter'` default (sharp) / `'round'` rounded / `'bevel'` flat-cut.

### Semantic thickness preset

`thickness` is a TikZ-style semantic preset — `'ultraThin'` / `'veryThin'` / `'thin'` / `'semithick'` / `'thick'` / `'veryThick'` / `'ultraThick'` — compiled to a numeric stroke width (`thin = 1` matches the default `strokeWidth`; the rest scale by TikZ's ratios). **Explicit `strokeWidth` always wins** — `thickness` only applies when `strokeWidth` is omitted:

## API Reference

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `stroke` | `string` | `currentColor` | Stroke color |
| `strokeWidth` | `number` | `1` | Stroke width |
| `strokeDasharray` | `string` | — | SVG `stroke-dasharray` pattern |
| `lineCap` | `'butt' \| 'round' \| 'square'` | `'butt'` | Stroke endpoint shape (SVG `stroke-linecap`) |
| `lineJoin` | `'miter' \| 'round' \| 'bevel'` | `'miter'` | Stroke corner shape (SVG `stroke-linejoin`) |
| `thickness` | `'ultraThin' \| 'veryThin' \| 'thin' \| 'semithick' \| 'thick' \| 'veryThick' \| 'ultraThick'` | — | Semantic stroke preset (`thin = 1`); explicit `strokeWidth` always wins |
| `arrow` | `'none' \| '->' \| '<-' \| '<->'` | — | Path-level arrow direction |
| `arrowDetail` | `{ shape?, scale?, length?, width?, color?, fill?, opacity?, lineWidth?, start?, end? }` | — | Detailed arrow config: top-level visual fields act as shared defaults; optional `start` / `end` sub-objects merge field-by-field |
| `fill` | `string` | `'none'` | Fill color of the closed region (any CSS color) |
| `fillRule` | `'nonzero' \| 'evenodd'` | `'nonzero'` | Fill rule; use `'evenodd'` for rings / holes |
| `opacity` | `number` | — | Whole-path opacity 0..1 (multiplies onto stroke and fill) |
| `fillOpacity` | `number` | — | Fill-only opacity 0..1 |
| `drawOpacity` | `number` | — | Stroke-only opacity 0..1 (TikZ `draw opacity`) |
| `children` | `Array<<Step>>` | — | At least 2; if the first isn't `move`, the compiler coerces it to `move`. `<Step>` supports `move` / `line` / `step` (folded) / `cycle` (close) / `curve` / `cubic` (Bezier) / `bend` (arc shorthand) / `arc` / `circlePath` / `ellipsePath` (circle / ellipse) |

### arrowDetail fields

The `arrowDetail` object has 8 visual fields plus optional `start` / `end` sub-objects. The sub-objects share the same field set (no recursion) and merge field-by-field into the top-level defaults:

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `shape` | `'normal' \| 'open' \| 'stealth' \| 'diamond' \| 'openDiamond' \| 'circle' \| 'openCircle'` | `'normal'` | Arrow tip shape |
| `scale` | `number > 0` | `1` | Uniform scale factor; multiplied after `length` / `width` (`length=10, scale=1.5` → effective 15) |
| `length` | `number ≥ 0` | `6` | Tip length (user units, along the path) |
| `width` | `number ≥ 0` | `6` | Tip width (user units, perpendicular to the path) |
| `color` | `string` | — | Stroke color override; omitted = `context-stroke` (inherits path stroke) |
| `fill` | `string` | `color` or `context-stroke` | Fill color override; **silent no-op on hollow shapes** (silently dropped when written) |
| `opacity` | `number ∈ [0, 1]` | — | Arrow-only opacity; omitted = inherits path opacity |
| `lineWidth` | `number ≥ 0` | `1.5` | Stroke width on hollow shapes; ignored on solid shapes |
| `start` | `ArrowEndDetail` (same field set minus `start` / `end`) | — | Start-end override (effective only when `arrow` includes start: `'<-'` / `'<->'`) |
| `end` | `ArrowEndDetail` (same field set minus `start` / `end`) | — | End-end override (effective only when `arrow` includes end: `'->'` / `'<->'`) |

> **Merge semantics**: `start` / `end` sub-object fields win first; missing sub-object fields fall back to the top-level defaults; missing top-level fields fall back to built-in defaults. e.g. `arrowDetail={{ shape: 'stealth', start: { color: 'red' } }}` → both ends use stealth, start end is red, end end inherits the path stroke.

## Path vs Draw

| | `<Draw>` | `<Path>` + `<Step>` |
| --- | --- | --- |
| Form | Single-line `way` array | Container + child nodes |
| Expressiveness | Equivalent to Path (compile-expanded) | All Kernel capability |
| Use when | 90% of connection cases | Explicit Step sequence, future extensions |

`<Draw>` compiles down to `<Path>`; tutorials should lead with `<Draw>` — `<Path>` is for users who want the underlying construct.
