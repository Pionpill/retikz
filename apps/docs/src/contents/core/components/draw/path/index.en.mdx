`<Path>` is the low-level path container: children must be `<Step>` nodes (at least 2) defining a sequence of move / line / step (folded) / cycle (close) actions. For everyday connections `<Draw>` is shorter; reach for `<Path>` when you need folds, closes, custom stroke, multi-segment polylines, or different step kinds.

## Usage

```tsx
import { Path, Step } from '@retikz/react';
```

```tsx
<Path stroke="currentColor">
  <Step kind="move" to="a" />
  <Step kind="line" to="b" />
</Path>
```

## Composition

`<Path>` requires `<Step>` children (at least 2); if the first isn't `move`, the compiler coerces it to `move`:

```tsx
<Path>
  <Step kind="move" to={...} />     {/* First: must be move */}
  <Step kind="line" to={...} />     {/* Then any number of line / move / step */}
  ...
</Path>
```

`<Step>` supports four kinds — `move` / `line` / `step` / `cycle` — the first two draw straight segments, `step` folds through a right angle, `cycle` closes back to the start. See [Step](/core/components/draw/step) for details.

### Filling

`fill` colors the closed region (default `'none'` — stroke only, backward compatible); `fillRule` controls how self-intersecting / nested sub-paths fill (`'nonzero'` default / `'evenodd'` for ring & hole shapes). Pairs with `cycle`:

### Arrows

`arrow` picks direction (`'->'` end / `'<-'` start / `'<->'` both / `'none'` or omitted), `arrowShape` picks the tip shape (default `'normal'`):

| Shape | Description | Typical use |
| --- | --- | --- |
| `'normal'` | Filled triangle | Flow chart default |
| `'open'` | Hollow triangle | UML generalization / inheritance |
| `'stealth'` | Sharp barbed triangle | Modern flow charts |
| `'diamond'` | Filled diamond | UML composition |
| `'openDiamond'` | Hollow diamond | UML aggregation |
| `'circle'` | Filled circle | Connection / dot end |
| `'openCircle'` | Hollow circle | White circle / o-end |

<ComponentPreview name="path-arrow" />

> Multi-node paths (e.g. A → B → C) clip per-segment, but the arrow only attaches to the overall **start** / **end** — never to intermediate nodes. The marker uses `context-stroke` so its color tracks the stroke; size scales with `strokeWidth`.

## API Reference

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `stroke` | `string` | `currentColor` | Stroke color |
| `strokeWidth` | `number` | `1` | Stroke width |
| `strokeDasharray` | `string` | — | SVG `stroke-dasharray` pattern |
| `arrow` | `'none' \| '->' \| '<-' \| '<->'` | — | Path-level arrow direction |
| `arrowShape` | `'normal' \| 'open' \| 'stealth' \| 'diamond' \| 'openDiamond' \| 'circle' \| 'openCircle'` | `'normal'` | Arrow tip shape |
| `fill` | `string` | `'none'` | Fill color of the closed region (any CSS color) |
| `fillRule` | `'nonzero' \| 'evenodd'` | `'nonzero'` | Fill rule; use `'evenodd'` for rings / holes |
| `children` | `Array<<Step>>` | — | At least 2; if the first isn't `move`, the compiler coerces it to `move`. `<Step>` supports `move` / `line` / `step` (folded) / `cycle` (close) |

## Path vs Draw

| | `<Draw>` | `<Path>` + `<Step>` |
| --- | --- | --- |
| Form | Single-line `way` array | Container + child nodes |
| Expressiveness | Equivalent to Path (compile-expanded) | All Kernel capability |
| Use when | 90% of connection cases | Explicit Step sequence, future extensions |

`<Draw>` compiles down to `<Path>`; tutorials should lead with `<Draw>` — `<Path>` is for users who want the underlying construct.
