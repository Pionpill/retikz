Arrows are driven by two independent props:

- **`arrow`** ŌĆö direction enum: `'->'` end / `'<-'` start / `'<->'` both / `'none'` (default) = none
- **`arrowDetail`** ŌĆö visual detail object, 8 top-level fields + optional `start` / `end` sub-objects. **Omit it for the default "stealth sharp barb inheriting the path stroke / opacity" behavior**

Both props sit on `<Draw>` and `<Path>` with identical syntax and effect ŌĆö the examples below use `<Draw>`; the `<Path>` equivalent is the same.

An arrow attaches to a path endpoint: it does not change the original path geometry, and it is not repeated at intermediate nodes. Very short segments, two-ended arrows, or large `length` / `width` values can overlap visually; reduce the arrow size or create room with node `margin` / explicit coordinates.

## Usage

<ComponentPreview name="arrow-default" size="xs" />

```tsx
<Draw way={['a', 'b']} arrow="->" />
```

With `arrowDetail` omitted, the arrow defaults to a stealth tip inheriting the path stroke color and using 6 user units for both length and width.

## Examples

### Shapes

`arrowDetail.shape` picks the tip shape (default `'stealth'`):

| Solid | Hollow | Typical use |
| --- | --- | --- |
| `'normal'` filled triangle | `'open'` hollow triangle | Basic triangle / UML generalization |
| `'stealth'` sharp barb | - | Default arrow, modern flow charts |
| `'diamond'` filled diamond | `'openDiamond'` hollow diamond | UML composition / aggregation |
| `'circle'` filled dot | `'openCircle'` hollow dot | Connection dot / open endpoint |

<ComponentPreview name="arrow-shapes" />

> Multi-node paths (e.g. a ŌåÆ b ŌåÆ c) clip per-segment, but the arrow only attaches to the overall **start** / **end** ŌĆö never to intermediate nodes.

### Per-end overrides

`arrowDetail.start` / `arrowDetail.end` sub-objects **merge field-by-field** into the top-level defaults ŌĆö omitted fields inherit, present fields override. The classic UML "composition + inheritance" shows up as different `start.shape` + `end.shape`:

<ComponentPreview name="arrow-uneven" size="xs" />

> **Merge semantics**: sub-object fields win first; fields not in the sub-object fall back to the top-level defaults; fields not at the top level fall back to the built-in defaults. Example: `arrowDetail={{ shape: 'stealth', start: { color: 'red' } }}` ŌåÆ both ends use stealth, start in red, end inherits the path stroke.

### Style

#### Color override

`color` overrides the arrow stroke / solid fill instead of inheriting the path stroke color; `fill` is only effective on solid shapes. `start.color` / `end.color` give per-end overrides ŌĆö handy for "light line, dark arrow" or two-color endpoints:

<ComponentPreview name="arrow-color" size="xs" />

> **Hollow shapes silently ignore `fill`**: when `shape Ōłł { open / openDiamond / openCircle }`, the `fill` field is silently dropped. Hollow shapes are driven entirely by `color` (stroke) + `lineWidth` (stroke width); writing `fill='red'` will not render a red fill.

#### Sizing: scale / length / width

- `scale` is the uniform arrow scale factor (default 1), multiplied **after** `length` / `width` ŌĆö `length=10, scale=1.5` ŌåÆ effective arrow length 15
- `length` controls the arrow dimension along the path direction (default 6)
- `width` controls the arrow dimension perpendicular to the path (default 6)

<ComponentPreview name="arrow-scale" size="sm" />

#### Translucent

`opacity` 0..1 applies to the whole arrow (independent of the path's own `opacity` / `drawOpacity`):

## API Reference

### arrow

| Property | Type | Default | Description |
| --- | --- | --- | --- |
| `arrow` | `'none' \| '->' \| '<-' \| '<->'` | ŌĆö | Path-level arrow direction |
| `arrowDetail` | `{ shape?, scale?, length?, width?, color?, fill?, opacity?, lineWidth?, start?, end? }` | ŌĆö | Arrow visual config; 8 top-level fields all optional + optional `start` / `end` sub-objects that merge field-by-field |

### arrowDetail fields

`arrowDetail` carries 8 visual fields + optional `start` / `end` sub-objects. The `start` / `end` sub-objects use the same field set (non-recursive) and merge field-by-field into the top-level defaults:

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `shape` | `'normal' \| 'open' \| 'stealth' \| 'diamond' \| 'openDiamond' \| 'circle' \| 'openCircle' \| string` | `'stealth'` | Arrow tip shape; one of the 7 built-ins or a custom arrow name registered via `<Layout arrows>` / `CompileOptions.arrows` |
| `scale` | `number > 0` | `1` | Uniform scale factor; multiplied **after** `length` / `width` (`length=10, scale=1.5` ŌåÆ effective 15) |
| `length` | `number Ōēź 0` | `6` | Arrow length along the path direction (user units) |
| `width` | `number Ōēź 0` | `6` | Arrow width perpendicular to the path (user units) |
| `color` | `string` | ŌĆö | Stroke color override; default inherits the path stroke color |
| `fill` | `string` | `color` or path stroke color | Fill color override; **silent no-op on hollow shapes** (written value is dropped) |
| `opacity` | `number Ōłł [0, 1]` | ŌĆö | Arrow-only opacity; default inherits path opacity |
| `lineWidth` | `number Ōēź 0` | `1.5` | Stroke width for hollow shapes; ignored on solid shapes |
| `start` | `ArrowEndDetail` (same field set, no `start` / `end`) | ŌĆö | Start-end override (only effective when `arrow` includes the start: `'<-'` / `'<->'`) |
| `end` | `ArrowEndDetail` (same field set, no `start` / `end`) | ŌĆö | End-end override (only effective when `arrow` includes the end: `'->'` / `'<->'`) |
