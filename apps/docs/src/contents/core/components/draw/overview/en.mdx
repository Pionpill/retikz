---
title: Draw
description: Sugar component — way array threads through ids / Cartesian / polar / fold targets, expands at compile time to Path + Step.
---

`<Draw>` is the go-to for 90% of connections. `way` is an array of targets joined as a path; at compile time it expands to an equivalent Kernel `<Path>` + `<Step>` tree, so `<Draw>` adds no capability the Kernel can't already express.

## Usage

```tsx
import { Draw } from '@retikz/react';
```

```tsx
<Draw way={['a', 'b']} />
```

### Four forms of way item

Each item in `way` accepts four forms — the first three are direct targets (matching `<Step>.to`), the fourth is a fold object.

| Form | Example | Meaning |
| --- | --- | --- |
| Node id | `'a'` | Reference a Node in the same Tikz; endpoint auto-attaches to border |
| Cartesian | `[120, 0]` | Literal `[x, y]` |
| Polar | `{ origin: 'a', angle: 60, radius: 80 }` | Polar (see [Positioning](/core/concepts/positioning)) |
| Fold | `{ via: '-\|' \| '\|-', to: target }` | Right-angle fold segment (TikZ `-\|` / `\|-`) |

### Folded segments

## API Reference

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `way` | `Array<WayItem>` | — | At least 2 items; first is always `move`, rest are `line` or `step` (fold) depending on item form |
| `stroke` | `string` | `currentColor` | Stroke color |
| `strokeWidth` | `number` | `1` | Stroke width |
| `strokeDasharray` | `string` | — | SVG `stroke-dasharray` pattern (e.g. `"4 2"`) |
