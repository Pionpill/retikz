`<Draw>` 是 90% 用户连线的首选。`way` 是一个目标点数组，依次串成路径；编译期它会展开为一组等价的 Kernel `<Path>` + `<Step>`，所以 `<Draw>` 不引入任何 Kernel 表达不了的能力。

## 用法

```tsx
import { Draw } from '@retikz/react';
```

```tsx
<Draw way={['a', 'b']} />
```

### way 的 4 种 item

`way` 数组里的每一项接受四种形态：前三种是直接 target（与 `<Step>.to` 一致），第四种是折角对象。

| 形态 | 例 | 含义 |
| --- | --- | --- |
| 节点 id | `'a'` | 引用同一 Tikz 内的 Node；端点自动贴 border |
| 笛卡尔 | `[120, 0]` | 直接给坐标 |
| 极坐标 | `{ origin: 'a', angle: 60, radius: 80 }` | 极坐标（详见 [定位](/core/concepts/positioning)） |
| 折角 | `{ via: '-\|' \| '\|-', to: target }` | 直角拐点段（TikZ `-\|` / `\|-`） |

## API 参考

| 属性 | 类型 | 默认值 | 描述 |
| --- | --- | --- | --- |
| `way` | `Array<WayItem>` | — | 至少 2 个点；首项总是 `move`，其余按 item 形态决定 `line` 或 `step`（折角） |
| `stroke` | `string` | `currentColor` | 描边色 |
| `strokeWidth` | `number` | `1` | 描边宽度 |
| `strokeDasharray` | `string` | — | SVG `stroke-dasharray` 模式（如 `"4 2"`） |
