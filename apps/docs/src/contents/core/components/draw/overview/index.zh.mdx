---
title: Draw
description: Sugar 组件——用 way 数组串起节点、坐标、折角、曲线、闭合、边标注和样式，编译期展开为等价的 Path + Step。
---

`<Draw>` 是大多数连线的首选入口。它接受一个 `way` 数组，按顺序把目标点与 infix 算子展开成 Kernel `<Path>` + `<Step>`，所以它不引入新的底层能力，只减少常见路径的书写成本。

## 用法

```tsx
import { Draw } from '@retikz/react';
```

```tsx
<Draw way={['a', 'b']} />
```

## API 参考

| 属性 | 类型 | 默认值 | 描述 |
| --- | --- | --- | --- |
| `way` | `Array<WayItem>` | — | 至少 2 个点；首项总是 `move`，其余按 item 形态决定 `line` / `step` / `cycle` 等动作 |
| `stroke` | `string` | `currentColor` | 描边色 |
| `strokeWidth` | `number` | `1` | 描边宽度 |
| `dashPattern` | `Array<number>` | — | dash / gap 长度序列，如 `[4, 2]` |
| `lineCap` | `'butt' \| 'round' \| 'square'` | `'butt'` | 描边端点形状（TikZ `line cap`） |
| `lineJoin` | `'miter' \| 'round' \| 'bevel'` | `'miter'` | 描边拐点形状（TikZ `line join`） |
| `thickness` | `'ultraThin' \| 'veryThin' \| 'thin' \| 'semithick' \| 'thick' \| 'veryThick' \| 'ultraThick'` | — | TikZ 风格语义 stroke 档位；显式 `strokeWidth` 优先 |
| `arrow` | `'none' \| '->' \| '<-' \| '<->'` | — | 路径级箭头方向 |
| `arrowDetail` | `{ shape?, scale?, length?, width?, color?, fill?, opacity?, lineWidth?, start?, end? }` | — | 箭头详细配置；详见 [Arrow](/core/components/draw/arrow) |
| `fill` | `string` | `'none'` | 闭合区域填充色 |
| `fillRule` | `'nonzero' \| 'evenodd'` | `'nonzero'` | 填充规则 |
| `opacity` | `number` | — | 整 path 透明度 0..1 |
| `fillOpacity` | `number` | — | 仅 fill 透明度 0..1 |
| `drawOpacity` | `number` | — | 仅 stroke 透明度 0..1（TikZ `draw opacity`） |
