title: Draw
description: Sugar component — way array threads through ids / Cartesian / polar / fold / cycle / curves (curve / cubic / bend) items, expands at compile time to Path + Step.
---

`<Draw>` is the go-to for 90% of connections. `way` is an array of targets (with infix operators) joined as a path; at compile time it expands to an equivalent Kernel `<Path>` + `<Step>` tree, so `<Draw>` adds no capability the Kernel can't already express.

## Usage

```tsx
import { Draw } from '@retikz/react';
```

```tsx
<Draw way={['a', 'b']} />
```

### Eight forms of way item

Each item in `way` accepts eight forms — the first three are direct targets (matching `<Step>.to`), the rest are infix operators / keywords.

| Form | Example | Meaning |
| --- | --- | --- |
| Node id | `'a'` | Reference a Node in the same Tikz; endpoint auto-attaches to border |
| Cartesian | `[120, 0]` | Literal `[x, y]` |
| Polar | `{ origin: 'a', angle: 60, radius: 80 }` | Polar (see [Positioning](/core/concepts/positioning)) |
| Fold operator | `'-\|'` / `'\|-'` (or `DrawWay.hv` / `DrawWay.vh`) | Turn the "previous → next" segment into a fold (TikZ `-\|` / `\|-` infix) |
| Cycle | `DrawWay.cycle` | Close back to way's first item (equivalent to TikZ `cycle` / SVG `Z`) |
| Quadratic curve | `{ curve: [cx, cy] }` | Merge with next item into a curve step; one control point (TikZ `..controls(B)..`) |
| Cubic curve | `{ cubic: [[c1x, c1y], [c2x, c2y]] }` | Merge with next item into a cubic step; two control points (TikZ `..controls(B) and (C)..`) |
| Bend | `{ bend: 'left' \| 'right', angle?: number }` | Merge with next item into a bend step; direction + optional angle (default 30°) |

### DrawWay keyword constants

`DrawWay` exposes named constants for special way-array items:

```tsx
import { Draw, DrawWay } from '@retikz/react';

<Draw way={['a', 'b', 'c', DrawWay.cycle]} />            {/* close */}
<Draw way={['a', '-|', 'b']} />                          {/* fold via raw literal */}
<Draw way={['a', DrawWay.hv, 'b']} />                    {/* fold via constant */}
```

| Constant | Raw literal | Meaning |
| --- | --- | --- |
| `DrawWay.cycle` | (**const-only**) | Close keyword. The underlying string is intentionally ugly (`'retikz-keyword_cycle'`) so it can never collide with a node id. |
| `DrawWay.hv` | `'-\|'` | Fold: horizontal-then-vertical (infix operator) |
| `DrawWay.vh` | `'\|-'` | Fold: vertical-then-horizontal (infix operator) |

**Why cycle bans the raw literal**: `'cycle'` is a common English word, very likely to collide with a real node id. Cycle therefore goes **only** through `DrawWay.cycle`; a bare string `'cycle'` in `way` is just a node id reference.

The fold values `'-\|'` / `'\|-'` contain reserved punctuation, so they cannot collide with node ids — **both raw literal and constant work**, pick whichever fits.

### Folded segments

Drop `'-|'` or `'|-'` directly into `way` (infix operator) and the "previous → next" segment becomes a fold — matches TikZ's `(A) -| (B)` form, equivalent to `<Step kind="step" via to>`. Essential for flow charts and UML relationships:

<ComponentPreview name="draw-fold" />

### Closing the loop

End `way` with `DrawWay.cycle` to close the last segment back to the first item — matches TikZ's native form `(A) -- (B) -- cycle`, equivalent to `<Step kind="cycle" />`:

<ComponentPreview name="draw-cycle" />

### Quadratic curve

Drop `{ curve: [cx, cy] }` (infix operator) into `way` and the "previous → next" segment becomes a quadratic Bezier — matching TikZ's `..controls(B)..`, equivalent to `<Step kind="curve">`:

<ComponentPreview name="draw-curve" />

### Cubic curve

`{ cubic: [[c1x, c1y], [c2x, c2y]] }` draws a cubic Bezier — two control points shape the start and end tangents independently, matching TikZ's `..controls(B) and (C)..`, equivalent to `<Step kind="cubic">`. Common for S-shaped or complex curves:

<ComponentPreview name="draw-cubic" />

### Bend (arc shorthand)

`{ bend: 'left' | 'right', angle?: number }` is the easier curve form — give a direction and angle (default 30°), control points are computed at compile time and lowered to a cubic. Equivalent to `<Step kind="bend">`:

<ComponentPreview name="draw-bend" />

### Arrows

The `arrow` prop controls path-end arrows: `'->'` / `'<-'` / `'<->'` / `'none'` (default). For multi-node paths, the arrow attaches only to the overall start / end — never on intermediate nodes:

<ComponentPreview name="draw-arrow" />

### Filling

`fill` colors the closed region (default `'none'` — stroke only); `fillRule` controls how nested sub-paths fill. Pairs with `DrawWay.cycle` at the end of `way` to draw filled shapes:

## API Reference

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `way` | `Array<WayItem>` | — | At least 2 items; first is always `move`, rest are `line` / `step` (fold) / `cycle` (close) depending on item form |
| `stroke` | `string` | `currentColor` | Stroke color |
| `strokeWidth` | `number` | `1` | Stroke width |
| `strokeDasharray` | `string` | — | SVG `stroke-dasharray` pattern (e.g. `"4 2"`) |
| `arrow` | `'none' \| '->' \| '<-' \| '<->'` | — | Path-level arrow direction |
| `arrowShape` | `'normal' \| 'open' \| 'stealth' \| 'diamond' \| 'openDiamond' \| 'circle' \| 'openCircle'` | `'normal'` | Arrow tip shape |
| `fill` | `string` | `'none'` | Fill color of the closed region |
| `fillRule` | `'nonzero' \| 'evenodd'` | `'nonzero'` | Fill rule |
