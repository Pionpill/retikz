---
title: Draw
description: Sugar component — a way array threads through nodes, coordinates, folds, curves, close operations, edge labels, and styling, then expands to equivalent Path + Step.
---

`<Draw>` is the default entry point for most connections. It accepts a `way` array and expands targets, folds, cycles, curves, shapes, and edge labels into Kernel `<Path>` + `<Step>`. Stroke, fill, arrow, and opacity props match `<Path>`.

Choosing rule: use `<Draw>` for ordinary relationship lines, flowchart edges, and arrowed connections; switch to `<Path>` + `<Step>` when you need multiple `move`s, precise per-segment `kind` control, or IR-close debugging; work with IR directly only at AI / persistence boundaries.

## Usage

```tsx
import { Draw } from '@retikz/react';
```

```tsx
<Draw way={['a', 'b']} />
```

### Edge Labels

#### Basic Form

`{ label }` is an infix label operator in Draw way. It decorates the next produced segment by setting `step.label`; it does not produce a segment itself. The shorthand `{ label: 'ok' }` is equivalent to `{ label: { text: 'ok' } }`.

<ComponentPreview name="draw-label" size="lg" />

#### Position Rules

`label.position` accepts seven keywords or a numeric `0..1` t. The geometry of t depends on step kind: line / fold use normalized path position, curve / cubic / bend use Bezier t, and arc / circle / ellipse use angular parametrization.

### Style

#### Arrows

`arrow` controls endpoint arrows: `'->'` / `'<-'` / `'<->'` / `'none'`. For shape, size, color, and per-end overrides, see [Arrow](/core/components/draw/arrow).

#### Fill

`fill` colors closed regions, and `fillRule` controls nested sub-path filling. It is commonly used with `DrawWay.Cycle`.

#### Dash

`dashPattern` uses `Array<number>` for dash / gap lengths. `[4, 2]` means draw 4, skip 2, in user units.

## API Reference

### Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `way` | `Array<WayItem>` | — | At least 2 items; first is always `move`, rest become `line` / `step` / `cycle` and related actions based on item form |
| `stroke` | `string` | `currentColor` | Stroke color |
| `strokeWidth` | `number` | `1` | Stroke width |
| `dashPattern` | `Array<number>` | — | Dash / gap length sequence, for example `[4, 2]` |
| `lineCap` | `'butt' \| 'round' \| 'square'` | `'butt'` | Stroke endpoint shape |
| `lineJoin` | `'miter' \| 'round' \| 'bevel'` | `'miter'` | Stroke corner shape |
| `thickness` | `'ultraThin' \| 'veryThin' \| 'thin' \| 'semithick' \| 'thick' \| 'veryThick' \| 'ultraThick'` | — | Semantic stroke preset; explicit `strokeWidth` wins |
| `arrow` | `'none' \| '->' \| '<-' \| '<->'` | — | Path-level arrow direction |
| `arrowDetail` | `{ shape?, scale?, length?, width?, color?, fill?, opacity?, lineWidth?, start?, end? }` | — | Detailed arrow configuration; see [Arrow](/core/components/draw/arrow) |
| `fill` | `string` | `'none'` | Fill color for closed regions |
| `fillRule` | `'nonzero' \| 'evenodd'` | `'nonzero'` | Fill rule |
| `opacity` | `number` | — | Whole-path opacity, 0..1 |
| `fillOpacity` | `number` | — | Fill-only opacity, 0..1 |
| `drawOpacity` | `number` | — | Stroke-only opacity, 0..1 |

<Comparison target="tikz" title="Style Naming">

`lineCap` / `lineJoin` map to TikZ `line cap` / `line join`; `drawOpacity` maps to `draw opacity`; `thickness` presets reuse semantic stroke-width names from `ultra thin` through `ultra thick`.

</Comparison>

### Curve / Shape / Label Way Items

| Way item | Parameters | Consumes next item | Equivalent Step | Description |
| --- | --- | --- | --- | --- |
| `{ curve }` | `[cx, cy]` | Yes | `kind="curve"` | Quadratic Bezier control point |
| `{ cubic }` | `[[c1x, c1y], [c2x, c2y]]` | Yes | `kind="cubic"` | Two cubic Bezier control points |
| `{ bend }` | `'left' \| 'right'` | Yes | `kind="bend"` | Bend direction, default angle 30 |
| `{ bend, angle }` | `number` | Yes | `kind="bend"` | Override the bend angle |
| `{ arc }` | `{ startAngle, endAngle, radius }` | No | `kind="arc"` | Draw an arc around the current point |
| `{ circle }` | `{ radius }` | No | `kind="circlePath"` | Draw a full circle around the current point |
| `{ ellipse }` | `{ radiusX, radiusY }` | No | `kind="ellipsePath"` | Draw a full ellipse centered on the current point |
| `{ label }` | `string \| StepLabel` | Yes | `step.label` | Decorates the next segment; string shorthand equals `{ text }` |

### StepLabel

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `text` | `string` | — | Label text |
| `position` | `StepLabelPosition \| number` | `'midway'` | Keyword or numeric `0..1` t value |
| `side` | `'above' \| 'below'` | `'above'` | Which side of the line to place the text on |
