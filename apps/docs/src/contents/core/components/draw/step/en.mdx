`<Step>` describes a single path action: `move` lifts the pen to the target, `line` draws a straight line from cursor to target, `step` folds through a single right-angle corner. It is **only** valid as a direct child of `<Path>`; placed elsewhere it is silently ignored.

## Usage

```tsx
import { Path, Step } from '@retikz/react';
```

```tsx
<Path>
  <Step kind="move" to="a" />
  <Step kind="line" to="b" />
</Path>
```

## API Reference

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `kind` | `'move' \| 'line' \| 'step'` | `'line'` | `'move'` lifts the pen; `'step'` folds through a corner |
| `via` | `'-\|' \| '\|-'` | â€” | Required for `kind="step"`: `-\|` horizontal-then-vertical; `\|-` vertical-then-horizontal |
| `to` | `IRTarget` | â€” | Target: node id, Cartesian `[x, y]`, or polar object |

## Three kinds

| kind | Behavior | SVG / TikZ analogue |
| --- | --- | --- |
| `'move'` | Move cursor, **no draw** | path "M" |
| `'line'` | Straight line from cursor to target | path "L" / TikZ `--` |
| `'step'` | Right-angle fold through one corner | TikZ `-\|` / `\|-` |

The first step of a `<Path>` must be `move` â€” if you write `'line'` or `'step'`, the compiler coerces it to `move`.

## Two via values

Only meaningful when `kind="step"`. Given cursor `(px, py)` and target `(tx, ty)`:

| via | Corner | Path |
| --- | --- | --- |
| `'-\|'` | `(tx, py)` | Horizontal to `tx`, then vertical to `ty` |
| `'\|-'` | `(px, ty)` | Vertical to `ty`, then horizontal to `tx` |

## Three forms of `to`

`to` matches [Draw's way item](/core/components/draw/overview#three-forms-of-way-item) exactly:

| Form | Example | Behavior |
| --- | --- | --- |
| Node id | `'a'` | Auto-attaches to Node border (see [Anchors](/core/concepts/anchors)) |
| Cartesian | `[60, 0]` | Literal `[x, y]` |
| Polar | `{ origin: 'a', angle: 30, radius: 50 }` | Polar (see [Positioning](/core/concepts/positioning)) |
