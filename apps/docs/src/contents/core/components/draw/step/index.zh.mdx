---
title: Step
description: Path 的子节点——一个 step = 一个动作（move / line / step 折角 / cycle 闭合 / curve / cubic / bend 曲线 / arc 弧段 / circlePath 整圆 / ellipsePath 整椭圆）；`to` 支持节点 id / 笛卡尔 / 极坐标 / 偏移定位（of+offset）/ 相对偏移（relative / relativeAccumulate）；只能作为 `<Path>` 的直接 children。
---

`<Step>` 描述路径上的一个动作：`move` 抬笔到目标点，`line` 落笔从游标画直线，`step` 经一个直角拐点折到目标点，`cycle` 闭合回 path 起点，`curve` / `cubic` / `bend` 画各类曲线，`arc` / `circlePath` / `ellipsePath` 以游标为圆心画弧段 / 整圆 / 整椭圆。它**只能**作为 `<Path>` 的直接 children；放在其它地方会被忽略。

## 用法

```tsx
import { Path, Step } from '@retikz/react';
```

```tsx
<Path>
  <Step kind="move" to="a" />
  <Step kind="line" to="b" />
</Path>
```

### 折角（step）

`kind="step"` 在游标与目标点之间插一个直角拐点，对应 TikZ 的 `(A) -| (B)` / `(A) |- (B)` 写法。流程图常用：

<ComponentPreview name="step-fold" />

### 闭合（cycle）

`kind="cycle"` 把当前子路径闭合回最近一次 `move` 的起点——对应 TikZ 的 `cycle` / SVG path 的 `Z`。`cycle` 没有 `to` 字段（目标隐式 = 起点）。比手动 `line` 回起点少写一个端点：

<ComponentPreview name="step-cycle" />

### 二次贝塞尔（curve）

`kind="curve"` 用一个控制点画二次贝塞尔曲线，对应 TikZ 的 `.. controls (B) ..` / SVG path 的 `Q`。控制点决定曲线的弯曲方向与幅度（**alpha.3 仅支持笛卡尔 `[x, y]` 控制点**）：

<ComponentPreview name="step-curve" />

### 三次贝塞尔（cubic）

`kind="cubic"` 用两个控制点画三次贝塞尔曲线，对应 TikZ 的 `.. controls (B) and (C) ..` / SVG path 的 `C`。`control1` 影响起点切线方向，`control2` 影响终点切线方向；常用于 S 形曲线、复杂连接：

<ComponentPreview name="step-cubic" />

### 弧形简记（bend）

`kind="bend"` 是 TikZ `to[bend left=N]` / `to[bend right=N]` 的简记——只给方向 + 角度，编译期自动算控制点退到 cubic。`bendDirection` 是相对 from→to 路径方向的视觉左 / 右；`bendAngle` 缺省 30：

<ComponentPreview name="step-bend" size="lg" />

### 偏移定位（offset）

`to` 也接受 **`{ of, offset }`** 形态——基准 + `(dx, dy)` 任意偏移，对应 TikZ `calc` 的 `($(of) + (dx, dy)$)`。`of` 三态：节点 id、笛卡尔字面值、PolarPosition（递归极坐标）；和 `<Node position>` 的偏移定位完全对称：

<ComponentPreview name="step-offset-target" size="lg" />

### 相对坐标（relative / relativeAccumulate）

`to` 还可以是相对偏移对象：`{ relative: [dx, dy] }` 与 `{ relativeAccumulate: [dx, dy] }` 都基于**前一 step 的终点**（`prevEnd`，无前段时回退到 `[0, 0]`）解析；区别只是**是否推进 prevEnd**：

- `{ relative }`：等价 TikZ `(+x, +y)`——多段链式偏移都基于**同一**锚点，**不**更新 prevEnd
- `{ relativeAccumulate }`：等价 TikZ `(++x, ++y)`——每段累积，prevEnd 跟着推进

也可写作 sugar 字符串 `'+dx,dy'` / `'++dx,dy'`（`<Step to="+1,0" />`），React adapter 会就地转成对应 IR 对象。下例两条路径 step 序列字面相同，仅差 target 形态——实线（relative）走出 L 形，虚线（relativeAccumulate）走出阶梯：

<ComponentPreview name="step-relative" />

### 弧段（arc）

`kind="arc"` 以**游标为圆心**，按起末角度 + 半径画一段圆弧——对应 TikZ 的 `arc[start angle=…, end angle=…, radius=…]`。角度沿 retikz polar 约定：0° 朝 +x，90° 视觉朝下；画完画笔停在弧的终点：

<ComponentPreview name="step-arc" />

### 整圆（circlePath）

`kind="circlePath"` 以**游标为圆心**，按 `radius` 画一个完整闭合的圆——画完画笔回到圆心。常用于画端点圆点、圆形节点边框等场景：

<ComponentPreview name="step-circle" />

### 整椭圆（ellipsePath）

`kind="ellipsePath"` 以**游标为圆心**，按 `radiusX` / `radiusY` 画一个完整闭合的椭圆——画完画笔回到圆心。`radiusX === radiusY` 时退化为圆：

<ComponentPreview name="step-ellipse" />

### 边标注（label）

给 `<Step>` 挂一个 `label` 就能在该段画线的中部 / 起点 / 终点附近放一段文字——对应 TikZ 的 `(a) -- node[midway, above] {accept} (b)`。`label` 是个对象：

- `text`：必填，标签文字（单行）
- `position`：`'at-start' | 'very-near-start' | 'near-start' | 'midway' | 'near-end' | 'very-near-end' | 'at-end'` 七个 keyword，或 0..1 的数值 t（任意精细位置），缺省 `'midway'`（t=0.5）
- `side`：`'above'` / `'below'` / `'left'` / `'right'` / `'sloped'`，缺省 `'above'`；`'sloped'` 让标签沿段切线方向旋转贴线

两种等价写法——prop 形态 `<Step label={{...}} />`，或 sugar 形态 `<Step><EdgeLabel>...</EdgeLabel></Step>`（需 `import { EdgeLabel } from '@retikz/react'`）；**两者并存时 prop 优先**。`'move'` 与 `'cycle'` 不挂 label——前者不画线、后者是闭合标记，挂上去会被静默忽略：

<ComponentPreview name="step-label" />

#### label.position：7 个 keyword + 任意数值 t

`position` 接受 7 个 keyword（TikZ 标准词汇）或 0..1 的数值 t，keyword 是 t 的语义糖：

| keyword | 对应 t | 说明 |
| --- | --- | --- |
| `'at-start'` | `0` | 段起点 |
| `'very-near-start'` | `0.125` | 起点附近 |
| `'near-start'` | `0.25` | 起点 1/4 处 |
| `'midway'` | `0.5` | 段中点（缺省） |
| `'near-end'` | `0.75` | 终点 1/4 处 |
| `'very-near-end'` | `0.875` | 终点附近 |
| `'at-end'` | `1` | 段终点 |

`position={0.3}` 这种任意数值与 TikZ `pos=0.3` 对齐——LLM 训练数据里最常见。**数值超出 `[0, 1]` 区间在 schema 阶段直接拒绝，不在编译阶段 clamp**（`position={-0.1}` / `position={1.2}` 均会让 IR 校验失败）。

#### t 在每种 step kind 上的几何含义

t 是沿整段的归一化参数，不同 kind 解释**与 TikZ 对齐**：

| Step kind | 参数化方式 | t=0.25 | t=0.5 | t=0.75 |
| --- | --- | --- | --- | --- |
| `'line'` | 归一化弧长 | 1/4 处 | 中点 | 3/4 处 |
| `'step'`（fold N=2） | 每段占 `1/N` t 区间——拐角恒在 `t = j/N`，**与段长无关** | 段 1 中点 | **拐角** | 段 2 中点 |
| `'curve'` / `'cubic'` | Bezier 参数 t（**非弧长**） | Bezier t=0.25 位置 | Bezier t=0.5 位置（通常**非视觉中点**） | Bezier t=0.75 位置 |
| `'bend'` | 内部 lower 成 cubic 后 Bezier t | 同 cubic | 同 cubic | 同 cubic |
| `'arc'` | t 线性映射 `startAngle..endAngle`（度） | 起末角度 1/4 处 | 起末中点角度 | 起末角度 3/4 处 |
| `'circlePath'` | 沿整圆 t∈[0, 1]：t=0 为 angle 0（+x），CCW 增长 | 圆周 90°（视觉下方） | 圆周 180°（-x） | 圆周 270°（视觉上方） |
| `'ellipsePath'` | 同 circlePath：t=0 在 angle 0，CCW 增长 | 椭圆 90° 处 | 椭圆 180° 处（**非弧长中点**） | 椭圆 270° 处 |

## API 参考

| 属性 | 类型 | 默认值 | 描述 |
| --- | --- | --- | --- |
| `kind` | `'move' \| 'line' \| 'step' \| 'cycle' \| 'curve' \| 'cubic' \| 'bend' \| 'arc' \| 'circlePath' \| 'ellipsePath'` | `'line'` | 见下方"十种取值"表 |
| `via` | `'-\|' \| '\|-'` | — | 仅 `kind="step"` 必填：`-\|` 先水平后垂直；`\|-` 先垂直后水平 |
| `to` | `IRTarget` | — | `kind="cycle"` / `'arc'` / `'circlePath'` / `'ellipsePath'` 时不需要；其余 kind 必填——目标点：节点 id、笛卡尔 `[x, y]`、极坐标对象、偏移定位对象 `{ of, offset }`、相对偏移对象 `{ relative }` / `{ relativeAccumulate }`，或 sugar 字符串 `'+dx,dy'` / `'++dx,dy'` |
| `control` | `[number, number]` | — | 仅 `kind="curve"` 必填：二次贝塞尔的单个控制点 |
| `control1` | `[number, number]` | — | 仅 `kind="cubic"` 必填：三次贝塞尔的第一控制点（影响起点切线） |
| `control2` | `[number, number]` | — | 仅 `kind="cubic"` 必填：三次贝塞尔的第二控制点（影响终点切线） |
| `bendDirection` | `'left' \| 'right'` | — | 仅 `kind="bend"` 必填：相对 from→to 路径方向的视觉左 / 右 |
| `bendAngle` | `number` | `30` | 仅 `kind="bend"` 可选：弯角度（度） |
| `startAngle` | `number` | — | 仅 `kind="arc"` 必填：弧的起始角度（度，0° 朝 +x，90° 视觉朝下） |
| `endAngle` | `number` | — | 仅 `kind="arc"` 必填：弧的终止角度（度） |
| `radius` | `number` | — | `kind="arc"` / `'circlePath'` 必填：圆 / 弧的半径 |
| `radiusX` | `number` | — | 仅 `kind="ellipsePath"` 必填：椭圆 x 轴半径 |
| `radiusY` | `number` | — | 仅 `kind="ellipsePath"` 必填：椭圆 y 轴半径 |
| `label` | `IRStepLabel` | — | 边标注（`'move'` / `'cycle'` 外八种 kind 通用）：`{ text, position?, side? }`；等价 sugar 写法 `<Step><EdgeLabel>…</EdgeLabel></Step>`，prop 优先 |

## kind 的十种取值

| kind | 行为 | 类比 SVG / TikZ |
| --- | --- | --- |
| `'move'` | 移动游标，**不画线** | path "M" |
| `'line'` | 从当前游标画直线到目标点 | path "L" / TikZ `--` |
| `'step'` | 经一个直角拐点折到目标点 | TikZ `-\|` / `\|-` |
| `'cycle'` | 闭合回最近一次 move 起点（无 `to`） | path "Z" / TikZ `cycle` |
| `'curve'` | 二次贝塞尔（一个控制点） | path "Q" / TikZ `.. controls (B) ..` |
| `'cubic'` | 三次贝塞尔（两个控制点） | path "C" / TikZ `.. controls (B) and (C) ..` |
| `'bend'` | 弧形简记（方向 + 角度），编译期退到 cubic | TikZ `to[bend left=N]` |
| `'arc'` | 以游标为圆心，按起末角度 + 半径画弧 | path "A" / TikZ `arc[start angle…]` |
| `'circlePath'` | 以游标为圆心画整圆，画完回到圆心 | path "A"×2 / TikZ `circle[radius]` |
| `'ellipsePath'` | 以游标为圆心画整椭圆，画完回到圆心 | path "A"×2 / TikZ `ellipse[…]` |

`<Path>` 第一个 step 必须是 `move`——如果你写 `'line'` / `'step'` / `'cycle'`，编译器会自动改成 `move`（cycle 无 `to`，回退到原点 `[0, 0]`）。

## via 的两种取值

仅 `kind="step"` 时生效，决定折角中间点的位置。设当前游标 `(px, py)`，目标 `(tx, ty)`：

| via | 中间点 | 走向 |
| --- | --- | --- |
| `'-\|'` | `(tx, py)` | 先水平到 `tx`，再垂直到 `ty` |
| `'\|-'` | `(px, ty)` | 先垂直到 `ty`，再水平到 `tx` |

## label 的字段

`label` 是单个对象，对所有可挂 label 的 kind 透明：

| 字段 | 类型 | 默认 | 描述 |
| --- | --- | --- | --- |
| `text` | `string` | — | 必填，标签文字（单行） |
| `position` | 7 keyword 之一 或 `number`（0..1） | `'midway'` | 段上归一化位置 t——keyword 是 t 的语义糖（at-start=0 / very-near-start=0.125 / near-start=0.25 / midway=0.5 / near-end=0.75 / very-near-end=0.875 / at-end=1）；超出 [0, 1] 由 schema 拒绝 |
| `side` | `'above' \| 'below' \| 'left' \| 'right' \| 'sloped'` | `'above'` | 锚点 + 偏移方向；`above` / `below` 偏 y 轴、`left` / `right` 偏 x 轴；`sloped` 不偏移、按段切线方向旋转 |

不同 step kind 上 t 的几何解释见上方 "[t 在每种 step kind 上的几何含义](#t-在每种-step-kind-上的几何含义)" 表。要点：line / fold 用归一化弧长（fold 拐角恒在 t=0.5），curve / cubic / bend 用 Bezier 参数 t（**非弧长**——t=0.5 通常不是视觉中点），arc 用角度参数化，circlePath / ellipsePath 沿整圆从 angle 0（+x）CCW 增长。

## to 的六种形态

| 形态 | 例 | 行为 |
| --- | --- | --- |
| 节点 id | `'a'` | 自动贴 Node 边界（参看 [锚点](/core/concepts/anchors)） |
| 笛卡尔 | `[60, 0]` | 直接坐标 |
| 极坐标 | `{ origin: 'a', angle: 30, radius: 50 }` | 极坐标（参看 [定位](/core/concepts/positioning)） |
| 偏移定位 | `{ of: 'a', offset: [10, 5] }` | 基准 + `(dx, dy)`；`of` 三态——节点 id / 笛卡尔字面值 / PolarPosition |
| 相对偏移 (relative) | `{ relative: [10, 0] }` 或 `'+10,0'` | 基于前一 step 终点 + `[dx, dy]`；**不**更新 prevEnd（TikZ `(+x, +y)`） |
| 相对偏移 (relativeAccumulate) | `{ relativeAccumulate: [10, 0] }` 或 `'++10,0'` | 基于前一 step 终点 + `[dx, dy]`；**更新** prevEnd（TikZ `(++x, ++y)`） |
