title: Step
description: Path 的子节点——一个 step = 一个动作（move / line / step 折角 / cycle 闭合 / curve / cubic / bend 曲线 / arc 弧段 / circlePath 整圆 / ellipsePath 整椭圆）；`to` 支持节点 id / 笛卡尔 / 极坐标 / 相对偏移（rel / relAccumulate）；只能作为 `<Path>` 的直接 children。
---

`<Step>` 描述路径上的一个动作：`move` 抬笔到目标点，`line` 落笔从游标画直线，`step` 经一个直角拐点折到目标点，`cycle` 闭合回 path 起点，`curve` / `cubic` / `bend` 画各类曲线，`arc` / `circlePath` / `ellipsePath` 以游标为圆心画弧段 / 整圆 / 整椭圆。它**只能**作为 `<Path>` 的直接 children；放在其它地方会被忽略。

## 用法

```tsx
import { Path, Step } from '@retikz/react';
```

```tsx
<Path>
  <Step kind="move" to="a" />
  <Step kind="line" to="b" />
</Path>
```

### 折角（step）

`kind="step"` 在游标与目标点之间插一个直角拐点，对应 TikZ 的 `(A) -| (B)` / `(A) |- (B)` 写法。流程图常用：

<ComponentPreview name="step-fold" />

### 闭合（cycle）

`kind="cycle"` 把当前子路径闭合回最近一次 `move` 的起点——对应 TikZ 的 `cycle` / SVG path 的 `Z`。`cycle` 没有 `to` 字段（目标隐式 = 起点）。比手动 `line` 回起点少写一个端点：

<ComponentPreview name="step-cycle" />

### 二次贝塞尔（curve）

`kind="curve"` 用一个控制点画二次贝塞尔曲线，对应 TikZ 的 `.. controls (B) ..` / SVG path 的 `Q`。控制点决定曲线的弯曲方向与幅度（**alpha.3 仅支持笛卡尔 `[x, y]` 控制点**）：

<ComponentPreview name="step-curve" />

### 三次贝塞尔（cubic）

`kind="cubic"` 用两个控制点画三次贝塞尔曲线，对应 TikZ 的 `.. controls (B) and (C) ..` / SVG path 的 `C`。`control1` 影响起点切线方向，`control2` 影响终点切线方向；常用于 S 形曲线、复杂连接：

<ComponentPreview name="step-cubic" />

### 弧形简记（bend）

`kind="bend"` 是 TikZ `to[bend left=N]` / `to[bend right=N]` 的简记——只给方向 + 角度，编译期自动算控制点退到 cubic。`bendDirection` 是相对 from→to 路径方向的视觉左 / 右；`bendAngle` 缺省 30：

<ComponentPreview name="step-bend" />

### 相对坐标（rel / relAccumulate）

`to` 还可以是相对偏移对象：`{ rel: [dx, dy] }` 与 `{ relAccumulate: [dx, dy] }` 都基于**前一 step 的终点**（`prevEnd`，无前段时回退到 `[0, 0]`）解析；区别只是**是否推进 prevEnd**：

- `{ rel }`：等价 TikZ `(+x, +y)`——多段链式偏移都基于**同一**锚点，**不**更新 prevEnd
- `{ relAccumulate }`：等价 TikZ `(++x, ++y)`——每段累积，prevEnd 跟着推进

也可写作 sugar 字符串 `'+dx,dy'` / `'++dx,dy'`（`<Step to="+1,0" />`），React adapter 会就地转成对应 IR 对象。下例两条路径 step 序列字面相同，仅差 target 形态——实线（rel）走出 L 形，虚线（relAccumulate）走出阶梯：

<ComponentPreview name="step-rel" />

### 弧段（arc）

`kind="arc"` 以**游标为圆心**，按起末角度 + 半径画一段圆弧——对应 TikZ 的 `arc[start angle=…, end angle=…, radius=…]`。角度沿 retikz polar 约定：0° 朝 +x，90° 视觉朝下；画完画笔停在弧的终点：

<ComponentPreview name="step-arc" />

### 整圆（circlePath）

`kind="circlePath"` 以**游标为圆心**，按 `radius` 画一个完整闭合的圆——画完画笔回到圆心。常用于画端点圆点、圆形节点边框等场景：

<ComponentPreview name="step-circle" />

### 整椭圆（ellipsePath）

`kind="ellipsePath"` 以**游标为圆心**，按 `radiusX` / `radiusY` 画一个完整闭合的椭圆——画完画笔回到圆心。`radiusX === radiusY` 时退化为圆：

<ComponentPreview name="step-ellipse" />

### 样式

`<Step>` 自身没有 stroke 类视觉属性——样式都挂在父 `<Path>` 上，整条 Path 的所有 Step 共享同一套 `stroke` / `strokeWidth` / `strokeDasharray`：

## API 参考

| 属性 | 类型 | 默认值 | 描述 |
| --- | --- | --- | --- |
| `kind` | `'move' \| 'line' \| 'step' \| 'cycle' \| 'curve' \| 'cubic' \| 'bend' \| 'arc' \| 'circlePath' \| 'ellipsePath'` | `'line'` | 见下方"十种取值"表 |
| `via` | `'-\|' \| '\|-'` | — | 仅 `kind="step"` 必填：`-\|` 先水平后垂直；`\|-` 先垂直后水平 |
| `to` | `IRTarget` | — | `kind="cycle"` / `'arc'` / `'circlePath'` / `'ellipsePath'` 时不需要；其余 kind 必填——目标点：节点 id、笛卡尔 `[x, y]`、极坐标对象、相对偏移对象 `{ rel }` / `{ relAccumulate }`，或 sugar 字符串 `'+dx,dy'` / `'++dx,dy'` |
| `control` | `[number, number]` | — | 仅 `kind="curve"` 必填：二次贝塞尔的单个控制点 |
| `control1` | `[number, number]` | — | 仅 `kind="cubic"` 必填：三次贝塞尔的第一控制点（影响起点切线） |
| `control2` | `[number, number]` | — | 仅 `kind="cubic"` 必填：三次贝塞尔的第二控制点（影响终点切线） |
| `bendDirection` | `'left' \| 'right'` | — | 仅 `kind="bend"` 必填：相对 from→to 路径方向的视觉左 / 右 |
| `bendAngle` | `number` | `30` | 仅 `kind="bend"` 可选：弯角度（度） |
| `startAngle` | `number` | — | 仅 `kind="arc"` 必填：弧的起始角度（度，0° 朝 +x，90° 视觉朝下） |
| `endAngle` | `number` | — | 仅 `kind="arc"` 必填：弧的终止角度（度） |
| `radius` | `number` | — | `kind="arc"` / `'circlePath'` 必填：圆 / 弧的半径 |
| `radiusX` | `number` | — | 仅 `kind="ellipsePath"` 必填：椭圆 x 轴半径 |
| `radiusY` | `number` | — | 仅 `kind="ellipsePath"` 必填：椭圆 y 轴半径 |

## kind 的十种取值

| kind | 行为 | 类比 SVG / TikZ |
| --- | --- | --- |
| `'move'` | 移动游标，**不画线** | path "M" |
| `'line'` | 从当前游标画直线到目标点 | path "L" / TikZ `--` |
| `'step'` | 经一个直角拐点折到目标点 | TikZ `-\|` / `\|-` |
| `'cycle'` | 闭合回最近一次 move 起点（无 `to`） | path "Z" / TikZ `cycle` |
| `'curve'` | 二次贝塞尔（一个控制点） | path "Q" / TikZ `.. controls (B) ..` |
| `'cubic'` | 三次贝塞尔（两个控制点） | path "C" / TikZ `.. controls (B) and (C) ..` |
| `'bend'` | 弧形简记（方向 + 角度），编译期退到 cubic | TikZ `to[bend left=N]` |
| `'arc'` | 以游标为圆心，按起末角度 + 半径画弧 | path "A" / TikZ `arc[start angle…]` |
| `'circlePath'` | 以游标为圆心画整圆，画完回到圆心 | path "A"×2 / TikZ `circle[radius]` |
| `'ellipsePath'` | 以游标为圆心画整椭圆，画完回到圆心 | path "A"×2 / TikZ `ellipse[…]` |

`<Path>` 第一个 step 必须是 `move`——如果你写 `'line'` / `'step'` / `'cycle'`，编译器会自动改成 `move`（cycle 无 `to`，回退到原点 `[0, 0]`）。

## via 的两种取值

仅 `kind="step"` 时生效，决定折角中间点的位置。设当前游标 `(px, py)`，目标 `(tx, ty)`：

| via | 中间点 | 走向 |
| --- | --- | --- |
| `'-\|'` | `(tx, py)` | 先水平到 `tx`，再垂直到 `ty` |
| `'\|-'` | `(px, ty)` | 先垂直到 `ty`，再水平到 `tx` |

## to 的五种形态

`to` 与 [Draw 的 way item](/core/components/draw/overview#way-的-13-种-item) 前五种完全一致：

| 形态 | 例 | 行为 |
| --- | --- | --- |
| 节点 id | `'a'` | 自动贴 Node 边界（参看 [锚点](/core/concepts/anchors)） |
| 笛卡尔 | `[60, 0]` | 直接坐标 |
| 极坐标 | `{ origin: 'a', angle: 30, radius: 50 }` | 极坐标（参看 [定位](/core/concepts/positioning)） |
| 相对偏移 (rel) | `{ rel: [10, 0] }` 或 `'+10,0'` | 基于前一 step 终点 + `[dx, dy]`；**不**更新 prevEnd（TikZ `(+x, +y)`） |
| 相对偏移 (relAccumulate) | `{ relAccumulate: [10, 0] }` 或 `'++10,0'` | 基于前一 step 终点 + `[dx, dy]`；**更新** prevEnd（TikZ `(++x, ++y)`） |
