`<Step>` describes a single path action: `move` lifts the pen, `line` draws a straight line, `step` folds through a single right-angle corner, `cycle` closes the path, `curve` / `cubic` / `bend` draw curves of various shapes, and `arc` / `circlePath` / `ellipsePath` use the cursor as the center to draw an arc / full circle / full ellipse. It is **only** valid as a direct child of `<Path>`; placed elsewhere it is silently ignored.

## Usage

```tsx
import { Path, Step } from '@retikz/react';
```

```tsx
<Path>
  <Step kind="move" to="a" />
  <Step kind="line" to="b" />
</Path>
```

### Offset positioning

`to` also accepts the **`{ of, offset }`** form â€” a base point plus an arbitrary `(dx, dy)`, mirroring TikZ `calc` syntax `($(of) + (dx, dy)$)`. `of` has three shapes â€” node id, Cartesian literal, or PolarPosition (recursive polar) â€” symmetric with `<Node position>`'s offset form:

### Relative coordinates (rel / relAccumulate)

`to` also accepts relative-offset objects: `{ rel: [dx, dy] }` and `{ relAccumulate: [dx, dy] }` both resolve against the **previous step's endpoint** (`prevEnd`, falling back to `[0, 0]` when there is none). The only difference is **whether prevEnd advances**:

- `{ rel }` â€” equivalent to TikZ `(+x, +y)`: chained offsets all anchor on the **same** point; prevEnd does **not** update
- `{ relAccumulate }` â€” equivalent to TikZ `(++x, ++y)`: each segment accumulates and prevEnd moves with it

You can also write the sugar strings `'+dx,dy'` / `'++dx,dy'` (e.g. `<Step to="+1,0" />`); the React adapter rewrites them in place to the matching IR objects. In the demo below, the two paths have identical step sequences and only the target form differs â€” the solid line (rel) traces an L, while the dashed line (relAccumulate) climbs a staircase:

<ComponentPreview name="step-rel" />

### Edge label

Attach a `label` to a `<Step>` to drop a single line of text near the segment's start / middle / end â€” the analogue of TikZ's `(a) -- node[midway, above] {accept} (b)`. `label` is an object:

- `text`: required, single-line label content
- `position`: `'midway'` / `'near-start'` / `'near-end'`, default `'midway'` (segment midpoint)
- `side`: `'above'` / `'below'` / `'left'` / `'right'` / `'sloped'`, default `'above'`; `'sloped'` rotates the label along the tangent so it sits on the line

Two equivalent forms â€” the prop form `<Step label={{...}} />`, or the sugar form `<Step><EdgeLabel>...</EdgeLabel></Step>` (needs `import { EdgeLabel } from '@retikz/react'`); **the prop wins when both are set**. `'move'` and `'cycle'` cannot carry a label â€” they don't draw a segment, so any label on them is silently dropped:

<ComponentPreview name="step-label" />

## Ten kinds

| kind | Behavior | SVG / TikZ analogue |
| --- | --- | --- |
| `'move'` | Move cursor, **no draw** | path "M" |
| `'line'` | Straight line from cursor to target | path "L" / TikZ `--` |
| `'step'` | Right-angle fold through one corner | TikZ `-\|` / `\|-` |
| `'cycle'` | Close back to most recent `move` (no `to`) | path "Z" / TikZ `cycle` |
| `'curve'` | Quadratic Bezier (one control point) | path "Q" / TikZ `.. controls (B) ..` |
| `'cubic'` | Cubic Bezier (two control points) | path "C" / TikZ `.. controls (B) and (C) ..` |
| `'bend'` | Arc shorthand (direction + angle), lowered to cubic | TikZ `to[bend left=N]` |
| `'arc'` | Cursor as center; arc by start / end angle + radius | path "A" / TikZ `arc[start angleâ€¦]` |
| `'circlePath'` | Cursor as center; full circle, pen returns to center | path "A"Ă—2 / TikZ `circle[radius]` |
| `'ellipsePath'` | Cursor as center; full ellipse, pen returns to center | path "A"Ă—2 / TikZ `ellipse[â€¦]` |

The first step of a `<Path>` must be `move` â€” if you write `'line'` / `'step'` / `'cycle'`, the compiler coerces it to `move` (cycle has no `to`, so it falls back to origin `[0, 0]`).

## Two via values

Only meaningful when `kind="step"`. Given cursor `(px, py)` and target `(tx, ty)`:

| via | Corner | Path |
| --- | --- | --- |
| `'-\|'` | `(tx, py)` | Horizontal to `tx`, then vertical to `ty` |
| `'\|-'` | `(px, ty)` | Vertical to `ty`, then horizontal to `tx` |

## Fields of `label`

`label` is a single object, applied uniformly across all kinds that accept it:

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `text` | `string` | â€” | Required label text (single line) |
| `position` | `'midway' \| 'near-start' \| 'near-end'` | `'midway'` | Position along the segment â€” `midway` = midpoint (t=0.5); `near-start` = t=0.25; `near-end` = t=0.75 |
| `side` | `'above' \| 'below' \| 'left' \| 'right' \| 'sloped'` | `'above'` | Anchor + offset direction; `above` / `below` shift along y, `left` / `right` along x; `sloped` keeps the anchor and rotates the label along the tangent |

For folded segments (`kind="step"`), t maps as tâˆˆ[0, 0.5] â†’ first leg (`fromâ†’corner`), tâˆˆ(0.5, 1] â†’ second leg; the default `midway` lands exactly on the corner, with the tangent taken from the first leg.

## Six forms of `to`

| Form | Example | Behavior |
| --- | --- | --- |
| Node id | `'a'` | Auto-attaches to Node border (see [Anchors](/core/concepts/anchors)) |
| Cartesian | `[60, 0]` | Literal `[x, y]` |
| Polar | `{ origin: 'a', angle: 30, radius: 50 }` | Polar (see [Positioning](/core/concepts/positioning)) |
| Offset | `{ of: 'a', offset: [10, 5] }` | Base + `(dx, dy)`; `of` accepts node id, Cartesian literal, or PolarPosition |
| Relative (rel) | `{ rel: [10, 0] }` or `'+10,0'` | Previous step's endpoint + `[dx, dy]`; does **not** update prevEnd (TikZ `(+x, +y)`) |
| Relative (relAccumulate) | `{ relAccumulate: [10, 0] }` or `'++10,0'` | Previous step's endpoint + `[dx, dy]`; **updates** prevEnd (TikZ `(++x, ++y)`) |
