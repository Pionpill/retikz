---
title: Step
description: Path's child node ‚Äî one step = one action (move / line / step fold / cycle close / curve / cubic / bend); only valid as a direct child of `<Path>`.
---

`<Step>` describes a single path action: `move` lifts the pen, `line` draws a straight line, `step` folds through a single right-angle corner, `cycle` closes the path, and `curve` / `cubic` / `bend` draw curves of various shapes. It is **only** valid as a direct child of `<Path>`; placed elsewhere it is silently ignored.

## Usage

```tsx
import { Path, Step } from '@retikz/react';
```

```tsx
<Path>
  <Step kind="move" to="a" />
  <Step kind="line" to="b" />
</Path>
```

### Folded (step)

`kind="step"` inserts a single right-angle corner between cursor and target ‚Äî the TikZ `(A) -| (B)` / `(A) |- (B)` form. A flow-chart staple:

### Closing (cycle)

`kind="cycle"` closes the current sub-path back to the most recent `move` target ‚Äî the TikZ `cycle` keyword / SVG path `Z`. `cycle` carries **no** `to` field (the destination is implicit). Saves writing one explicit endpoint vs. drawing a `line` back to the start:

### Quadratic curve

`kind="curve"` draws a quadratic Bezier with one control point ‚Äî TikZ's `.. controls (B) ..` / SVG path `Q`. The control point governs the bend direction and amplitude (**alpha.3 supports Cartesian `[x, y]` control points only**):

### Cubic curve

`kind="cubic"` draws a cubic Bezier with two control points ‚Äî TikZ's `.. controls (B) and (C) ..` / SVG path `C`. `control1` shapes the start tangent; `control2` shapes the end tangent. Common for S-shaped curves and complex connectors:

<ComponentPreview name="step-cubic" />

### Bend (arc shorthand)

`kind="bend"` is shorthand for TikZ's `to[bend left=N]` / `to[bend right=N]` ‚Äî give a direction and angle, control points are computed at compile time and lowered to a cubic. `bendDirection` is the visual left/right relative to the from‚Üíto path; `bendAngle` defaults to 30:

## API Reference

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `kind` | `'move' \| 'line' \| 'step' \| 'cycle' \| 'curve' \| 'cubic' \| 'bend'` | `'line'` | See the "Seven kinds" table below |
| `via` | `'-\|' \| '\|-'` | ‚Äî | Required for `kind="step"`: `-\|` horizontal-then-vertical; `\|-` vertical-then-horizontal |
| `to` | `IRTarget` | ‚Äî | Required except for `kind="cycle"`. Target: node id, Cartesian `[x, y]`, or polar object |
| `control` | `[number, number]` | ‚Äî | Required for `kind="curve"`: single control point of the quadratic Bezier |
| `control1` | `[number, number]` | ‚Äî | Required for `kind="cubic"`: first control point (shapes start tangent) |
| `control2` | `[number, number]` | ‚Äî | Required for `kind="cubic"`: second control point (shapes end tangent) |
| `bendDirection` | `'left' \| 'right'` | ‚Äî | Required for `kind="bend"`: visual left / right relative to from‚Üíto direction |
| `bendAngle` | `number` | `30` | Optional for `kind="bend"`: bend angle in degrees |

## Seven kinds

| kind | Behavior | SVG / TikZ analogue |
| --- | --- | --- |
| `'move'` | Move cursor, **no draw** | path "M" |
| `'line'` | Straight line from cursor to target | path "L" / TikZ `--` |
| `'step'` | Right-angle fold through one corner | TikZ `-\|` / `\|-` |
| `'cycle'` | Close back to most recent `move` (no `to`) | path "Z" / TikZ `cycle` |
| `'curve'` | Quadratic Bezier (one control point) | path "Q" / TikZ `.. controls (B) ..` |
| `'cubic'` | Cubic Bezier (two control points) | path "C" / TikZ `.. controls (B) and (C) ..` |
| `'bend'` | Arc shorthand (direction + angle), lowered to cubic | TikZ `to[bend left=N]` |

The first step of a `<Path>` must be `move` ‚Äî if you write `'line'` / `'step'` / `'cycle'`, the compiler coerces it to `move` (cycle has no `to`, so it falls back to origin `[0, 0]`).

## Two via values

Only meaningful when `kind="step"`. Given cursor `(px, py)` and target `(tx, ty)`:

| via | Corner | Path |
| --- | --- | --- |
| `'-\|'` | `(tx, py)` | Horizontal to `tx`, then vertical to `ty` |
| `'\|-'` | `(px, ty)` | Vertical to `ty`, then horizontal to `tx` |

## Three forms of `to`

`to` matches [Draw's way item](/core/components/draw/overview#three-forms-of-way-item) exactly:

| Form | Example | Behavior |
| --- | --- | --- |
| Node id | `'a'` | Auto-attaches to Node border (see [Anchors](/core/concepts/anchors)) |
| Cartesian | `[60, 0]` | Literal `[x, y]` |
| Polar | `{ origin: 'a', angle: 30, radius: 50 }` | Polar (see [Positioning](/core/concepts/positioning)) |
