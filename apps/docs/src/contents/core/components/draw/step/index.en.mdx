description: Path's child node ŌĆö one step = one action (move / line / step fold / cycle close); only valid as a direct child of `<Path>`.

`<Step>` describes a single path action: `move` lifts the pen to the target, `line` draws a straight line from cursor to target, `step` folds through a single right-angle corner, `cycle` closes the path back to its starting point. It is **only** valid as a direct child of `<Path>`; placed elsewhere it is silently ignored.

## Usage

```tsx
import { Path, Step } from '@retikz/react';
```

```tsx
<Path>
  <Step kind="move" to="a" />
  <Step kind="line" to="b" />
</Path>
```

`kind="cycle"` closes the current sub-path back to the most recent `move` target ŌĆö the TikZ `cycle` keyword / SVG path `Z`. `cycle` carries **no** `to` field (the destination is implicit). Saves writing one explicit endpoint vs. drawing a `line` back to the start:

## API Reference

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `kind` | `'move' \| 'line' \| 'step' \| 'cycle'` | `'line'` | `'move'` lifts the pen; `'step'` folds; `'cycle'` closes back to start |
| `via` | `'-\|' \| '\|-'` | ŌĆö | Required for `kind="step"`: `-\|` horizontal-then-vertical; `\|-` vertical-then-horizontal |
| `to` | `IRTarget` | ŌĆö | Required except for `kind="cycle"`. Target: node id, Cartesian `[x, y]`, or polar object |

## Four kinds

| kind | Behavior | SVG / TikZ analogue |
| --- | --- | --- |
| `'move'` | Move cursor, **no draw** | path "M" |
| `'line'` | Straight line from cursor to target | path "L" / TikZ `--` |
| `'step'` | Right-angle fold through one corner | TikZ `-\|` / `\|-` |
| `'cycle'` | Close back to most recent `move` (no `to`) | path "Z" / TikZ `cycle` |

The first step of a `<Path>` must be `move` ŌĆö if you write `'line'` / `'step'` / `'cycle'`, the compiler coerces it to `move` (cycle has no `to`, so it falls back to origin `[0, 0]`).

## Two via values

Only meaningful when `kind="step"`. Given cursor `(px, py)` and target `(tx, ty)`:

| via | Corner | Path |
| --- | --- | --- |
| `'-\|'` | `(tx, py)` | Horizontal to `tx`, then vertical to `ty` |
| `'\|-'` | `(px, ty)` | Vertical to `ty`, then horizontal to `tx` |

## Three forms of `to`

`to` matches [Draw's way item](/core/components/draw/overview#three-forms-of-way-item) exactly:

| Form | Example | Behavior |
| --- | --- | --- |
| Node id | `'a'` | Auto-attaches to Node border (see [Anchors](/core/concepts/anchors)) |
| Cartesian | `[60, 0]` | Literal `[x, y]` |
| Polar | `{ origin: 'a', angle: 30, radius: 50 }` | Polar (see [Positioning](/core/concepts/positioning)) |
