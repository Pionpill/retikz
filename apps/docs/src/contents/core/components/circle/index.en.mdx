`<Circle>` draws a circle in one line: give a center + radius (or diameter / diameter endpoints / bounding-box corners); add angles for a partial circle (half circle / segment). It expands to `<Path>` + `<Step kind="circlePath">` — following "sugar introduces no new capability".

Difference from `<Node shape="circle">`: `<Circle>` is a **plain, text-less drawing path**; a circle Node is an **object** with text / anchors that edges can connect to. For a labelled, connectable circle node use [`<Node>`](/core/components/node/overview).

## Usage

```tsx
import { Circle } from '@retikz/react';
```

```tsx
<Circle center={[0, 0]} radius={40} />
```

## Forms

One of four ways to set center + radius:

| Form | prop | Point type |
| --- | --- | --- |
| center + radius | `{ center, radius }` | `center` any Target |
| center + diameter | `{ center, diameter }` | `center` any Target |
| diameter endpoints | `{ from, to }` | literal `[x, y]` |
| bounding box | `{ corner1, corner2 }` (radius = min(\|dx\|,\|dy\|)/2) | literal `[x, y]` |

Forms that compute coordinates (`from/to`, `corner1/corner2`) only accept **literal Cartesian `[x, y]`**; passing a node id / polar point throws.

## Partial cropping

Give any **two** of `startAngle` / `endAngle` / `sweepAngle` for a partial circle:

- `closed="chord"` (default): chord-closed → half circle / segment
- `closed="open"`: open arc (equivalent to [`<Arc>`](/core/components/arc))

No angles = full circle.

## API reference

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `center` | `Target` | — | Center (`{center,radius}` / `{center,diameter}` forms) |
| `radius` | `number` | — | Radius |
| `diameter` | `number` | — | Diameter (= radius × 2) |
| `from` / `to` | `[x, y]` | — | Diameter endpoints (literal) |
| `corner1` / `corner2` | `[x, y]` | — | Bounding-box corners (literal) |
| `startAngle` / `endAngle` / `sweepAngle` | `number` | — | Partial-circle angles (any two) |
| `closed` | `'chord' \| 'open'` | `'chord'` | Partial-circle closing mode |
| Visual props | same as `<Path>` | — | `fill` / `stroke` / `strokeWidth` / `dashPattern` / `opacity` / `zIndex` etc. |
