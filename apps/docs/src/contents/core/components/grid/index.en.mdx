`<Grid>` draws a background grid: give a bounding box (two corners or center + size) + a step, and it expands to **multiple** `<Path>` elements (vertical + horizontal lines, each move + line, none closed). Visual props are forwarded to every line.

Often paired with `zIndex` (negative to sink to the back) or placed before content as a backdrop. Points must be literal Cartesian (the component computes every line endpoint).

## Usage

```tsx
import { Grid } from '@retikz/react';
```

```tsx
<Grid corner1={[0, 0]} corner2={[100, 60]} step={10} stroke="#ddd" />
```

## Forms and step

| Form | prop |
| --- | --- |
| two corners | `{ corner1, corner2 }` |
| center + size | `{ center, width, height }` |

Step (all points literal `[x, y]`):

- `step`: single value, xStep = yStep
- `xStep` + `yStep`: per-axis (overrides `step`)

**Boundary rule**: start from `corner1` (or `center - size/2`), step strictly; the last line is omitted if it doesn't coincide with the far corner (no trailing line). Vertical lines: `floor(|x1-x0|/xStep)+1`; horizontal likewise.

## API reference

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `corner1` / `corner2` | `[x, y]` | — | Bounding-box corners (literal) |
| `center` | `[x, y]` | — | Center (center form, literal) |
| `width` / `height` | `number` | — | Size (center form) |
| `step` | `number` | — | Single step value |
| `xStep` / `yStep` | `number` | — | Per-axis step (overrides step) |
| Visual props | same as `<Path>` | — | `stroke` / `strokeWidth` / `dashPattern` / `zIndex` etc. |
