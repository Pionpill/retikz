title: Camera & Stagger
description: cameraTo animates the scene viewbox for pan / zoom (on <Layout>); stagger adds increasing delay across a group for a one-by-one entrance.
---

Camera and stagger are the two animations that act **on the whole, not on a single element's look**: `cameraTo` animates the scene's viewbox, while `stagger` adjusts the start times of a group of animations. For the common options and the full preset table, see the [Overview](/core/components/animation).

## Camera

<ComponentPreview name="camera-to" size="md" />

`cameraTo({ from, to })`: animates the scene's viewbox `[x, y, w, h]` for pan / zoom. Attach it to `<Layout animations>` (the camera belongs to the scene root, not an element); both endpoints are explicit. It defaults to `duration` 800ms and `easing` `ease-in-out`, both overridable like the other common options.

It expands to a single `viewBox`-channel track:

```json
{
  "property": "viewBox",
  "keyframes": [
    { "at": 0, "value": [0, 0, 100, 100] },
    { "at": 1, "value": [25, 25, 50, 50] }
  ],
  "duration": 800,
  "easing": "ease-in-out"
}
```

## Stagger

<ComponentPreview name="stagger" size="sm" />

`stagger(tracks, stepMs, startMs = 0)`: **rewrites each track's `delay` to `startMs + i×stepMs`** (overriding any existing `delay`) for a one-by-one entrance. The third argument `startMs` defaults to 0 and is the group's start offset. It choreographs the start times of the element animations ([Intro](/core/components/animation/entrance) / [Emphasis & Loop](/core/components/animation/emphasis)); it doesn't change any single animation's shape.
