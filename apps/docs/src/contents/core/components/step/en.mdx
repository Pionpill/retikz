---
title: Step
description: Path's child node — one step = one action (move / line); only valid as a direct child of `<Path>`.
---

`<Step>` describes a single path action: `move` lifts the pen to the target, `line` draws a straight line from cursor to target. It is **only** valid as a direct child of `<Path>`; placed elsewhere it is silently ignored.

## Usage

```tsx
import { Path, Step } from '@retikz/react';
```

```tsx
<Path>
  <Step kind="move" to="a" />
  <Step kind="line" to="b" />
</Path>
```

## API Reference

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `kind` | `'move' \| 'line'` | `'line'` | `'move'` lifts the pen |
| `to` | `IRTarget` | — | Target: node id, Cartesian `[x, y]`, or polar object |

## Two kinds

| kind | Behavior | SVG analogue |
| --- | --- | --- |
| `'move'` | Move cursor, **no draw** | path "M" |
| `'line'` | Straight line from cursor to target | path "L" |

The first step of a `<Path>` must be `move` — if you write `'line'`, the compiler coerces it to `move`.

## Three forms of `to`

`to` matches [Draw's way item](/core/components/draw#three-forms-of-way-item) exactly:

| Form | Example | Behavior |
| --- | --- | --- |
| Node id | `'a'` | Auto-attaches to Node border (see [Anchors](/core/concepts/anchors)) |
| Cartesian | `[60, 0]` | Literal `[x, y]` |
| Polar | `{ origin: 'a', angle: 30, radius: 50 }` | Polar (see [Positioning](/core/concepts/positioning)) |
