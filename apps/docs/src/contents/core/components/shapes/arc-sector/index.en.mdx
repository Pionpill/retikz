---
title: Arc / Sector
description: Two ways to make an arc or sector — draw an angle span with sugar, or give a node a sector / arc boundary.
---

Arcs and sectors are both angle-based, and retikz gives you two forms:

- **Draw an outline** — the sugar components `<Arc>` (open arc) / `<Sector>` (closed wedge) draw an angle span at a position, expanding to an underlying `<Path>`.
- **A node with that boundary** — `<Node shape={{ type:'sector', params }}>` gives a node bounded by a sector / arc: it can hold content, be referenced by a connection, and snap endpoints to geometric anchors.

In short: reach for `<Arc>` / `<Sector>` when you just need an arc / sector line, and for `<Node shape>` when you need a connectable sector / arc element.

## Usage

```tsx
import { Arc, Sector, Node } from '@retikz/react';
```

```tsx
{/* Draw an outline */}
<Arc center={[0, 0]} radius={40} startAngle={0} endAngle={90} />
<Sector center={[0, 0]} radius={40} startAngle={0} endAngle={90} fill="darkorange" />

{/* A node with a sector boundary */}
<Node shape={{ type: 'sector', params: { innerRadius: 0, outerRadius: 40, startAngle: 0, endAngle: 90 } }} />
```

## Examples

### Arc

`<Arc>` draws an open arc by default. It is useful for curved arrows, ring markers, and decorative strokes.

| Form | Best for |
| --- | --- |
| `{ center, radius, startAngle, endAngle }` | Circular arc |
| `{ center, radiusX, radiusY, startAngle, endAngle }` | Elliptical arc |

Angles start from +x; `90` points downward on screen. Angles take any two of `startAngle` / `endAngle` / `sweepAngle`, e.g. `{ startAngle: 30, sweepAngle: 90 }`. `center` accepts any target (node id / polar / Cartesian).

<ComponentPreview name="arc-basic" size="sm" />

#### Visual props

`<Arc>` forwards the full `<Path>` visual prop set: `stroke` / `strokeWidth` / `dashPattern` / `arrow` / `marks` / `opacity` / `rotate` / `scale`, and more. Common ones are arrowed arcs (`arrow="->"`), dashed arcs (`dashPattern`), and mid-path direction marks (`marks`, oriented along the tangent).

<ComponentPreview name="arc-visual" size="sm" />

#### Closing modes

Arcs are open by default; pass `close` to turn the arc into a fillable region:

| `close` | Meaning |
| --- | --- |
| omitted / `'open'` | Open arc, not closed, not filled |
| `'chord'` | Connect both ends with a chord (segment) |
| `'sector'` | Connect both ends back to the center (wedge) |

<ComponentPreview name="arc-close" size="sm" />

#### Labels on the arc

`label` forwards to the underlying arc step; the text is positioned along the arc (`position` defaults to `midway`, mapped linearly across `startAngle..endAngle`), just like an edge label. Sector supports it too — see the pie example below.

### Sector

`<Sector>` starts from the center, draws an arc, then closes back to the center. Add `innerRadius` for a hollow circular sector; for an elliptical hollow sector, use `innerRadiusX` + `innerRadiusY`.

| Form | Best for |
| --- | --- |
| `{ center, radius, startAngle, endAngle }` | Circular sector |
| `{ center, radiusX, radiusY, startAngle, endAngle }` | Elliptical sector |
| add `innerRadius` | Hollow circular sector |
| add `innerRadiusX` + `innerRadiusY` | Hollow elliptical sector |

<ComponentPreview name="sector-basic" size="sm" />

#### Center on a node, labeled slices

A filled sector's center is the cursor, so `center` can take a node id (or polar coordinate) to pin the sector onto a node; combined with `sweepAngle` and `label`, a few slices make a labeled pie chart.

> A hollow sector (with an inner radius) needs to compute the inner/outer arc endpoints, so its `center` must still be a literal Cartesian `[x, y]`.

<ComponentPreview name="sector-label" size="sm" />

#### Whole-shape transform

`rotate` / `scale` forward to `<Path>`, transforming the whole sector around its bounding-box center (no `<Scope>` wrapper needed). `rotate` tilts an axis-aligned elliptical sector — currently the way to draw a "tilted elliptical sector".

<ComponentPreview name="sector-transform" size="sm" />

### As a node shape: shape="sector" / "arc"

When you need a sector / arc **node** (referenceable by a connection, usable as an icon glyph) rather than a line, set `shape={{ type, params }}` on `<Node>`. These shapes are sized by the radii / angles in `params`, **independent of any text content**.

| shape | params | Notes |
| --- | --- | --- |
| sector | `{ innerRadius, outerRadius, startAngle, endAngle }` | Ring wedge; `innerRadius:0` is a solid pie slice |
| arc | `{ radius, startAngle, endAngle, close? }` | Single-radius arc; `close:true` closes it into a segment |

<ComponentPreview name="node-arc-sector" size="md" />

Besides the node's 9 standard anchors (`north` / `center` …), these shapes provide geometry-named anchors for precise connections via `{ id, anchor }`:

- **sector**: `outer-arc-mid` / `inner-arc-mid` / `apex` / `centroid` / `start-edge-mid` / `end-edge-mid`
- **arc**: `arc-mid` / `start` / `end` / `center`

For the full `<Node>` props see [Node](/core/components/node/overview).

## How it works

> Optional for everyday use; revisit this section when angle direction or closing behavior looks surprising.

### Angles follow screen coordinates

The docs examples use screen coordinates: x points right and y points down. So `0` degrees points right, `90` points down, and `-90` points up.

### Arc and Sector close differently

`<Arc>` is open by default, so it does not create a fillable region on its own; `<Sector>` closes a region and can be filled directly. For a chord-closed or sector-closed partial circle, pass `<Arc close="chord" | "sector">`, or use the angle + `closed` forms on [Circle and Ellipse](../circle-ellipse) (all three share the same underlying path). `<Node shape="sector">` is a different route: it is not a Path but a node with a sector boundary that connections can snap to.

## API Reference

### As a shape: Arc / Sector

| Component | Combination | Meaning |
| --- | --- | --- |
| `<Arc>` | `center` + `radius` + angles | Circular arc (open by default; angles required) |
| | `center` + `radiusX` + `radiusY` + angles | Elliptical arc |
| `<Sector>` | `center` + `radius` + angles | Circular sector (angles required) |
| | `center` + `radiusX` + `radiusY` + angles | Elliptical sector |
| | + `innerRadius?` | Hollow circular sector |
| | + `innerRadiusX?` + `innerRadiusY?` | Hollow elliptical sector |

Shared props (forwarded to the underlying `<Path>`):

| Prop | Type | Applies to | Notes |
| --- | --- | --- | --- |
| angles | `startAngle` / `endAngle` / `sweepAngle` | Arc / Sector | Any two, in degrees; `0`=+x, `90`=screen-down |
| `center` | target | Arc / filled Sector | Node id / polar / Cartesian; a hollow Sector must use a literal `[x, y]` |
| `close` | `'open' \| 'chord' \| 'sector'` | Arc | Closing mode, defaults to `'open'` |
| `label` | edge-label object | Arc / Sector | Positioned along the arc, `position` defaults to `midway` |
| `rotate` / `scale` | `number` / `{ x, y }` | Arc / Sector | Whole-shape transform around the bounding-box center |
| `marks` | `{ pos, mark }[]` | Arc / Sector | Mid-path direction markers along the path |
| visual | `stroke` / `strokeWidth` / `dashPattern` / `arrow` / `fill` / `opacity` … | Arc / Sector | Forwards the full `<Path>` visual prop set |

### As a node shape

| `shape` | params | Boundary / use |
| --- | --- | --- |
| `{ type: 'sector', params }` | `innerRadius` / `outerRadius` / `startAngle` / `endAngle` | Ring wedge (connectable; `innerRadius:0` is a solid slice) |
| `{ type: 'arc', params }` | `radius` / `startAngle` / `endAngle` / `close?` | Arc glyph (connectable) |

> For the full `<Node>` props see [Node](/core/components/node/overview).
