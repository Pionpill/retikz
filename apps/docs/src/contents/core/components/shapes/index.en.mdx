---
title: Shapes
description: Two ways to make the same geometric shape — draw an outline with sugar, or give a node that shape's boundary.
---

This group covers the common geometric shapes: circle, ellipse, arc, sector, rectangle, polygon, and star. In retikz each shape has **two forms**, and every page puts them side by side:

- **Draw an outline** — use a sugar component (`<Circle>` / `<Sector>` …) to draw a shape line at a position; it expands to an underlying `<Path>`. Pure graphics, not part of any connection.
- **A node with that boundary** — use `<Node shape="circle">` to get a node bounded by that shape: it can hold text, be referenced by `<Path>` / `<Draw>` through its id, and have connection endpoints snap to its edge.

In short: reach for sugar when you just need a shape line, and for a node shape when you need a connectable, labelable shape element.

## What it contains

| Shape | Outline (sugar) | Node boundary (Node shape) |
| --- | --- | --- |
| Circle / Ellipse | `<Circle>` · `<Ellipse>` | `shape="circle"` · `"ellipse"` |
| Arc / Sector | `<Arc>` · `<Sector>` | `shape="arc"` · `"sector"` |
| Rectangle | `<Rectangle>` | default / `shape="rectangle"` |
| Polygon | `<RegularPolygon>` | `shape="polygon"` · `"diamond"` |
| Star | `<Star>` | `shape="star"` |

> Background grids and reference lines — **a field of regular lines** rather than a single closed shape — are not in this group; see `<Grid>` under [Helpers](/core/components/helpers).
