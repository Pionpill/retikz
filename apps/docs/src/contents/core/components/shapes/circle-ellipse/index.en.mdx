Circles and ellipses are retikz geometric shapes with two faces: draw an outline with `<Circle>` / `<Ellipse>` (sugar), or bound a node with `shape="circle"` / `"ellipse"`. See [Shapes](/core/components/shapes) for the full definition. Reach for sugar to draw a line, and for `<Node shape>` to get a connectable, labelable node.

## As a Path shape (sugar)

`<Circle>` / `<Ellipse>` draw a circle / ellipse outline at a position, expanding to an underlying `<Path>`. They fit cases where you know the center and radius, or only have a box.

### Usage

```tsx
import { Circle, Ellipse } from '@retikz/react';
```

```tsx
<Circle center={[0, 0]} radius={32} fill="none" />
<Ellipse center={[0, 0]} radiusX={48} radiusY={28} fill="none" />
```

### Examples

The common form is `center + radius`. If you already have a rectangular area, you can also use `corner1 + corner2` or `box`.

| Form | Best for |
| --- | --- |
| `{ center, radius }` | Known center and radius |
| `{ center, diameter }` | Known diameter |
| `{ from, to }` | Diameter endpoints |
| `{ corner1, corner2 }` | Fit inside a box |
| `{ box }` | Describe the box with `x/y/origin + width + height` |
| + `fit="contain" \| "cover"` | Fit the circle inside or outside the box |
| + `inset` / `outset` | Shrink or expand the box before fitting |
| + angles + `closed="sector"` | Draw a partial circle whose arc ends connect to the center |

Ellipse works like Circle, but the radius is split into `radiusX` and `radiusY`. It also supports `corner1 + corner2` and `box`.

| Form | Best for |
| --- | --- |
| `{ center, radiusX, radiusY }` | Known center and radii |
| `{ center, diameterX, diameterY }` | Known diameters |
| `{ corner1, corner2 }` | Fit inside a box |
| `{ box }` | Describe the box with `x/y/origin + width + height` |
| + `inset` / `outset` | Shrink or expand the box before fitting |
| + angles + `closed="sector"` | Draw a partial ellipse whose arc ends connect to the center |

### API Reference

| Component | Combination | Meaning |
| --- | --- | --- |
| `<Circle>` | `center` + `radius` | Circle from center and radius |
| | `center` + `diameter` | Circle from center and diameter |
| | `from` + `to` | Circle from two points (diameter endpoints) |
| | `corner1` + `corner2` | Largest circle inscribed in the box |
| | `box` | Circle from a box, with `fit` / `inset` / `outset` |
| | + `fit?` | `contain` uses the shorter side, `cover` uses the longer side |
| | + `inset?` / `outset?` | Shrink or expand the box before fitting |
| | + angles (`startAngle` / `endAngle` / `sweepAngle`, any two) + `closed?` | Partial circle: `chord` / `sector` / `open` |
| `<Ellipse>` | `center` + `radiusX` + `radiusY` | Ellipse from center and two radii |
| | `center` + `diameterX` + `diameterY` | Ellipse from center and two diameters |
| | `corner1` + `corner2` | Ellipse inscribed in the box |
| | `box` | Ellipse from a box, with `inset` / `outset` |
| | + `inset?` / `outset?` | Shrink or expand the box before fitting |
| | + angles + `closed?` | Partial ellipse: `chord` / `sector` / `open` |

## As a node shape

When you need a circular / elliptical node rather than a line (to hold text, to be referenced by a connection), set `shape` on `<Node>`.

### Usage

```tsx
import { Node } from '@retikz/react';
```

```tsx
<Node shape="circle">label</Node>
<Node shape="ellipse">label</Node>
```

### Examples

The boundary circumscribes "text + padding": `circle` takes the circle that fits the text, `ellipse` takes the circumscribed ellipse (each axis × √2). The bare strings `"circle"` / `"ellipse"` need no extra parameters; `ellipse` also has one optional param `circumscribe` (`'proportional'` (default, each axis × √2) / `'equal'` (isotropic, radius = half the inner box diagonal)), and `circle` is exactly the `circumscribe:'equal'` preset of `ellipse`.

<ComponentPreview name="node-circle-ellipse" size="sm" />

Connection endpoints snap to the circle / ellipse edge, and the node's 9 standard anchors (`north` / `center` …) still apply. For the full `<Node>` props (`position` / `fill` / `padding` / `label` …) see [Node](/core/components/node/overview).

### API Reference

| `shape` | Boundary | Parameters |
| --- | --- | --- |
| `"circle"` | Circle circumscribing the text (= the `circumscribe:'equal'` preset of `ellipse`) | None |
| `"ellipse"` or `{ type:'ellipse', params }` | Circumscribed ellipse | `circumscribe?`: `'proportional'` (default, each axis × √2) / `'equal'` (isotropic, radius = half the inner box diagonal) |

## How it works

> Optional for everyday use; revisit when a point form throws an error, or when you want to understand the difference between sugar and Node.

### Some forms pass points through, others compute first (sugar)

`<Circle>`'s `center` passes the point directly to the underlying Path, so it accepts node ids, polar points, or plain `[x, y]` coordinates:

```tsx
<Circle center="A" radius={24} />
```

Other forms must compute a midpoint, size, or bounding box in React first, so they only accept literal `[x, y]` coordinates — all of `<Ellipse>`'s point inputs (including `center` / `corner1` / `corner2`), plus `<Circle>`'s `from` / `to` / `corner1` / `corner2`:

```tsx
<Circle from={[-40, 0]} to={[40, 0]} />
<Ellipse corner1={[-40, -20]} corner2={[40, 20]} />
```

Passing a node id to a computed `<Circle>` form throws an error; `<Ellipse>`'s corner branch is currently unchecked, so a non-`[x, y]` input computes to NaN — always pass literal coordinates.

### Partial shapes have three closing modes (sugar)

When angles are present, `closed` decides how the arc ends are connected:

| Value | Effect |
| --- | --- |
| `chord` | Default, draw a straight chord between the arc ends |
| `sector` | Draw from the arc end to the center, then back to the arc start |
| `open` | Draw only the arc |

### Sugar and Node are two routes

`<Circle>` / `<Ellipse>` expand to an equivalent `<Path>` / `<Step>` sequence — pure graphics that can use `stroke`, `fill`, `arrow`, and `zIndex` like any path. `<Node shape>` is not a Path but a node with a boundary circumscribed from the text content that connections can snap to. For a circle that both holds text and connects, use `<Node>` rather than `<Circle>`.
