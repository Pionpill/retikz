title: Circle / Ellipse
description: Sugar forms for circles and ellipses, with support for box-based fitting and box adjustments.

`<Circle>` and `<Ellipse>` are sugar forms that expand common geometry into `<Path>`. They fit cases where you know the center and radius, only have a box, or want to pad a circle or ellipse in or out from that box.

## Usage

```tsx
import { Circle, Ellipse } from '@retikz/react';
```

```tsx
<Circle center={[0, 0]} radius={32} fill="none" />
<Ellipse center={[0, 0]} radiusX={48} radiusY={28} fill="none" />
```

## Examples

### Circle

The common form is `center + radius`. If you already have a rectangular area, you can also use `corner1 + corner2`, `box`, or `boundingBox`.

| Form | Best for |
| --- | --- |
| `{ center, radius }` | Known center and radius |
| `{ center, diameter }` | Known diameter |
| `{ from, to }` | Diameter endpoints |
| `{ corner1, corner2 }` | Fit inside a box |
| `{ box }` / `{ boundingBox }` | Describe the box with `x/y/origin + width + height` |
| + `fit="contain" \| "cover"` | Fit the circle inside or outside the box |
| + `inset` / `outset` | Shrink or expand the box before fitting |
| + angles + `closed="sector"` | Draw a partial circle whose arc ends connect to the center |

### Ellipse

Ellipse works like Circle, but the radius is split into `radiusX` and `radiusY`. It also supports `corner1 + corner2`, `box`, and `boundingBox`.

| Form | Best for |
| --- | --- |
| `{ center, radiusX, radiusY }` | Known center and radii |
| `{ center, diameterX, diameterY }` | Known diameters |
| `{ corner1, corner2 }` | Fit inside a box |
| `{ box }` / `{ boundingBox }` | Describe the box with `x/y/origin + width + height` |
| + `inset` / `outset` | Shrink or expand the box before fitting |
| + angles + `closed="sector"` | Draw a partial ellipse whose arc ends connect to the center |

## How it works

> Optional for everyday use; revisit this section when a point form throws an error, or when you want to mix these shapes with `<Path>`.

### Some forms pass points through, others compute first

Some forms pass the point directly to the underlying Path. They can use node ids, polar points, or plain `[x, y]` coordinates:

```tsx
<Circle center="A" radius={24} />
<Ellipse corner1="A" corner2="B" />
```

Other forms must compute a midpoint, size, or bounding box in React first. These forms only accept literal `[x, y]` coordinates:

```tsx
<Circle from={[-40, 0]} to={[40, 0]} />
```

Passing a node id to a computed form throws an error instead of guessing a result.

### Partial shapes have three closing modes

When angles are present, `closed` decides how the arc ends are connected:

| Value | Effect |
| --- | --- |
| `chord` | Default, draw a straight chord between the arc ends |
| `sector` | Draw from the arc end to the center, then back to the arc start |
| `open` | Draw only the arc |

### They are still Path

Every circle or ellipse expands to an equivalent `<Path>` / `<Step>` sequence. That means they can use `stroke`, `fill`, `arrow`, and `zIndex` like any other path.

## API Reference

| Component | Combination | Meaning |
| --- | --- | --- |
| `<Circle>` | `center` + `radius` | Circle from center and radius |
| | `center` + `diameter` | Circle from center and diameter |
| | `from` + `to` | Circle from two points (diameter endpoints) |
| | `corner1` + `corner2` | Largest circle inscribed in the box |
| | `box` / `boundingBox` | Circle from a box, with `fit` / `inset` / `outset` |
| | + `fit?` | `contain` uses the shorter side, `cover` uses the longer side |
| | + `inset?` / `outset?` | Shrink or expand the box before fitting |
| | + angles (`startAngle` / `endAngle` / `sweepAngle`, any two) + `closed?` | Partial circle: `chord` / `sector` / `open` |
| `<Ellipse>` | `center` + `radiusX` + `radiusY` | Ellipse from center and two radii |
| | `center` + `diameterX` + `diameterY` | Ellipse from center and two diameters |
| | `corner1` + `corner2` | Ellipse inscribed in the box |
| | `box` / `boundingBox` | Ellipse from a box, with `inset` / `outset` |
| | + `inset?` / `outset?` | Shrink or expand the box before fitting |
| | + angles + `closed?` | Partial ellipse: `chord` / `sector` / `open` |
