A regular polygon (equal sides and angles) has two faces: draw an outline with `<RegularPolygon>` (sugar), or bound a node with `shape={{ type:'polygon', params:{ sides } }}`. `diamond` is the preset for `polygon{ sides:4, rotate:45 }`. See [Shapes](/core/components/shapes) for the full definition. Reach for sugar to draw a polygon outline, and for `<Node shape>` to get a connectable, labelable polygon node.

## As a Path shape (sugar)

`<RegularPolygon>` draws an equal-sided polygon at a position, expanding to an underlying `<Path>`.

### Usage

```tsx
import { RegularPolygon } from '@retikz/react';
```

```tsx
<RegularPolygon center={[0, 0]} radius={36} sides={6} />
```

### Examples

`<RegularPolygon>` uses either a circumradius or a side length. `sides` must be at least `3`, and `rotate` changes the overall orientation.

| Form | Best for |
| --- | --- |
| `{ center, radius, sides }` | Known circumradius |
| `{ center, sideLength, sides }` | Known side length |

### API Reference

| Component | Combination | Meaning |
| --- | --- | --- |
| `<RegularPolygon>` | `center` + `radius` + `sides` | Polygon from circumradius and side count |
| | `center` + `sideLength` + `sides` | Polygon from side length and side count |
| | + `rotate?` | Overall start angle |

## As a node shape

When you need a polygon **node** (to hold text, to be referenced by a connection) rather than a line, set `shape={{ type:'polygon', params }}` on `<Node>`. The boundary takes the regular polygon that circumscribes "text + padding", sized by the text.

### Usage

```tsx
import { Node } from '@retikz/react';
```

```tsx
<Node shape={{ type: 'polygon', params: { sides: 6 } }}>label</Node>
<Node shape="diamond">label</Node>
```

### Examples

`diamond` is the 4-side, 45°-rotated preset, writable as the bare string `shape="diamond"`, equivalent to `{ type:'polygon', params:{ sides:4, rotate:45 } }`.

<ComponentPreview name="node-polygon" size="sm" />

Give `params.cornerRadius` a positive number to round every vertex; omitting it or `0` keeps sharp corners. The rounding is connection-aware: connection endpoints land on the fillet arc rather than the sharp corner that has been rounded away.

<ComponentPreview name="node-polygon-corner-radius" size="sm" />

Connection endpoints snap to the polygon edge, and the node's 9 standard anchors (`north` / `center` …) still apply. For the full `<Node>` props see [Node](/core/components/node/overview).

### API Reference

| `shape` | Boundary | Parameters |
| --- | --- | --- |
| `{ type: 'polygon', params }` | Regular polygon circumscribing the text | `sides` (≥3) / `rotate?` / `cornerRadius?` |
| `"diamond"` | = `polygon{ sides:4, rotate:45 }` preset | None |

`cornerRadius` (user units) rounds every vertex; omitted / `0` = sharp corners, clamped per corner to the largest non-self-intersecting radius.

## How it works

> Optional for everyday use; revisit when a point form throws an error or the rotation direction looks surprising.

### RegularPolygon computes vertices first (sugar)

`<RegularPolygon>` computes all vertices from the center, radius or side length, then expands to a closed Path. `rotate` changes the starting vertex angle; it is not an SVG transform.

### diamond is a preset of polygon

`<Node shape="diamond">` normalizes at compile time to `polygon{ sides:4, rotate:45 }` — a square box rotated 45°. For a non-square text box it stays a square (rather than a stretched rhombus); the bare string `"diamond"` and the explicit `{ type:'polygon', params:{ sides:4, rotate:45 } }` are equivalent.
