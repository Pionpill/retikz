A star alternates points between an outer and an inner radius, with two faces: draw an outline with `<Star>` (sugar), or bound a node with `shape={{ type:'star', params }}`. See [Shapes](/core/components/shapes) for the full definition. Reach for sugar to draw a star outline, and for `<Node shape>` to get a connectable star element.

## As a Path shape (sugar)

`<Star>` alternates points on an outer and inner radius, connects them into a closed path, and expands to an underlying `<Path>`. Useful for rating stars, emphasis marks, and decorative multi-point shapes.

### Usage

```tsx
import { Star } from '@retikz/react';
```

```tsx
<Star center={[0, 0]} outerRadius={36} innerRadius={16} points={5} />
```

### Examples

`points` is the number of star points. Give `innerRadius` directly, or use `innerRatio` to derive it from the outer radius.

| Form | Best for |
| --- | --- |
| `{ center, outerRadius, innerRadius, points }` | Control both radii |
| `{ center, outerRadius, points, innerRatio? }` | Derive inner radius from a ratio |

## As a node shape

When you need a star **node** (referenceable by a connection, usable as an icon glyph) rather than a line, set `shape={{ type:'star', params }}` on `<Node>`. It is sized by the radius parameters, **independent of any text content**.

### Usage

```tsx
import { Node } from '@retikz/react';
```

```tsx
<Node shape={{ type: 'star', params: { points: 5, innerRadius: 16, outerRadius: 40 } }} />
```

### Examples

| shape | params | Notes |
| --- | --- | --- |
| star | `{ points, innerRadius, outerRadius, rotate? }` | Star with alternating inner / outer radii |

<ComponentPreview name="node-star" size="sm" />

Give `params.cornerRadius` a positive number to round both the outer tips and the inner notches at once; omitting it or `0` keeps sharp corners. The rounding is connection-aware: an endpoint aimed at a tip lands on the fillet arc rather than the sharp point that has been rounded away.

<ComponentPreview name="node-star-corner-radius" size="sm" />

Besides the node's 9 standard anchors (`north` / `center` â€¦), the star provides geometry-named anchors for precise connections via `{ id, anchor }`: `tip-N` (the Nth point) / `notch-N` (the Nth notch) / `center`. These named anchors stay at the original tip / notch logical positions and do not move inward with `cornerRadius`. For the full `<Node>` props see [Node](/core/components/node/overview).

### API Reference

| `shape` | params | Boundary / use |
| --- | --- | --- |
| `{ type: 'star', params }` | `points` / `innerRadius` / `outerRadius` / `rotate?` / `cornerRadius?` | Connectable star glyph |

`cornerRadius` (user units) rounds both tips and notches; omitted / `0` = sharp corners, clamped per corner.

## How it works

> Optional for everyday use; revisit when the orientation or inner-radius ratio looks surprising.

`<Star>` computes `points * 2` vertices: outer vertices land on `outerRadius`, and inner vertices land on `innerRadius`. The default `innerRatio` is `0.5`, so the inner radius is half of the outer radius. `rotate` changes the starting angle of the first outer vertex; by default one outer point faces upward, and in screen coordinates positive angles rotate clockwise.

`<Node shape="star">` shares the same geometry as `<Star>`; the only difference is that the former is a node with a boundary that connections can snap to.
