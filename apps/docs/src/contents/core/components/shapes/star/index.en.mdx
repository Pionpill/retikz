title: Star
description: Two ways to make a star — draw an outline with Star, or give a node a star boundary.
---

A star alternates points between an outer and an inner radius, and retikz gives you two forms:

- **Draw an outline** — the sugar component `<Star>` draws a star at a position, expanding to an underlying `<Path>`.
- **A node with that boundary** — `<Node shape={{ type:'star', params }}>` gives a node bounded by a star, referenceable by a connection and usable as a rating / emphasis glyph.

In short: reach for `<Star>` when you just need a star outline, and for `<Node shape>` when you need a connectable star element.

## Usage

```tsx
import { Star, Node } from '@retikz/react';
```

```tsx
{/* Draw an outline */}
<Star center={[0, 0]} outerRadius={36} innerRadius={16} points={5} />

{/* A node with a star boundary */}
<Node shape={{ type: 'star', params: { points: 5, innerRadius: 16, outerRadius: 40 } }} />
```

## Examples

### As a shape: Star

`points` is the number of star points. Give `innerRadius` directly, or use `innerRatio` to derive it from the outer radius.

| Form | Best for |
| --- | --- |
| `{ center, outerRadius, innerRadius, points }` | Control both radii |
| `{ center, outerRadius, points, innerRatio? }` | Derive inner radius from a ratio |

### As a node shape: shape="star"

When you need a star **node** (referenceable by a connection, usable as an icon glyph) rather than a line, set `shape={{ type:'star', params }}` on `<Node>`. It is sized by the radius parameters, **independent of any text content**.

| shape | params | Notes |
| --- | --- | --- |
| star | `{ points, innerRadius, outerRadius, rotate? }` | Star with alternating inner / outer radii |

<ComponentPreview name="node-star" size="sm" />

Besides the node's 9 standard anchors (`north` / `center` …), the star provides geometry-named anchors for precise connections via `{ id, anchor }`: `tip-N` (the Nth point) / `notch-N` (the Nth notch) / `center`. For the full `<Node>` props see [Node](/core/components/node/overview).

## How it works

> Optional for everyday use; revisit this section when the orientation or inner-radius ratio looks surprising.

`<Star>` computes `points * 2` vertices: outer vertices land on `outerRadius`, and inner vertices land on `innerRadius`. The default `innerRatio` is `0.5`, so the inner radius is half of the outer radius.

`rotate` changes the starting angle of the first outer vertex. By default, one outer point faces upward; in screen coordinates, positive angles rotate clockwise. `<Node shape="star">` shares the same geometry as `<Star>`; the only difference is that the former is a node with a boundary that connections can snap to.

## API Reference

### As a shape: Star

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `center` | `[number, number]` | — | Star center |
| `outerRadius` | `number` | — | Outer radius |
| `innerRadius` | `number` | — | Inner radius; wins when provided |
| `innerRatio` | `number` | `0.5` | Inner radius as a ratio of `outerRadius` when `innerRadius` is omitted |
| `points` | `number` | — | Number of star points |
| `rotate` | `number` | `-90` | Starting angle |

### As a node shape

| `shape` | params | Boundary / use |
| --- | --- | --- |
| `{ type: 'star', params }` | `points` / `innerRadius` / `outerRadius` / `rotate?` | Connectable star glyph |

> For the full `<Node>` props see [Node](/core/components/node/overview).
