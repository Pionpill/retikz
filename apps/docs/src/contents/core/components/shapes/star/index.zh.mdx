星形在外半径和内半径之间交替取点，有两副面孔：用 `<Star>` 画一个星形轮廓（Sugar），或用 `shape={{ type:'star', params }}` 让节点带上星形边界。完整定义见 [形状](/core/components/shapes)。要画一个星形，用 Sugar；要一个可连接的星形图元，用 `<Node shape>`。

## 作为 Path 图形（Sugar）

`<Star>` 在外半径和内半径之间交替取点，连成闭合路径、展开成底层 `<Path>`。适合评级星、强调标记、装饰性多角形。

### 用法

```tsx
import { Star } from '@retikz/react';
```

```tsx
<Star center={[0, 0]} outerRadius={36} innerRadius={16} points={5} />
```

### 例子

`points` 是星角数量。你可以显式给 `innerRadius`，也可以用 `innerRatio` 按比例自动算内半径。

| 写法 | 适合 |
| --- | --- |
| `{ center, outerRadius, innerRadius, points }` | 自己控制内外半径 |
| `{ center, outerRadius, points, innerRatio? }` | 用比例自动计算内半径 |

<ComponentPreview name="star-basic" size="sm" />

### API 参考

| 属性 | 类型 | 默认值 | 描述 |
| --- | --- | --- | --- |
| `center` | `[number, number]` | — | 星形中心 |
| `outerRadius` | `number` | — | 外半径 |
| `innerRadius` | `number` | — | 内半径；显式给出时优先 |
| `innerRatio` | `number` | `0.5` | 未给 `innerRadius` 时，内半径相对外半径的比例 |
| `points` | `number` | — | 星角数量 |
| `rotate` | `number` | `-90` | 起始角度 |

## 作为 Node 形状

当你要的是一个星形**节点**（可被连线引用、当图标 glyph）而不是一条线时，给 `<Node>` 设 `shape={{ type:'star', params }}`。尺寸由半径参数定、**不依赖文字内容**。

### 用法

```tsx
import { Node } from '@retikz/react';
```

```tsx
<Node shape={{ type: 'star', params: { points: 5, innerRadius: 16, outerRadius: 40 } }} />
```

### 例子

| shape | params | 说明 |
| --- | --- | --- |
| star | `{ points, innerRadius, outerRadius, rotate? }` | 内外半径交替的星形 |

<ComponentPreview name="node-star" size="sm" />

给 `params.cornerRadius` 一个正数会同时磨圆外凸尖角（tip）和内凹槽（notch），省略 / `0` 是硬角。倒角是连接感知的：朝某个尖角方向**自动贴边**（不写 anchor）的连线端点会落在圆角弧上、停在尖角之前，而不是原始尖尖；显式连命名锚点 `tip-N` 则**仍落在原始尖角**（恒在逻辑顶点、不随倒角内移）。

<ComponentPreview name="node-star-corner-radius" size="md" />

除节点的 9 个标准 anchor（`north` / `center` …）外，星形还提供按几何命名的 anchor，可用 `{ id, anchor }` 精确连线：`tip-N`（第 N 个尖角）/ `notch-N`（第 N 个凹角）/ `center`。这些命名锚点固定在原始尖角 / 凹角的逻辑位置，不随 `cornerRadius` 内移。完整的 `<Node>` 属性见 [Node](/core/components/node/overview)。

### API 参考

| `shape` | params | 边界 / 用途 |
| --- | --- | --- |
| `{ type: 'star', params }` | `points` / `innerRadius` / `outerRadius` / `rotate?` / `cornerRadius?` | 可连接的星形 glyph |

`cornerRadius`（用户单位）同时给尖角与凹角倒角，省略 / `0` = 硬角，逐角夹紧。

## 技术原理

> 日常使用可以跳过本节；只有在星形朝向或内外半径比例不符合预期时再回来读。

`<Star>` 会先计算 `points * 2` 个顶点：外点落在 `outerRadius` 上，内点落在 `innerRadius` 上。默认 `innerRatio` 是 `0.5`，也就是内半径为外半径的一半。`rotate` 改变第一个外点的起始角度；默认起始角让一个外点朝上，在屏幕坐标里正角度顺时针旋转。

`<Node shape="star">` 与 `<Star>` 几何一致，区别只是前者是有边界、可连线的节点。
