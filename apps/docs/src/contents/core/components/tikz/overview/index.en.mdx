`<TikZ>` is retikz's only required top-level wrapper. Every Kernel / Sugar component must live inside a `<TikZ>`.

`<TikZ>` is the entry point between the DSL and the renderer: with children, it converts the inner component tree into IR; with `ir`, it uses that pre-built IR directly. It then compiles the IR into a Scene and hands it to the current renderer. `width` / `height` control the rendered output's displayed size in the page; internal coordinates stay in retikz user units, and the compiler uses layout bounds to compute the visible range.

## Usage

```tsx
import { TikZ } from '@retikz/react';
```

```tsx
<TikZ width={300} height={80}>
  {/* Kernel / Sugar children */}
</TikZ>
```

## Composition

`<TikZ>` is a container; children are any Kernel / Sugar component tree:

```tsx
<TikZ>
  <Node id="..." position={...}>label</Node>
  <Path>
    <Step kind="move" to="..." />
    <Step kind="line" to="..." />
  </Path>
  <Draw way={[...]} />
</TikZ>
```

### Render from IR

## API Reference

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `width` | `number \| string` | ‚Äî | Rendered output width |
| `height` | `number \| string` | ‚Äî | Rendered output height |
| `children` | `ReactNode` | ‚Äî | Kernel / Sugar component tree (mutually exclusive with `ir`) |
| `ir` | `IR` | ‚Äî | Pre-built IR JSON (mutually exclusive with `children`) |
| `className` | `string` | ‚Äî | Forwarded to the render root |
| `style` | `CSSProperties` | ‚Äî | Forwarded to the render root |

### Feeding IR Directly

The `ir` prop fits scenarios where IR JSON is already in hand ‚Äî AI generation, JSON editors, URL deserialization. Pair with `convertReactNodeToIR` / `convertIRToReactNode` for a DSL ‚Üî IR bridge:

```tsx
import { TikZ, Node, convertReactNodeToIR } from '@retikz/react';

const ir = convertReactNodeToIR(
  <Node id="a" position={[0, 0]}>A</Node>,
);
// Serialize, send to LLM, store in DB...
<TikZ ir={ir} width={200} height={80} />;
```

`ir` and `children` are mutually exclusive; if both are supplied, `ir` wins.

If a diagram looks clipped or has too much empty space, inspect **View Code ‚Üí IR** first: the usual causes are internal coordinates far away from visible primitives, invisible coordinates that do not extend bounds, or explicit coordinates without a visible boundary around them.
