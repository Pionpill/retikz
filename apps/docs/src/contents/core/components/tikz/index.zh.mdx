`<TikZ>` 是 retikz 唯一必需的顶层包装，所有 Kernel / Sugar 组件都必须放在它里面。

## 用法

```tsx
import { TikZ } from '@retikz/react';
```

```tsx
<TikZ width={300} height={80}>
  {/* Kernel / Sugar children */}
</TikZ>
```

## 组合

`<TikZ>` 是容器；children 是任意 Kernel / Sugar 组件树：

```tsx
<TikZ>
  <Node id="..." position={...}>label</Node>
  <Path>
    <Step kind="move" to="..." />
    <Step kind="line" to="..." />
  </Path>
  <Draw way={[...]} />
</TikZ>
```

### 从 IR 渲染

`<TikZ>` 接受 `ir` prop——直接把一段已经持久化的 IR JSON 渲染成图，**不写一行 JSX children**。来源不限：LLM 生成、JSON 编辑器、URL 反序列化、数据库存档都可以。

## API 参考

| 属性 | 类型 | 默认值 | 描述 |
| --- | --- | --- | --- |
| `width` | `number \| string` | — | SVG 元素宽度 |
| `height` | `number \| string` | — | SVG 元素高度 |
| `children` | `ReactNode` | — | Kernel / Sugar 组件树（与 `ir` 二选一） |
| `ir` | `IR` | — | 已构造好的 IR JSON（与 `children` 二选一） |
| `className` | `string` | — | 透传到 `<svg>` |
| `style` | `CSSProperties` | — | 透传到 `<svg>` |

## 直接喂 IR

`ir` prop 适合"已经有 IR JSON"的场景——AI 生成、JSON 编辑器、URL 反序列化。配合 `convertReactNodeToIR` / `convertIRToReactNode` 做 DSL ↔ IR 双向桥：

```tsx
import { TikZ, Node, convertReactNodeToIR } from '@retikz/react';

const ir = convertReactNodeToIR(
  <Node id="a" position={[0, 0]}>A</Node>,
);
// 序列化、传 LLM、存数据库……
<TikZ ir={ir} width={200} height={80} />;
```

`ir` 与 `children` 互斥；同时传时优先 `ir`。
