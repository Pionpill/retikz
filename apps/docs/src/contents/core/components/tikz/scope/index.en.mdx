`<Scope>` groups a set of IR children at the IR layer and applies a local transform chain to them.

## Usage

```tsx
import { Scope } from '@retikz/react';
```

```tsx
<Scope transforms={[{ kind: 'translate', x: 50, y: 0 }]}>
  {/* Node / Path / Coordinate / nested Scope */}
</Scope>
```

A `<Scope></Scope>` with no props and no children ŌåÆ compile emits no Scene primitive; any non-empty field ŌåÆ a `GroupPrim` is emitted, no empty-container overhead.

### When to use Scope

When your design has a "grouping" concept ŌĆö a set of nodes / paths that conceptually belong to the same sub-diagram, need to be transformed as a unit, or need to be referenced / positioned as a whole from outside via `scope.id`.

<ComponentAlert
  type="warn"
  title="Don't wrap a Scope just for code-level organization"
  description="Wrapping a Scope without a shared transform or external reference handle only adds namespace pitfalls and gives no geometric or referencing benefit."
/>

### The 6 transform variants

The 4 translate variants mirror the 4 `<Node position>` shapes (Cartesian / polar / direction-relative / offset) one-to-one.

| kind | Fields | Semantics |
|---|---|---|
| `translate` | `{ x, y }` | Cartesian translation in user units |
| `polar-translate` | `{ origin?, angle, radius }` | Polar translation; `origin` accepts `id` / `[x, y]` / `PolarPosition`; defaults to origin point |
| `at-translate` | `{ direction, of, distance? }` | Place scope at `distance` from referent node along `direction`; `distance` defaults to `<TikZ nodeDistance>` |
| `offset-translate` | `{ of, offset? }` | Scope origin = referent global position + `[dx, dy]`; `offset` defaults to `[0, 0]` |
| `rotate` | `{ degrees, cx?, cy? }` | Rotate by degrees; `cx/cy` default to scope local origin |
| `scale` | `{ x, y? }` | Scale; `y` defaults to `x` (uniform) |

Array order is application order ŌĆö the 0-th entry takes effect first (innermost). `transforms: [translate, rotate]` means translate then rotate; compile flattening / lowering details are in ┬¦ How it works ┬¦ŃĆītransforms lowering to SceneŃĆŹ.

<Comparison target="tikz" title="Scope mapping">

`<Scope transforms={[{ kind: 'translate', x: 50, y: 0 }]}>...</Scope>` maps to TikZ `\begin{scope}[shift={(50,0)}] ... \end{scope}`: a local transform is pushed at the start of the subdiagram and popped at the end. TikZ stacks CM in the order options appear; retikz applies `transforms` in array index order, reusing the SVG / Scene convention.

### Style inheritance

A scope doubles as a style-default anchor, exposing three style surfaces to its children: a **master color `color`** plus shared sub-fields that cascade to every element; **four typed defaults** `nodeDefault` / `pathDefault` / `labelDefault` / `arrowDefault` dispatched by element type, flat and independent; and **`resetStyle`** to cut outer inheritance.

#### Master color cascade

`<Scope color>` cascades the current color to every inner element ŌĆö node border / text, path stroke / arrow / edge label all default to it; individual fields (`stroke` / `fill` / `textColor`) override the master when set explicitly:

<ComponentPreview name="scope-style-cascade" size="md" />

The master color also tints node `fill`; the anchor nodes above set `fill="white"` to cut that fill so their text stays readable. What follows the color is the **master `color`**, not `stroke` ŌĆö `<Path color>` colors the label / arrow too, while `<Path stroke>` only paints the line and leaves the label unchanged.

#### Typed defaults

`nodeDefault` / `pathDefault` / `labelDefault` / `arrowDefault` each attach defaults to one element type, flat and independent. `nodeDefault` carries node-only fields like shape / spacing (a bare `color` can't reach `shape` ŌĆö use `nodeDefault`); arrows go through the separate `arrowDefault`, so `pathDefault` excludes them:

<ComponentPreview name="scope-style-typed-defaults" size="md" />

#### resetStyle inheritance barrier

`resetStyle` cuts the outer scope's inheritance (master-color cascade + typed defaults) back to the built-in baseline; the scope's own values and inner scopes still apply. `true` cuts all; pass a channel array (`['node' | 'path' | 'label' | 'arrow']`) to cut per type:

<ComponentPreview name="scope-style-reset" size="md" />

`resetStyle` only cuts the scope inheritance, **not the structural relation between an element and its host** ŌĆö labels / arrows still follow their owning path / node's resolved color: `resetStyle={['label']}` only ignores the outer `labelDefault`, the label still follows its line. Priority chain / master-color expansion / opacity-does-not-compound details are in ┬¦ How it works ┬¦ŃĆīStyle inheritance resolutionŃĆŹ.

## How it works

<ComponentAlert
  type="tip"
  title="Optional on first read"
  description="Revisit when you hit edge cases or want to see how scope flattens transforms, registers namespaces, and isolates / cascades ids at compile time."
/>

### Namespace stack and lookup

A scope does **not** introduce a namespace boundary by default ŌĆö child node / coordinate / nested scope.id ids all register into the current top frame; ids from multiple stacked scopes flow up to the `<TikZ>` root frame, and outer paths can reference them directly. Only `localNamespace: true` makes the scope push its own frame and isolate the subtree.

<ComponentPreview name="scope-namespace-stack" hideCode />

<Comparison target="tikz" title="Default namespace behavior">

TikZ pgf's `\begin{scope}` does not introduce a local namespace ŌĆö a `\node (foo) {...}` registered inside is globally visible, and outer code can `\draw (foo) -- ...` directly. retikz keeps this convention so v0.1 IR continues to work without changing id references after being wrapped in a scope; opt in to `localNamespace` only when subdiagram reuse or template-style isolation is needed.

</Comparison>

| Mechanism | Behavior |
|---|---|
| **Inside-out lookup** | An inner path's id lookup walks the frame stack from top to bottom ŌĆö inner code **can** see outer ids; outer code **cannot** see inner ids |
| **Shadowing** | Registering the same id in an inner frame is not a conflict and emits no warning; lookups hit the inner copy first; the outer id becomes visible again once the frame is popped |
| **scope.id always registers in the parent frame** | scope.id is the external handle and is **not** affected by its own `localNamespace`; outer paths can always reference `<Scope id="X">` as `X`, regardless of whether the scope uses local namespacing internally |
| **Stack frames** | Every `<Scope localNamespace>` pushes a child frame; the frame is popped when the scope subtree finishes compiling ŌĆö id references beyond the level naturally break |

### Duplicate id and conflict handling

When the same id is registered more than once in the same frame (any combination of node ├Ś node, node ├Ś coordinate, node ├Ś scope.id), compile emits a `DUPLICATE_NODE_ID` warning, and **the later definition overrides the earlier one** (last-wins).

| Rule | Behavior |
|---|---|
| Trigger | The 2nd and subsequent registers of the same id within the same frame |
| Handling | Warning is collected via `CompileOptions.onWarn`; compile **does not throw**; nodeIndex keeps the last definition |
| Warn unit | Every duplicate emits its own warning (not merged); each carries two IR locators (earlier and current) plus the current frame depth |
| Same id across frames | Counts as shadowing rather than duplicate, **no warning** is emitted |
| scope.id self-reference | bbox is computed only after the scope subtree finishes ŌĆö `at.of='self'` / `offset.of='self'` etc. fail forward-resolution and throw (not subject to last-wins) |

### Relative positioning projection

When a `<Node>` / `<Coordinate>` / `<Path>` `Step.to` inside a scope uses `PolarPosition` / `AtPosition` / `OffsetPosition`, retikz resolves it under the following rules:

1. **Referent is global**: the `polar.origin` / `at.of` / `offset.of` referent (node id / scope.id / coordinate literal) is resolved to its **global** coordinate
2. **Relative part is measured in the current scope's local frame**: polar's `(angle, radius)`, at's `(direction, distance)`, offset's `[dx, dy]` are all expressed in the **current node's enclosing scope's local coordinate system**
3. **The end result is projected back to global via the current scope's chain**: the referent's global point is first inversely projected to the current scope's local frame, the relative part is added, then the sum is forward-projected back to global

Math: `final = scopeTransform(inverseScopeTransform(referent_global) + relative_part)`

<Comparison target="tikz" title="Relative positioning inside a scope">

Maps to TikZ `\begin{scope}[rotate=90] \node at ($(hub) + (50:0)$) {orbit}; \end{scope}` ŌĆö `(hub)` resolves to a global coordinate (unaffected by scope rotate), and `+ (50:0)` is measured in the current scope's local frame. retikz reuses this convention: referents are globally visible, while relative parts rotate / scale together with the scope.

</Comparison>

Edge cases:

| Case | Behavior |
|---|---|
| **Nested polar / nested offset** | The whole nested chain is measured in the **current node's enclosing scope's local frame** ŌĆö the basis scope does not switch as you recurse |
| **Cartesian `Position` literal** | Inside a scope, `position={[10, 0]}` keeps v0.1 behavior ŌĆö `[10, 0]` is measured in the current scope's local frame and projected to global via the scope chain |
| **Scope translate does not change relative vectors** | translate is neither rotation nor scaling ŌĆö `relative=(10, 0)` inside a `translate(50, 0)` scope keeps the same visual direction and length |
| **Scope `scale=0`** | The inverse projection is undefined; retikz falls back to local origin (0, 0) to avoid NaN ŌĆö avoid zero scale in practice |
| **Relative `Step.to` in a path** | Same rules as `Node.position` ŌĆö when a step.to is polar / offset / `{ relative: [dx, dy] }`, the relative part is measured in the path's enclosing scope's local frame |

### scope.id synthetic bbox

When `<Scope id="X">` is set, compile collects the axis-aligned **global bounding box** of all child layouts (nodes / coordinates / nested scope.id bboxes) at the end of Pass 1 and registers a synthetic `rectangle` `NodeLayout` named `X` into the **parent** namespace frame.

| Item | Behavior |
|---|---|
| **Compute timing** | End of Pass 1 recursion of the scope subtree (after every inner node / coordinate / nested scope.id is registered) |
| **bbox input** | The 4 corners (with accumulated rotate / scale in global coordinates) of every `NodeLayout` in the subtree, fed into an AABB |
| **Not counted in bbox** | Path endpoints (move / line `to`) ŌĆö matches TikZ `local bounding box`'s default |
| **Register location** | **Parent frame** (not affected by its own `localNamespace`) ŌĆö scope.id is the external handle |
| **Shape** | Synthetic rectangle `NodeLayout` with `rect.rotate=0` (axis-aligned) |
| **As anchor referent** | All 9 named anchors plus numeric angles share the rect anchor path ŌĆö `'X.north'` / `'X.30'` / no-suffix uses `boundaryPoint` clip |
| **As polar / at / offset referent** | Resolves to bbox center, same as a normal rectangle node |
| **Nesting** | Each scope.id keeps its own bbox ŌĆö outer bbox covers all inner layouts; inner bbox covers only its own subtree |
| **Empty scope.id** | If the subtree has no layouts, the bbox degenerates to a 0├Ś0 placeholder (the scope's local origin projected to global via the chain) |

<Comparison target="tikz" title="scope.id mapping">

`<Scope id="cluster">` maps to TikZ `\begin{scope}[local bounding box=cluster] ... \end{scope}`: pgf collects the axis-aligned bounding box of everything drawn inside and registers a rectangle node named `cluster`. retikz reuses the existing scope `id` field as the single entry point ŌĆö no extra `boundingBox` flag ŌĆö so the field stays a one-job thing (set id ŌćÆ get a referenceable bbox handle).

</Comparison>

### transforms lowering to Scene

The 6 IR variants are uniformly flattened to 3 Scene variants (Cartesian translate / rotate / scale) at compile time, then lowered into `GroupPrim.transforms`:

| IR shape | Compile handling | Scene shape |
|---|---|---|
| `translate` | Pass through | `translate(x, y)` |
| `polar-translate` | `resolvePosition` resolves origin global + (angle, radius) ŌåÆ Cartesian (x, y) | `translate(x, y)` |
| `at-translate` | `resolvePosition` resolves referent global + direction + distance ŌåÆ Cartesian (x, y) | `translate(x, y)` |
| `offset-translate` | `resolvePosition` resolves referent global + (dx, dy) ŌåÆ Cartesian (x, y) | `translate(x, y)` |
| `rotate` | Pass through | `rotate(degrees, cx, cy)` |
| `scale` | Pass through | `scale(x, y)` |

The `transforms` array is applied in order ŌĆö 0-th first (innermost), matching SVG `transform="t1 t2 t3"` / Scene `GroupPrim.transforms`. `transforms: [translate, rotate]` is equivalent to SVG `transform="translate(...) rotate(...)"`: translate first, then rotate.

External nodeIndex sees node layouts already in **global coordinates** (compile has projected through the chain); where a path primitive inside a scope lands depends on that scope's transform chain: in a **transform-free scope** the path is backfilled into that scope's `GroupPrim.children`, preserving declaration order relative to sibling nodes; in a **transformed scope** the endpoints are already global, so the path is still hoisted to the top-level `primitives` (not into the GroupPrim, to avoid double application of the transform), and its cross-frame z-order does not fully match IR order (a known limitation).

### Style inheritance resolution

Compile keeps a style frame stack, pushing one frame per scope, and resolves each element's final style **inside-out, per field**. Priority chain (per field, nearest wins):

```
element field > element color > typed-default field > typed-default color > scope cascade field > scope color > built-in
```

Inner overrides outer; within one frame the typed default overrides the cascade.

| Mechanism | Behavior |
|---|---|
| **Master-color expansion** | Within each style source, a sub-color (stroke / fill / textColor, arrow color) not set explicitly takes that source's `color` ŌĆö same-source fields override the master |
| **Color cascade** | A container's resolved color flows down as the child's color default: node.color ŌåÆ text / edge label; path.color ŌåÆ stroke / arrow / step label |
| **Missing vs explicit** | Decided by presence (not truthiness): missing defers outward; explicit `none` / `0` / `false` cuts at this layer |
| **opacity replaces, not compounds** | Nested scope opacity replaces (0.5 ŌŖā 0.5 ŌåÆ 0.5 each, not 0.25), matching TikZ default; only label ├Ś element opacity multiplies within an element |
| **resetStyle keeps the host axis** | Only the scope inheritance axis is cut; labels / arrows still follow the host path / node resolved color, never orphaned |
