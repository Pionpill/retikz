title: Scope
description: 局部 transform 容器——把一组 children 包起来批量平移 / 旋转 / 缩放，对应 TikZ `\begin{scope}[...]`。
---

`<Scope>` 是 retikz Kernel 的第四件（与 `<Node>` / `<Path>` / `<Coordinate>` 平级）。它在 IR 层把一组子元素聚成容器，对它们应用局部 transform 链；对应 TikZ `\begin{scope}[shift=..., rotate=..., scale=...] ... \end{scope}`。

## 用法

```tsx
import { Scope } from '@retikz/react';
```

```tsx
<Scope transforms={[{ kind: 'translate', x: 50, y: 0 }]}>
  {/* Node / Path / Coordinate / 嵌套 Scope */}
</Scope>
```

裸 `<Scope></Scope>`（无 children、无 transforms、无 id）→ compile 不发任何 Scene primitive；任一非空 → 发 `GroupPrim`，不会留下空容器开销。

### 何时用 Scope

- **批量平移子图**：一组节点 + path 形成 cluster，整体移动只改 scope 的 transform，组内节点 position 不动
- **整体旋转**：把一段子图绕 scope 局部原点旋转，组内连线 / 标签自动跟随
- **嵌套坐标系**：外层定位 cluster，内层再定位 cluster 内子节点，层级表达比手算绝对坐标清晰

### transforms 6 变体

4 个 translate 变体一一镜像 `<Node position>` 的 4 种形态（笛卡尔 / 极坐标 / 方向相对 / 偏移），在 compile 时统一展平为笛卡尔 translate 再下沉到 Scene `GroupPrim`；`rotate` / `scale` 直接透传。

| kind | 字段 | 语义 |
|---|---|---|
| `translate` | `{ x, y }` | 笛卡尔平移，user units |
| `polar-translate` | `{ origin?, angle, radius }` | 极坐标平移，`origin` 为 `id` / `[x,y]` / `PolarPosition`，缺省 = 原点 |
| `at-translate` | `{ direction, of, distance? }` | 落到 referent 节点的某方向 distance 处，`distance` 缺省走 `<TikZ nodeDistance>` |
| `offset-translate` | `{ of, offset? }` | scope 原点 = referent 全局坐标 + `[dx, dy]`，`offset` 缺省 = `[0, 0]` |
| `rotate` | `{ degrees, cx?, cy? }` | 旋转 degrees 度，`cx/cy` 缺省 = scope 局部原点 |
| `scale` | `{ x, y? }` | 缩放，`y` 缺省 = `x`（等比） |

数组顺序应用——第 0 个先生效（最内层），与 SVG transform list / Scene `GroupPrim` 一致。`transforms: [translate, rotate]` 表示先 translate 再 rotate；与 SVG `transform="translate(...) rotate(...)"` 同语义。

<Comparison target="tikz" title="Scope 对照">

`<Scope transforms={[{ kind: 'translate', x: 50, y: 0 }]}>...</Scope>` 对应 TikZ 的 `\begin{scope}[shift={(50,0)}] ... \end{scope}`：在子图开始处压一层局部 transform，结束时弹出。TikZ 多个 option 写一起按出现顺序叠加 CM；retikz `transforms` 数组按 index 顺序应用，复用 SVG / Scene 约定。

</Comparison>

### 作为引用整体

`<Scope id="cluster">` 设值后，compile 在 Pass 1 子树结束时收集子树**全部** layout（node / coordinate / 嵌套 scope.id bbox）的轴对齐**全局 bounding box**，注册为一条 `rectangle` synthetic `NodeLayout`（id = `cluster`）到**父 namespace frame**。从外部看，`cluster` 与普通 rectangle 节点完全一致——path 可指 `cluster.north` / `cluster.east` / `cluster.30`，其他 node 可把 `cluster` 用作 polar / direction / offset 定位 referent。

<Comparison target="tikz" title="scope.id 对照">

`<Scope id="cluster">` 对应 TikZ 的 `\begin{scope}[local bounding box=cluster] ... \end{scope}`：pgf 收集 scope 内所有元素的轴对齐 bbox 并注册一个名为 `cluster` 的矩形节点。retikz 复用已有的 scope `id` 字段作为唯一入口——不另加 `boundingBox` flag——保持字段单一职责（设 id ⇒ 拿到可引用的 bbox 句柄）。

</Comparison>

外部 path 用 `'cluster.<anchor>'` 指到**整体** cluster 视觉边界，而非任何单个内部节点：

<ComponentPreview name="scope-id-reference" size="md" />

- bbox 锚点（`.north` / `.east` / `.south-west` / `.30` 等）与普通 rectangle 节点完全一致——9 个命名 anchor + 数字角度均走共享的 rect anchor 路径
- `'cluster'` 不带 anchor 后缀时用 `boundaryPoint` 朝源方向 clip
- path 走线端点（move/line `to`）**不计入** bbox——只有 node / coordinate / 嵌套 scope.id bbox 参与，与 TikZ `local bounding box` 默认行为一致

嵌套时每条 scope.id 各算各的 bbox；外层 scope.id 包含全部内层 layout，内层 scope.id 独立——outer bbox 含 `A` 加 `inner` 子树全部内容；`inner.bbox` 仅含 `B` 与 `C`：

<ComponentPreview name="scope-id-nested" size="md" />

scope.id 用作 `polar.origin` / `at.of` / `offset.of` referent 时取 bbox **中心**——与普通 rectangle 节点一致：

<ComponentPreview name="scope-id-referent" size="md" />

> scope.id 与 `node.id` / `coordinate.id` 共享扁平全局命名空间——同名 `<Node id="cluster">` 或兄弟 `<Scope id="cluster">` 触发 `DUPLICATE_NODE_ID` warn 走 last-wins（见 [重复 id 处理](#重复-id-处理)）。前向引用解析失败抛错，因为 bbox 在 scope 子树完成后才能算出。

### 内部相对定位

scope 内 `<Node>` / `<Coordinate>` / `<Path>` `Step.to` 使用 `PolarPosition` / `AtPosition` / `OffsetPosition` 时按以下规则解析：

1. **referent 取全局**：`polar.origin` / `at.of` / `offset.of` 引用的 referent（节点 id / scope.id / 坐标字面量）取**全局**坐标
2. **relative 部分在当前 scope 局部度量**：polar 的 `(angle, radius)` / at 的 `(direction, distance)` / offset 的 `[dx, dy]` 都在**当前 node 所属 scope 局部坐标系**内表达
3. **末端 apply 当前 scope chain 投回全局**：referent 全局点先反向投影到当前 scope 局部，加 relative，再正向投影回全局

数学：`final = scopeTransform(inverseScopeTransform(referent_global) + relative_part)`

<Comparison target="tikz" title="scope 内相对定位">

对应 TikZ `\begin{scope}[rotate=90] \node at ($(hub) + (50:0)$) {orbit}; \end{scope}`——`(hub)` 引用解析为全局坐标（不受 scope rotate 影响），`+ (50:0)` 在当前 scope 局部坐标系内度量——前者全局、后者局部。retikz 复用此约定：referent 全局可见，relative 跟着 scope rotate / scale 一起旋转 / 缩放。

</Comparison>

referent `hub` 在 scope 外（全局原点）；scope `rotate(45°)` 内的 `orbit` 用 polar `{ origin: 'hub', angle: 0, radius: 80 }`——relative 局部右 80，scope rotate 后视觉落到右下 45 度方向。同样 polar 写在 scope 外（参考 `ref` 节点）则直接在 hub 全局右 80：

<ComponentPreview name="scope-position-polar-in-rotate" size="md" />

`direction: 'right'` 是**当前 scope 局部**的 right——scope rotate 90 度后视觉变 down，B 落在 A 下方 60；scope 外的 `B-ref` 在 A 全局右侧 60：

<ComponentPreview name="scope-position-at-cross-scope" size="md" />

`offset: [40, 0]` 在 scope 局部度量；scope `scale(2)` 后视觉偏移 (80, 0)——B 落在 anchor 右 80。scope 外的 `B-ref` offset 量按未缩放计算：

<ComponentPreview name="scope-position-offset-with-scale" size="md" />

细节规则：

- **嵌套 polar / 嵌套 offset**：整条嵌套链都在**当前 node 所属 scope 局部**度量——不切换基准 scope
- **`Position` 笛卡尔字面量**：scope 内 `position={[10, 0]}` 与 v0.1 一致——`[10, 0]` 在当前 scope 局部度量，经 scope chain 投影到全局
- **scope translate 不改 relative 矢量**：translate 不旋转 / 不缩放——`relative=(10, 0)` 在 translate(50, 0) 内视觉仍是 (10, 0) 方向 / 长度
- **scope `scale=0`**：反向投影未定义——retikz 退化为局部 (0, 0) 防 NaN；实践中避免 0 scale
- **path `step.to` 内的相对定位**：与 node position 同规则——path step.to 用 polar / offset / 笛卡尔 / `{ relative: [dx, dy] }` 字面量时，relative 部分在当前 path 所属 scope 局部度量

## 命名空间与隔离

scope 默认**不引入命名空间边界**——子 node / coordinate / 嵌套 scope.id 全部注册到当前栈顶 frame；多层 scope 串起来 id 仍汇到 `<TikZ>` 根 frame，外部 path 可直接引用。

<Comparison target="tikz" title="命名空间默认行为">

TikZ pgf 的 `\begin{scope}` 不引入局部命名空间——scope 内 `\node (foo) {...}` 注册的 `foo` 全局可见、scope 外可直接 `\draw (foo) -- ...`。retikz 沿用此约定，让 v0.1 已写好的 IR 嵌入 scope 后无需改 id 引用即可继续工作；需要"子图复用 / 模板化"再 opt-in `localNamespace`。

</Comparison>

需要"子图复用 / 模板化"时显式 `localNamespace: true` 让 scope 创建本地 frame，内部 id 与外部隔离：

<ComponentPreview name="scope-local-namespace-basic" size="md" />

- **inside-out lookup**：内层 path 引用字符串 id 时从当前 frame 向栈底逐层查找——内部**可见**外部 id（shadowing），外部**不可见**内部 id
- **shadowing**：内层 frame 注册同名 id 不算冲突、不发 warn；lookup 命中内层；frame 退出后外层 id 重新可见
- **scope.id 始终在父 frame 注册**：scope 是外部句柄，不受自身 `localNamespace` 影响——外部 path 永远可以引用 `<Scope id="X">` 的 `X`，无论其内部是否启用本地命名空间

嵌套 localNamespace 形成栈式 namespace frame：

<ComponentPreview name="scope-local-namespace-nested" size="md" />

最内层 path 引用 `L2-node` / `L1-node` / `root-anchor` 分别由最内层 / 中层 / 根 frame 命中——栈搜索遇到第一个匹配即返回。

当每层 `localNamespace` 都注册同名 id 时，inside-out lookup 自动让内层 shadowing 外层；跨 frame 同 id 不算 duplicate、不发 warn：

<ComponentPreview name="scope-local-namespace-shadowing" size="md" />

## 重复 id 处理

同一 frame 内重复出现的 id（node × node、node × coordinate、node × scope.id 任意组合）触发 `DUPLICATE_NODE_ID` warning（通过 `CompileOptions.onWarn` 收集），**后定义覆盖前定义**（last-wins）：

<ComponentPreview name="scope-duplicate-id-warn" size="md" />

- compile 不抛错——LLM 生成 / 模板复制场景下能跑出可用结果
- 每次重复都单独发一条 warn（不合并），含两条 IR locator 区分前后位置 + 当前 frame 深度
- 跨 frame 同 id（如外层 + `<Scope localNamespace>` 内同名）是 shadowing 不是 duplicate，**不发 warn**

> v0.1 行为：同名 id 直接 throw。alpha.1 起改为 warn + last-wins，迁移时 v0.1 代码 0 改动；如需 strict 行为，可在 `onWarn` 里检测 `code === 'DUPLICATE_NODE_ID'` 主动 throw。

## API 参考

| 属性 | 类型 | 默认值 | 描述 |
| --- | --- | --- | --- |
| `transforms` | `Array<Transform>` | — | 局部 transform 列表，6 变体；数组顺序应用（第 0 个最先生效） |
| `children` | `ReactNode` | — | scope 内子元素：`<Node>` / `<Path>` / `<Coordinate>` / 嵌套 `<Scope>` |
| `id` | `string` | — | 可选 scope 引用 id；设值后 scope 子树全部 layout 的**轴对齐全局 bbox** 注册为 synthetic rectangle layout 到**父 frame**，外部 path 可用 `'cluster'`（boundary clip 后的 bbox 边界点）/ `'cluster.<anchor>'`（9 个命名 anchor）/ `'cluster.<deg>'`（数字角度）引用。其他 node 可把 scope.id 用作 `polar.origin` / `at.of` / `offset.of` referent（取 bbox 中心） |
| `localNamespace` | `boolean` | `false` | 是否创建本地命名空间：`true` 时 scope 推一个子 frame，内部 id 与外部隔离（外部不可见、内部可 inside-out 看外部）；缺省 false = 全局扁平 |
