`<Node>` 是有命名、有位置的最基本图元。其它路径用它的 `id` 引用，连线时按 shape 自动贴边界。

## 用法

```tsx
import { Node } from '@retikz/react';
```

```tsx
<Node id="a" position={[0, 0]}>
  Label
</Node>
```

### shape 多态

`shape` 控制节点形状：`'rectangle'`（默认）/ `'circle'` / `'ellipse'` / `'diamond'`。任意 shape 的节点都被 path / draw 当作有边界的图元，连线端点自动贴对应形状的边：

<ComponentPreview name="node-shapes" />

> 形状的"外接边界"包住"text 矩形 + padding"——
> - rectangle: 边界 = 内框
> - circle: 外接圆，半径 = √(innerHalfW² + innerHalfH²)
> - ellipse: 外接椭圆，rx = innerHalfW × √2、ry = innerHalfH × √2
> - diamond: 外接菱形，halfA = 2 × innerHalfW、halfB = 2 × innerHalfH

### 多行文本

`text` 既可以是字符串（单行），也可以是字符串数组（一元素一行）。`align` 控制多行块内各行水平对齐方式（`'left'` / `'center'`（默认）/ `'right'`），`lineHeight` 控制相邻行的间距，未设走 `font.size × 1.2`。

<ComponentPreview name="node-multiline" />

> 选数组而非 `'\n'` 字符串：JSON 友好（无 escape 困扰）；行级属性扩展空间预留。

也可以用 `children` 直接写文字，三种姿势随便挑：

```tsx
{/* 字符串带 \n */}
<Node>{'Line 1\nLine 2'}</Node>

{/* 模板字面量（真换行） */}
<Node>{`Line 1
Line 2`}</Node>

{/* 数组 */}
<Node>{['Line 1', 'Line 2']}</Node>
```

`text` prop 优先级高于 children；二者都给时取 `text`。

### 行级样式

每行可独立覆盖 `fill` / `opacity` / `font`（family / size / weight / style 任意子集），未填字段继承块级。两条等价路径：

1. **`text` 数组里混 LineSpec 对象**——AI / JSON 友好，schema 直接表达
2. **`<Text>` 子元素**——JSX 写起来顺手；和字符串 children 平等参与，按 JSX 顺序排列

```tsx
{/* 路径 1：text 数组 + LineSpec 对象 */}
<Node text={[
  { text: 'Heading', font: { weight: 'bold', size: 16 }, fill: '#dc2626' },
  'body line 1',
  { text: 'note', font: { style: 'italic' }, opacity: 0.6 },
]} />

{/* 路径 2：<Text> 子元素 */}
<Node>
  <Text fill="#dc2626" font={{ weight: 'bold' }}>Heading</Text>
  body line 1
  <Text font={{ style: 'italic' }} opacity={0.6}>note</Text>
</Node>
```

> 块级 `align` / `lineHeight` 不能行级覆盖（多行块整体属性）。

### 内 / 外边距

内 / 外边距支持**对称别名**与**轴特化**两套写法（解析顺序：轴特化 ?? 别名 ?? 默认）：

- 内边距：`padding`（对称） · `innerXSep` / `innerYSep`（分轴）
- 外边距：`margin`（对称） · `outerSep`（分轴目前只有 outer 一个；TikZ 也是单值）

90% 场景写 `padding={N}` 就够；做 button-like 扁宽节点等少数场景再用轴特化。

## API 参考

| 属性 | 类型 | 默认值 | 描述 |
| --- | --- | --- | --- |
| `id` | `string` | — | 唯一标识；被 `<Path>` / `<Draw>` 用来引用 |
| `shape` | `'rectangle' \| 'circle' \| 'ellipse' \| 'diamond'` | `'rectangle'` | 节点形状 |
| `position` | `[x, y] \| PolarPosition` | — | 节点中心位置（笛卡尔或极坐标） |
| `children` / `text` | `string \| Array<string>` | — | 节点内可见文字；字符串单行 / 数组多行；`text` 优先于 `children` |
| `align` | `'left' \| 'center' \| 'right'` | `'center'` | 多行文本块内各行水平对齐方式 |
| `lineHeight` | `number` | `font.size × 1.2` | 多行块的行高（user units） |
| `rotate` | `number` | `0` | 绕中心旋转角度（度，正值顺时针） |
| `fill` | `string` | — | 背景色（任意 CSS 颜色） |
| `stroke` | `string` | `currentColor` | 边框色 |
| `strokeWidth` | `number` | `1` | 边框宽度（user units） |
| `innerXSep` | `number` | — | 横向内边距（text → 左右 border）；不填走 `padding` 兜底 |
| `innerYSep` | `number` | — | 纵向内边距（text → 上下 border）；不填走 `padding` 兜底 |
| `outerSep` | `number` | — | 外边距（border → path 附着点）；不填走 `margin` 兜底 |
| `padding` | `number` | `8` | 内边距对称别名；轴特化字段优先 |
| `margin` | `number` | `0` | 外边距对称别名；`outerSep` 优先；**不影响 border 位置**，只影响连线起止 |
| `font` | `{ family?, size?, weight?, style? }` | — | 字体规格；嵌套对象，所有字段可选 |

## margin 与 padding 的区别

| | 影响什么 | 视觉表现 |
| --- | --- | --- |
| `padding` | 文字到 border 的距离 | border 变大 |
| `margin` | border 到 path 端点的距离 | border 不变；连线提前 margin 停 |

`margin` 用来在多节点流程图里给连线"留呼吸空间"——不让线贴死 border。
