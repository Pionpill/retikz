`<Node>` is the basic positioned, named primitive. Paths reference it by `id` and auto-attach to the per-shape border.

## Usage

```tsx
import { Node } from '@retikz/react';
```

```tsx
<Node id="a" position={[0, 0]}>
  Label
</Node>
```

### Shape polymorphism

`shape` picks the visual shape: `'rectangle'` (default) / `'circle'` / `'ellipse'` / `'diamond'`. Any shape works as an attachable target — connecting lines clip to that shape's boundary automatically:

<ComponentPreview name="node-shapes" />

> Each shape's outer boundary contains "text rectangle + padding":
> - rectangle: boundary = inner rect
> - circle: circumscribed, radius = √(innerHalfW² + innerHalfH²)
> - ellipse: circumscribed, rx = innerHalfW × √2, ry = innerHalfH × √2
> - diamond: circumscribed, halfA = 2 × innerHalfW, halfB = 2 × innerHalfH

### Multi-line text

`text` accepts a string (one line) or an array of strings (one line each). `align` controls horizontal alignment within the text block (`'left'` / `'center'` (default) / `'right'`), and `lineHeight` controls vertical spacing between lines, defaulting to `font.size × 1.2`.

<ComponentPreview name="node-multiline" />

> Arrays beat `'\n'`-delimited strings: JSON-friendly (no escape headaches), and leaves room for per-line attributes later.

You can also pass the text as `children` — pick whichever form reads best:

```tsx
{/* string with \n */}
<Node>{'Line 1\nLine 2'}</Node>

{/* template literal (real newlines) */}
<Node>{`Line 1
Line 2`}</Node>

{/* array */}
<Node>{['Line 1', 'Line 2']}</Node>
```

`text` wins over `children` whenever both are supplied.

### Per-line styling

Each line can override `fill` / `opacity` / `font` (any subset of family / size / weight / style); unset fields inherit block defaults. Two equivalent paths:

1. **Mix LineSpec objects into the `text` array** — JSON / AI-friendly, schema-expressible
2. **`<Text>` child elements** — natural JSX; participates equally with string children, JSX order preserved

```tsx
{/* Path 1: text array with LineSpec objects */}
<Node text={[
  { text: 'Heading', font: { weight: 'bold', size: 16 }, fill: '#dc2626' },
  'body line 1',
  { text: 'note', font: { style: 'italic' }, opacity: 0.6 },
]} />

{/* Path 2: <Text> children */}
<Node>
  <Text fill="#dc2626" font={{ weight: 'bold' }}>Heading</Text>
  body line 1
  <Text font={{ style: 'italic' }} opacity={0.6}>note</Text>
</Node>
```

<ComponentPreview name="node-styled-lines" />

> Block-level `align` / `lineHeight` cannot be overridden per line — they apply to the whole multi-line block.

### Inner / outer sep

Both **symmetric aliases** and **axis-specific** forms are supported (resolution order: axis-specific ?? alias ?? default):

- Inner: `padding` (symmetric) · `innerXSep` / `innerYSep` (per-axis)
- Outer: `margin` (symmetric) · `outerSep` (single — TikZ also keeps a single outer)

90% of the time `padding={N}` is enough; reach for axis-specific only when you need button-like wide-but-flat nodes etc.

### Colors / opacity

- `textColor` — text color (block default; per-line `LineSpec.fill` still overrides)
- `opacity` — whole-node 0..1 (applies to shape + text)
- `fillOpacity` / `drawOpacity` — fill-only / stroke-only opacity

### Size constraints + scale

- `roundedCorners` — corner radius (only on `rectangle` shape)
- `minimumWidth` / `minimumHeight` / `minimumSize` — floor the bbox without shrinking natural text width
- `scale` / `xScale` / `yScale` — uniform / per-axis scale; affects bbox, font size, padding, and path attachment

## API Reference

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `id` | `string` | — | Unique identifier; referenced by `<Path>` / `<Draw>` |
| `shape` | `'rectangle' \| 'circle' \| 'ellipse' \| 'diamond'` | `'rectangle'` | Node shape |
| `position` | `[x, y] \| PolarPosition` | — | Node center (Cartesian or polar) |
| `children` / `text` | `string \| Array<string>` | — | Visible label; string = one line, array = one element per line; `text` wins over `children` |
| `align` | `'left' \| 'center' \| 'right'` | `'center'` | Horizontal alignment of lines within the multi-line text block |
| `lineHeight` | `number` | `font.size × 1.2` | Line height of the multi-line block (user units) |
| `rotate` | `number` | `0` | Rotation around center, degrees, positive = clockwise |
| `fill` | `string` | — | Background color (any CSS color) |
| `stroke` | `string` | `currentColor` | Border color |
| `strokeWidth` | `number` | `1` | Border width (user units) |
| `innerXSep` | `number` | — | Horizontal inner padding (text → left/right border); falls back to `padding` |
| `innerYSep` | `number` | — | Vertical inner padding (text → top/bottom border); falls back to `padding` |
| `outerSep` | `number` | — | Outer margin (border → path attachment point); falls back to `margin` |
| `padding` | `number` | `8` | Symmetric inner padding alias; axis-specific fields take precedence |
| `margin` | `number` | `0` | Symmetric outer margin alias; `outerSep` takes precedence; **does not affect border position**, only path endpoints |
| `font` | `{ family?, size?, weight?, style? }` | — | Font spec; nested object, all fields optional |
| `textColor` | `string` | `currentColor` | Text color (block default; per-line override via `LineSpec.fill`) |
| `opacity` | `number` | — | Whole-node 0..1, applies to both shape and text |
| `fillOpacity` | `number` | — | Fill-only opacity 0..1 |
| `drawOpacity` | `number` | — | Stroke-only opacity 0..1 (TikZ `draw opacity`) |
| `dashed` | `boolean` | — | Dashed border preset ("4 2") |
| `dotted` | `boolean` | — | Dotted border preset ("1 2") |
| `dashArray` | `string` | — | Explicit SVG `stroke-dasharray`; highest priority |
| `roundedCorners` | `number` | — | Corner radius (only on `rectangle` shape) |
| `minimumWidth` / `minimumHeight` | `number` | — | Floors the bbox dimensions |
| `minimumSize` | `number` | — | Symmetric minimum-size alias; axis-specific fields take precedence |
| `scale` | `number` | `1` | Uniform scale (includes font, padding, path attachment) |
| `xScale` / `yScale` | `number` | — | Per-axis scale; takes precedence over `scale` |

## margin vs padding

| | Affects | Visual effect |
| --- | --- | --- |
| `padding` | Distance from text to border | Border grows |
| `margin` | Distance from border to path endpoint | Border unchanged; lines stop `margin` short |

Use `margin` to give connecting lines breathing room in multi-node diagrams — keeps lines from touching the border directly.
