A node is what the rest of a figure *references*: it has an id, a position, and a shape boundary, and connectors auto-attach to that boundary by id. This group gathers the node itself, the per-line text styling that lives inside it, and the placeholder point that names a spot without drawing anything.

<ComponentPreview name="node-family" size="md" hideCode />

## What each part does

| Component | Type | Responsibility | Use it when |
| --- | --- | --- | --- |
| Node | Kernel | Named primitive with position / shape / text; paths attach to its boundary by id | Flowchart nodes, shapes with text, anything a connector must reference |
| Text | Sugar | Lives inside a Node and overrides fill / opacity / font for a single line | Mixing several styles within one line of text |
| Coordinate | Kernel | Names a point without drawing any shape | Pure reference anchors, helper positioning, no visible geometry needed |
