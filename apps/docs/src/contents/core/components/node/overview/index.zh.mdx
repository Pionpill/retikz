title: Node
description: 可绑定 id 的命名图元：拥有位置、形状边界与可选文本，Path / Draw 会通过 id 引用并自动贴合它的边界。
---

`<Node>` 是有命名、有位置、有几何边界的最基本图元。其它路径用它的 `id` 引用，连线时按 shape 自动贴合对应边界。

## 用法

```tsx
import { Node } from '@retikz/react';
```

```tsx
<Node id="a" position={[0, 0]}>
  Label
</Node>
```

先按图元角色选择写法：流程图节点通常用默认 rectangle + `padding`；纯文字标签用 `stroke="none"` / `fill="none"`；按钮感节点加 `cornerRadius` + `minimumWidth`；说明框适合多行 `text` + `align="left"`；如果只需要一个可被引用的点，而不需要可见形状，用 [Coordinate](/core/components/node/coordinate) 更合适。

### 定位

#### 相对定位

`position` 除了笛卡尔 / 极坐标还接受 **`{ direction, of, distance? }`** 形态；不需要手算每个绝对坐标，描述节点关系即可。

- `direction` 接 8 方向：`above` / `below` / `left` / `right` / `above-left` / `above-right` / `below-left` / `below-right`
- `of` 是被参照节点 / coordinate 的 id（要求先出现在 IR 中，前向引用拒绝）
- `distance` 缺省时取 `<Layout nodeDistance>`，再缺省回退到 1

<Comparison target="tikz" title="相对定位">

`{ direction: 'above', of: 'a', distance: 20 }` 对应 TikZ positioning library 的 `[above=20 of a]` 一类写法。`<Layout nodeDistance={...}>` 对应 TikZ 的全局 `node distance` 缺省值。

</Comparison>

<ComponentPreview name="node-at-relative" size="sm" />

#### 偏移定位

`position` 还接受 **`{ of, offset }`** 形态——任意 `(dx, dy)` 相对基准点偏移。当 `at` 的 8 方向 + 单标量距离表达不下需求时（"A 右 30 下 10" 这种），用 offset 就够。

- `of` 是基准点，三种形态（与 `PolarPosition.origin` 同形）：
  - 字符串 id——引用命名节点 / coordinate（前向引用拒绝）
  - `[x, y]` 笛卡尔字面值——直接坐标基准，无需任何节点定义
  - `PolarPosition`（`{ angle, radius, origin? }`）——极坐标作基准，递归解析
- `offset` 是 `[dx, dy]` 二元组，user units

<Comparison target="tikz" title="偏移定位">

`{ of: 'a', offset: [10, 5] }` 对应 TikZ `calc` library 的 `($(a) + (10,5)$)`。

</Comparison>

<ComponentPreview name="node-offset-position" size="sm" />

#### 比例定位

`position` 还接受 **`{ between, t }`**：把节点放在两个端点连线的比例位置上，`t=0.5` 就是中点。端点可用节点 id、坐标、极坐标、offset 或嵌套 between；更适合辅助点和中点标注的完整示例见 [Coordinate](/core/components/node/coordinate)。

### 形状

节点的几何形状决定它的视觉外观，也决定连线如何贴边。

#### shape 多态

形状在 retikz 有两副面孔——既能用 Sugar 画一条 Path，也能作为 Node 边界，完整定义见 [形状](/core/components/shapes)。本节只讲它作为节点形状的用法。

`shape` 控制节点形状，两种写法：

- **裸字符串**（无参形状）：`'rectangle'`（默认）/ `'circle'` / `'ellipse'` / `'diamond'` / `'polygon'` / `'sector'` / `'arc'` / `'star'`，或经 `<Layout shapes>` / `CompileOptions.shapes` 注册的自定义名。
- **`{ type, params }` 对象**（带参形状）：当形状需要额外几何参数（边数 / 内外半径 / 起止角…）时，用 `type` 选形状、`params` 给参数。

不论哪种写法，节点都被 path / draw 当作有边界的图元，连线端点自动贴对应形状的边。

按"尺寸从哪来"分两类：**文本容器形状**（rectangle / circle / ellipse / polygon / diamond）边界包住「文字 + padding」、尺寸由文字驱动；**params 驱动的几何形状**（sector / arc / star）尺寸由半径 / 角度定、不依赖文字，可当作可连接的图形 glyph。

<ComponentPreview name="node-shapes" size="md" />

<ComponentPreview name="node-shape-params" size="md" />

每种形状的参数、anchor 和示例见对应形状页：

| shape | 写法 | 文档 |
| --- | --- | --- |
| rectangle | 默认 / `'rectangle'` | [Rectangle](/core/components/shapes/rectangle) |
| circle / ellipse | `'circle'` / `'ellipse'` | [Circle / Ellipse](/core/components/shapes/circle-ellipse) |
| sector / arc | `{ type:'sector' \| 'arc', params }` | [Arc / Sector](/core/components/shapes/arc-sector) |

| star | `{ type:'star', params }` | [Star](/core/components/shapes/star) |

`shape` 还接受内置之外的注册名（hexagon / cylinder / cloud…）——任意字符串通过 schema 校验，定义经 `CompileOptions.shapes` / `<Layout shapes>` 注入，未注册名编译期报错。带参注册形状用 `{ type, params }` 把参数传进去。见 [自定义形状](/core/reference/extending/shape-registry)。

#### 连接面 boundary

默认情况下，连线贴的就是节点视觉形状的边界。当你想让边按另一种形状贴边、却不改节点真实占位时，用 `boundary` 指定一个独立的"连接面"。常见场景是星形 / 多边形这类带凹角的形状——你想保留尖角的视觉，但希望连线像贴圆一样均匀汇入，而不是钻进凹槽。

`boundary` 取值与 `shape` 同形。常用是前四个免参取值，另可借任意已注册形状：

| 取值 | 连接面 | 效果 |
| --- | --- | --- |
| `'shape'`（默认） | 视觉形状自身 | 边贴真实轮廓——星形 / 多边形会落在尖端或钻进凹边 |
| `'circle'` | 真圆，半径 = 节点外接盒较长半轴 | 边均匀贴一整圈，不受凹角 / 旋转影响 |
| `'rectangle'` | 外接矩形（AABB） | 边贴水平 / 垂直盒边 |
| `'ellipse'` | 按 AABB 宽高比的椭圆 | 边贴椭圆，随宽高比拉伸 |
| 注册 shape 名 / `{ type, params }` | 借该形状的边界套在节点外接盒上 | 借 `polygon` / `star` 等带参形状需自给 params；按 AABB 自动适配尺寸不在范围内 |

同一五角星分设四种连接面，各引一条边从左下 45° 斜贴上来——`shape` 落在星形真实轮廓（最内）、`rectangle` 顶到外接盒角（最外）、`circle` / `ellipse` 居中（近正方节点下两者接近）。斜向才看得出区别：竖直 / 水平轴向上椭圆与矩形必然同点、圆又取较长半轴，三者会重合：

<ComponentPreview name="node-boundary-surfaces" size="sm" />

连接面**不改节点占位**：节点的包围盒、视觉形状、布局位置都不变，只改"边在哪贴、compass 锚点怎么解析"。形状专属锚点（如星形的 `tip-0`）和边上比例点始终走视觉形状，不受 `boundary` 影响。

下例五角星 `boundary="circle"`：默认实线连到外接圆上；端点写 `boundary="shape"` 的虚线则改贴真实尖端。

<ComponentPreview name="node-boundary" size="sm" />

单条边想覆盖目标节点的 `boundary` 时，在 path 端点对象写 `boundary`——只影响这一个端点，取值同节点 `boundary`。详见 [Path](/core/components/draw/path) 的端点 `boundary` 说明。

### 文本

#### 字体

`font` 是嵌套对象 `{ family?, size?, weight?, style? }`——粗体 / 斜体 / 字族都在一个字段里设：

<ComponentPreview name="node-font" size="xs" />

#### 多行文本

`text` 可以是字符串，也可以是字符串数组（一元素一行）。字符串里的 `'\n'` 是硬换行——一个字符串也能渲染成多行。`align` 控制多行块内各行水平对齐方式（`'left'` / `'center'`（默认）/ `'right'`），`lineHeight` 控制相邻行的间距，未设走 `font.size × 1.2`。

<ComponentPreview name="node-multiline" size="xs" />

> 选数组而非 `'\n'` 字符串：JSON 友好（无 escape 困扰）；行级属性扩展空间预留。

也可以用 `children` 直接写文字，三种姿势随便挑：

```tsx
{/* 字符串带 \n */}
<Node>{'Line 1\nLine 2'}</Node>

{/* 模板字面量（真换行） */}
<Node>{`Line 1
Line 2`}</Node>

{/* 数组 */}
<Node>{['Line 1', 'Line 2']}</Node>
```

`text` prop 优先级高于 children；二者都给时取 `text`。

#### 自动换行

`maxTextWidth` 给文本设一个**折行阈值**（user units）：超过阈值才折行（西文按词、CJK 按字），短文本盒宽收缩到实际内容、不撑满。它**不是固定段落宽**——这点与 TikZ `text width` 不同。

<ComponentPreview name="node-max-text-width" size="sm" />

> 一个不可断的长词（无空白的长串）超过阈值时按整体溢出，不强行截断。

#### 行级样式

每行可独立覆盖 `fill` / `opacity` / `font`（family / size / weight / style 任意子集），未填字段继承块级。两条等价路径：

1. **`text` 数组里混 LineSpec 对象**——AI / JSON 友好，schema 直接表达
2. **`<Text>` 子元素**——JSX 写起来顺手；详见 [Text 章节](/core/components/node/text)

<ComponentPreview name="node-styled-lines" size="xs" />

> 块级 `align` / `lineHeight` 不能行级覆盖（多行块整体属性）。

多行文本套到不同 shape 上，外接边界沿用各自 shape 多态规则（外接圆 / 外接椭圆 / 多边形外接）：

### 节点标签

`label` 在节点边界外挂额外文字。单对象或数组形态；`position` 接 8 方向枚举或数字角度；`font` / `textColor` 缺省时从 node 继承。

<Comparison target="tikz" title="节点标签">

`label={{ text: 'foo', position: 'above' }}` 对应 TikZ `[label=above:foo]`；`position: 30` 对应 `label=30:foo`。

</Comparison>

<ComponentPreview name="node-label-basic" size="xs" />

`distance` 控制标签从节点边界继续向外推开的距离，单位仍是 retikz user units；省略时默认为 `12`。例如 `position: 'above', distance: 20` 会从节点上边界继续向上放置标签。

<ComponentPreview name="node-label-distance" size="xs" />

#### 朝向

`label.rotate` 让标签文字绕**自身中心**自旋：`'radial'` 沿"节点中心 → 标签中心"方向、`'tangent'` 取其切向（radial + 90°）、数字为显式角度（度）；`keepUpright` 在文字会倒置时翻 180° 保持可读。`rotate` 只改朝向，不改由 `position` / `distance` 决定的位置。

<ComponentPreview name="node-label-rotate" size="sm" />

#### 引脚

`label.pin` 给标签加一条**引线**连回节点边界（TikZ `pin`）：`pin: true` 是默认细实线（继承标签色）；`pin: { stroke, strokeWidth, dashPattern }` 自定义引线样式；不写 / `false` 则无引线。标签摆放仍由 `position` / `distance` 决定，pin 只是多画一条引线。

<Comparison target="tikz" title="引脚">

`label={{ text: 'foo', position: 'right', pin: true }}` 对应 TikZ `[pin=right:foo]`。

</Comparison>

<ComponentPreview name="node-pin" size="md" />

### 尺寸与变换

#### 内 / 外边距

内 / 外边距支持**对称别名**与**轴特化**两套写法。若同时提供，分轴字段优先；没有分轴字段时使用对称别名；两者都没有时使用默认值：

- 内边距：`padding`（对称） · `innerXSep` / `innerYSep`（分轴）
- 外边距：`margin`（对称） · `outerSep`（分轴目前只有 outer 一个）

多数节点只需要写 `padding={N}`。当水平留白和垂直留白需要不同数值时，再改用 `innerXSep` / `innerYSep`。

<ComponentPreview name="node-sep" size="xs" />

<Comparison target="tikz" title="Sep 命名">

`innerXSep` / `innerYSep` / `outerSep` 沿用 TikZ 的 sep 词汇；retikz 另外保留 `padding` / `margin` 作为更接近 CSS 的对称别名。

</Comparison>

#### 尺寸约束 + 缩放

- `cornerRadius`——圆角半径。`rectangle` / `polygon` / `star` / `sector` 支持圆角（`ellipse` 等不支持），写在形状 params 里：`shape={{ type:'polygon', params:{ sides:6, cornerRadius:8 } }}`、`star` / `sector` / `rectangle` 同理。顶层 `cornerRadius` 只对默认 `rectangle` 生效，两者并存时 params 优先
- `minimumWidth` / `minimumHeight` / `minimumSize`——撑开 bbox（不缩水 text 自然宽度）
- `scale` / `xScale` / `yScale`——均匀 / 分轴缩放，影响 bbox + 字号 + padding + 路径附着点

<ComponentPreview name="node-size-constraints" size="xs" />

#### 旋转

`rotate` 绕节点中心旋转；连线会跟着新边界点贴：

<ComponentPreview name="node-rotated" size="xs" />

#### 栈序（zIndex）

默认渲染层叠 = 声明顺序（SVG 只认绘制顺序）。`zIndex` 显式覆盖：大者在上、缺省 0、同值保持声明顺序；只在同层（同 scope / 顶层）子节点间生效。例如把某个高亮节点 `zIndex={10}` 浮到所有连线之上，而不必把它挪到 JSX 末尾。`<Path>` / `<Scope>` 同样支持 `zIndex`（scope 的 `zIndex` 作用于整组在父层的位置，不影响 scope 内部相对栈序）。

### 样式

`fill` / `stroke` / `strokeWidth` 等样式 prop 控制单个节点的观感：

<ComponentPreview name="node-styled" size="xs" />

#### 颜色 / 不透明度

- `textColor`——文字颜色（块级默认；行级 LineSpec.fill 仍可覆盖）
- `opacity`——整节点 0~1（同时作用于 shape + text）
- `fillOpacity` / `drawOpacity`——分别只透明填充或描边

<ComponentPreview name="node-color-opacity" size="xs" />

#### 渐变填充（Paint）

`fill` 除纯色字符串外还接 **PaintSpec 对象**——线性 / 径向渐变、图案 pattern、图片 image。相同 PaintSpec 自动去重，渲染为 SVG `<defs>` 里的 paint server。`<Path fill>` 同样支持。

- 线性：`{ kind: 'linearGradient', angle?, stops }`——`angle` 度数（0 = 左→右，90 = 上→下）
- 径向：`{ kind: 'radialGradient', center?, radius?, stops }`——`center` / `radius` 用 0..1 相对值
- `stops`：`{ offset (0..1), color, opacity? }` 数组，至少 2 个；`color` 可用 `currentColor`

<ComponentPreview name="node-paint" size="sm" />

`stop` 的 `opacity` 可做"实色 → 透明"渐隐；`color` 用 `currentColor` 让渐变跟随元素 `color`、随主题切换：

<ComponentPreview name="node-paint-stops" size="sm" />

`fill` 还接 **图案 pattern** 和 **图片 image**：

- 图案：`{ kind:'pattern', shape:'lines'|'dots'|'grid', color?, background?, size?, lineWidth?, rotation? }`——`color` 缺省 `currentColor`、`size` 是 tile 周期、`rotation` 旋转整片
- 图片：`{ kind:'image', href, fit? }`——`href` 为 URL（含 data URI），`fit`：`cover`（默认）/ `contain` / `fill`

<ComponentPreview name="node-pattern-image" size="sm" />

`href` 接任意外部图片 URL（http(s)）或 data URI；`fit` 控制图片如何映射到形状盒——同一张外部横图（`picsum.photos`，320×200）按三种 fit 填进方形盒：

<ComponentPreview name="node-image-fit" size="sm" />

> `fit` 按形状盒归一计算：方形盒时与 CSS `object-fit` 一致；强非方形盒下 cover/contain 为近似（按归一方形而非真实长宽比）。源图与盒同形（都方/都横）时三种 fit 视觉无差异。
>
> image 走运行期 `<image href>`，外部 URL 在浏览器侧加载；SSR / 导出 PNG 时的图片内联待后续。

`dashed` / `dotted` 是预设；`dashPattern` 是显式 dash/gap 长度序列（优先级：`dashPattern` > `dashed` > `dotted`）：

## API 参考

| 属性 | 类型 | 默认值 | 描述 |
| --- | --- | --- | --- |
| `id` | `string` | — | 唯一标识；被 `<Path>` / `<Draw>` 用来引用 |
| `meta` | `Record<string, JSON 值>` | — | provenance 元数据：随该节点原样透传到渲染产物，renderer 忽略、不参与布局 / 连接 / 样式；典型由 plot 等高层注入（标记来自哪个 datum / series / layer），供交互层命中图元后反查来源。须为 JSON 可序列化对象 |
| `shape` | `'rectangle' \| 'circle' \| 'ellipse' \| 'diamond' \| 'polygon' \| 'sector' \| 'arc' \| 'star' \| string`<br />`\| { type, params }` | `'rectangle'` | 节点形状：裸字符串（无参形状 / 注册名）或 `{ type, params }`（带参形状）；内置见上，或经 `<Layout shapes>` / `CompileOptions.shapes` 注册 |
| `boundary` | `'shape' \| 'circle' \| string \| { type, params }` | `'shape'` | 连接面：边在此节点如何贴边、compass 锚点如何解析，独立于视觉 `shape`；`'shape'` = 视觉形状自身，`'circle'` = 真圆（半径 = 外接盒较长半轴），其它走借用的 shape 边界。不改节点占位；形状专属锚点与边上比例点仍走视觉形状。单条边可用端点 `boundary` 覆盖 |
| `position` | `[x, y] \| PolarPosition \| { direction, of, distance? } \| { of, offset } \| { between, t }` | — | 节点中心位置：笛卡尔 / 极坐标 / 相对定位 / 偏移定位 / 比例定位；`of` 三态：节点 id、笛卡尔、PolarPosition |
| `children` / `text` | `string \| Array<string>` | — | 节点内可见文字；字符串里 `'\n'` 是硬换行 / 数组一元素一行；`text` 优先于 `children` |
| `align` | `'left' \| 'center' \| 'right'` | `'center'` | 多行文本块内各行水平对齐方式 |
| `lineHeight` | `number` | `font.size × 1.2` | 多行块的行高（user units） |
| `maxTextWidth` | `number` | — | 折行阈值（user units）；超过才折行（西文按词 / CJK 按字），短文本盒宽收缩；非固定段落宽 |
| `rotate` | `number` | `0` | 绕中心旋转角度（度，正值顺时针） |
| `color` | `string` | — | 主色（TikZ `color=`）；`stroke` / `fill` / `textColor` 未单设则随它，并级联到内部文字与边 label |
| `fill` | `string \| PaintSpec` | — | 背景填充：纯色 CSS 串，或 PaintSpec（渐变 linear / radial、图案 pattern、图片 image） |
| `stroke` | `string` | `currentColor` | 边框色 |
| `strokeWidth` | `number` | `1` | 边框宽度（user units） |
| `innerXSep` | `number` | — | 横向内边距（text → 左右 border）；不填走 `padding` 兜底 |
| `innerYSep` | `number` | — | 纵向内边距（text → 上下 border）；不填走 `padding` 兜底 |

### margin 与 padding 的区别

| | 影响什么 | 视觉表现 |
| --- | --- | --- |
| `padding` | 文字到 border 的距离 | 视觉形状变大 |
| `margin` | 连接面外的均匀外偏移（TikZ outer sep） | 视觉形状不变；所有 border 锚点与连线端点外移 margin，节点占位也含 margin |

`padding` 撑大节点本身；`margin` 不动视觉形状，而是把节点的连接/引用边界整体外推 margin（含显式 `.north` 这类锚点），并让节点在布局里多占 margin 的空间——既给连线"留呼吸空间"，也和 TikZ `outer sep` 一致。`margin` 缺省 0，不设则零影响。
