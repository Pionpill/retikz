title: Node
description: Named primitive with an id, position, shape boundary, and optional text; Path / Draw reference it by id and auto-attach to its border.

`<Node>` is the basic positioned, named primitive with its own geometric boundary. Paths reference it by `id` and auto-attach to the per-shape border.

## Usage

```tsx
import { Node } from '@retikz/react';
```

```tsx
<Node id="a" position={[0, 0]}>
  Label
</Node>
```

Choose the form by the role in the diagram: flowchart nodes usually use the default rectangle + `padding`; plain text labels use `stroke="none"` / `fill="none"`; button-like nodes add `roundedCorners` + `minimumWidth`; annotation cards fit multi-line `text` + `align="left"`; and if you only need a reference point with no visible shape, use [Coordinate](/core/components/node/coordinate) instead.

### Shape polymorphism

Shapes have two faces in retikz ‚Äî drawn as a Path with sugar, or used as a Node boundary; see [Shapes](/core/components/shapes) for the full definition. This section covers the Node-shape side.

`shape` picks the visual shape in two forms:

- **Bare string** (parameterless shapes): `'rectangle'` (default) / `'circle'` / `'ellipse'` / `'diamond'` / `'polygon'` / `'sector'` / `'arc'` / `'star'`, or a custom name registered via `<Layout shapes>` / `CompileOptions.shapes`.
- **`{ type, params }` object** (parametric shapes): when a shape needs extra geometry (sides / inner & outer radii / start & end angles‚Ä¶), pick it with `type` and pass the params.

Either way, the node works as an attachable target ‚Äî connecting lines clip to that shape's boundary automatically.

Shapes split into two families by where their size comes from: **text-container shapes** (rectangle / circle / ellipse / polygon / diamond) wrap "text + padding" and are sized by the text; **params-driven geometric shapes** (sector / arc / star) are sized by radii / angles, independent of the text, and work as attachable graphic glyphs.

<ComponentPreview name="node-shape-params" size="md" />

For each shape's parameters, anchors, and examples, see its shape page:

| shape | form | docs |
| --- | --- | --- |
| rectangle | default / `'rectangle'` | [Rectangle](/core/components/shapes/rectangle) |
| circle / ellipse | `'circle'` / `'ellipse'` | [Circle / Ellipse](/core/components/shapes/circle-ellipse) |

`shape` also accepts registered names beyond the built-ins (hexagon / cylinder / cloud‚Ä¶) ‚Äî any string passes schema validation, the definition is injected via `CompileOptions.shapes` / `<Layout shapes>`, and unregistered names throw at compile time. Parametric registered shapes pass their params through `{ type, params }`. See [Custom Shapes](/core/reference/extending/shape-registry).

### Positioning

#### Relative positioning

In addition to Cartesian / polar, `position` accepts **`{ direction, of, distance? }`**. No more hand-computing absolute coordinates; describe node relationships instead.

- `direction`: 8 directions ‚Äî `above` / `below` / `left` / `right` / `above-left` / `above-right` / `below-left` / `below-right`
- `of`: id of the referenced node / coordinate (must be defined earlier in the IR; forward references rejected)
- `distance`: falls back to `<Layout nodeDistance>` then to 1 when omitted

<Comparison target="tikz" title="Relative Positioning">

`{ direction: 'above', of: 'a', distance: 20 }` maps to TikZ positioning-library forms like `[above=20 of a]`. `<Layout nodeDistance={...}>` maps to TikZ's global `node distance` default.

</Comparison>

<ComponentPreview name="node-at-relative" size="sm" />

#### Offset positioning

`position` also accepts **`{ of, offset }`** ‚Äî an arbitrary `(dx, dy)` offset from a base point. Use offset whenever the 8-direction + scalar-distance `at` form can't express what you want (e.g. "A right 30 down 10").

- `of` is the base point in three shapes (matching `PolarPosition.origin`):
  - string id ‚Äî references a named node / coordinate (forward references rejected)
  - `[x, y]` Cartesian literal ‚Äî direct coordinate base, no node definition needed
  - `PolarPosition` (`{ angle, radius, origin? }`) ‚Äî polar expression as base, recursively resolved
- `offset` is a `[dx, dy]` tuple in user units

<Comparison target="tikz" title="Offset Positioning">

`{ of: 'a', offset: [10, 5] }` maps to TikZ `calc` syntax `($(a) + (10,5)$)`.

</Comparison>

<ComponentPreview name="node-offset-position" size="sm" />

#### Proportional positioning

Since v0.2, `position` also accepts **`{ between, t }`**: place the node at a proportional point between two endpoints, with `t=0.5` as the midpoint. Endpoints may be node ids, coordinates, polar positions, offsets, or nested between positions; see [Coordinate](/core/components/node/coordinate) for a fuller midpoint-helper example.

### Node labels

`label` attaches extra text outside the node border. Single object or array form; `position` accepts the 8-direction enum or a numeric angle; `font` / `textColor` inherit from the node when omitted.

<Comparison target="tikz" title="Node Labels">

`label={{ text: 'foo', position: 'above' }}` maps to TikZ `[label=above:foo]`; `position: 30` maps to `label=30:foo`.

</Comparison>

<ComponentPreview name="node-label-basic" size="xs" />

`distance` controls how far the label is pushed outward from the node border, still in retikz user units. It defaults to `12`; for example, `position: 'above', distance: 20` places the label further above the node's top border.

#### Multi-line text

`text` accepts a string (one line) or an array of strings (one line each). `align` controls horizontal alignment within the text block (`'left'` / `'center'` (default) / `'right'`), and `lineHeight` controls vertical spacing between lines, defaulting to `font.size √ó 1.2`.

<ComponentPreview name="node-multiline" size="xs" />

> Arrays beat `'\n'`-delimited strings: JSON-friendly (no escape headaches), and leaves room for per-line attributes later.

You can also pass the text as `children` ‚Äî pick whichever form reads best:

```tsx
{/* string with \n */}
<Node>{'Line 1\nLine 2'}</Node>

{/* template literal (real newlines) */}
<Node>{`Line 1
Line 2`}</Node>

{/* array */}
<Node>{['Line 1', 'Line 2']}</Node>
```

`text` wins over `children` whenever both are supplied.

#### Auto-wrap

`maxTextWidth` sets a **wrap threshold** (user units): text wraps past the threshold (Western on word boundaries, CJK per character), and short text shrinks the box to its actual content instead of filling the width. It is **not a fixed paragraph width** ‚Äî that is where it differs from TikZ `text width`.

<ComponentPreview name="node-max-text-width" size="sm" />

> An unbreakable long token (a long run with no whitespace) that exceeds the threshold overflows as a whole rather than being hard-cut.

#### Per-line styling

Each line can override `fill` / `opacity` / `font` (any subset of family / size / weight / style); unset fields inherit block defaults. Two equivalent paths:

1. **Mix LineSpec objects into the `text` array** ‚Äî JSON / AI-friendly, schema-expressible
2. **`<Text>` child elements** ‚Äî natural JSX; see the [Text page](/core/components/node/text) for the dedicated docs

<ComponentPreview name="node-styled-lines" size="xs" />

> Block-level `align` / `lineHeight` cannot be overridden per line ‚Äî they apply to the whole multi-line block.

Multi-line text combined with each shape ‚Äî outer boundary follows the shape-specific circumscription (circle / ellipse / polygon):

<ComponentPreview name="node-multiline-shapes" size="xs" />

#### Inner / outer sep

Both **symmetric aliases** and **axis-specific** forms are supported. If both are provided, axis-specific fields win; otherwise retikz falls back to the symmetric alias, then to the default:

- Inner: `padding` (symmetric) ¬∑ `innerXSep` / `innerYSep` (per-axis)
- Outer: `margin` (symmetric) ¬∑ `outerSep` (single)

Most nodes only need `padding={N}`. Use `innerXSep` / `innerYSep` when horizontal and vertical spacing need different values.

<ComponentPreview name="node-sep" size="xs" />

<Comparison target="tikz" title="Sep Naming">

`innerXSep` / `innerYSep` / `outerSep` reuse TikZ's sep vocabulary; retikz also keeps `padding` / `margin` as CSS-like symmetric aliases.

</Comparison>

#### Colors / opacity

- `textColor` ‚Äî text color (block default; per-line `LineSpec.fill` still overrides)
- `opacity` ‚Äî whole-node 0..1 (applies to shape + text)
- `fillOpacity` / `drawOpacity` ‚Äî fill-only / stroke-only opacity

<ComponentPreview name="node-color-opacity" size="xs" />

#### Gradient fill (Paint)

Besides a plain color string, `fill` accepts a **PaintSpec object** ‚Äî linear / radial gradient, pattern, or image. Identical PaintSpecs are deduplicated at compile time into a Scene resource table and rendered as a paint server inside SVG `<defs>`. `<Path fill>` supports it too.

- Linear: `{ type: 'linearGradient', angle?, stops }` ‚Äî `angle` in degrees (0 = left‚Üíright, 90 = top‚Üíbottom)
- Radial: `{ type: 'radialGradient', center?, radius?, stops }` ‚Äî `center` / `radius` in 0..1 relative units
- `stops`: array of `{ offset (0..1), color, opacity? }`, at least 2; `color` may be `currentColor`

<ComponentPreview name="node-paint" size="sm" />

A stop's `opacity` gives a "solid ‚Üí transparent" fade; `color` set to `currentColor` makes the gradient follow the element's `color` and react to theme changes:

<ComponentPreview name="node-paint-stops" size="sm" />

`fill` also accepts **pattern** and **image**:

- Pattern: `{ type:'pattern', shape:'lines'|'dots'|'grid', color?, background?, size?, lineWidth?, rotation? }` ‚Äî `color` defaults to `currentColor`, `size` is the tile period, `rotation` rotates the whole pattern
- Image: `{ type:'image', href, fit? }` ‚Äî `href` is a URL (incl. data URI), `fit`: `cover` (default) / `contain` / `fill`

<ComponentPreview name="node-pattern-image" size="sm" />

`href` takes any external image URL (http(s)) or a data URI; `fit` controls how the image maps into the shape box ‚Äî the same external landscape image (`picsum.photos`, 320√ó200) under all three fits in a square box:

<ComponentPreview name="node-image-fit" size="sm" />

> `fit` is computed against the normalized shape box: it matches CSS `object-fit` for square boxes; for strongly non-square boxes cover/contain are approximate (against the normalized square, not the true aspect ratio). When the source and box share a shape (both square / both landscape) the three fits look identical.
>
> Image fill renders via a runtime `<image href>`, so external URLs load on the browser side; inlining for SSR / PNG export is deferred.

#### Border style

`dashed` / `dotted` are presets; `dashArray` is an explicit dash/gap length sequence (priority: `dashArray` > `dashed` > `dotted`):

<ComponentPreview name="node-stroke-style" size="xs" />

#### Size constraints + scale

- `roundedCorners` ‚Äî corner radius (only on `rectangle` shape); can also be written as `shape={{ type:'rectangle', params:{ roundedCorners } }}`, with the top-level `roundedCorners` still supported (params wins when both are set)
- `minimumWidth` / `minimumHeight` / `minimumSize` ‚Äî floor the bbox without shrinking natural text width
- `scale` / `xScale` / `yScale` ‚Äî uniform / per-axis scale; affects bbox, font size, padding, and path attachment

<ComponentPreview name="node-size-constraints" size="xs" />

#### Rotation

`rotate` rotates around the node center; attached paths follow the new border:

<ComponentPreview name="node-rotated" size="xs" />

#### Stacking order (zIndex)

By default stacking = declaration order (SVG only honors paint order). `zIndex` overrides it explicitly: higher draws on top, omitted = 0, ties keep declaration order; effective only among siblings at the same level (same scope / top level). For example, lift a highlighted node with `zIndex={10}` above all connecting paths without moving it to the end of the JSX. `<Path>` / `<Scope>` support `zIndex` too (a scope's `zIndex` positions the whole group among its siblings without affecting the relative stacking inside it).

<ComponentPreview name="node-z-index" size="sm" />

#### Label rotation

`label.rotate` spins the label text around **its own center**: `'radial'` along the node-center ‚Üí label-center direction, `'tangent'` is its tangent (radial + 90¬∞), a number is an explicit angle (degrees); `keepUpright` flips it 180¬∞ when it would otherwise read upside-down. `rotate` only changes orientation, not the placement determined by `position` / `distance`.

<ComponentPreview name="node-label-rotate" size="sm" />

#### Pin

`label.pin` adds a **leader line** from the node border to the label (TikZ `pin`): `pin: true` is a default thin solid line (inherits the label color); `pin: { stroke, strokeWidth, dashPattern }` customizes it; omitted / `false` = no leader. The label is still placed by `position` / `distance` ‚Äî pin only adds the leader.

<Comparison target="tikz" title="Pin">

`label={{ text: 'foo', position: 'right', pin: true }}` maps to TikZ `[pin=right:foo]`.

</Comparison>

<ComponentPreview name="node-pin" size="md" />

## API Reference

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `id` | `string` | ‚Äî | Unique identifier; referenced by `<Path>` / `<Draw>` |
| `shape` | `'rectangle' \| 'circle' \| 'ellipse' \| 'diamond' \| 'polygon' \| 'sector' \| 'arc' \| 'star' \| string`<br />`\| { type, params }` | `'rectangle'` | Node shape: bare string (parameterless / registered name) or `{ type, params }` (parametric shape); built-ins above, or registered via `<Layout shapes>` / `CompileOptions.shapes` |
| `position` | `[x, y] \| PolarPosition \| { direction, of, distance? } \| { of, offset } \| { between, t }` | ‚Äî | Node center: Cartesian / polar / relative / offset / proportional placement; `of` accepts node id, Cartesian literal, or PolarPosition |
| `children` / `text` | `string \| Array<string>` | ‚Äî | Visible label; string = one line, array = one element per line; `text` wins over `children` |
| `align` | `'left' \| 'center' \| 'right'` | `'center'` | Horizontal alignment of lines within the multi-line text block |
| `lineHeight` | `number` | `font.size √ó 1.2` | Line height of the multi-line block (user units) |
| `maxTextWidth` | `number` | ‚Äî | Wrap threshold (user units); text wraps past it (Western per word / CJK per char) and the box shrinks for short text; not a fixed paragraph width |
| `rotate` | `number` | `0` | Rotation around center, degrees, positive = clockwise |
| `color` | `string` | ‚Äî | Master color (TikZ `color=`); `stroke` / `fill` / `textColor` default to it unless individually set, and it cascades to inner text and edge labels |
| `fill` | `string \| PaintSpec` | ‚Äî | Background fill: a plain CSS color string, or a PaintSpec (gradient linear / radial, pattern, or image) |
| `stroke` | `string` | `currentColor` | Border color |
| `strokeWidth` | `number` | `1` | Border width (user units) |
| `innerXSep` | `number` | ‚Äî | Horizontal inner padding (text ‚Üí left/right border); falls back to `padding` |
| `innerYSep` | `number` | ‚Äî | Vertical inner padding (text ‚Üí top/bottom border); falls back to `padding` |
| `outerSep` | `number` | ‚Äî | Outer margin (border ‚Üí path attachment point); falls back to `margin` |
| `padding` | `number` | `8` | Symmetric inner padding alias; axis-specific fields take precedence |
| `margin` | `number` | `0` | Symmetric outer margin alias; `outerSep` takes precedence; **does not affect border position**, only path endpoints |
| `font` | `{ family?, size?, weight?, style? }` | ‚Äî | Font spec; nested object, all fields optional |
| `textColor` | `string` | `currentColor` | Text color (block default; per-line override via `LineSpec.fill`) |
| `opacity` | `number` | ‚Äî | Whole-node 0..1, applies to both shape and text |
| `fillOpacity` | `number` | ‚Äî | Fill-only opacity 0..1 |
| `drawOpacity` | `number` | ‚Äî | Stroke-only opacity 0..1 |
| `dashed` | `boolean` | ? | Dashed border preset (`[4, 2]`) |
| `dotted` | `boolean` | ? | Dotted border preset (`[1, 2]`) |
| `dashArray` | `Array<number>` | ? | Explicit dash/gap lengths; highest priority |
| `roundedCorners` | `number` | ‚Äî | Corner radius (only on `rectangle` shape); also settable via `shape={{ type:'rectangle', params:{ roundedCorners } }}` (params wins) |
| `minimumWidth` / `minimumHeight` | `number` | ‚Äî | Floors the bbox dimensions |
| `minimumSize` | `number` | ‚Äî | Symmetric minimum-size alias; axis-specific fields take precedence |
| `scale` | `number` | `1` | Uniform scale (includes font, padding, path attachment) |
| `xScale` / `yScale` | `number` | ‚Äî | Per-axis scale; takes precedence over `scale` |
| `zIndex` | `number` (integer) | `0` | Explicit stacking order: higher draws on top; ties keep declaration order; effective only among siblings at the same level |
| `label` | `NodeLabel \| Array<NodeLabel>` | ‚Äî | Extra label(s) attached to the node border; single object or array; `position` accepts the 8-direction enum or a numeric angle, `distance` defaults to 12, `font` / `textColor` inherit from the node; `rotate` (`'none'` / `'radial'` / `'tangent'` / a number of degrees) + `keepUpright` control label spin; `pin` (`true` default line / `{ stroke, strokeWidth, dashPattern }` styled / omitted = none) draws a leader |

### margin vs padding

| | Affects | Visual effect |
| --- | --- | --- |
| `padding` | Distance from text to border | Border grows |
| `margin` | Distance from border to path endpoint | Border unchanged; lines stop `margin` short |

Use `margin` to give connecting lines breathing room in multi-node diagrams ‚Äî keeps lines from touching the border directly.
