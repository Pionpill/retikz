title: Coordinate
description: 占位节点——命名一个点不画任何图形，给后续 path 和 node 相对定位（`{ of }`）引用。
---

`<Coordinate>` 用来在 IR 中登记一个可命名的引用点。它和 `<Node>` 一样可以被路径与相对定位引用，但不会在画面上渲染形状或文字；编译时只登记它的 `id` 与位置，供后续 `<Path>` 端点和 `<Node position>` 的相对 / 偏移定位（`{ of }`）引用。

## 用法

```tsx
import { Coordinate } from '@retikz/react';
```

```tsx
<Coordinate id="hub" position={[0, 0]} />
```

`position` 接受 `<Node>` 完全相同的五种形态：笛卡尔 `[x, y]` / 极坐标 `{ angle, radius, origin? }` / 相对定位 `{ direction, of, distance? }` / 偏移定位 `{ of, offset }` / 比例定位 `{ between, t }`。

### 为什么需要它

要在某处放一个引用点，最朴素的做法是 fake 一个 `stroke="transparent" fill="transparent"` 的空 `<Node>`——但空 Node 仍会按 padding 占一块不可见的方块、把外框撑大，AI / 序列化往返时也分不清它是真节点还是占位。

`<Coordinate>` 把"命名点"做成一等概念：字段最少（只有 `id` + `position`）、不画任何东西、也不会撑大画布。背后的占位语义见 [技术原理](#技术原理)。

### 比例 partway 定位

`{ between: [A, B], t }` 在 A、B 两端点的连线上按比例 t（0..1）取点——`lerp(A, B, t)`，对应 TikZ `($(A)!t!(B)$)`。放"连线中点 / 1/3 处"的点比手算坐标更省、且随 A、B 移动自动保持比例。端点是**自包含 AbsoluteTarget**（笛卡尔 / 极坐标 / 节点引用 `{ id, anchor? }` / `{ of, offset }` / 嵌套 between，**不含** path-relative）。同一形态可用于 `<Coordinate position>` / `<Node position>` / `<Step to>`：

<ComponentPreview name="coordinate-between" size="xs" />

> `t` 严格 [0,1]（外插暂不支持）；端点引用未定义节点时，Coordinate / Node 位置解析失败抛错、path `Step.to` 发 warn 跳过——与其它定位形态同口径。

## 技术原理

> 上手阶段可跳过本节；想理解 coordinate 为何不撑大画布、anchor 为何都落在中心时再回来读。

Coordinate 在内部以一个 **0×0 的点**登记，由此带来三条可观察行为：

- **不画任何图元**——没有形状、没有文字，渲染产物里完全不出现。
- **不撑大画布**——远处的 coordinate 不会让最终 viewBox 自动变大；必须有实际 `<Node>` / `<Path>` 落在那片区域，画布才覆盖过去（调试定位时常踩这点）。
- **所有 anchor 退化为中心**——尺寸是 0，`hub.north` / `hub.south` 等方位 anchor 都等于中心点；需要方向 anchor 就改用 `<Node>`。

## API 参考

| 属性 | 类型 | 默认值 | 描述 |
| --- | --- | --- | --- |
| `id` | `string` | — | **必填**——占位的全部意义就在于被 id 引用 |
| `position` | `[x, y] \| PolarPosition \| { direction, of, distance? } \| { of, offset } \| { between, t }` | — | 占位点位置；与 `<Node position>` 完全一致——含偏移定位 `{ of, offset }` 与比例 partway `{ between: [A, B], t }` 形态 |

## 相关

### 与 `<Node>` / `<Path>` 的关系

| | `<Coordinate>` | `<Node>` |
|---|---|---|
| 渲染图形 | ❌ | ✅（按 shape） |
| 参与 layout bounds | ❌ | ✅ |
| 进 nodeIndex | ✅ | ✅ |
| path target 引用 | ✅ | ✅ |
| 相对 / 偏移定位（`{ of }`）引用 | ✅ | ✅ |
| 必须有 `id` | ✅ | 可选 |

简单原则：需要画形状 + 文字 → `<Node>`；只要一个引用点 → `<Coordinate>`。

### 不适用场景

- **要给点配文字标签** → 用 `<Node>` 加 `label?`，coordinate 不带文字字段
- **要画一个圆点 / 标记位置** → 用 `<Node shape="circle">` + 小尺寸；coordinate 永远不画
- **anchor 指向**（`hub.north`）→ coordinate 所有 anchor 都退化为中心，要方向 anchor 用 `<Node>`
