description: 占位节点——命名一个点不画任何图形，给后续 path 和 node `at.of` 引用。等价于 TikZ `\coordinate (id) at (x, y);`。
---

`<Coordinate>` 是 retikz kernel 的第三件（与 `<Node>` / `<Path>` 平级）。它在画面上不渲染任何东西，但参与 `nodeIndex` 注册——让后续 `<Path>` 端点和 `<Node>` 的 `at.of` 都能用 `id` 引用它。

## 用法

```tsx
import { Coordinate } from '@retikz/react';
```

```tsx
<Coordinate id="hub" position={[0, 0]} />
```

`position` 接受 `<Node>` 完全相同的四种形态：笛卡尔 `[x, y]` / 极坐标 `{ angle, radius, origin? }` / 相对定位 `{ direction, of, distance? }` / 偏移定位 `{ of, offset }`。

## 为什么需要它

要在某处放一个引用点，不带 `<Coordinate>` 的话只能 fake 一个 `stroke="transparent" fill="transparent"` 的空 `<Node>`——但空 Node 仍然会：

- 走 layoutNode 流程，按 padding 撑出一个不可见的矩形 layout
- 参与 viewBox 扩展（让外框变大）
- 让 AI / TikZ codec 反推时分不清"真节点 vs fake 占位"

`<Coordinate>` 把"命名点"升级为一等概念：

- IR 字段最少（只有 `id` + `position`）
- **不发任何 primitive**——没有形状、没有文字
- **不参与 viewBox**——远端 coordinate 不会撑大画面
- 在 nodeIndex 中以 0×0 矩形 layout 形式注册——boundaryPoint 始终返回中心，正好符合"无形状边界"语义

## API 参考

| 属性 | 类型 | 默认值 | 描述 |
| --- | --- | --- | --- |
| `id` | `string` | — | **必填**——占位的全部意义就在于被 id 引用 |
| `position` | `[x, y] \| PolarPosition \| { direction, of, distance? } \| { of, offset }` | — | 占位点位置；与 `<Node position>` 完全一致——包含偏移定位 `{ of, offset }` 形态 |

## 与 `<Node>` / `<Path>` 的关系

| | `<Coordinate>` | `<Node>` |
|---|---|---|
| 渲染图形 | ❌ | ✅（按 shape） |
| 参与 viewBox | ❌ | ✅ |
| 进 nodeIndex | ✅ | ✅ |
| path target 引用 | ✅ | ✅ |
| `at.of` 引用 | ✅ | ✅ |
| 必须有 `id` | ✅ | 可选 |

简单原则：需要画形状 + 文字 → `<Node>`；只要一个引用点 → `<Coordinate>`。

## 不适用场景

- **要给点配文字标签** → 用 `<Node>` 加 `label?`，coordinate 不带文字字段
- **要画一个圆点 / 标记位置** → 用 `<Node shape="circle">` + 小尺寸；coordinate 永远不画
- **anchor 指向**（`m.north`）→ coordinate 是 0×0 占位，所有 anchor 都退化为中心；要用方向 anchor 还是用 `<Node>`
