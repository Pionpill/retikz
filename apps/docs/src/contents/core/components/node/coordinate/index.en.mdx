---
title: Coordinate
description: Placeholder node — names a point without drawing anything, so paths and `at.of` can reference it by id. Equivalent to TikZ `\coordinate (id) at (x, y);`.
---

`<Coordinate>` is retikz's third kernel component (alongside `<Node>` / `<Path>`). It renders nothing visible, but registers into `nodeIndex` so downstream `<Path>` endpoints and `<Node>` `at.of` references can target it by `id`.

## Usage

```tsx
import { Coordinate } from '@retikz/react';
```

```tsx
<Coordinate id="hub" position={[0, 0]} />
```

`position` accepts the same four forms as `<Node>`: Cartesian `[x, y]` / polar `{ angle, radius, origin? }` / relative `{ direction, of, distance? }` / offset `{ of, offset }`.

### Why it exists

Without `<Coordinate>`, anchoring "a named point" forces you to fake an empty `<Node>` with `stroke="transparent" fill="transparent"` — but an empty Node still:

- Goes through layoutNode and gets a non-visible rect layout sized by padding
- Participates in layout bounds expansion (enlarges the outer frame)
- Confuses AI / TikZ codecs round-tripping IR ("real node vs invisible placeholder?")

`<Coordinate>` makes "named point" a first-class concept:

- Smallest IR shape (just `id` + `position`)
- **Emits zero primitives** — no shape, no text
- **Does not extend layout bounds** — far-away coordinates don't bloat the frame
- Registers as a 0×0 rectangle layout in nodeIndex — `boundaryPoint` always returns the center, matching "no shape boundary" semantics

## API Reference

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `id` | `string` | — | **Required** — a placeholder's entire purpose is to be referenced by id |
| `position` | `[x, y] \| PolarPosition \| { direction, of, distance? } \| { of, offset }` | — | Placeholder position; same shapes as `<Node position>` — includes the offset `{ of, offset }` form |

## Related

### Relationship with `<Node>` / `<Path>`

| | `<Coordinate>` | `<Node>` |
|---|---|---|
| Renders shape | ❌ | ✅ (by `shape`) |
| Participates in layout bounds | ❌ | ✅ |
| Registered in nodeIndex | ✅ | ✅ |
| Referenced as path target | ✅ | ✅ |
| Referenced via `at.of` | ✅ | ✅ |
| Required `id` | ✅ | optional |

Rule of thumb: need a shape + text → `<Node>`; need only a reference point → `<Coordinate>`.

### When not to use

- **Need a labelled point** → use `<Node>` with `label?`; coordinate has no text field
- **Need a small visible dot** → use `<Node shape="circle">` with a small size; coordinate never draws
- **Direction anchors** (`m.north`) → coordinate is a 0×0 placeholder, all anchors collapse to center; use `<Node>` if you need directional anchors
