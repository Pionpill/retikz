---
title: Coordinate
description: Placeholder node — names a point without drawing anything, so paths and `at.of` can reference it by id.
---

`<Coordinate>` registers a named reference point in IR. Like `<Node>`, it can be targeted by paths and relative positioning, but it renders no visible shape or text; during compilation, only its `id` and position are registered, for downstream `<Path>` endpoints and `<Node>` `at.of` references.

## Usage

```tsx
import { Coordinate } from '@retikz/react';
```

```tsx
<Coordinate id="hub" position={[0, 0]} />
```

`position` accepts the same four forms as `<Node>`: Cartesian `[x, y]` / polar `{ angle, radius, origin? }` / relative `{ direction, of, distance? }` / offset `{ of, offset }`.

### Why it exists

The plainest way to anchor "a named point" is to fake an empty `<Node>` with `stroke="transparent" fill="transparent"` — but an empty Node still takes up an invisible box sized by padding, enlarges the outer frame, and confuses AI / serialization round-trips ("real node or placeholder?").

`<Coordinate>` makes "named point" a first-class concept: the smallest possible shape (just `id` + `position`), draws nothing, and never enlarges the canvas. See [How it works](#how-it-works) for the placeholder semantics.

### Proportional partway positioning

`{ between: [A, B], t }` takes a point along the A→B segment at proportion t (0..1) — `lerp(A, B, t)`, matching TikZ `($(A)!t!(B)$)`. Placing a "midpoint / one-third" point this way beats hand-computing coordinates and keeps the proportion as A / B move. Endpoints are a self-contained **AbsoluteTarget** (Cartesian / polar / node reference `{ id, anchor? }` / `{ of, offset }` / nested between — **excluding** path-relative). The same shape works in `<Coordinate position>` / `<Node position>` / `<Step to>`:

> `t` is strict [0,1] (extrapolation not yet supported); when an endpoint references an undefined node, Coordinate / Node position resolution throws while a path `Step.to` warns and skips — the same as other positioning forms.

## How it works

> Optional for everyday use; revisit when you want to understand why a coordinate never enlarges the canvas and why all its anchors land at the center.

Internally a Coordinate registers as a **0×0 point**, which gives three observable behaviors:

- **Draws no primitive** — no shape, no text; it never appears in the rendered output.
- **Never enlarges the canvas** — a far-away coordinate won't grow the final viewBox on its own; an actual `<Node>` / `<Path>` must land in that region for the canvas to cover it (a common gotcha when debugging placement).
- **All anchors collapse to the center** — with zero size, directional anchors like `hub.north` / `hub.south` all equal the center point; use `<Node>` when you need directional anchors.

## API Reference

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `id` | `string` | — | **Required** — a placeholder's entire purpose is to be referenced by id |
| `position` | `[x, y] \| PolarPosition \| { direction, of, distance? } \| { of, offset } \| { between, t }` | — | Placeholder position; same shapes as `<Node position>` — includes the offset `{ of, offset }` and proportional partway `{ between: [A, B], t }` forms |

## Related

### Relationship with `<Node>` / `<Path>`

| | `<Coordinate>` | `<Node>` |
|---|---|---|
| Renders shape | ❌ | ✅ (by `shape`) |
| Participates in layout bounds | ❌ | ✅ |
| Registered in nodeIndex | ✅ | ✅ |
| Referenced as path target | ✅ | ✅ |
| Referenced via `at.of` | ✅ | ✅ |
| Required `id` | ✅ | optional |

Rule of thumb: need a shape + text → `<Node>`; need only a reference point → `<Coordinate>`.

### When not to use

- **Need a labelled point** → use `<Node>` with `label?`; coordinate has no text field
- **Need a small visible dot** → use `<Node shape="circle">` with a small size; coordinate never draws
- **Direction anchors** (`hub.north`) → a coordinate's anchors all collapse to the center; use `<Node>` for directional anchors
