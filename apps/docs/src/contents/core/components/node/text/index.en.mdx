---

`<Text>` is Sugar ‚Äî it renders nothing on its own. At buildIR time it is recognized and lowered to an `IRLineSpec` object inside `<Node>`'s `text` array.

It's an **equivalent** alternative to writing LineSpec objects directly into the `text` prop:

| Path | Form | Strength |
|---|---|---|
| `<Text>` Sugar | `<Node><Text fill="red">x</Text>...</Node>` | Natural JSX, mixes well with string children |
| `text` array | `<Node text={[{ text: 'x', fill: 'red' }, ...]} />` | JSON-friendly; LLMs can emit IR JSON directly |

> Block-level props (`align` / `lineHeight`) go on `<Node>`; per-line overrides (`fill` / `opacity` / `font`) go on `<Text>`.

Think of `<Text>` as "add one styled line to a Node", not as an HTML inline `<span>`. It cannot mix several colors inside the same line; for multi-color explanations, split the content into separate lines or use multiple independent `<Node>`s.

## Usage

```tsx
import { Node, Text } from '@retikz/react';
```

```tsx
<Node>
  <Text fill="red" font={{ weight: 'bold' }}>Heading</Text>
  body line
</Node>
```

## API Reference

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `children` | `string` | ‚Äî | Line content; must be a single string (other child types are ignored) |
| `fill` | `string` | ‚Äî | Per-line color override; falls back to Node `textColor` block default |
| `opacity` | `number` | ‚Äî | Per-line opacity 0..1 |
| `font` | `{ family?, size?, weight?, style? }` | ‚Äî | Per-line font override; missing fields inherit Node block-level `font` |

### Constraints

- **Must live inside `<Node>`**: a `<Text>` placed at the top level under `<Layout>` is silently ignored.
- **`children` must be a string**: nested elements or expressions are not supported. A line is a line.
- **Block-level props can't be overridden**: `align` / `lineHeight` only go on `<Node>`.
