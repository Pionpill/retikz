## Installation

The React adapter is `@retikz/react`, with peer dependencies on `react` / `react-dom` (>= 18).

Note: retikz itself is framework-agnostic. You can also use `@retikz/vanilla` or `@retikz/core` / `@retikz/render` to build and render diagrams with plain JavaScript. The prose uses the React API so examples can focus on primitives, paths, and IR; open **View Code** on a demo to switch to the equivalent Vanilla API.

<PackageManagerInstall packages="@retikz/react react react-dom" />

The underlying `@retikz/core` (IR + compiler, zero React / zero DOM) is pulled in transitively — no need to install it directly.

## Step by Step

### 1. The container `<Layout />` + one `<Node />`

A minimal diagram starts with a container and a primitive. `<Layout>` holds the whole diagram, sets the SVG's rendered size, and renders the primitives inside it. `<Node>` draws a positioned, named element with visible text. This first step places one node to confirm both the canvas and the basic primitive are working.

<ComponentPreview name="get-start-step-1" size="xs" />

### 2. More `<Node />`s

Add more `<Node>`s to place the entities in the diagram before drawing relationships between them. You can add as many nodes as you like, but each `id` must be unique — `<Draw>` will find them by id later. `position` is the node center in canvas coordinates.

| Prop | Meaning |
| --- | --- |
| `id` | Unique identifier, referenced by `<Path>` / `<Draw>` |
| `position` | `[x, y]` user-unit coordinates for the node center |
| children | The node's visible content (label) |

<ComponentPreview name="get-start-step-2" size="xs" />

### 3. Connect them with `<Draw />`

`<Draw>` draws connector lines. The `way` array lists the node ids to pass through in order, and retikz attaches the line to node borders instead of text centers. `arrow="->"` adds an arrowhead at the end.

<ComponentPreview name="get-start-final" size="xs" />

Three nodes joined by two lines — the static structure now matches the example at the top.

### 4. Add Animation and a Callback

A figure can also respond to user input. Add `animations` to describe animation tracks, then add an `onClick` callback. The callback's second argument, `context`, carries the current element information and animation controls; calling `context.animation.restart()` replays the animation from the event.

<ComponentPreview name="get-start-step-4" size="xs" interactive replayable />

This animation uses `trigger: 'manual'`, so it does not autoplay. It waits for the click callback, and the callback can update your React state at the same time.

Keep four rules in mind before moving on:

- Canvas coordinates grow right on x and down on y; `[0, 0]` is the origin in retikz user units
- `<Layout width height>` controls the rendered SVG size; it does not rewrite internal coordinates into CSS pixels
- A node `id` is the name used by connectors, and `<Draw way={['a', 'b']}>` targets node borders rather than text centers
- Use **View Code → IR** to inspect the compiled diagram data; it is the quickest way to debug connections
