## 安装

retikz 的 React 适配器是 `@retikz/react`，peer 依赖 `react` / `react-dom`（>= 18）。

<PackageManagerInstall packages="@retikz/react react react-dom" />

底层 `@retikz/core`（IR + 编译器，零 React / 零 DOM）会作为 `@retikz/react` 的依赖自动装上，无需单独安装。

## 一步步搭建

### 1. 容器 `<TikZ />` + 一个 `<Node />`

画一张图至少要两件东西：容器和图元。`<TikZ>` 是顶层容器，扫描 children 构造 IR、编译 Scene、渲染 SVG；`width` / `height` 决定 SVG 元素的渲染尺寸。`<Node>` 是最常用的图元，有 `id`、`position`、可见 label。

<ComponentPreview name="get-start-step-1" size="xs" />

### 2. 多个 `<Node />`

节点数量随便加，但 `id` 必须全局唯一——后面 `<Draw>` 会按 id 引用它们。`position` 是画布坐标（像素）。

| Prop | 含义 |
| --- | --- |
| `id` | 唯一标识，被 `<Path>` / `<Draw>` 用来引用 |
| `position` | `[x, y]` 画布坐标（像素） |
| children | 节点的可见内容（label） |

<ComponentPreview name="get-start-step-2" size="xs" />

### 3. 用 `<Draw />` 连起来

`<Draw>` 是 Sugar 组件，`way` 数组按 id 引用上面的节点，编译期展开为 Kernel 的 `<Path>` + `<Step>`——所以 `<Draw>` 不会引入任何 Kernel 表达不了的能力。点开 **View Code** 切到 IR 标签，能直接看到 `<Draw>` 在 IR 里就是一个 `path`。

<ComponentPreview name="get-start-final" size="xs" />
