## 安装

retikz 的 React 适配器是 `@retikz/react`，peer 依赖 `react` / `react-dom`（>= 18）。

备注：retikz 本身与框架解耦，也可以用 `@retikz/vanilla` 或 `@retikz/core` / `@retikz/render` 通过纯 JS 组织和渲染图形。文档正文统一使用 React API，是为了聚焦图元、路径和 IR 能力；点开 demo 的 **View Code** 后，可以切换到 Vanilla API 查看等价写法。

<PackageManagerInstall packages="@retikz/react react react-dom" />

底层 `@retikz/core`（IR + 编译器，零 React / 零 DOM）会作为 `@retikz/react` 的依赖自动装上，无需单独安装。

## 如何学习

建议先把这页跑通，再按“心智模型 → 组件细节 → 完整示例 → 参考契约”的顺序读：

| 分组 | 读什么 |
| --- | --- |
| 简介 / 快速开始 | 建立整体印象，知道 retikz 为什么以 Node、Path、IR 为中心。 |
| 概念 | 学定位、锚点、分层和动画；这些是所有图形复用的基础规则。 |
| 组件 | 查 `<Layout>`、`<Node>`、`<Draw>`、形状和辅助组件的写法。 |
| 示例 | 看完整图形如何从 prompt / 想法一步步拆成图元。 |
| 参考 | 查扩展点、IR schema、renderer、runtime；写 adapter、持久化或调试底层行为时再深入。 |

如果你想边读边问，可以点顶部的 AI 按钮打开 AI 面板，把当前页、选中的代码或 API 名称作为上下文提问。文档站是纯静态站点，不提供 LLM 服务；你需要在面板里填自己的 API Key，站点只在浏览器里把请求转发给你选择的模型供应商。

## 一步步搭建

### 1. 容器 `<Layout />` + 一个 `<Node />`

最小图形从容器和基础图元开始。`<Layout>` 负责承载整张图，决定 SVG 的显示尺寸，并把内部图元渲染出来；`<Node>` 负责画一个有位置、有名字、可显示文字的节点。这里先放一个节点，确认画布和基础图元都能工作。

<ComponentPreview name="get-start-step-1" size="xs" />

### 2. 多个 `<Node />`

继续添加多个 `<Node>`，相当于先把图里的实体摆好。节点数量随便加，但 `id` 必须全局唯一——后面 `<Draw>` 会按 id 找到它们；`position` 是节点中心在画布里的坐标。

| Prop | 含义 |
| --- | --- |
| `id` | 唯一标识，被 `<Path>` / `<Draw>` 用来引用 |
| `position` | `[x, y]` 用户单位坐标，表示节点中心位置 |
| children | 节点的可见内容（label） |

<ComponentPreview name="get-start-step-2" size="xs" />

### 3. 用 `<Draw />` 连起来

`<Draw>` 负责画连接线。`way` 数组按顺序写要经过的节点 id，retikz 会自动把线贴到节点边界，而不是连到文字中心；`arrow="->"` 给末端加箭头。

<ComponentPreview name="get-start-final" size="xs" />

3 个节点用 2 条线串起，静态结构已经与最上方的例子一致。

### 4. 添加动画与回调函数

图形也可以响应用户操作。给节点加 `animations` 描述动画轨道，再加 `onClick` 回调；回调第二个参数 `context` 会带上当前图元的信息和动画控制器，调用 `context.animation.restart()` 就能从事件里重播动画。

<ComponentPreview name="get-start-step-4" size="xs" interactive replayable />

这里的动画设置为 `trigger: 'manual'`，所以它不会自动播放，而是等点击节点时由回调触发；回调里也可以同步更新你的 React 状态。

到这里先记住 4 个规则：

- 画布坐标 x 向右、y 向下；`[0, 0]` 是 retikz 内部 user units 的原点
- `<Layout width height>` 控制最终 SVG 显示尺寸，不等于强行把内部坐标改成 CSS 像素
- 节点 `id` 是连线的名字，`<Draw way={['a', 'b']}>` 引用的是节点边界，不是文本中心
- 点开 **View Code → IR** 可以看到编译后的图形数据，调试连线时优先看这里
