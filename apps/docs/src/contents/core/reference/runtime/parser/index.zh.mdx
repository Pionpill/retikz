---
title: 解析器
description: Sugar 输入到 IR 片段的纯函数转换
---

Parser reference 覆盖 `@retikz/core` 公开的 sugar 解析函数。它们不依赖 React、DOM 或 nodeIndex，适合 adapter、测试和自定义 DSL 复用。

## 函数

| API | Signature | 用途 |
| --- | --- | --- |
| `parseWay` | `(way: WayDSL) => Array<IRStep>` | 把 `<Draw way={...}>` 的 way 数组转成 path step 序列 |
| `parseTargetSugar` | `(input: unknown) => IRTarget` | 把 React DSL 字符串 shorthand 转成 IR target：`'+dx,dy'` / `'++dx,dy'` 变相对偏移，节点 ref 字符串委托 `parseNodeTarget`；**非字符串**输入（笛卡尔 / 极坐标 / 已是对象）原样返回。注意：非法字符串会抛错而非原样返回（如 `'+x,'`、未知 anchor、空 id、纯小数串） |
| `parseNodeTarget` | `(s: string) => IRNodeTarget` | 节点 ref 字符串 → NodeTarget 对象：`'A'`→`{id:'A'}`、`'A.north'`→命名 anchor、`'A.30'`→角度 anchor；按第一个 `.` 切分，含 `.` 的 id / shape 自定义 anchor 须写对象形态；空 id / 未知 anchor / 纯小数串抛错 |

## 画线方式常量

`DrawWay` 是 way DSL 的关键字集合。不要硬编码内部字符串，使用常量引用。

| Constant | 值 / 语义 | 说明 |
| --- | --- | --- |
| `DrawWay.Cycle` | cycle | 闭合回 path 起点 |
| `DrawWay.Hv` | `'-|'` | 折角：先水平后垂直 |
| `DrawWay.Vh` | `'\|-'` | 折角：先垂直后水平 |
| `DrawWay.Relative` | relative 类型标记 | 对象式相对偏移，不更新 prevEnd |
| `DrawWay.Accumulate` | accumulate 类型标记 | 对象式累积相对偏移，更新 prevEnd |

## 路径项

`WayDSL` 是 `Array<WayItem>`。常见 item 形态：

| Shape | 产物 | 说明 |
| --- | --- | --- |
| `IRTarget` / 字符串 shorthand | `move` / `line` | 第一项成为 move，后续 target 成为 line；字符串会经 `parseTargetSugar` 转成 IR 对象 |
| `{ position, type }` | `relative` / `relativeAccumulate` target | TS-friendly 的相对偏移对象 |
| `DrawWay.Hv` / `DrawWay.Vh` | `kind: 'fold'` | 与下一项 target 合并成折角 step |
| `DrawWay.Cycle` | `kind: 'cycle'` | 闭合当前子路径 |
| `{ curve }` | `kind: 'curve'` | 与下一项 target 合并成二次贝塞尔 |
| `{ cubic }` | `kind: 'cubic'` | 与下一项 target 合并成三次贝塞尔 |
| `{ bend }` | `kind: 'bend'` | 与下一项 target 合并成弯曲段；`angle` 可选 |
| `{ arc }` | `kind: 'arc'` | 以上一项为圆心，不消耗下一项 |
| `{ circle }` | `kind: 'circlePath'` | 以上一项为圆心画整圆 |
| `{ ellipse }` | `kind: 'ellipsePath'` | 以上一项为中心画整椭圆 |
| `{ label }` | step label | 修饰下一个产生绘制段的 item |

## 路径解析规则

| Rule | 行为 |
| --- | --- |
| 最少长度 | `way.length < 2` 抛错 |
| 第一项 | 必须是 target（move 起点）；`way[0]` 是 label 算子或任何其他算子都直接抛错，不降级（`[0,0]` 回退是相对 target 作首项时的基准语义，不是首项算子的降级） |
| target | 首项转 `move`，后续 target 转 `line` |
| fold / curve / cubic / bend | 必须跟随 target；解析后会消费下一项 |
| arc / circle / ellipse | 以上一项为中心，不消费下一项 |
| label | 修饰下一个产生绘制段的 item |
| label + cycle | 抛错；cycle 不承载 label |
| 连续 label / 末尾 label | 抛错 |

`parseWay` 的输出是 IR step 数组，可以直接放入 `PathSchema.children`。

## 相对目标点语法糖

`parseTargetSugar` 只处理 React DSL 的字符串 shorthand；非字符串输入原样返回，已经是对象的 target（含 `{ id, anchor? }`、`{ of, offset }`、`{ between, t }`）不会被重写。合法的节点 ref / 偏移字符串按下表转换，非法字符串（如 `'+x,'`、未知 anchor、空 id、纯小数串）抛错：

| Input | Output | 含义 |
| --- | --- | --- |
| `'+10,0'` | `{ relative: [10, 0] }` | 相对前一终点，不更新 prevEnd |
| `'++10,0'` | `{ relativeAccumulate: [10, 0] }` | 相对前一终点，并更新 prevEnd |
| `'node-id'` | `{ id: 'node-id' }` | 普通节点引用 |
| `'node-id.north'` | `{ id: 'node-id', anchor: 'north' }` | 命名 anchor |
| `'node-id.30'` | `{ id: 'node-id', anchor: 30 }` | 数字角度 anchor |
| `[10, 20]` | `[10, 20]` | 非字符串输入原样返回 |
| `{ between: [{ id: 'a' }, { id: 'b' }], t: 0.5 }` | `{ between: [{ id: 'a' }, { id: 'b' }], t: 0.5 }` | 对象 target 原样返回；后续 compile 再解析端点 |

正则允许空格和负数，例如 `'+ -4, 8'` / `'++-4,8'`。

## 错误边界

| Case | Message 前缀 |
| --- | --- |
| way 太短 | `parseWay: way must contain at least 2 items` |
| 第一项是 label | `parseWay: way[0] must be a target` |
| via 在末尾 | `parseWay: via operator ... at end of way` |
| via 后不是 target | `parseWay: via operator ... must be followed by a target` |
| curve-like 在末尾 | `parseWay: curve operator at end of way` |
| curve-like 后不是 target | `parseWay: curve operator must be followed by a target` |
| cycle 承载 label | `parseWay: cycle step cannot carry a label` |
| 连续 label | `parseWay: label operator ... cannot directly follow another label operator` |
| 末尾 label | `parseWay: label operator at end of way` |

这些错误属于 sugar 输入错误，应该在 adapter / 开发期尽早暴露，而不是降级成 warning。
