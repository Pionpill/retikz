---
title: 编译
description: IR 到 Scene 的编译入口、布局选项、文字度量与 warning 合同
---

`@retikz/core` 的 compile 层把 JSON 化的 IR 转成已布局的 `Scene`。React 渲染器、未来的 SSR / Canvas adapter、以及任何自定义导出流程都应该从这里接入。

## 函数

| API | Signature | 用途 |
| --- | --- | --- |
| `compileToScene` | `(ir: IR, options?: CompileOptions) => Scene` | 主编译入口；处理 node / coordinate / path，输出 primitive 列表和 layout |
| `computeLayout` | `(points, padding, round) => Layout` | 从 bbox 候选点计算场景边界；通常由 `compileToScene` 内部调用 |
| `fallbackMeasurer` | `(text, font) => TextMetrics` | 默认文字度量；不准但可运行，适合无 DOM 环境的兜底 |

`compileToScene` 分两步：先按 IR 顺序布局 node / coordinate 并注册 id，再按原始 children 顺序发出 primitive。这样路径可以引用任意已经注册的 node / coordinate，最终渲染顺序仍保持用户写入顺序。

## 编译选项

| Option | Type | Default | 说明 |
| --- | --- | --- | --- |
| `measureText` | `TextMeasurer` | `fallbackMeasurer` | 注入文字度量函数；React adapter 使用浏览器能力，SSR 可以接 fontkit / opentype |
| `padding` | `number` | `10` | layout 外围留白，单位为 user units |
| `precision` | `number` | `2` | 输出 Scene primitive / path d / layout 的小数位；内部几何计算保持双精度 |
| `nodeDistance` | `number` | `1` | `AtPosition` 未写 `distance` 时使用的默认距离 |
| `onWarn` | `(warning) => void` | dev: `console.warn`<br />production: silent | 收集可恢复的编译期 warning |
| `shapes` | `Record<string, ShapeDefinition>` | `—` | 运行时注入自定义 node shape；同名覆盖内置发 `SHAPE_OVERRIDES_BUILTIN`。见 [自定义形状](/core/reference/extending/shape-registry) |
| `arrows` | `Record<string, ArrowDefinition>` | `—` | 运行时注入自定义箭头；同名覆盖内置发 `ARROW_OVERRIDES_BUILTIN`。见 [自定义箭头](/core/reference/extending/custom-arrow) |
| `patterns` | `Record<string, PatternDefinition>` | `—` | 运行时注入自定义填充图案 motif；同名覆盖内置发 `PATTERN_OVERRIDES_BUILTIN`。见 [自定义图案](/core/reference/extending/custom-pattern) |
| `pathGenerators` | `Record<string, PathGeneratorDefinition>` | `—` | 运行时注入自定义曲线生成器；generator step 按 `name` 查表，未注册编译期 throw。见 [路径生成器](/core/reference/extending/path-generator) |
| `composites` | `Array<CompositeDefinition>` | `—` | 运行时注入 Tier 2 composite 展开逻辑；编译第一步把 composite 节点展开成 Tier 1，未注册发 `COMPOSITE_NOT_REGISTERED` 并跳过该节点 |
| `maxCompositeDepth` | `number` | `32` | composite 嵌套展开的最大深度（防环 / 防失控递归），超限或成环 throw |

## 文字度量

`TextMeasurer` 是 compile 层唯一需要外部环境帮助的地方：

```ts
type TextMeasurer = (text: string, font: FontSpec) => TextMetrics;
```

| Type | 字段 | 说明 |
| --- | --- | --- |
| `FontSpec` | `family?` / `size` / `weight?` / `style?` | 传给 measurer 的字体信息；`size` 必填 |
| `TextMetrics` | `width` / `height` / `ascent?` / `descent?` | 文本盒尺寸；ascent / descent 可选 |

如果不传 `measureText`，`fallbackMeasurer` 用 `text.length * size * 0.55` 估算宽度、`size * 1.2` 估算高度。它能保证编译可跑，但不适合做精确排版。

## 警告

`CompileWarning` 是机器可读的同步 warning：

| Field | Type | 说明 |
| --- | --- | --- |
| `code` | string | warning 类型代码 |
| `message` | string | 英文人类可读消息 |
| `path` | string | IR locator，例如 `children[3].path.children[1].to` |

当前内置 code（机器可读常量 `CompileWarningCode` 已公开导出，可按枚举引用而非硬编码字符串）：

| Code | 含义 |
| --- | --- |
| `UNRESOLVED_NODE_REFERENCE` | 路径或位置引用了不存在的节点 / coordinate |
| `PATH_TOO_SHORT` | path 不足 2 步 / 缺前一终点，无法形成有效绘制段，整条 path 跳过 |
| `OFFSET_BASE_UNRESOLVED` | OffsetPosition 的基准 `of` 无法解析 |
| `POLAR_ORIGIN_UNRESOLVED` | PolarPosition 的 `origin` 无法解析 |
| `AT_TARGET_UNRESOLVED` | AtPosition 的 `of` 无法解析 |
| `DUPLICATE_NODE_ID` | 同一命名空间帧内重复 id；后定义覆盖先定义（last-wins） |
| `SHAPE_OVERRIDES_BUILTIN` | `CompileOptions.shapes` 同名 key 覆盖了内置 shape |
| `ARROW_OVERRIDES_BUILTIN` | `CompileOptions.arrows` 同名 key 覆盖了内置箭头 |
| `PATTERN_OVERRIDES_BUILTIN` | `CompileOptions.patterns` 同名 key 覆盖了内置图案 |
| `COMPOSITE_NOT_REGISTERED` | composite 节点的 `namespace.type` 未注册，跳过该节点 |
| `ANIMATION_INVALID_PROPERTY` | 动画 track 的 property 非法（如 scene-root 非 viewBox 属性），该 track 丢弃 |
| `ARC_MISSING_RADIUS` | arc step 缺 `radius`（圆）或 `radiusX`+`radiusY`（椭圆），整条 path 跳过 |
| `PARTIAL_ARC_NEEDS_BOTH_ANGLES` | circlePath / ellipsePath 部分弧需同时给 `startAngle` 与 `endAngle`，否则按完整圆 / 椭圆处理 |
| `PARTIAL_ARC_CLOSED_INVALID` | 带角度的部分圆 / 椭圆不能用 `closed:'closed'`，退回 `'chord'` |

大多数 warning 不会中断编译。但 coordinate / node 自身位置无法解析时**直接抛错、不发 warning**（源码刻意如此：warn 后立即 throw 会让收集器记录一条永不产出 Scene 的死告警），因为它属于「定义位置」的实体，会破坏后续引用合同。

## 输出

`compileToScene` 返回：

| Field | Type | 说明 |
| --- | --- | --- |
| `primitives` | `Array<ScenePrimitive>` | 已布局图元，顺序就是渲染顺序 |
| `layout` | `Layout` | 场景边界 `{ x, y, width, height }` |
| `resources?` | `Array<SceneResource>` | 渲染无关资源表（渐变 / pattern / clip 等 paint server）；adapter 物化（SVG → `<defs>`）。无资源时省略 |
| `animations?` | `Array<IRAnimationTrack>` | scene 根（镜头）时间轴动画 tracks（viewBox property），从 IR 根 animations 透传；无则省略 |

自定义 renderer 必须消费 `resources` 与 `animations`，否则会丢渐变 / 裁剪 / 镜头动画。如果没有任何 bbox 候选点，`computeLayout` 返回 `100×100` 的兜底 layout。
