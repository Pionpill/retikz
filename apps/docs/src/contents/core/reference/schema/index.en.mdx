This group is the data contract of the IR (intermediate representation): the JSON structure a retikz figure ultimately serializes to. AI generation, database storage, and custom converters all build on this schema. From the scene root downward come the named entities, the drawn paths, and the positioning fields that run through them.

> This is the reference layer, not component docs; to author figures, start from [Components](/core/components/tikz).

## What each part covers

| Topic | Responsibility | Reach for it when |
| --- | --- | --- |
| Scene | The IR root: a JSON-serializable description of one figure | Understanding the top-level structure, serializing a whole figure |
| Entity | Named members in a scene: Node and Coordinate | Working with the IR shape of nodes / coordinates |
| Path | The stroke shell, step action sequence, and step targets | Working with the IR shape of paths / connectors |
| Placement | The four forms of a position field: Cartesian / polar / relative / offset | Parsing or generating any positioning field |
