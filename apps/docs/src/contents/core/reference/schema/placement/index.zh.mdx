`position` 字段可填的四种形态。四者都会在 Scene 编译阶段解析为笛卡尔。

## Position

`PositionSchema` 是元组 —— 最简单的位置形态。

<ZodSchema name="PositionSchema" description="笛卡尔位置元组 `[x, y]`。" />

## PolarPosition

`PolarPositionSchema` 是递归的 —— `origin` 字段本身可以又是一个 `PolarPosition`（链式极坐标）。

<ZodSchema name="PolarPositionSchema" description="极坐标位置；origin 可省略、引用节点或继续嵌套极坐标。" descriptions={{
  origin: '原点引用：node id 字符串、笛卡尔 [x,y]、或嵌套 PolarPosition（递归）；省略则取 [0, 0]',
  angle: '角度（°），逆时针为正',
  radius: '半径 / 距离，用户单位',
}} />

## AtPosition

`AtPositionSchema` 是节点间相对定位对象。

<ZodSchema name="AtPositionSchema" description="相对另一个命名实体的方向定位。" descriptions={{
  direction: '从被引用节点指向本节点的方向，视觉约定（above = 视觉向上，screen y-）',
  of: '被引用 node / coordinate 的 id；必须在 IR 中先于本节点出现（不接受前向引用）',
  distance: '从被引用节点中心到本节点中心的距离（用户单位）；默认走 nodeDistance 上下文，再回 1',
}} />

<Comparison target="tikz" title="相对定位">

`AtPositionSchema` 对应 TikZ positioning library 的 `[<direction>=<distance> of <id>]`。`nodeDistance` 上下文对应 TikZ 的全局 `node distance` 缺省值。

</Comparison>

## OffsetPosition

`OffsetPositionSchema` 表示任意 `(dx, dy)` 偏移相对基准点；`of` 三态对称 `PolarPosition.origin`。

<ZodSchema name="OffsetPositionSchema" description="从一个基准点出发再加 `(dx, dy)` 偏移的位置。" descriptions={{
  of: '基准点：节点 id 字符串（前向引用拒绝）/ 笛卡尔 [x, y] 字面值（无需任何节点先定义）/ PolarPosition（递归极坐标）；与 PolarPosition.origin 同形',
  offset: '相对基准点的 (dx, dy) 偏移，用户单位',
}} />

<Comparison target="tikz" title="偏移定位">

`OffsetPositionSchema` 对应 TikZ `calc` library 的 `($(of) + (dx, dy)$)`。

</Comparison>
