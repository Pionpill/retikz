A `position` field can be filled with any of four forms. All resolve to Cartesian at compile time.

## Position

`PositionSchema` is a tuple ‚Äî the simplest position form.

<ZodSchema name="PositionSchema" />

## PolarPosition

`PolarPositionSchema` is recursive ‚Äî the `origin` field may itself be another `PolarPosition` (chained polars).

<ZodSchema name="PolarPositionSchema" />

## AtPosition

`AtPositionSchema` describes relative positioning between named entities.

<ZodSchema name="AtPositionSchema" />

<Comparison target="tikz" title="Relative Positioning">

`AtPositionSchema` maps to TikZ positioning-library forms like `[<direction>=<distance> of <id>]`. The `nodeDistance` context maps to TikZ's global `node distance` default.

</Comparison>

## OffsetPosition

`OffsetPositionSchema` describes an arbitrary `(dx, dy)` offset from a base point; the `of` union shape matches `PolarPosition.origin`.

<ZodSchema name="OffsetPositionSchema" />

<Comparison target="tikz" title="Offset Positioning">

`OffsetPositionSchema` maps to TikZ `calc` library syntax `($(of) + (dx, dy)$)`.

</Comparison>
