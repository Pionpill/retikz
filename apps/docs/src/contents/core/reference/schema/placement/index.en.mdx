A `position` field can be filled with any of four forms. All resolve to Cartesian at compile time.

## Position

`PositionSchema` is a tuple — the simplest position form.

## PolarPosition

`PolarPositionSchema` is recursive — the `origin` field may itself be another `PolarPosition` (chained polars).

<ZodSchema name="PolarPositionSchema" />

## AtPosition

`AtPositionSchema` describes relative positioning between named entities.

<ZodSchema name="AtPositionSchema" />

<Comparison target="tikz" title="Relative Positioning">

`AtPositionSchema` maps to TikZ positioning-library forms like `[<direction>=<distance> of <id>]`. The `nodeDistance` context maps to TikZ's global `node distance` default.

</Comparison>

## OffsetPosition

`OffsetPositionSchema` describes an arbitrary `(dx, dy)` offset from a base point; the `of` union shape matches `PolarPosition.origin`.

<ZodSchema name="OffsetPositionSchema" />

<Comparison target="tikz" title="Offset Positioning">

`OffsetPositionSchema` maps to TikZ `calc` library syntax `($(of) + (dx, dy)$)`.

</Comparison>

## BetweenPosition

`BetweenPositionSchema` takes a point along the segment between two endpoints at proportion `t` (0..1) — `lerp(A, B, t)`. Endpoints use the self-contained `AbsoluteTarget`.

<ZodSchema name="BetweenPositionSchema" />

<span id="absolutetarget"></span>

## AbsoluteTarget

`AbsoluteTargetSchema` is the endpoint type used by `between`: Cartesian / polar / node reference / offset / nested between, **excluding** path-relative (`{ relative }` / `{ relativeAccumulate }`) — which avoids a `Position` ↔ `Target` schema cycle.

<ZodSchema name="AbsoluteTargetSchema" />

<Comparison target="tikz" title="Proportional partway">

`BetweenPositionSchema` maps to TikZ `calc` library syntax `($(A)!t!(B)$)`. The same form works in `Node.position` / `Coordinate.position` / path `Step.to`.

</Comparison>
