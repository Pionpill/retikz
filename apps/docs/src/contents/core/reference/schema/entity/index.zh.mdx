`entity` 涵盖**有 id**、**放在某个位置**、**自身不画线**的 IR 元素。两者注册到同一个 nodeIndex，所以 Node 的 id 与 Coordinate 的 id 共享命名空间。

<span id="node"></span>

## 节点

`NodeSchema` 描述一个被定位的视觉原件。它是 IR 里字段最多的实体 —— 30+ 字段覆盖形状、填充 / 描边、padding、文本、字体、缩放、附加标签。

## 坐标

`CoordinateSchema` 必须有 `id`（这是它存在的全部意义），不参与 layout 扩展，与 Node 共享 nodeIndex。

<Comparison target="tikz" title="Coordinate 对照">

`{ type: 'coordinate', id: 'm', position: [3, 2] }` 对应 TikZ 的 `\coordinate (m) at (3,2);`。

</Comparison>

## 文本块

`TextBlockSchema` 是节点文本块的结构：单字符串表示单行；数组表示多行，数组项可用 `LineSpec` 做行级样式覆盖。
