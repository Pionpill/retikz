`entity` covers IR primitives that **have an id**, are **placed at a position**, and **don't draw paths themselves**. Both register into the same node index, so a Node id and a Coordinate id share the same namespace.

## Node

`NodeSchema` describes one positioned visual primitive. It's the heaviest IR entity — 30+ fields covering shape, fill / stroke style, padding, text content, font, scaling, and attached labels.

<ZodSchema name="NodeSchema" />

## Coordinate

`CoordinateSchema` requires an `id` (its whole purpose), never contributes to layout, and shares the node index with `Node`.

<Comparison target="tikz" title="Coordinate Mapping">

`{ type: 'coordinate', id: 'm', position: [3, 2] }` maps to TikZ `\coordinate (m) at (3,2);`.

</Comparison>

## NodeLabel

`NodeLabelSchema` describes extra text attached around a node border. `Node.label` accepts either one object or an array.

## TextBlock

`TextBlockSchema` is the node text block shape: a single string for one line, or an array for multiple lines where each item may use `LineSpec` for per-line styling.
