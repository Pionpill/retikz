`path` 是一条笔画的外壳容器；它的 `children` 字段是一个 `Step` 动作有序数组，渲染器顺序处理它们产出 SVG path 命令。

## 目标点

`TargetSchema` 是路径端点的 union：笛卡尔 / 极坐标 / 节点引用对象 `{ id, anchor?, offset?, boundary? }` / 偏移定位 `{ of, offset }` / 比例定位 `{ between, t }` / 相对偏移（relative / relativeAccumulate）。节点引用对象里 `boundary?` 单边覆盖本端点贴向的连接面（缺省取节点自身 `boundary`，默认 `shape`）；`anchor?` 缺省时自动贴边裁剪。字符串节点 id 是 React DSL 的便捷写法，会在进入 IR 前解析成对象：

<ZodSchema name="TargetSchema" description="路径目标点：支持笛卡尔、极坐标、节点引用对象 `{ id, anchor?, offset?, boundary? }`、偏移定位、比例定位和相对偏移；字符串 shorthand 只存在于 React DSL。" />

比例定位的完整端点约束见 [定位参考](/core/reference/schema/placement#betweenposition)：`between` 两端必须是自包含端点，不接收 path-relative。
