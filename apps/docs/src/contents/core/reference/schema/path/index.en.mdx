A `path` is the outer container for a drawn line. Its `children` field is an ordered array of `Step` actions; the renderer walks them sequentially to produce SVG path commands. Each step's destination is a `Target`.

## Target

`TargetSchema` is the union for path endpoints: Cartesian / polar / node target object `{ id, anchor?, offset?, boundary? }` / offset placement `{ of, offset }` / proportional placement `{ between, t }` / relative offsets (relative / relativeAccumulate). In the node target object, `boundary?` overrides the connection surface this single endpoint clips to (defaults to the node's own `boundary`, i.e. `shape`); `anchor?` auto-clips to the boundary when omitted. Node id strings are React DSL shorthand and are parsed into objects before reaching the IR:

For the endpoint rules of proportional placement, see [Placement](/core/reference/schema/placement#betweenposition): `between` endpoints must be self-contained and do not accept path-relative targets.
