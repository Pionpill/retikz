This group is about **extending** retikz: capabilities the built-in components don't cover, filled in through runtime-injected extension points. The audience is people shipping third-party packages, doing host integration, or adding new primitives to retikz — for everyday figures, start from [Components](/core/components/layout).

> Extension points are **host / compile-time capabilities and never enter the IR** — the IR stays serializable plain data (LLM / persistence friendly); extension definitions are injected at runtime via `CompileOptions` / component props.

## What each part covers

| Topic | Extends | Reach for it when |
| --- | --- | --- |
| Custom Shapes | Registering new node shapes (`ShapeDefinition` + `CompileOptions.shapes`) | You need hexagon / cylinder / cloud beyond the built-in four |
| Custom Arrows | Registering new arrow styles (`ArrowDefinition` + `CompileOptions.arrows`) | You need bracket / domain-specific end caps |
| Custom Patterns | Registering new fill patterns (`PatternDefinition` + `CompileOptions.patterns`) | You need cross / stripes / special motifs |
| Path Generators | Registering new curve generators (`PathGeneratorDefinition` + `CompileOptions.pathGenerators`) | You need parabola / waves / custom curves |
