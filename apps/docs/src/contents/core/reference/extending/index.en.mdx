This group is about **extending** retikz: capabilities the built-in components don't cover, filled in through runtime-injected extension points. The audience is people shipping third-party packages, doing host integration, or adding new primitives to retikz — for everyday figures, start from [Components](/core/components/tikz).

> Extension points are **host / compile-time capabilities and never enter the IR** — the IR stays serializable plain data (LLM / persistence friendly); extension definitions are injected at runtime via `CompileOptions` / component props.
