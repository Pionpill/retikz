This group is about **extending** retikz: capabilities the built-in components don't cover, filled in through runtime-injected extension points. The audience is people shipping third-party packages, doing host integration, or adding new primitives to retikz â€” for everyday figures, start from [Components](/core/components/layout).

> Extension points are **host / compile-time capabilities and never enter the IR** â€” the IR stays serializable plain data (LLM / persistence friendly); extension definitions are injected at runtime via `CompileOptions` / component props.

## What each part covers

| Topic | Extends | Reach for it when |
| --- | --- | --- |
| Custom Shapes | Registering new node shapes (`ShapeDefinition` + `CompileOptions.shapes`) | You need cylinder / cloud beyond the 6 registered built-ins + 2 presets (hexagon is already expressible via the built-in `polygon{ sides:6 }`) |
| Custom Arrows | Registering new arrow styles (`defineArrow` / `ArrowDefinition` + `CompileOptions.arrows`) | You need bracket / domain-specific end caps |
| Custom Patterns | Registering new fill patterns (`definePattern` / `PatternDefinition` + `CompileOptions.patterns`) | You need cross / stripes / special motifs |
| Path Generators | Registering new curve generators (`PathGeneratorDefinition` + `CompileOptions.pathGenerators`) | You need parabola / waves / custom curves |
| Custom Animation | Registering new easing curves / animation property channels (`<Layout>`'s `easings` / `animationProperties`) | You need spring easing, blur, or other animation properties beyond the built-ins |
