这一分组讲怎么**扩展** retikz：内置组件覆盖不到的能力，通过运行时注入的扩展点补上。读者是发第三方包、做宿主集成、或给 retikz 加新基元的人——日常画图从 [组件](/core/components/layout) 开始即可。

> 扩展点都是**宿主 / 编译期能力，不写进 IR**——IR 始终是可序列化的纯数据（LLM / 持久化友好），扩展定义经 `CompileOptions` / 组件 props 在运行时注入。

## 职责一览

| 主题 | 扩展什么 | 何时用 |
| --- | --- | --- |
| 自定义形状 | 注册新的节点形状（`ShapeDefinition` + `CompileOptions.shapes`）| 内置 6 注册形状 + 2 preset 之外要 cylinder / cloud 等（hexagon 已可用内置 `polygon{ sides:6 }` 表达） |
| 自定义箭头 | 注册新的箭头样式（`defineArrow` / `ArrowDefinition` + `CompileOptions.arrows`） | 需要 bracket / 业务箭头等新端点样式 |
| 自定义图案 | 注册新的填充图案（`definePattern` / `PatternDefinition` + `CompileOptions.patterns`） | 需要 cross / 条纹 / 特殊 motif |
| 路径生成器 | 注册新的曲线生成器（`PathGeneratorDefinition` + `CompileOptions.pathGenerators`） | 需要 parabola / 波形 / 自定义曲线 |
| 自定义动画 | 注册新的缓动曲线 / 动画属性通道（`<Layout>` 的 `easings` / `animationProperties`） | 需要 spring 缓动、blur 等内置之外的动画属性 |
