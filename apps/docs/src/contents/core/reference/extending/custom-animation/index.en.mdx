An [animation](/core/concepts/primitive-model/animation)'s `easing` and `property` are both **open strings**: beyond the built-ins you can register your own easing curves and property channels. The two injection points are two `<Layout>` props — the same idea as the [shape](/core/reference/extending/shape-registry) / [arrow](/core/reference/extending/custom-arrow) extensions: the IR / preset only carries a name, the definition is injected at `<Layout>`.

## Custom easing (`easings`)

`<Layout easings={{ name: curve }}>` registers an easing; a preset's or track's `easing` referencing that name takes effect:

<ComponentPreview name="custom-easing" size="xs" />

```tsx
<Layout easings={{ spring: [0.34, 1.56, 0.64, 1] }}>
  <Node animations={[scaleIn({ from: 0.4, easing: 'spring' })]}>spring</Node>
</Layout>
```

Two registered-value forms:

| Form | Type | Backend support |
| --- | --- | --- |
| cubic-bezier tuple | `[x1, y1, x2, y2]` | SVG (becomes CSS `cubic-bezier()`) + Canvas |
| easing function | `(t: number) => number` | **Canvas only**; can't enter CSS → falls back to `linear` with a warning |

`y` may exceed `[0, 1]` (e.g. `1.56`) for overshoot / spring effects. An unregistered easing name → warns and falls back to `linear`.

## Custom property channel (`animationProperties`)

`property` is open too: register a channel name plus its interpolator and how to apply it, and you can animate properties beyond the built-ins (e.g. a `blur` filter). Below registers `blur` and runs a focus-in entrance (click ↺ at the top-left to replay):

<ComponentPreview name="custom-property" size="xs" />

`AnimationPropertyDefinition`:

| Field | Type | Description |
| --- | --- | --- |
| `interpolate` | `(from, to, t) => value` | interpolates adjacent keyframe values at segment progress `t∈[0,1]` (fed to the evaluation engine) |
| `applyCanvas` | `(ctx, prim, value) => void` | applies the current value to the Canvas 2D context (called within the primitive's `save/restore` scope) |

<ComponentAlert type="warn" title="Custom property channels are Canvas-only for now" description="Generic interpolation of a custom property runs in JS (applyCanvas), which lives in the Canvas backend. SVG has no built-in mapping, so an unrecognized property warns and skips that track (rendering the resting state). Use the canvas renderer for custom property channels." />

## Backend support

| Extension | SVG | Canvas |
| --- | --- | --- |
| custom easing (cubic-bezier) | ✅ | ✅ |
| custom easing (function) | ⚠️ falls back to linear + warns | ✅ |
| custom property channel | ⚠️ warns + skips | ✅ |

Any unregistered custom easing / property degrades diagnosably: a warning plus the track falling back to the resting state — the figure is never dropped.
