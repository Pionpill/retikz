retikz 自带的 `rectangle` / `circle` / `ellipse` / `diamond` 也是通过 Shape Registry 注册出来的形状。也就是说，内置形状和你自己写的形状走同一套机制：给一个形状名，再提供它如何包住文字、如何计算边界、如何画出来。

第三方包（例如 `@retikz/shapes-flow` / `-uml` 之类）或宿主代码可以用同样的方式注册新形状，不需要修改 core。

> 这里是参考层。日常写图只用内置形状看 [节点](/core/components/node/overview) 即可；需要 hexagon / cylinder / cloud 这类形状再来这里。

## 核心约束：IR 里的形状名永远是字符串

`node.shape` 只保存一个名字，比如 `'hexagon'`。这个名字可以安全写进 JSON，适合持久化、JSON Patch，也适合 LLM 生成。

真正的形状定义叫 `ShapeDefinition`，里面会包含函数，因此**不会进入 IR**。编译时再通过 `CompileOptions.shapes` 把这些函数交给 retikz。schema 只检查 `shape` 是非空字符串；如果名字没有注册，错误会在 **compile 阶段**抛出，并列出当前可用的形状名。

## 形状定义

一个自定义形状需要回答四个问题。retikz 已经负责文字测量、样式合并、旋转分组和最终渲染；形状作者只需要描述这个形状自己的几何和绘制方式。

表里的几个类型先按直觉理解即可：

- `Rect` 是“这个节点最终占用的外接矩形”：包含中心点 `x/y`、宽高 `width/height`，以及可选旋转角度 `rotate`。
- `Position` 是一个点坐标，形如 `[x, y]`。
- `ScenePrimitive` 是最终交给渲染器的基础图元，例如 `path`、`rect`、`ellipse`。

| 方法 | 参数与返回值 | 作用 |
| --- | --- | --- |
| `circumscribe`<br />`(innerHalfW, innerHalfH) => { halfWidth, halfHeight }` | `innerHalfW` / `innerHalfH`：文字和 padding 已经占用的半宽、半高。<br />`halfWidth` / `halfHeight`：外层形状最终需要的半宽、半高。 | 节点布局时调用，回答“为了包住文字和 padding，形状外接矩形需要多大”。例如菱形通常要放大，矩形通常原样返回。 |
| `boundaryPoint`<br />`(rect, toward) => Position` | `rect`：节点外接矩形。<br />`toward`：从节点中心看出去的目标点。<br />返回值：从中心朝 `toward` 连线时，与形状边界相交的点。 | 路径连接节点边界、解析 `A.30` 这种数字角度时调用，决定线条应该停在形状边上的哪里。 |
| `anchor`<br />`(rect, name) => Position \| undefined` | `rect`：节点外接矩形。<br />`name`：锚点名，例如 `north`、`south east`、`center`。<br />返回值：认识这个锚点就返回坐标；不认识就返回 `undefined`。 | 用户写 `A.north`、`A.center` 或自定义锚点名时调用，决定命名锚点在哪里。未知锚点由编译层统一报 Unknown anchor。 |
| `emit`<br />`(rect, style, round) => Iterable<ScenePrimitive>` | `rect`：节点外接矩形。<br />`style`：已经合并好的填充、描边、透明度等样式。<br />`round`：用于统一坐标精度的函数。<br />返回值：一个或多个要画出来的基础图元。 | 编译节点视觉内容时调用，决定这个形状最终画成哪些基础图元。简单形状通常返回一个 path 或 rect；复杂形状可以返回多个 primitive，比如“外框 + 两条竖线”。 |

数字角度（`A.30`）不需要单独实现。只要形状实现了 `boundaryPoint`，编译层会把 `30` 度换成一个方向点，再调用你的 `boundaryPoint`。

## 注入形状定义

```ts
import { compileToScene, type ShapeDefinition } from '@retikz/core';
import { worldToLocal, localToWorld } from '@retikz/core';

// factory 只是一个普通函数：传入参数，返回一份 ShapeDefinition
const createPolygonShape = (sides: number): ShapeDefinition => ({
  circumscribe: (hw, hh) => {
    const r = Math.hypot(hw, hh);
    return { halfWidth: r, halfHeight: r };
  },
  boundaryPoint: (rect, toward) => {
    /* 先把 toward 转到局部坐标，再计算射线和多边形边界的交点 */
  },
  anchor: (rect, name) => (name === 'center' ? [rect.x, rect.y] : undefined),
  *emit(rect, style, round) {
    yield {
      type: 'path',
      commands: [/* n 顶点 + close */],
      fill: style.fill ?? 'transparent',
      fillOpacity: style.fillOpacity,
      stroke: style.stroke ?? 'currentColor',
      strokeOpacity: style.strokeOpacity,
      strokeWidth: style.strokeWidth ?? 1,
      dashPattern: style.dashPattern,
      opacity: style.opacity,
    };
  },
});

// IR 里只写 shape: 'hexagon'，具体定义由 options 提供
const scene = compileToScene(ir, { shapes: { hexagon: createPolygonShape(6) } });
```

在 React 里，把同一份 `shapes` 传给 `<Layout>`：

```tsx
<Layout shapes={{ hexagon: createPolygonShape(6) }}>
  <Node shape="hexagon" position={[0, 0]} text="准备" />
</Layout>
```

## 实现清单

写第三方 shape 时，优先逐项核对这几个点：

- `circumscribe` 回答“为了包住文字和 padding，外接矩形需要多大”，不要直接返回视觉 path 的原始尺寸。
- `boundaryPoint` 要处理 `toward` 刚好等于中心点的情况；常见做法是选一个固定默认方向。
- `boundaryPoint` 和 `anchor` 如果涉及旋转，先用 `worldToLocal` 进入形状自己的局部坐标，算完后再用 `localToWorld` 返回世界坐标。
- `anchor` 只处理自己认识的名字；未知名返回 `undefined`，让编译层抛统一错误。
- `emit` 里用传入的 `round` 处理坐标与尺寸，并尽量透传相关样式字段：`fillOpacity` / `strokeOpacity` / `dashPattern` / `opacity` / `roundedCorners`。

## 覆盖内置与未知名

- **同名覆盖**：如果写 `shapes: { rectangle: myRect }`，新的定义会覆盖内置 rectangle。编译期会通过 `onWarn` 发出 `SHAPE_OVERRIDES_BUILTIN`，提醒你正在替换内置行为。
- **未知 shape**：如果 `node.shape` 写了一个没有注册过的名字，compile 会抛出 `Unknown shape '<name>'; registered shapes: <sorted list>`。因为 retikz 不知道这个形状的边界和锚点，不能安全降级成别的形状。

## 公开导出

`@retikz/core` 为第三方形状作者导出：`ShapeDefinition` / `ShapeStyle` / `BUILTIN_SHAPES` / `BuiltinShapeName` / `Rect` / `Position` / `worldToLocal` / `localToWorld`。
