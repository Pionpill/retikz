> 这里是参考层,不是组件文档;写图请从 [组件](/core/components/layout) 开始。

Canvas 后端把图画成像素位图,而不是 DOM。它和 SVG 渲染同一份 Scene、视觉一致,区别在于整张图是一块画布而非一棵元素树:没有逐图元的 DOM 节点,因此图元成百上千、或要高频重绘 / 动画时,它比 SVG 更稳。代价是失去了 CSS 命中和无障碍可读性——这两件事 retikz 用坐标命中查询和统一的水合语义来补。

## 怎么用

**React**——给 `<Layout>` 显式传 `renderer="canvas"`:

```tsx
import { Layout, Node } from '@retikz/react';

<Layout renderer="canvas">
  <Node id="a">hello</Node>
</Layout>
```

**Vanilla(浏览器)**——`mountCanvas` 把图挂成 `<canvas>`,返回的 view 比 `mountSvg` 多了 canvas 侧的水合与坐标映射:

```ts
import { mountCanvas } from '@retikz/vanilla';

const view = mountCanvas(container, ir);
// view.root 是 <canvas>;view.update(next) 重绘;view.dispose() 卸载
```

入参和 SVG 一致——接收 `Scene`、`IR` 或 `Figure`;`devicePixelRatio` 缺省读 `globalThis.devicePixelRatio`、再回退 1,高清屏自动清晰。

## 给 Canvas 图加交互

Canvas 没有 DOM 图元,定位靠**坐标命中查询**:把指针的 client 像素逆映射回 Scene 坐标,再做几何命中得到图元 `id`。这套定位封装在 `view.hydrate` 里,写法和 SVG 完全一样:

```ts
const view = mountCanvas(container, ir);
const handle = view.hydrate({
  handlers: { box: { click: event => toggle('box') } },
});
// handle.dispose() 解绑
```

同一张 `handlers` 表在 SVG 与 Canvas 下行为完全一致——详见 [水合](/core/reference/renderer/hydration)。

## Node 图片导出

服务端或 CLI 里要直接生成 PNG / JPEG / WebP 时,使用独立入口 `@retikz/render/canvas-node`。它不会把 native canvas 打进浏览器包;调用前需要在应用侧安装可选 peer:

```bash
pnpm add @napi-rs/canvas
```

```ts
import { compileToScene } from '@retikz/core';
import { renderSceneToImage } from '@retikz/render/canvas-node';

const scene = compileToScene(ir);
const png = await renderSceneToImage(scene, {
  width: 960,
  height: 540,
  devicePixelRatio: 2,
  format: 'png',
  background: '#fff',
});
```

`renderSceneToImage` 消费已编译的 `Scene`,与浏览器 Canvas 一样走 `drawScene`,所以渐变、图案、裁剪、marker、动画截帧等语义保持一致。`width` / `height` 是 CSS 尺寸,实际位图尺寸为它们乘以 `devicePixelRatio`。

## Canvas 还是 SVG

| | Canvas | SVG |
| --- | --- | --- |
| 输出 | 像素位图 | 矢量 DOM 节点 |
| 大量图元 | 即时模式更稳 | 节点多时变慢 |
| 缩放 | 受分辨率限制 | 任意倍数清晰 |
| CSS / 选择器 | 整块画布 | 每个图元可命中 |
| 默认 | 显式 `renderer="canvas"` | 是 |

图元成百上千或要高频重绘 / 动画时用 Canvas;图元不多、要清晰矢量或 CSS 交互时用默认的 [SVG](/core/reference/renderer/svg)。两者渲染同一份 Scene、水合写法也一样,可以随时切换。
