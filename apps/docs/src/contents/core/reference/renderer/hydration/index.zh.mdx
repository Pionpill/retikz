> 这里是参考层，不是组件文档；先会画图（从 [组件](/core/components/layout) 开始）再回来给图加交互。

retikz 的图是先画出来、再加交互的：服务端或静态先渲染出一张完整的图（SSR / 字符串输出），客户端拿到这张图后，按图元 id 把你的事件 handler 绑回去。这个绑定步骤叫**水合（hydration）**。它不重建组件树、不接管应用状态，只做一件事：把「图元 ↔ 用户函数」对应起来。

之所以分两步，是因为事件 handler 是函数、不能被序列化——它不进 IR，只活在 runtime。IR 里只留一个稳定的挂点：元素的 `id`。渲染时这个 id 会被带到图元上（SVG 写成 `data-retikz-id`，Canvas 留作命中查询的返回值），客户端就能按 id + 事件名把函数绑上。SVG 与 Canvas 走的是**同一套绑定语义**，差别只在「怎么把一次指针事件定位到图元」这一层。

## React：在组件上写事件 props

给 `<Node>` / `<Path>` / `<Scope>` 加 `on<事件>` props 即可——前提是元素有 `id`。handler 签名是 `(event, context) => void`：第一参是原生 DOM `Event`，第二参 `context` 是 renderer 无关的 runtime 上下文（见 [下文](#handler-的-context)）。只用 `event` 的旧式 `event => …` 写法照常工作。

<ComponentPreview name="hydration-click" size="md" interactive />

<p className="text-center text-sm text-muted-foreground">点击橙色节点切换高亮，旁边的计数同步累加。</p>

```tsx
import { Layout, Node } from '@retikz/react';

<Layout>
  <Node id="box" onClick={event => toggle('box')} onPointerEnter={highlight}>
    点我
  </Node>
</Layout>
```

可用事件 props：

| Prop | 触发时机 |
| --- | --- |
| `onClick` | 单击图元 |
| `onDoubleClick` | 双击图元 |
| `onRightClick` | 右键图元（默认不抑制浏览器菜单，需自行 `event.preventDefault()`） |
| `onPointerDown` / `onPointerUp` | 指针在图元上按下 / 抬起 |
| `onPointerMove` | 指针在图元上移动 |
| `onPointerEnter` / `onPointerLeave` | 指针进入 / 离开图元（跨子元素不重复触发） |
| `onWheel` | 在图元上滚轮 |

几条约束：

- handler 只在元素**有 `id`** 时可绑；无 id 带 handler 会被开发模式警告并跳过。
- `<Coordinate>` 是命名点、不绘制任何图形、没有可点面积，**不接** handler。
- `id` 重复时同 id 的不同事件会合并、同事件后者覆盖前者（开发模式给警告）。

### ir 模式：用 Layout 的 handlers prop

直接喂 IR（`<Layout ir={...}>`、没有 JSX children）时，没有组件 props 可写，改用 `<Layout>` 的 `handlers` prop，按 id 提供：

```tsx
<Layout
  ir={ir}
  handlers={{
    box: { click: event => toggle('box') },
    edge1: { click: selectEdge },
  }}
/>
```

## Vanilla：hydrate 与 mountCanvas

无框架场景下，先渲染出图、再调一个水合函数。SVG 与 Canvas 各有入口，但传入的 `handlers` 形态一致。

**SVG**——已挂好的 `<svg>`（`mountSvg` 或注入 SSR 字符串产物）上根级委托绑定：

```ts
import { mountSvg, hydrate } from '@retikz/vanilla';

const view = mountSvg(container, ir);
const handle = hydrate(view.root, {
  handlers: { box: { click: event => toggle('box') } },
});
// handle.dispose() 解绑；view.dispose() 一并解绑
```

**Canvas**——`mountCanvas` 把 IR 挂成 `<canvas>`，返回的 view 自带坐标映射，`view.hydrate` 用命中查询定位图元：

```ts
import { mountCanvas } from '@retikz/vanilla';

const view = mountCanvas(container, ir);
const handle = view.hydrate({
  handlers: { box: { click: event => toggle('box') } },
});
```

`handlers` 是一张 `{ [id]: { 事件名: handler } }` 表，事件名用 `click` / `doubleClick` / `rightClick` / `pointerDown` / `pointerUp` / `pointerMove` / `pointerEnter` / `pointerLeave` / `wheel`（与 React 的 `on<事件>` 一一对应，去掉 `on` 前缀、首字母小写）。

SSR 后用独立的 `hydrate(root, options)` 入口绑定（容器里已是 SSR 注入的 `<svg>` 字符串）。要让 context 拿到 meta / 几何 / 动画控制，把 `scene`（或 `toScene(ir)`）一并传入；不传则只给最小 context（见下）：

```ts
import { hydrate, mountSvg } from '@retikz/vanilla';

// mountSvg / mountCanvas 的 view 自带 view.hydrate，context 自动用本 view 的 Scene
mountSvg(container, ir).hydrate({ handlers: { box: { click: (event, context) => context.animation.restart() } } });

// SSR 后独立水合：传 scene 才得富 context（meta / 几何 / 动画）
hydrate(root, { handlers, scene });
```

## handler 的 context

handler 第二参 `context` 把「命中的那个语义元素」补齐成一个 renderer 无关的对象——不论 SVG 还是 Canvas，写法一致。它表达的是**按 `id` 聚合的语义元素**（一个多形状 Node 的多片图元共享同一 id，算「一个元素」），不是底层单个图元。

| 字段 | 含义 |
| --- | --- |
| `id` | 命中的语义元素 id |
| `meta` | 该元素的 provenance（来自 `<Node meta={…}>`，认 datum / series / layer）；无则 `undefined` |
| `renderer` | `'svg'` \| `'canvas'` |
| `element` | 命中的 DOM 元素：SVG = 被点中那片 `data-retikz-id` 图元；Canvas = `null`（无逐元素 DOM） |
| `root` | figure 根（`<svg>` 或 `<canvas>`） |
| `point` | 指针在 scene user units 的坐标（逆 meet-fit）；非指针事件 `null` |
| `geometry` | 该 id 全部图元的并集包围盒 `bbox` + 中心 `center`（scene user units）；无 scene 时 `undefined` |
| `animation` | 动画控制：`play` / `pause` / `restart` / `stop` / `seek`（缺省作用于命中元素，传 id 控别的元素） |
| `scene` | 当前 Scene（逃生舱）；standalone `hydrate` 未传 scene 时 `undefined` |

**回调触发动画**：给图元配 `trigger: 'manual'` 的动画（不自播、等命令），handler 内 `context.animation.restart()` 即可点一下重播：

<ComponentPreview name="hydration-context" size="md" interactive />

```tsx
import { Layout, Node, pulse } from '@retikz/react';

<Layout>
  <Node
    id="ball"
    meta={{ label: '小球' }}
    animations={[{ ...pulse({ peak: 1.4 }), trigger: 'manual', iterations: 1 }]}
    onClick={(event, context) => {
      console.log(context.meta);     // { label: '小球' }
      context.animation.restart();   // 点一下重播本节点动画
    }}
  >
    点我
  </Node>
</Layout>
```

<ComponentAlert title="context 永远传入" description="信息不全只表现为字段缺省（meta / geometry / scene 可能 undefined、animation 在无 scene 时为 no-op），而不是没有 context。只用 event 的 handler 可以完全忽略它。" />

## 双模等价

同一张 `handlers` 表，在 `renderer="svg"` 与 `renderer="canvas"` 下行为完全一致——共用同一注册表与分发逻辑，只有「怎么定位图元」这层不同：SVG 用 `data-retikz-id` 反查最近祖先，Canvas 用几何命中查询返回图元 id。切换 renderer 不需要改任何 handler。

| 模式 | 定位方式 |
| --- | --- |
| `renderer="svg"` | 事件目标向上找最近带 `data-retikz-id` 的祖先 |
| `renderer="canvas"` | 指针坐标经命中查询定位到图元 id |

context 的 `id` / `meta` / `geometry` / `point` 在两后端一致；`context.animation` 也两后端都是 **per-id 精确控制**（缺省作用于命中元素、传 id 控别的）：SVG 经 `getAnimations()` 控该元素的 WAAPI / CSS 动画，Canvas 经 per-id 虚拟时钟（在单 rAF 共享时钟上给每个 id 叠加独立 offset / 暂停 / 激活态）。唯一差异：Canvas 无逐元素 DOM，故 `context.element` 恒为 `null`（用 `root` + `point`）。
