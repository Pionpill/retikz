SVG 是 retikz 的默认渲染器。它把编译好的图画成一棵真实的 `<svg>` DOM:每个节点、每条路径都是页面上的元素,能被 CSS 命中、被选择器抓取、被无障碍工具读取。矢量输出在任意缩放下都清晰,适合绝大多数图表和示意图——除非图元成百上千或要高频重绘,否则就用它。

## 怎么用

**React**——`<Layout>` 默认就是 SVG,`renderer` 可省:

```tsx
import { Layout, Node } from '@retikz/react';

<Layout renderer="svg">      {/* renderer="svg" 是默认值,可省 */}
  <Node id="a">hello</Node>
</Layout>
```

**Vanilla(浏览器)**——`mountSvg` 把图挂成 DOM,返回的 view 里 `root` 是 `<svg>` 元素、`dispose` 卸载:

```ts
import { mountSvg } from '@retikz/vanilla';

const view = mountSvg(container, ir);
// view.root 是 <svg>;view.update(next) 整图重渲染;view.dispose() 卸载
```

**Vanilla(SSR / 构建期)**——`renderToSvgString` 直接产 HTML 字符串,不碰 DOM,可在 Node 里跑:

```ts
import { renderToSvgString } from '@retikz/vanilla';

const svg = renderToSvgString(ir);   // '<svg ...>...</svg>',塞进 HTML 即可
```

三个入口都接收已编译的 `Scene`、待编译的 `IR`,或命令式 builder 产出的 `Figure`。

## 给 SVG 图加交互

SVG 图元渲染时会带上元素 `id`(写成 `data-retikz-id` 属性),客户端据此把事件 handler 绑回去——见 [水合](/core/reference/renderer/hydration)。

## SVG 还是 Canvas

| | SVG | Canvas |
| --- | --- | --- |
| 输出 | 矢量 DOM 节点 | 像素位图 |
| 缩放 | 任意倍数清晰 | 受分辨率限制 |
| CSS / 选择器 | 每个图元可命中 | 整块画布 |
| 大量图元 | 节点多时变慢 | 即时模式更稳 |
| 默认 | 是 | 显式 `renderer="canvas"` |

图元不多、要清晰矢量或 CSS 交互时用 SVG(默认);图元成百上千或高频重绘时换 [Canvas](/core/reference/renderer/canvas)。两者渲染同一份 Scene、视觉一致,水合写法也完全一样,可以随时切换。
