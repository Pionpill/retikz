`@retikz/math` is retikz's **zero-dependency pure computational geometry** package: vector math, affine transforms, arc primitives, intersections, triangle in/circumcircles, point-in-polygon, and convex hull. Everything is a pure function over plain `Position = [number, number]` arrays — no classes, no IR, no zod, no runtime dependencies.

`@retikz/core` depends on it for low-level computation. The common primitives (`Position` / `point` / `lerp` / `localToWorld` / `worldToLocal`) are **re-exported by `@retikz/core`** — for everyday drawing just use core; you don't need to install math separately. Depend on `@retikz/math` directly when you need capabilities core does not re-export (intersections, in/circumcircles, hull, …).

> This is the reference layer, not component docs; to draw, start from [Components](/core/components/layout).

## Install

```bash
pnpm add @retikz/math
```
