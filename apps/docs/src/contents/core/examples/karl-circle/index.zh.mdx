## 引言

这是 [TikZ 入门教程](https://tikz.dev/tutorial) 最后那张「Karl 给女儿讲三角函数」的图：单位圆 + 30° 角下 sin / cos / tan 的几何关系示意。

本页把它拆成 7 个累加 step：从一个孤零零的圆开始，每步加一点东西、对应学一点 retikz 能力——结束时既得到完整图，也顺手看完了最适合新手上手的三个 sugar：[Circle](/core/components/shapes/circle-ellipse) / [Grid](/core/components/helpers/grid) / [Sector](/core/components/shapes/arc-sector)。它们把原本要手写的 Path / Step 片段压短了，后面的 [Draw](/core/components/draw/overview) / [Coordinate](/core/components/node/coordinate) / [Node](/core/components/node/overview) 则继续把完整图拼起来，顺带练到极坐标、多 Draw 协同、Node 多行多色文本等高频技巧。

<ComponentPreview name="karl-circle-07-info" size="lg" />

如果你是从 TikZ 迁移过来，这页重点看 4 个差异：retikz 坐标 y 向下；圆 / 网格 / 扇形现在可以直接用 sugar；投影 / 交点目前需要手算或命名 Coordinate；`foreach` 风格重复图元通常在 React 里用 `.map()` 生成。

## 过程

### 步骤 1：画单位圆

最小起步——一个原点为中心的圆。现在直接用 [Circle](/core/components/shapes/circle-ellipse) 更短：`center + radius` 就能把这一步写完；底层仍会展开成等价的 Path / Step，但读代码时一眼就能看懂“这是个圆”。`lineCap="round"` 让圆轮廓端点圆润一点。

<ComponentPreview name="karl-circle-01-circle" size="lg" />

### 步骤 2：加坐标轴 + 端点标签 + 命名锚

x 轴和 y 轴用 [Draw](/core/components/draw/overview) 配 `arrow="->"` 一键画带箭头的线段。轴端的 `x` / `y` 文字用 [Node](/core/components/node/overview)（`stroke="none"` 去掉默认外框），独立放在轴外侧。[Coordinate](/core/components/node/coordinate) 给两个端点命名（`x-axis` / `y-axis`），后续 step 可以用 id 字符串引用——本例其实没引用到，但保留以贴合 TikZ 的 `coordinate(x axis)` 风格习惯。

<ComponentPreview name="karl-circle-02-axes" diffFrom="karl-circle-01-circle" size="lg" />

### 步骤 3：加网格 + 刻度

背景网格现在直接用 [Grid](/core/components/helpers/grid) sugar；它会展开成多条线，不再需要手写 5 横 5 竖，所以新手读起来更像“我要一张网格背景”，而不是“我要拼 10 条线”。刻度同样靠数组遍历，每条短线配一个无背景的 [Node](/core/components/node/overview)；label 位置在 tick 基础上**轻微偏移**（x 轴左偏 10px / y 轴下偏 10px），避开圆周线与轴线交叠。

retikz 坐标统一是 SVG 像素，y 向下为正——所以「向上 1 单位」写成 `[0, -100]`（本例取 100px = 单位圆半径作为长度基准）。顶层 `<Layout>` 不接 picture-level scale + 不翻 y，TikZ 教程里的数学坐标得手动给 y 加负号。

<ComponentPreview name="karl-circle-03-ticks" diffFrom="karl-circle-02-axes" size="lg" />

### 步骤 4：画 30° 扇形 + α 标签

扇形现在直接用 [Sector](/core/components/shapes/arc-sector) sugar。它会从圆心出发、画弧、再闭合回圆心，所以这一步比手写 `arc` + `line` 更接近“画一个扇形”的直觉，也更适合刚接触 retikz 的读者先记住一个清楚的 API。

α 文字标签用 [Node](/core/components/node/overview) 配极坐标位置（`{ angle, radius }`）——`{ angle: -15, radius: 22 }` 把它放在扇形的视觉中央（角度负、朝屏幕上方）。

<ComponentPreview name="karl-circle-04-wedge" diffFrom="karl-circle-03-ticks" size="lg" />

### 步骤 5：sin α 红线 + cos α 蓝线

sin 是 (30°, 100px) 垂直落到 x 轴；cos 是落点回原点。两段都靠 [Draw](/core/components/draw/overview)，way 数组里中间插一个 `{ label: { text, side } }` 算子给段挂边标注。`thickness="thick"` 是语义 stroke 档位，省得写具体像素值。

注：原 TikZ 用 `(30:1cm |- x axis)` 直接表达"30°,1cm 点与 x 轴的投影交点"。retikz 暂无投影 target，几何上等于 (cos30°, 0)，手算坐标喂进去。

<ComponentPreview name="karl-circle-05-sin-cos" diffFrom="karl-circle-04-wedge" size="lg" />

### 步骤 6：tan α 橙线 + 辅助射线

tan 线从 (1, 0) 垂直到 (1, tan30°)。原图用 TikZ 的 named path + intersections 求"x=1 竖线与原点 30° 射线"的交点；retikz 暂无这类构造，手算 (1, tan30°) 并用 [Coordinate](/core/components/node/coordinate) 命名为 `t`，下方延伸到 `t` 的辅助射线就能直接引用这个 id。

注意"多个 [Draw](/core/components/draw/overview) 协同 + 共享命名锚"——这是 retikz 把"一张图的多条 path 串起来"的标准套路。

<ComponentPreview name="karl-circle-06-tan" diffFrom="karl-circle-05-sin-cos" size="lg" />

### 步骤 7：右侧多色信息说明框

最后加一个图右侧的彩色说明框。[Node](/core/components/node/overview) 的 `text` prop 接字符串数组，**每行可以是 string 或 `{ text, fill }`**——后者让该行单独换色。视觉分组靠「彩色公式行 + 默认色注释行」的交替表达，不再额外加空行间隔（空 `''` 行会让 textHeight 把 lineHeight 算进盒高、但 SVG 渲染上空 tspan 的 dy 在浏览器间不稳，外框会撑出明显的下边空白）。

`shape="rectangle"` + `cornerRadius` + `align="left"` 把它装成一个左对齐的圆角卡片，`stroke="lightgray"` + `dashed` 加一圈浅灰虚线外框（无填充，主题切换都看得见）。注意行内片段着色（同一行里某几个词换色）目前不支持，只能整行换色——见「已知限制」。

## 限制

| 能力 | 计划 | 现状 / 兜底 |
| --- | --- | --- |
| 数学 y-up 坐标系 | 🚧 未来支持 | 缩放本身可用 Scope 的 `scale` 变换；但还没有 Layout 级的「y 轴朝上」开关（`scale` 的 y 取负会连文字一起翻转），本例直接写像素坐标 + 手动给 y 加负号 |
| 投影 target（`(A \|- B)`） | 🚧 未来支持 | 手算 `[cos30 * 100, 0]` |
| named path + intersection target | 🚧 未来支持 | 手算 `[100, -tan30 * 100]` |
| LineSpec 行内 spans | 🚧 未来支持 | 只能整行换色；未来 LineSpec 加 `spans` 字段 |
| inline LaTeX 数学排版（`$\frac{1}{2}$`） | ❌ 不支持 | 用纯文本字符兜底（`1/2` / `sin α`） |
