电表、电源、开关、电阻、滑动变阻器这些课本符号全都做成自定义形状，靠固定端点参与连线。下面按「先做形状 → 再摆元件 → 最后连线标注」的顺序一步步搭。

## 过程

### 步骤 1：把电表做成自定义形状

电压表、电流表是一个圆形表头加左右两根引线，只在左右两个端点接线（和 [Shape Registry](/core/reference/extending/shape-registry) 里的「语义端点：二极管」一个思路）。把它做成自定义形状 `circuit-meter`，先平铺两个（A / V）认识一下。

<ComponentPreview
  name="circuit-01-meters"
  sourceFiles={['circuit-01-meters.meter.tsx']}
  size="xs"
/>

### 步骤 2：平铺其余自定义符号

电源、开关、电阻、滑动变阻器也不是 core 内置形状，用 [Shape Registry](/core/reference/extending/shape-registry) 自定义，集中放在 `circuitShapes.tsx`。它们都只认左右两个端点（电源长板正极在右、短板负极在左）；极板、触点、矩形框、斜箭头都画在形状内部。

<ComponentPreview
  name="ohms-law-circuit-02-custom-shapes"
  sourceFiles={['circuitShapes.tsx']}
  size="xs"
/>

### 步骤 3：将元件放到电路位置

先把电源、开关、电流表、滑动变阻器摆到最终位置——电阻和电压表这一步先不画，留到下一步讲怎么组合。电源竖放在左侧（rotate 90），开关和电流表在上排，滑动变阻器在下排。

<ComponentPreview
  name="ohms-law-circuit-03-placement"
  sourceFiles={['circuitShapes.tsx', 'circuit-01-meters.meter.tsx']}
  size="sm"
/>

### 步骤 4：用 Scope 组合并复用测量单元

一段电阻 + 正下方一个电压表，两端各用折线竖直相连，整组收进一个 [Scope](/core/components/layout/scope)——这就是一个「测量单元」。第二组不重写，直接用 `Scope` 的 `translate` 平移复制、只换 id 前缀。diff 高亮的正是新增的这两组单元。

### 步骤 5：用折角连成闭合回路

元件就位后，用 [Draw](/core/components/draw/overview) 把它们依次连起来：电源顶端 → 开关 → 电流表 → R1 → 沿右侧竖线下来 → R2 → 滑动变阻器 → 回到电源底端。导线全走横平竖直的折角（`|- -|`），并引用元件端点而不是手猜坐标。

最后补上 `E`、`S`、`R1` / `R2`、`U1` / `U2` 和变阻器名等文字标注，图就读得通了（电流表 / 电压表的 A、V 已经写在表头里）。变阻器名是唯一需要本地化的文本，直接写在中英两份 demo 里。

## 能力

| 组件 | 本例中的角色 | 主要步骤 |
| --- | --- | --- |
| [Layout](/core/components/layout/overview) | 固定 1280 x 760 视图，让所有 demo 共用同一套坐标系 | [1](#步骤-1把电表做成自定义形状), [2](#步骤-2平铺其余自定义符号), [3](#步骤-3将元件放到电路位置) |
| [Scope](/core/components/layout/scope) | 统一样式默认值，并把测量单元平移复制复用 | [4](#步骤-4用-scope-组合并复用测量单元) |
| [Node](/core/components/node/overview) | 承载电表、电源、开关、电阻、滑动变阻器和文字标注 | [1](#步骤-1把电表做成自定义形状), [3](#步骤-3将元件放到电路位置), [4](#步骤-4用-scope-组合并复用测量单元), [6](#步骤-6标注元件) |
| [Draw](/core/components/draw/overview) | 用折线接出测量单元、用折角连成主回路 | [4](#步骤-4用-scope-组合并复用测量单元), [5](#步骤-5用折角连成闭合回路) |
| [ShapeDefinition](/core/reference/extending/shape-registry) | 把电表、电源、开关、电阻和滑动变阻器做成有固定端点的自定义形状 | [1](#步骤-1把电表做成自定义形状), [2](#步骤-2平铺其余自定义符号) |

## 限制

| 能力 / 限制 | 计划 | 现状 / 兜底 |
| --- | --- | --- |
| 电路专用组件库 | ❌ 不支持 | 这些符号只是本例里的自定义形状，适合展示底层能力，不适合直接当大规模电路编辑器用 |
| 电阻 / 变阻器画法拆成路径生成器 | 🔧 优化方案 | 当前矩形框、斜箭头都画在 shape 的 emit 里；可改用[路径生成器](/core/reference/extending/path-generator)或封装专门组件，本页为聚焦端点连线流程未拆 |
| 电学校验 | ❌ 不支持 | retikz 只负责几何和渲染，不判断电压表接法或开关状态是否符合实验规范 |

## 扩展阅读

- **更多扩展点**：除自定义形状（[Shape Registry](/core/reference/extending/shape-registry)）外，retikz 还开放了[自定义箭头](/core/reference/extending/custom-arrow)、[自定义图案](/core/reference/extending/custom-pattern)、[自定义路径生成器](/core/reference/extending/path-generator)。
- **进阶 / 优化**：本页只是一个使用用例；反复画这类电路时，把它封装成专门的（更高 tier 的）组件，既能减少重复代码，也能精简 IR 持久化的内容。
- **让 AI 更懂图**：在 IR 里给节点 / 路径补上 `description` 等元数据，能让 AI 更准确地理解图的语义内容。
- **相关组件 / 概念**：[Node](/core/components/node/overview) / [Scope](/core/components/layout/scope) / [Draw](/core/components/draw/overview)，以及[定位](/core/concepts/positioning)（E 标签用到的 `of` 关系定位）。
- [Karl 单位圆](/core/examples/karl-circle) - 另一个由多种基础组件拼成完整图的示例。
