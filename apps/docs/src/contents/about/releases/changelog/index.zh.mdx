## v0.2.0-alpha.4

v0.2 第四个 alpha：内部 bug 修复——修复 compile 管线重写后 z-order 退化。同层 node / path 交错声明时，path 不再被统一顶到末尾，而是恢复按 IR 声明顺序层叠（SVG 只认绘制顺序、无 z-index 能力）。是后续显式 `zIndex` / Grid 底纹等特性的前置。

### `@retikz/core`

- **transform-free frame 的 IR 声明序回归**：顶层与无 transform 的 scope 内，path primitive 在 Pass 1 于本层占一个位、Pass 2 原位回填，恢复与同层 node 的声明顺序；无 transform 的 scope 仅含 path 时不再被误剪枝。
- **已知限制**：带 transform 的 scope 内 path 端点已是全局坐标，仍 hoist 到顶层 `primitives`（避免 transform 二次 apply），其跨 frame z-order 不完全等于 IR 顺序——留待后续坐标系方案 / 显式 `zIndex` 覆盖。
- 纯 compile 内部修复：IR / schema / 公开 API 零变化；内部占位类型不泄漏到 Scene 输出（返回前无条件校验）。

### 文档站

- Scope 技术原理订正「scope 内 path 落点」描述：无 transform → 回填 `GroupPrim.children`；带 transform → hoist 顶层（已知限制）。

## v0.2.0-alpha.3

v0.2 第三个 alpha：Shape Registry——把 node 形状从内置 4 种（硬编码 switch）推进到「可注册、可第三方注入」。IR 里 `shape` 仍是字符串名，定义经 `CompileOptions.shapes` 运行时注入；第三方能发 `@retikz/shapes-flow` 之类形状库，不依赖改 core。

### `@retikz/core`

- **`ShapeDefinition` 扩展面**：四方法 `circumscribe` / `boundaryPoint` / `anchor` / `emit`，统一操作外接 `Rect`；plain object（factory 友好），含函数、不进 IR。新建 `shapes/`，内置 4 shape 改造为注册项（无内置特权）。
- **`shape` 字段开放为字符串**：`NodeSchema.shape` 由闭合枚举改 `z.string().min(1)`；schema 只校验非空，未注册名 compile 期 throw（列出可用名）。`BuiltinShapeName`（内置 4 名，Record key）与 `NodeShape`（开放名）分离。
- **`CompileOptions.shapes` 运行时注入**：有效表 = `{ ...BUILTIN_SHAPES, ...shapes }`，同名覆盖内置经 `onWarn` 发 `SHAPE_OVERRIDES_BUILTIN`（用户传 onWarn 时即使 production 也收到）。
- **数字角度 generic**：任何实现 `boundaryPoint` 的 shape 免费获得 `.30` 角度锚点；emit 收轴对齐 rect（rotate 由外层 GroupPrim 统一施加），消除 diamond 的 unrotated 特例。
- **公开 API**：`ShapeDefinition` / `ShapeStyle` / `BUILTIN_SHAPES` / `BuiltinShapeName` / `worldToLocal` / `localToWorld`。
- 内置 4 shape 改造前后 Scene 逐字节相等（纯重构无行为漂移）。

### `@retikz/react`

- **`<TikZ shapes={...}>`**：透传 `CompileOptions.shapes`，自定义 shape 端到端可用；`<Node shape>` 接受任意字符串名。

### 文档站

- **新增「自定义形状」参考页**（reference/extending/shape-registry）：ShapeDefinition 四方法、注入 / 覆盖 / 未知名行为、factory 模式 + 自定义 hexagon live demo；显著提示 emit 收 rotate=0 轴对齐 rect、boundaryPoint/anchor 收带 rotate rect 两套坐标系。
- Node / schema entity / compile options 文档补 shape 开放 string + 注入扩展点说明。

## v0.2.0-alpha.2

v0.2 第二个 alpha：把 `<Scope>` 升级为样式默认值挂点——主色 `color` 级联 + 四类默认样式（`nodeDefault` / `pathDefault` / `labelDefault` / `arrowDefault`）；StepLabel 补齐 `textColor` / `opacity` / `font`，彩色线终于能配同色标注。

### `@retikz/core`

- **主色 `color`（TikZ `color=`）上 Scope / Node / Path**：设当前色后 stroke / fill / 文字 / 箭头 / 标注未单设则随它；分项（`stroke` / `fill` / `textColor`）显式给则覆盖主色。颜色跟随的是主色而非 `stroke`——`<Path color>` 标注 / 箭头同色，`<Path stroke>`（只染线）不跟（与 TikZ `draw=` 一致）。
- **Scope 级联 graphic state**：`color` + 跨类共享分项（`stroke` / `fill` / `strokeWidth` / `opacity` / `fillOpacity` / `drawOpacity`）级联到 scope 内全部元素。
- **四类默认样式**：`nodeDefault` / `pathDefault` / `labelDefault` / `arrowDefault`，按元素类型分发、扁平独立；各从对应元素 schema `.omit()` 派生（单一真源），顶层 `.strict()` 严拒未知 / 被排除字段。
- **`resetStyle` 继承屏障**：`boolean | ('node'|'path'|'label'|'arrow')[]`，切外层 scope 继承轴回内置基线；只切 scope 继承轴、不碰实例-host 轴——label / arrow 仍跟所属 path / node 已解析色，不成孤岛。
- **StepLabel 加 `textColor` / `opacity` / `font`**：边标注文字色 / 透明度 / 字体；继承顺序 label 显式 > scope `labelDefault` > 宿主 path 主色 > currentColor；`opacity` 与所属 path opacity 相乘（元素内轴）。
- **样式解析机制**：compile 维护 style frame 栈，每元素按 inside-out per-field 解析（就近优先）；主色同源展开 + 颜色级联到子元素；opacity 嵌套替换不复合（TikZ 默认）。

### `@retikz/react`

- **`<Scope>` 加 12 个样式 props**：级联 graphic state（`color` / `stroke` / `fill` / `strokeWidth` / `opacity` / `fillOpacity` / `drawOpacity`）+ 四通道（`nodeDefault` / `pathDefault` / `labelDefault` / `arrowDefault`）+ `resetStyle`，全 optional。
- **`<Node>` / `<Path>` 加主色 `color`**，`<Step label>` 加 `textColor` / `opacity` / `font`，均 optional，IR ↔ JSX 双向透传。

### 文档站

- **Scope 样式继承章节**：主色级联 / 四通道 every-X / resetStyle 屏障 + 优先级链 / 主色展开 / opacity 不复合的技术原理，配 3 个 demo。
- **StepLabel 颜色与字体**：彩色线同色标注示例 + `textColor` / `opacity` / `font` 字段说明。
- **Node / Path 主色 + Reference 词典**：组件页 API 表与 schema 词典补 `color` / StepLabel 新字段。

## v0.2.0-alpha.1

v0.2 首个 alpha：引入 IR 层 `<Scope>` 分组容器，承接 TikZ `\begin{scope}` 的「分组 + 局部 transform」语义。样式继承留到 alpha.2。

### `@retikz/core`

- **新增 `<Scope>` IR 容器**：`IRScope` 作为第 4 类 IRChild，支持任意深度嵌套；compile 时下沉为 Scene `GroupPrim`，空 scope（无 transform / children / id）不发 primitive。
- **6 种 transform 变体**：`translate` / `polar-translate` / `at-translate` / `offset-translate` / `rotate` / `scale`，数组按顺序应用（第 0 个最内层）；polar / at / offset 在 compile 期展平为 Cartesian 再下沉。
- **scope.id 整体引用**：设 `id` 后 scope 子树的轴对齐全局 bbox 注册为 synthetic rectangle layout，外部可用 `id` / `id.<anchor>` / `id.<deg>` 引用，对应 TikZ `local bounding box`。
- **跨 scope 解析**：scope 内 node id 默认注册到全局 nodeIndex，外层 path / position 可引用；anchor 坐标经 transform 链投影到全局。
- **scope 内相对定位**：polar / at / offset 的 referent 取全局、relative 部分在当前 scope 局部度量，末端经 scope chain 投影回全局。
- **`localNamespace` 局部命名空间**：opt-in 后 scope 推子 frame 隔离内部 id，支持 inside-out lookup 与 shadowing。
- **重复 id 改为 warn + last-wins**：同 frame 重复 id 不再 throw，改发 `DUPLICATE_NODE_ID` warning + 后定义覆盖前定义。

### `@retikz/react`

- **新增 `<Scope>` Kernel 组件**：接收 `transforms` / `id` / `localNamespace` / children，builder 递归构造 IRScope。

## v0.1.0-rc.2

候选发布窗口第二轮：库代码限于渲染管线 3 处缺陷修复，重头戏在文档站——新增 Examples / Recipes 分组、搜索覆盖 mdx 正文、AI 助手 Copilot 风格重构 + 多会话 IDB 持久化、AI 直接渲染 retikz 图、新增 Blog 分区。无 schema / 公开 API 变更。

### `@retikz/core`

- **编译 z-order 跟随 IR / JSX 顺序**：旧 2-pass compile 先发所有 Node primitive、再发所有 Path primitive，导致 Path 永远在 Node 上层，无视 JSX 书写顺序。重构为「pass 1 仅 layout 注册 nodeIndex、pass 2 按 IR 顺序交错发 primitive」。现在 SVG z-order 严格等于 JSX 顺序，符合 DOM 直觉。

### `@retikz/react`

- **`buildIR` 透明展开 `<Fragment>`**：之前 `.map(...)` 包 `<></>` 形成的 Fragment children 被静默丢弃；`React.Fragment` 现在在 IR 构建时递归解开，子节点照原序进入。覆盖直接 Fragment / `.map` + Fragment / 嵌套 Fragment / 混合 .map 与直接子节点四种形态。
- **CSS `var()` 颜色走 inline `style`**：SVG2 spec 规定 `var()` 仅在 CSS 属性上下文展开，挂到 SVG attribute 上不解析。`renderPrim` 现在检测 `fill` / `stroke` 含 `var(` 时改走 inline `style`，覆盖 rect / ellipse / text + tspan / path 四类 primitive。用户写 `fill="var(--background)"` 等主题感知颜色直接生效。
- **`<TikZ>` arrow spec 收集加 null 防御**：`collectArrowSpecs` visitor 在 fragment / 空 children 场景下加 null 防护，避免 AI 流式生成中间态触发崩溃。

### 文档站

## v0.1.0-beta.2

第二轮 beta 命名收敛：继续把公开 API 从 SVG / 历史大小写细节中抽离出来，为 rc 前的冻结做准备。

### `@retikz/core`

- **BREAKING：Scene 边界从 `ViewBox` 改为 `Layout`**：`ViewBox` / `Scene.viewBox` / `computeViewBox` 分别改为 `Layout` / `Scene.layout` / `computeLayout`。SVG adapter 内部仍使用 SVG `viewBox` 属性；core 公开类型改用渲染目标无关的 layout 命名。
- **BREAKING：path dash 字段改为 `dashPattern`**：`IRPath.strokeDasharray`、`PathPrim.strokeDasharray`、`RectPrim.strokeDasharray`、`EllipsePrim.strokeDasharray` 改为 `dashPattern: Array<number>`。React/SVG renderer 内部仍把该字段映射到 SVG `stroke-dasharray` 属性。
- **端点箭头几何渲染目标无关化**：多 sub-path 端点箭头拆分不再在 core 中使用 SVG marker 术语；箭头 shrink 改用中性箭头几何，不再依赖 React/SVG `viewBox` / `refX` 表述。

### `@retikz/react`

- **BREAKING：顶层组件从 `<Tikz>` 改为 `<TikZ>`**：`Tikz` / `TikzProps` 分别改为 `TikZ` / `TikZProps`，与 LaTeX TikZ 原品牌大小写一致。迁移时替换 import 与 JSX 标签即可。
- **BREAKING：`<Path>` / `<Draw>` dash prop 改为 `dashPattern`**：把 `strokeDasharray="4 2"` 替换为 `dashPattern={[4, 2]}`。`Node.dashArray` 也从字符串改为 `Array<number>`，例如 `dashArray={[6, 2, 1, 2]}`。`lineCap` / `lineJoin` 保持不变，并继续对齐 TikZ `line cap` / `line join` 术语。

## v0.1.0-alpha.5

破坏性窗口最后一站：Scene primitive 去 SVG 字符串、Path 箭头视觉规格对象化、StepLabel 位置扩 7 keyword + 任意 t、新增 OffsetPosition 第 4 种相对定位、IRTarget 字段去缩写。本版发布后 IR schema 字段名 / 语义冻结，进入 beta 非破坏性窗口。配 4 篇 ADR 与 +250+ 个新测试用例（533 → 833）。

### `@retikz/core`

- **Scene PathPrim 结构化**：`PathPrim.d: string` 改为 `commands: Array<PathCommand>`（move / line / quad / cubic / arc / ellipseArc / close 七种 kind）；`GroupPrim.transform?: string` 改为 `transforms?: Array<Transform>`（translate / rotate / scale）。core 内部不再持 SVG mini-language，canvas / Skia / PDF 等 adapter 共用同一结构化数据；arc / ellipseArc 用 canvas 原生的 center-parameterization —— ADR-01
- **StepLabel.position 扩充**：从 3 个 keyword 升到 **7 keyword + 任意 t∈[0, 1]**（`at-start` / `very-near-start` / `near-start` / `midway` / `near-end` / `very-near-end` / `at-end`，数值与 TikZ `pos=<float>` 对齐）。Fold N 段每段等占 `1/N` t 区间（拐角恒在 `t = j/N`）、Bezier 用参数 t 非弧长、arc / ellipsePath 用角度参数化，全部与 TikZ 对齐 —— ADR-02
- **Path 箭头重设计**：删 `arrowShape`、加 `arrowDetail` 对象——一个字段承载 shape / scale / length / width / color / fill / opacity / lineWidth 全部视觉规格；起末两端通过 `start` / `end` 子对象**逐字段 merge** 顶层默认，支持异形 / 异色 / 异 scale。空心 shape 上 `fill` silent no-op（schema 不拒绝，render 丢弃）。所有 shape 的 line tip 都接到 back 接线点（含实心），低 opacity 下不再透出 line —— ADR-03
- **OffsetPosition 第 4 种相对定位**：新增 `{ of, offset: [dx, dy] }` 形态，`of` 三态——节点 id / 笛卡尔 / PolarPosition；同步进 **`IRTarget` union**——`step.to` 也接受 OffsetPosition，与 `Node.position` / `Coordinate.position` 对称。对应 TikZ `calc` 的 `($(A) + (dx, dy)$)` —— ADR-04
- **IRTarget 字段去缩写（BREAKING）**：`{ rel }` / `{ relAccumulate }` → `{ relative }` / `{ relativeAccumulate }`；导出名 `RelTargetSchema` / `IRRelTarget` 等同步全展。sugar 字符串 `'+x,y'` / `'++x,y'` 公开输入面不变
- **常量导出**：`ARROW_MARKER_DEFAULT_SIZE` / `ARROW_MARKER_HOLLOW_DEFAULT_LINE_WIDTH` / `HOLLOW_ARROW_SHAPES` 三处之前 react 端硬编码的箭头默认值进入公开 API

### `@retikz/react`

- `<Path arrowDetail={...}>` prop 接 `ArrowDetail` 对象（替代旧 `arrowShape` 单字段）；marker id 纳入 detail hash，同 path 不同 detail 不会复用错配 marker
- `<Step label={{ position: 0.3 }}>` / `<EdgeLabel position="very-near-start" />` 接受 7 keyword + 任意 0..1 数值
- `<Node position={{ of, offset }}>` / `<Step to={{ of, offset }}>` 接受 OffsetPosition 形态
- 新增 `path-d-builder` / `transform-builder` 两 helper，把结构化 PathCommand[] / Transform[] 翻译为 SVG `d` / `transform` 字符串；arc → A 命令的 large-arc-flag / sweep-flag 计算从 core 挪进来
- 文档站新增 **Arrow 专页**（`/core/components/draw/arrow`）讲箭头 shape / 颜色 / 起末异形 / 缩放 / 半透明；Path / Draw 页对应段精简引用 Arrow 专页
