## v0.1.0-beta.2

第二轮 beta 命名收敛：继续把公开 API 从 SVG / 历史大小写细节中抽离出来，为 rc 前的冻结做准备。

### `@retikz/core`

- **BREAKING：Scene 边界从 `ViewBox` 改为 `Layout`**：`ViewBox` / `Scene.viewBox` / `computeViewBox` 分别改为 `Layout` / `Scene.layout` / `computeLayout`。SVG adapter 内部仍使用 SVG `viewBox` 属性；core 公开类型改用渲染目标无关的 layout 命名。
- **BREAKING：path dash 字段改为 `dashPattern`**：`IRPath.strokeDasharray`、`PathPrim.strokeDasharray`、`RectPrim.strokeDasharray`、`EllipsePrim.strokeDasharray` 改为 `dashPattern`。React/SVG renderer 内部仍把该字段映射到 SVG `stroke-dasharray` 属性。
- **端点箭头几何渲染目标无关化**：多 sub-path 端点箭头拆分不再在 core 中使用 SVG marker 术语；箭头 shrink 改用中性箭头几何，不再依赖 React/SVG `viewBox` / `refX` 表述。

### `@retikz/react`

- **BREAKING：顶层组件从 `<Tikz>` 改为 `<TikZ>`**：`Tikz` / `TikzProps` 分别改为 `TikZ` / `TikZProps`，与 LaTeX TikZ 原品牌大小写一致。迁移时替换 import 与 JSX 标签即可。
- **BREAKING：`<Path>` / `<Draw>` dash prop 改为 `dashPattern`**：把 `strokeDasharray="4 2"` 替换为 `dashPattern="4 2"`。`lineCap` / `lineJoin` 保持不变，并继续对齐 TikZ `line cap` / `line join` 术语。

## v0.1.0-beta.1

beta 优化窗口第一版：在 alpha.5 schema 收尾后，集中做 renderer-neutral 注释清理、公开 union 类型与 JSDoc 补强、几何 transform 去重、unbuilder round-trip 覆盖、compile/path 拆分与性能优化、字段表类型互锁、builder cast 收敛、显式 warning 收集、边界测试补齐、命名清理与 core 测试 helper 去 renderer mirror。公开能力不新增，重点是降低回归风险、改善诊断和维护性。

### `@retikz/core`

- **Renderer-neutral core 契约**：清理 core 注释与 zod `.describe()` 中的 SVG-imposing 表述，让 IR / Scene primitive 描述保持渲染目标无关。
- **公开类型与 JSDoc 补强**：为 `PathCommand` / `Transform` / `TextLine` 等公开 union 拆出命名子类型并补字段 JSDoc，提升 IDE hover 与 wrapper 复用。
- **geometry transform 去重**：抽出 4 shape 共用的 local/world transform helper，清理无消费的 `*Anchor` 类型重复。
- **compile/path 维护性与性能**：拆分路径编译职责、收敛 `findPrev` O(n²) 热点、让 thickness 映射与枚举字段类型互锁。
- **诊断与测试加固**：增加 `CompileOptions.onWarn` warning 收集路径、补边界 / error path 测试，并清理 core 测试 helper 对 SVG renderer 的镜像依赖。
- **IR JSON 可序列化守门**：新增非有限数值测试，暴露并锁定 `Infinity` / `-Infinity` 不应进入 IR schema 的契约缺口。

### `@retikz/react`

- **unbuilder 覆盖补齐**：补 alpha.5 新增形态 round-trip 覆盖，包括 AtPosition / OffsetPosition / `arrowDetail` / 扩展 StepLabel position。
- **字段表类型互锁**：让 builder / unbuilder 镜像字段和 arrow marker `stableSpecKey` 字段表与类型同步，降低新增字段漏传风险。
- **builder 类型收敛**：减少 `_builder.ts` 分散 `as` cast，把不安全边界集中到更小范围。
- **命名清理**：整理 builder/unbuilder 文件命名与 `renderPrim` 上下文参数命名，保持公开表意一致。

## v0.1.0-alpha.5

破坏性窗口最后一站：Scene primitive 去 SVG 字符串、Path 箭头视觉规格对象化、StepLabel 位置扩 7 keyword + 任意 t、新增 OffsetPosition 第 4 种相对定位、IRTarget 字段去缩写。本版发布后 IR schema 字段名 / 语义冻结，进入 beta 非破坏性窗口。配 4 篇 ADR 与 +250+ 个新测试用例（533 → 833）。

### `@retikz/core`

- **Scene PathPrim 结构化**：`PathPrim.d: string` 改为 `commands: Array<PathCommand>`（move / line / quad / cubic / arc / ellipseArc / close 七种 kind）；`GroupPrim.transform?: string` 改为 `transforms?: Array<Transform>`（translate / rotate / scale）。core 内部不再持 SVG mini-language，canvas / Skia / PDF 等 adapter 共用同一结构化数据；arc / ellipseArc 用 canvas 原生的 center-parameterization —— ADR-01
- **StepLabel.position 扩充**：从 3 个 keyword 升到 **7 keyword + 任意 t∈[0, 1]**（`at-start` / `very-near-start` / `near-start` / `midway` / `near-end` / `very-near-end` / `at-end`，数值与 TikZ `pos=<float>` 对齐）。Fold N 段每段等占 `1/N` t 区间（拐角恒在 `t = j/N`）、Bezier 用参数 t 非弧长、arc / ellipsePath 用角度参数化，全部与 TikZ 对齐 —— ADR-02
- **Path 箭头重设计**：删 `arrowShape`、加 `arrowDetail` 对象——一个字段承载 shape / scale / length / width / color / fill / opacity / lineWidth 全部视觉规格；起末两端通过 `start` / `end` 子对象**逐字段 merge** 顶层默认，支持异形 / 异色 / 异 scale。空心 shape 上 `fill` silent no-op（schema 不拒绝，render 丢弃）。所有 shape 的 line tip 都接到 back 接线点（含实心），低 opacity 下不再透出 line —— ADR-03
- **OffsetPosition 第 4 种相对定位**：新增 `{ of, offset: [dx, dy] }` 形态，`of` 三态——节点 id / 笛卡尔 / PolarPosition；同步进 **`IRTarget` union**——`step.to` 也接受 OffsetPosition，与 `Node.position` / `Coordinate.position` 对称。对应 TikZ `calc` 的 `($(A) + (dx, dy)$)` —— ADR-04
- **IRTarget 字段去缩写（BREAKING）**：`{ rel }` / `{ relAccumulate }` → `{ relative }` / `{ relativeAccumulate }`；导出名 `RelTargetSchema` / `IRRelTarget` 等同步全展。sugar 字符串 `'+x,y'` / `'++x,y'` 公开输入面不变
- **常量导出**：`ARROW_MARKER_DEFAULT_SIZE` / `ARROW_MARKER_HOLLOW_DEFAULT_LINE_WIDTH` / `HOLLOW_ARROW_SHAPES` 三处之前 react 端硬编码的箭头默认值进入公开 API

### `@retikz/react`

- `<Path arrowDetail={...}>` prop 接 `ArrowDetail` 对象（替代旧 `arrowShape` 单字段）；marker id 纳入 detail hash，同 path 不同 detail 不会复用错配 marker
- `<Step label={{ position: 0.3 }}>` / `<EdgeLabel position="very-near-start" />` 接受 7 keyword + 任意 0..1 数值
- `<Node position={{ of, offset }}>` / `<Step to={{ of, offset }}>` 接受 OffsetPosition 形态
- 新增 `path-d-builder` / `transform-builder` 两 helper，把结构化 PathCommand[] / Transform[] 翻译为 SVG `d` / `transform` 字符串；arc → A 命令的 large-arc-flag / sweep-flag 计算从 core 挪进来
- 文档站新增 **Arrow 专页**（`/core/components/draw/arrow`）讲箭头 shape / 颜色 / 起末异形 / 缩放 / 半透明；Path / Draw 页对应段精简引用 Arrow 专页

## v0.1.0-alpha.4

节点关系层一次到位：节点间相对定位、Coordinate 占位节点、Node 边挂标签。配 3 篇 ADR 与 +33 个新测试用例。

### `@retikz/core`

- **节点间相对定位**：`Node.position` 接受 `{ direction, of, distance? }` 形态，对应 TikZ `[<direction>=<distance> of <id>]`；direction 走 8 方向枚举（4 主向 + 4 对角，对角分量 1/√2 保斜向距离一致）；`of` 字符串引用要求被引节点先在 IR 中定义（同 polar `origin` / Step `to`） —— ADR-01
- **TikZ 容器 `nodeDistance` prop**：注入 `CompileOptions.nodeDistance`，作为相对定位 distance 缺省值；自带 `distance` 优先，未配回退到 1（对应 TikZ `node distance=1cm`）
- **`<Coordinate>` 占位节点**：新增 `IRChild` discriminator `'coordinate'`（仅 id + position 两字段），与 Node / Path 在顶层平级；编译期 Pass 1 单独分支——不发任何 primitive、不参与 viewBox 扩展，但注册到 nodeIndex（0×0 矩形 layout，boundaryPoint 恒返中心），让 path target / `at.of` 字符串引用都能命中 —— ADR-02
- **Node 边挂标签**：`Node.label?` 接 `NodeLabel` 单对象或数组；每条 label 接 `text` / `position?`（8 方向枚举或数字角度，对应 TikZ `label=above:foo` / `label=30:foo`）/ `distance?`（缺省 4）/ `textColor?` / `opacity?` / `font?`；`font` / `textColor` 缺省自动继承 Node 块级值；emit 阶段每条 label 一个 TextPrim 放在 inner group 内，节点 rotate 时 label 跟着转 —— ADR-03

### `@retikz/react`

- `<TikZ>` 加 `nodeDistance` prop；`<Node position={{ direction, of, distance? }}>` 接受相对定位形态
- 新增 kernel 第三件 `<Coordinate id="..." position={...} />`（与 `<Node>` / `<Path>` 平级，IR 一对一）；builder / unbuilder 全链路 round-trip
- `<Node label={...}>` prop 接 NodeLabel 单对象或数组，与 IR 字段一致

## v0.1.0-alpha.3

Path 增强一波到位：曲线三件套、路径级形状、相对坐标、边标注、视觉属性补齐。配 4 篇 ADR 与 +120 个新测试用例。

### `@retikz/core`

- **Path 曲线三件套**：`Step.kind` 加 `'curve'`（quadratic）/ `'cubic'`（cubic）/ `'bend'`（弧形简记，方向 + 角度自动算控制点退到 cubic）—— ADR-01
- **Path-level 形状**：`Step.kind` 加 `'arc'` / `'circlePath'` / `'ellipsePath'`，以游标为圆心；arc 留笔在弧终点，circle / ellipse 留笔回圆心 —— ADR-02
- **相对坐标**：`IRTarget` 加 `{ rel: [dx, dy] }` / `{ relAccumulate: [dx, dy] }` 两个变体——`rel` 不推进 prevEnd（TikZ `(+x, +y)`），`relAccumulate` 累积（TikZ `(++x, ++y)`）；compile 入口 `normalizeRelativeTargets` 一次性解析 —— ADR-03
- **边标注**：除 `'move'` / `'cycle'` 外八种 step kind 加 `label?: { text, position?, side? }`；`position` ∈ `'midway' | 'near-start' | 'near-end'`，`side` ∈ `'above' | 'below' | 'left' | 'right' | 'sloped'`；compile 出 TextPrim（sloped 裹 group rotate）—— ADR-04
- **新增 geometry/segment.ts**：line / quad / cubic / fold / arc / circle / ellipse 七种段几何采样工具，给 label 算几何点 / 切线
- **Path 描边端点 / 拐点**：`lineCap` / `lineJoin`（TikZ `line cap` / `line join`）
- **语义 stroke 档位**：`thickness` ∈ `'ultraThin'` … `'ultraThick'`（7 档），`thin = 1` 与默认 `strokeWidth` 对齐；显式 `strokeWidth` 始终优先
- **Path 透明度**：`opacity` 整体 + `fillOpacity` 仅 fill + `drawOpacity` 仅 stroke（与 Node 命名一致）
- **Sugar way DSL 扩展**：`{ curve | cubic | bend }` infix 算子（与下一项合并）；`{ arc | circle | ellipse }` infix 算子（不消耗下一项）；`WayRelItem` 对象形态 `{ position, type: DrawWay.Relative | DrawWay.Accumulate }` + sugar 字符串 `'+1,0'` / `'++1,0'`；`{ label }` infix 算子修饰下一段

### `@retikz/react`

- 新增 sugar `<EdgeLabel>`：作为 `<Step>` child 等价于 `<Step label={...}>`，prop 优先；不可逆（unbuilder 只产 prop 形态）
- Step / Path / Draw 透传所有新字段；builder / unbuilder 全链路 round-trip
- Draw way DSL 一并支持 label / 曲线 / 形状 / 相对坐标 sugar，与 Kernel 字面等价

## v0.1.0-alpha.2

Node 美化层 P1 全集落地：多行文本、字体对象、分轴边距、颜色 / 不透明度 / 描边样式 / 尺寸约束 / 缩放全套。配 3 篇 ADR 与 +47 个新测试用例。

> **BREAKING**：`<Node fontSize={n}>` 改为 `<Node font={{ size: n }}>`，IR `NodeSchema.fontSize` 字段直接删除；alpha 期不留 deprecate 共存窗口（见 ADR-01）。

### `@retikz/core`

- **Node font 嵌套对象**：`font: { family?, size?, weight?, style? }` 取代标量 `fontSize` — ADR-01
- **Node 多行文本**：`text: string | Array<string | LineSpec>`，行级可覆盖 `fill` / `opacity` / `font`；配 `align`（left / center / right）+ `lineHeight`（user units，缺省 `font.size × 1.2`） — ADR-02
- **Node 分轴边距**：`innerXSep` / `innerYSep` / `outerSep` 三字段对齐 TikZ `inner xsep` / `inner ysep` / `outer sep`；`padding` / `margin` 保留为对称别名（轴特化字段优先） — ADR-03
- **Node 颜色与不透明度**：`textColor` / `opacity`（整节点）/ `fillOpacity` / `drawOpacity`
- **Node 描边样式**：`dashed`（预设 `4 2`）/ `dotted`（预设 `1 2`）/ `dashArray`（显式 dash pattern）；优先级 `dashArray` > `dashed` > `dotted`
- **Node 尺寸约束**：`roundedCorners`（rectangle 圆角）/ `minimumWidth` / `minimumHeight` + 对称别名 `minimumSize`
- **Node 缩放**：`scale` / `xScale` / `yScale`；等比时字号同步缩放，非等比时字号取 `min(sx, sy)` 保形
- **TextPrim 升级**：`lines: Array<TextLine>` 取代 `content: string`；行级字段 `fontSize` / `fontFamily` / `fontWeight` / `fontStyle` / `fill` / `opacity`

### `@retikz/react`

- 新增 sugar `<Text>` 组件——`<Node>` 内行级样式覆盖：`<Node><Text fill="red" font={{ weight: 'bold' }}>L1</Text>L2</Node>`
- `<Node>` children 直接支持多行：字符串内嵌 `\n`、数组、混 `<Text>` 三种写法均可
- builder / unbuilder 透传所有新字段；派生类型 `IRFont` / `IRLineSpec` / `NodeTextAlign` 从 core re-export
- renderer `<text>` 改为多 `<tspan>` 渲染，按 `baseline` 自动反推首行 `dy` 实现块级垂直对齐

## v0.1.0-alpha.1

P0 + P1 流程图 / UML / 状态机刚需能力一次到位：节点 4 形状、path 折角 / 闭合 / 多端箭头 / 区域填色、显式锚点引用。配 4 篇 ADR 与 ~180 个测试用例。

### `@retikz/core`

- **Step kind 扩到 4 种**：`'move'` / `'line'` / `'step'`（折角，TikZ `(A) -| (B)` 风格）/ `'cycle'`（闭合到起点）—— ADR-01
- **Path 路径级视觉**：`arrow`（4 方向）+ `arrowShape`（7 种：normal / open / stealth / diamond / openDiamond / circle / openCircle，含 hollow shape 自动 shrink）+ `fill` / `fillRule` —— ADR-02
- **Node shape 多态**：`rectangle` / `circle` / `ellipse` / `diamond` 4 种几何 + boundaryPoint 多态；外接形状严丝合缝包住"text + padding" —— ADR-03
- **Target 字符串扩展**：`'A'`（auto-clip）/ `'A.<anchor>'`（命名 anchor）/ `'A.<deg>'`（角度边界）—— ADR-04
- **Sugar way DSL 升级**：`DrawWay` 关键字常量（cycle / hv / vh）+ infix 折角算子 `'-|'` / `'|-'` + cycle 关键字
- **路径 clip 算法重构**：每段独立用节点中心算两端 boundary——避免线穿过中间节点本体（与 TikZ 语义一致）
- **新工具类型**：`ValueOf<T extends object> = T[keyof T]`，统一 `as const` 对象 + 派生枚举模式
- **新 geometry 模块**：`circle` / `ellipse` / `diamond` 各自的 `boundaryPoint` / `anchor` / `contains`

### `@retikz/react`

- Path / Draw / Step / Node 透传所有新 IR 字段；`DrawWay` 与 IR 字面量类型完全互通
- TikZ 容器按需注入 SVG `<defs><marker>`（每种用到的 arrow shape 一份，`useId()` 防多实例冲突）
- Renderer 新增 `'ellipse'` primitive case（circle / ellipse 节点共用）
- 渲染管线现支持 GroupPrim 把多 sub-path 包起来——多节点 path + 箭头时只首段 / 末段挂 marker

## v0.1.0-alpha.0

首版 npm 发布。retikz v0.1 重构基线，IR 居中——为后续 alpha 迭代打底。

### `@retikz/core`

- **IR + Scene 编译器**：framework-agnostic 的中间表示与编译器，零 React、零 DOM
- **持久化形式**：内部 React state → 可序列化 IR JSON，跨平台、可被 LLM 直接读写

### `@retikz/react`

- **架构拆分**：从单包 React 组件库改为薄 React adapter，core 与 React 解耦
- **Kernel + Sugar 双层 JSX**：Kernel 直对应 IR 节点，Sugar 编译期展开为 Kernel
- peerDep React 18+
