## v0.1.0-beta.2

Second beta naming pass: continue moving public APIs away from SVG-specific wording and historical casing mismatches before the rc API freeze.

### `@retikz/core`

- **BREAKING: Scene bounds are now `Layout` instead of `ViewBox`**: `ViewBox` / `Scene.viewBox` / `computeViewBox` become `Layout` / `Scene.layout` / `computeLayout`. The SVG adapter still writes the SVG `viewBox` attribute; core's public type layer now uses render-target-neutral layout naming.
- **BREAKING: path dash fields are now `dashPattern`**: `IRPath.strokeDasharray`, `PathPrim.strokeDasharray`, `RectPrim.strokeDasharray`, and `EllipsePrim.strokeDasharray` become `dashPattern`. The React/SVG renderer still maps the field to the SVG `stroke-dasharray` attribute internally.
- **Renderer-neutral endpoint arrow geometry**: multi-sub-path endpoint arrow splitting no longer uses SVG marker terminology in core, and arrow shrink now uses neutral arrow geometry instead of depending on React/SVG `viewBox` / `refX` wording.

### `@retikz/react`

- **BREAKING: the top-level component is now `<TikZ>` instead of `<Tikz>`**: `Tikz` / `TikzProps` become `TikZ` / `TikZProps`, matching the original LaTeX TikZ casing. Migration is a direct import and JSX tag rename.
- **BREAKING: `<Path>` / `<Draw>` dash props are now `dashPattern`**: replace `strokeDasharray="4 2"` with `dashPattern="4 2"`. `lineCap` / `lineJoin` stay unchanged and continue to align with TikZ `line cap` / `line join` terminology.

## v0.1.0-beta.1

First beta optimization release after the alpha.5 schema wrap-up. It focuses on renderer-neutral core wording, named public union types and JSDoc, shared geometry transforms, unbuilder round-trip coverage, compile/path modularization and performance, field-list type locks, builder cast narrowing, explicit compile warnings, boundary tests, naming cleanup, and removing renderer-mirror helpers from core tests. It adds no new drawing capability; the goal is lower regression risk, better diagnostics, and maintainability.

### `@retikz/core`

- **Renderer-neutral core contract**: clean SVG-imposing wording from core comments and zod `.describe()` text so IR / Scene primitive descriptions stay render-target agnostic.
- **Public types and JSDoc**: split public unions such as `PathCommand`, `Transform`, and `TextLine` into named variants with field-level JSDoc for better IDE hovers and wrapper reuse.
- **Shared geometry transforms**: extract the common local/world transform helper used by the four shapes and remove duplicated dead `*Anchor` types.
- **compile/path maintainability and performance**: split path compilation responsibilities, remove the `findPrev` O(nÂ²) hotspot, and type-lock the thickness mapping against its enum.
- **Diagnostics and test hardening**: add the `CompileOptions.onWarn` warning-collector path, cover boundary / error paths, and reduce core test helpers that mirror the SVG renderer.
- **IR JSON serializability guard**: add finite-number tests exposing and locking the contract gap that `Infinity` / `-Infinity` must not enter IR schemas.

### `@retikz/react`

- **unbuilder coverage**: add round-trip coverage for alpha.5 forms including AtPosition / OffsetPosition / `arrowDetail` / expanded StepLabel positions.
- **Field-list type locks**: tie builder / unbuilder mirrored field lists and arrow marker `stableSpecKey` fields to their TypeScript types.
- **Builder cast narrowing**: reduce scattered `as` casts in `_builder.ts` by concentrating unsafe boundaries.
- **Naming cleanup**: align builder/unbuilder file names and the `renderPrim` context parameter with public naming conventions.

## v0.1.0-alpha.4

Node-relations layer in one wave: relative positioning between nodes, coordinate placeholders, and node-attached labels. Backed by 3 ADRs and +33 new test cases.

### `@retikz/core`

- **Relative positioning between nodes**: `Node.position` accepts a `{ direction, of, distance? }` form mirroring TikZ `[<direction>=<distance> of <id>]`; `direction` ranges over the 8-direction enum (4 cardinals + 4 diagonals, with `1/âˆš2` components keeping diagonal distance consistent with the cardinal radius); `of` requires the referenced node to be defined earlier in the IR (same forward-reference rule as polar `origin` / Step `to`) â€” ADR-01
- **`nodeDistance` on the TikZ container**: feeds `CompileOptions.nodeDistance`, used as the default distance for relative positioning; an explicit `distance` on the position wins, otherwise falls back to 1 (matches TikZ `node distance=1cm`)
- **`<Coordinate>` placeholder**: new `IRChild` discriminator `'coordinate'` (only `id` + `position`), top-level peer of Node / Path; in compile Pass 1 it emits no primitive and does not extend the viewBox, but it does register in `nodeIndex` as a 0Ã—0 rect layout (boundaryPoint always returns the center), letting path targets and `at.of` string references resolve to it â€” ADR-02
- **Node-attached labels**: `Node.label?` takes a single `NodeLabel` object or an array; each label takes `text` / `position?` (8-direction enum or numeric angle â€” `label=above:foo` / `label=30:foo`) / `distance?` (default 4) / `textColor?` / `opacity?` / `font?`; missing `font` / `textColor` inherit from the parent node; emit produces one TextPrim per label, placed inside the inner group so labels rotate with the node â€” ADR-03

### `@retikz/react`

- `<TikZ>` gains a `nodeDistance` prop; `<Node position={{ direction, of, distance? }}>` accepts the relative-positioning form
- New third kernel component `<Coordinate id="..." position={...} />` (peer of `<Node>` / `<Path>`, one-to-one with IR); builder / unbuilder round-trip end-to-end
- `<Node label={...}>` prop takes a NodeLabel object or array, matching the IR field

## v0.1.0-alpha.3

Path enhancements land in one wave: curve trio, path-level shapes, relative coordinates, edge labels, visual-prop fill-out. Backed by 4 ADRs and +120 new test cases.

### `@retikz/core`

- **Path curve trio**: `Step.kind` adds `'curve'` (quadratic) / `'cubic'` (cubic) / `'bend'` (arc shorthand â€” direction + angle, lowered to cubic at compile time) â€” ADR-01
- **Path-level shapes**: `Step.kind` adds `'arc'` / `'circlePath'` / `'ellipsePath'`, centered on the cursor; `arc` leaves the pen at the arc end, `circle` / `ellipse` return it to the center â€” ADR-02
- **Relative coordinates**: `IRTarget` gains two variants â€” `{ rel: [dx, dy] }` (does not advance prevEnd, TikZ `(+x, +y)`) and `{ relAccumulate: [dx, dy] }` (accumulates, TikZ `(++x, ++y)`); compile entry `normalizeRelativeTargets` resolves them up-front â€” ADR-03
- **Edge labels**: every step kind except `'move'` / `'cycle'` accepts `label?: { text, position?, side? }`; `position` âˆˆ `'midway' | 'near-start' | 'near-end'`, `side` âˆˆ `'above' | 'below' | 'left' | 'right' | 'sloped'`; compiles to a TextPrim (sloped wrapped in a group rotate) â€” ADR-04
- **New `geometry/segment.ts`**: seven sampling helpers (line / quad / cubic / fold / arc / circle / ellipse) backing label position / tangent math
- **Path stroke caps / joins**: `lineCap` / `lineJoin` (TikZ `line cap` / `line join`)
- **Semantic stroke preset**: `thickness` âˆˆ `'ultraThin'` â€¦ `'ultraThick'` (7 levels), `thin = 1` matches the default `strokeWidth`; explicit `strokeWidth` always wins
- **Path opacity**: `opacity` overall + `fillOpacity` for fill only + `drawOpacity` for stroke only (matches Node naming)
- **Sugar way DSL extensions**: `{ curve | cubic | bend }` infix operators (consume the next item); `{ arc | circle | ellipse }` infix operators (do not consume the next item); `WayRelItem` object form `{ position, type: DrawWay.Relative | DrawWay.Accumulate }` + sugar strings `'+1,0'` / `'++1,0'`; `{ label }` infix operator decorates the next segment

### `@retikz/react`

- New sugar `<EdgeLabel>`: as a `<Step>` child it equals `<Step label={...}>`, with the prop winning on conflict; one-way (unbuilder only emits the prop form)

## v0.1.0-alpha.2

Node visual P1 lands in full: multi-line text, font object, axis-specific seps, color / opacity / stroke style / size constraints / scaling. Backed by 3 ADRs and +47 new test cases.

> **BREAKING**: `<Node fontSize={n}>` is now `<Node font={{ size: n }}>`; the IR `NodeSchema.fontSize` field is removed. No deprecation window during alpha (see ADR-01).

### `@retikz/core`

- **Node font as nested object**: `font: { family?, size?, weight?, style? }` replaces the `fontSize` scalar â€” ADR-01
- **Node multi-line text**: `text: string | Array<string | LineSpec>` with per-line `fill` / `opacity` / `font` overrides; complementary `align` (left / center / right) + `lineHeight` (user units, defaults to `font.size Ã— 1.2`) â€” ADR-02
- **Node axis-specific seps**: `innerXSep` / `innerYSep` / `outerSep`, mirroring TikZ `inner xsep` / `inner ysep` / `outer sep`; `padding` / `margin` are kept as symmetric aliases (axis-specific fields take precedence) â€” ADR-03
- **Node colors and opacity**: `textColor` / `opacity` (whole-node) / `fillOpacity` / `drawOpacity`

- **TextPrim upgrade**: `lines: Array<TextLine>` replaces `content: string`; per-line `fontSize` / `fontFamily` / `fontWeight` / `fontStyle` / `fill` / `opacity` overrides

### `@retikz/react`

- New sugar `<Text>` component for per-line overrides inside `<Node>`: `<Node><Text fill="red" font={{ weight: 'bold' }}>L1</Text>L2</Node>`
