## Short term: v0.x → v0.1 stable

Freeze Kernel and Sugar APIs, stabilize the IR schema.

| Goal | Content |
| --- | --- |
| Kernel convergence | 5–8 components: `<TikZ>` `<Node>` `<Path>` `<Step>` `<Anchor>` `<Scope>` |
| Sugar convergence | `<Draw way={[...]} />` and friends compile down to Kernel |
| Public IR schema | One zod source → TypeScript types + JSON Schema |
| AI integration spike | OpenAI Structured Outputs / Anthropic Tool Use emitting IR directly |

## Mid term: domain packages

| Version | Package | Content |

## Long term

| Topic | Where |
| --- | --- |
| `@retikz/codec` | v3.x, IR ↔ TikZ source bidirectional conversion |
| Cross-framework adapters | Vue / Svelte / Native (Skia or RN) |
| Text DSL | Adopt TikZ native syntax, reuse LLM training data |

## Not on the roadmap

To keep scope honest, things we are **not** doing:

- Built-in LaTeX math rendering — delegate to KaTeX / MathJax, retikz handles graphics only
- Keyframe animation DSL — basic transitions yes, full animation system no
- Dedicated SSR adapter — the renderer is already pure, SSR works generically
