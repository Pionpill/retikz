---
title: LineMark
description: Draw line charts with <LineMark>: connect points in order, support multiple series; add closed under polar for a radar.
---

`<LineMark>` connects data points into a line in order. `x` / `y` bind the position fields, `order` decides the connection order (defaults to data array order). Set `series` to split into multiple lines, set `closed` to join ends — under polar that's a radar.

## Usage

```tsx
import { Plot, LineMark } from '@retikz/plot-react';
```

```tsx
<Plot data={revenue}>
  <LineMark x="month" y="revenue" order="month" />
</Plot>
```

Set `order` explicitly — otherwise the line follows data row order and jumps around when that order changes.

## API

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `x` | `string` | — | Field path for the x channel (angular under polar) |
| `y` | `string` | — | Field path for the y channel (radial under polar) |
| `order` | `string?` | data order | Field driving the connection order |
| `series` | `string?` | — | Series field: split into multiple lines; omit for a single line |
| `closed` | `boolean?` | `false` | Connect the last point back to the first into a polygon (a radar outline under polar) |
| `color` | `string?` | `series` | Color field (categorical, auto ordinal color scale); with no explicit `series` it implicitly splits into one line per category; falls back to `series`. Continuous / temporal fields error (sequential color scales come later) |
| `id` | `string?` | — | Optional handle (reserved, not yet resolved) |
