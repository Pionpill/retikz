title: AreaMark
description: Draw area charts with <AreaMark>: a top line and a baseline enclose a filled region; add closed under polar for a filled radar.
---

`<AreaMark>` adds a return edge to a line: the top edge is the line connecting `x` / `y`, the bottom edge sticks to `baseline` (default `0`), and the two enclose a fillable region. Switch to a polar coordinate system and add `closed`, and the region closes into a polygon ‚Äî a filled radar.

## Usage

```tsx
import { Plot, AreaMark } from '@retikz/plot-react';
```

```tsx
<Plot data={activity}>
  <AreaMark x="day" y="users" order="day" />
</Plot>
```

`x` / `y` bind the position fields, `order` decides the top edge's connection order. The bottom defaults to `0`; set `baseline` for a different floor.

## API

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `x` | `string` | ‚Äî | Field path for the x channel (angular under polar) |
| `y` | `string` | ‚Äî | Field path for the y channel (radial under polar) |
| `angle` | `string?` | `x` | Explicit angular field under polar; defaults to `x` |
| `radius` | `string?` | `y` | Explicit radial field under polar; defaults to `y` |
| `order` | `string?` | data order | Field driving the top edge's connection order |
| `baseline` | `number?` | `0` | Baseline value the return edge sticks to |
| `closed` | `boolean?` | `false` | Connect the last point back to the first into a polygon (a filled radar under polar) |
| `series` | `string?` | ‚Äî | Series field: split into multiple areas; omit for a single area |
| `color` | `string?` | `series` | Color field (auto ordinal color scale); falls back to `series` |
| `id` | `string?` | ‚Äî | Optional handle (reserved, not yet resolved) |
