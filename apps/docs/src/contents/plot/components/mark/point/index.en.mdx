`<PointMark>` draws one dot per data row. Bind `x` / `y` to position fields. Give `size` a numeric field and the dot radius maps to the value as a bubble chart; give `color` a categorical field and dots are tinted by category.

Which position channels you bind depends on the [coordinate system](/plot/grammar/coordinate): 2D cartesian / polar use `x` / `y`, 1D (cartesian1D / polar1D) bind only `x`, and ternary (ternary2D) uses `a` / `b` / `c`. That's why `x` / `y` are optional — which ones are required, and whether a/b/c apply, is validated by the coordinate at compile time.

## Usage

```tsx
import { Plot, PointMark } from '@retikz/plot-react';
```

```tsx
<Plot data={cities}>
  <PointMark x="lng" y="lat" />
</Plot>
```

Scatter often overlays a line — draw `<LineMark>` first, then layer `<PointMark>` on the same data to mark each observation.

### Continuous color

Give `color` a continuous numeric field and dots take their color by value — a sequential color scale (viridis by default) maps low to high along one color band. Coloring is still per datum, as with categorical color.

A continuous field needs a `model` to declare its type; without one, `color` is treated as categorical and a continuous field errors — either declare the field as `continuous` in `model`, or use a categorical field instead.

### Opacity

Give `opacity` a continuous field and the opacity maps linearly to `[min, 1]` (the lower bound keeps small values visible). Useful for fading low-value points in dense overlap.

## API

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `x` | `string?` | — | Field bound to the x position channel (cartesian horizontal / polar angular / 1D single axis); required by cartesian·polar, unused by ternary |
| `y` | `string?` | — | Field bound to the y position channel (cartesian vertical / polar radial); required by cartesian·polar, omitted by 1D / ternary |
| `a` | `string?` | — | ternary2D a component field (top vertex); used with b / c for ternary scatter, auto-normalized |
| `b` | `string?` | — | ternary2D b component field (bottom-right vertex) |
| `c` | `string?` | — | ternary2D c component field (bottom-left vertex) |
| `size` | `string?` | — | Numeric field: mapped to dot radius via a square-root scale (area-perceptual); negative values error |
| `opacity` | `string?` | — | Continuous field: mapped to opacity via a clamped linear scale to `[min, 1]`; temporal / categorical fields error |
| `shape` | `string?` | — | Categorical field: mapped to a glyph palette (circle / rectangle / diamond, cycled); continuous / temporal fields error |
| `color` | `string?` | — | Color field, colored per datum: categorical → auto ordinal color scale; continuous / temporal needs a `model` declaration → auto sequential color scale (viridis) |
| `id` | `string?` | — | Optional handle (reserved, not yet resolved) |
