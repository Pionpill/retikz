---

`<Plot>` 是所有图表的根容器。它接收 `data`（裸数据行数组）和显示尺寸，把内部的 `<LineMark>` / `<BarMark>` / `<SectorMark>` / `<Axis>` 等子组件装配成一张图。`coordinate` 选坐标系：缺省平面直角，可切极坐标（`"polar2D"`）、一维（`"cartesian1D"` / `"polar1D"`）或三元（`"ternary2D"`）。数据不进 IR，只在编译期注入。`<Plot>` 是**薄容器**：自己不画坐标轴 / 网格，轴由你显式的 `<Axis>` 声明（scale 与坐标系仍自动推断）。

## 用法

```tsx
import { Axis, BarMark, Plot } from '@retikz/plot-react';
```

```tsx
<Plot data={revenue} width={360} height={220}>
  <BarMark x="quarter" y="value" />
  <Axis dimension="x" />
  <Axis dimension="y" grid />
</Plot>
```

`width × height` 是含坐标轴与刻度标签的外框。子组件只声明字段映射，不写像素坐标。

### 何时使用

`<Plot>` 是**底层容器**：把声明式子组件编译成一张图，但不替你做开箱即用的装饰。

- **会做**：收集 mark（`<LineMark>` / `<PointMark>` / `<BarMark>` / `<SectorMark>` / `<AreaMark>`）；从字段推断 scale（band / linear / time / log…）与坐标系；收集你显式写的 `<Axis>` / `<Legend>`；编译并渲染。
- **不会做**：自动出坐标轴 / 网格 / 图例——要轴就显式写 `<Axis>`；`bare` 则连显式装饰和边距都不要。

### 坐标系切换

`coordinate` 支持字符串简写和对象两种形态，覆盖坐标系族：

```tsx
<Plot coordinate="polar2D">                                    {/* 极坐标（饼 / 雷达 / 径向柱） */}
<Plot coordinate={{ type: 'polar2D', innerRadius: 0.5 }}>      {/* 挖中心成环图 */}
<Plot coordinate="cartesian1D">                                {/* 一维直线（rug / timeline） */}
<Plot coordinate="polar1D">                                    {/* 一维圆周（周期数据） */}
<Plot coordinate="ternary2D">                                  {/* 三元（重心坐标） */}
```

各坐标系的几何配置（polar 的 `innerRadius` / `startAngle` / `endAngle`、cartesian1D 的 `orientation`、polar1D 的 `radius` 等）与适用的 mark / 通道见 [坐标系](/plot/grammar/coordinate)。

## 入口形态

`<Plot>` 有两条入口，按是否传 `spec` 分流：

| 入口 | 必填 | 适用 |
| --- | --- | --- |
| 组合 DSL | `data`（裸数组）+ `children` | 手写 JSX 声明 mark / guide（最常用） |
| spec 入口 | `spec`（已构造 PlotSpec）+ `data`（数据集表） | 持久化 / AI 生成的完整 spec 直喂 |

## API

| 属性 | 类型 | 默认值 | 描述 |
| --- | --- | --- | --- |
| `data` | `Array<Row>` | — | 裸数据行数组（组合 DSL 入口）；数据不进 IR |
| `children` | `ReactNode` | — | mark / guide 子组件（组合 DSL 入口） |
| `model` | `DataModel` | — | 数据模型（字段名 + 类型）：声明则启用 strict 校验与 type-driven scale / guide 派生；详见[数据模型](/plot/data/model) |
| `fieldMap` | `Record<string, string>` | — | 逻辑字段名 → 数据里的实际路径（需配合 `model`）；只改名、不变换值；详见[数据处理](/plot/data/processing) |
| `coordinate` | 见[坐标系](/plot/grammar/coordinate) | cartesian2D | 坐标系：`"polar2D"` / `"cartesian1D"` / `"polar1D"` / `"ternary2D"` 简写或对应对象配置；缺省平面直角 |
| `scaleX` | `'linear' \| 'time' \| 'point' \| 'log' \| 'sqrt'` | `'linear'` | 连续 x scale 类型；含 `<BarMark>` 时强制 band，polar 下忽略。`log` / `sqrt` 仅 point / line（柱 / 面积 baseline 含 0，与之冲突会报错） |
| `scaleY` | `'linear' \| 'log' \| 'sqrt'` | `'linear'` | 连续 y（值轴）scale 类型；polar 下忽略。`log` / `sqrt` 仅 point / line（柱 / 面积会报错）。`log` 要求数据为正、`sqrt` 要求非负 |
| `bare` | `boolean?` | `false` | 只绘图、不出任何轴 / 网格；忽略 `<Axis>` |
| `width` | `number` | — | 外框宽度（含轴与标签） |
| `height` | `number` | — | 外框高度（含轴与标签） |
| `fontSize` | `number?` | — | 刻度标签字号 |
| `margin` | `{ top?; right?; bottom?; left? }` | — | 覆盖四周留白 |
| `spec` | `PlotSpec` | — | spec 入口：已构造好的 Plot IR 根（与 `children` 二选一） |

此外 `renderer` / `className` / `style` 透传给底层 `<Layout>`，分别选渲染后端、附加类名、内联样式。
