`<Plot>` is the root container for every chart. It takes `data` (an array of raw rows) and a display size, and assembles its `<LineMark>` / `<BarMark>` / `<SectorMark>` / `<Axis>` children into a chart. `coordinate` picks the coordinate system: cartesian2D by default, plus polar (`"polar2D"`), 1D (`"cartesian1D"` / `"polar1D"`), and ternary (`"ternary2D"`). Data never enters the IR; it's injected at compile time. `<Plot>` is a **thin container**: it draws no axes or grid on its own ‚Äî axes come from the `<Axis>` you declare (scales and the coordinate system are still inferred automatically).

## Usage

```tsx
import { Axis, BarMark, Plot } from '@retikz/plot-react';
```

```tsx
<Plot data={revenue} width={360} height={220}>
  <BarMark x="quarter" y="value" />
  <Axis dimension="x" />
  <Axis dimension="y" grid />
</Plot>
```

`width √ó height` is the outer box including axes and tick labels. Children only declare field mappings, never pixel coordinates.

`<Plot>` is a **low-level container**: it collects marks, infers scales and the coordinate system from fields, collects the `<Axis>` / `<Legend>` you write, then compiles and renders ‚Äî but it doesn't draw axes / grid / legend for you. Write `<Axis>` for axes; `bare` drops even explicit decoration and margins.

## Examples

### Axes

`coordinate` is a chart-level switch ‚Äî for the same `children`, the default is cartesian and adding `"polar2D"` makes it polar. Marks aren't split into two variants; the coordinate system decides the geometry.

`coordinate` accepts a string shorthand or an object, covering the coordinate family:

```tsx
<Plot coordinate="polar2D">                                    {/* polar (pie / radar / radial bars) */}
<Plot coordinate={{ type: 'polar2D', innerRadius: 0.5 }}>      {/* carve the center into a donut */}
<Plot coordinate="cartesian1D">                                {/* 1D line (rug / timeline) */}
<Plot coordinate="polar1D">                                    {/* 1D circle (cyclic data) */}
<Plot coordinate="ternary2D">                                  {/* ternary (barycentric) */}
```

See [Coordinates](/plot/grammar/coordinate) for each system's geometry options (polar's `innerRadius` / `startAngle` / `endAngle`, cartesian1D's `orientation`, polar1D's `radius`, ‚Ä¶) and which marks / channels apply.

## Entry forms

`<Plot>` has two entries, dispatched by whether `spec` is passed:

| Entry | Required | Use |
| --- | --- | --- |
| Composition DSL | `data` (raw array) + `children` | Declare marks / guides in JSX (the common case) |
| Spec entry | `spec` (built PlotSpec) + `data` (dataset table) | Feed a persisted / AI-generated spec directly |

## How it works

> Optional for everyday use; revisit when you want to understand how `<Plot>` is compiled, or when you write / generate the low-level IR directly.

`<Plot>` draws nothing on its own. It first assembles its children (marks / guides / coordinate) into a **PlotSpec** ‚Äî a high-level Tier 2 IR (fully JSON-serializable; data is not part of it). At compile time, **lowering** turns that PlotSpec into `@retikz/core`'s Tier 1 IR (`<Scope>` / `<Node>` / `<Path>`), which core then compiles into a Scene for the renderer. The spec entry skips assembly and feeds a PlotSpec directly.

Lowering isn't private to `<Plot>`: it's registered as a **composite** in core's compile pipeline. Core expands any PlotSpec node through it, the same way it handles every high-level node ‚Äî upper layers add no parallel rendering path.

### Each lowering step

Data enters lowering alongside the PlotSpec and becomes Tier 1 step by step:

| Step | Input | Output |
| --- | --- | --- |
| Resolve data | dataset via `data.reference` + `model` / `fieldMap` | normalized rows (type inference, renaming, transforms) |
| Infer scales | fields bound to each channel + coordinate | band / linear / time / log‚Ä¶ scales and the projection frame |
| Mark ‚Üí layer | each `<LineMark>` / `<BarMark>` ‚Ä¶ | one core `<Scope>` layer (of `<Node>` / `<Path>`) |
| Axis / grid / legend ‚Üí layer | explicit `<Axis>` / `<Legend>` | grid, axis, and legend layers (each a `<Scope>`) |
| Order z-index | all layers above | grid at the bottom ‚Üí marks ‚Üí axes ‚Üí legend |

Every layer is wrapped in one isolated-namespace `<Scope>`, so each chart's internal ids (datum / series) never collide.

### Composable panels

When the PlotSpec carries an `id` (spec entry or generated IR), lowering wraps an extra **externally visible** panel `<Scope>`: the panel bbox is registered as `<plotId>` and the drawing area as `<plotId>.plotArea`, both placed outside the isolated namespace. Sibling nodes can anchor to them to connect or annotate ‚Äî letting you compose several charts into one figure.

## API

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `data` | `Array<Row>` | ‚Äî | Raw data rows (composition DSL entry); data never enters the IR |
| `children` | `ReactNode` | ‚Äî | Mark / guide children (composition DSL entry) |
| `model` | `DataModel` | ‚Äî | Data model (field names + types): enables strict validation and type-driven scale / guide derivation; see [Data Model](/plot/data/model) |
| `fieldMap` | `Record<string, string>` | ‚Äî | Logical field name ‚Üí actual path in the data (requires `model`); renames only, no value transform; see [Processing](/plot/data/processing) |
| `coordinate` | see [Coordinates](/plot/grammar/coordinate) | cartesian2D | Coordinate system: `"polar2D"` / `"cartesian1D"` / `"polar1D"` / `"ternary2D"` shorthand or the matching object; cartesian2D by default |
| `scaleX` | `'linear' \| 'time' \| 'point' \| 'log' \| 'sqrt'` | `'linear'` | Continuous x scale type; forced to band with `<BarMark>`, ignored under polar. `log` / `sqrt` are point / line only (interval / area baselines include 0 and will error) |
| `scaleY` | `'linear' \| 'log' \| 'sqrt'` | `'linear'` | Continuous y (value-axis) scale type; ignored under polar. `log` / `sqrt` are point / line only (interval / area will error). `log` requires positive data, `sqrt` requires non-negative |
| `bare` | `boolean?` | `false` | Draw only the data, no axes / grid; ignores `<Axis>` |
| `width` | `number` | ‚Äî | Outer width (including axes and labels) |
| `height` | `number` | ‚Äî | Outer height (including axes and labels) |
| `fontSize` | `number?` | ‚Äî | Tick label font size |
| `margin` | `{ top?; right?; bottom?; left? }` | ‚Äî | Override the surrounding padding |
| `spec` | `PlotSpec` | ‚Äî | Spec entry: a prebuilt Plot IR root (mutually exclusive with `children`) |

In addition, `renderer` / `className` / `style` pass through to the underlying `<Layout>` ‚Äî picking the render backend, an extra class name, and inline styles respectively.
