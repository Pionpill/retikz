`<Plot>` is the root container for every chart. It takes `data` (an array of raw rows) and a display size, and assembles its `<LineMark>` / `<BarMark>` / `<SectorMark>` / `<Axis>` children into a chart. `coordinate` picks the coordinate system: cartesian2D by default, plus polar (`"polar2D"`), 1D (`"cartesian1D"` / `"polar1D"`), and ternary (`"ternary2D"`). Data never enters the IR; it's injected at compile time. `<Plot>` is a **thin container**: it draws no axes or grid on its own — axes come from the `<Axis>` you declare (scales and the coordinate system are still inferred automatically).

## Usage

```tsx
import { Axis, BarMark, Plot } from '@retikz/plot-react';
```

```tsx
<Plot data={revenue} width={360} height={220}>
  <BarMark x="quarter" y="value" />
  <Axis dimension="x" />
  <Axis dimension="y" grid />
</Plot>
```

`width × height` is the outer box including axes and tick labels. Children only declare field mappings, never pixel coordinates.

### When to use

`<Plot>` is a **low-level container**: it compiles declarative children into a chart, but does not add batteries-included decoration for you.

- **It does**: collect marks (`<LineMark>` / `<PointMark>` / `<BarMark>` / `<SectorMark>` / `<AreaMark>`); infer scales (band / linear / time / log…) and the coordinate system from fields; collect the `<Axis>` / `<Legend>` you write; compile and render.
- **It does not**: draw axes / grid / legend automatically — write `<Axis>` for axes; `bare` drops even explicit decoration and margins.

## Examples

### Switching coordinate systems

`coordinate` is a chart-level switch — for the same `children`, the default is cartesian and adding `"polar2D"` makes it polar. Marks aren't split into two variants; the coordinate system decides the geometry.

`coordinate` accepts a string shorthand or an object, covering the coordinate family:

```tsx
<Plot coordinate="polar2D">                                    {/* polar (pie / radar / radial bars) */}
<Plot coordinate={{ type: 'polar2D', innerRadius: 0.5 }}>      {/* carve the center into a donut */}
<Plot coordinate="cartesian1D">                                {/* 1D line (rug / timeline) */}
<Plot coordinate="polar1D">                                    {/* 1D circle (cyclic data) */}
<Plot coordinate="ternary2D">                                  {/* ternary (barycentric) */}
```

See [Coordinates](/plot/grammar/coordinate) for each system's geometry options (polar's `innerRadius` / `startAngle` / `endAngle`, cartesian1D's `orientation`, polar1D's `radius`, …) and which marks / channels apply.

## Entry forms

`<Plot>` has two entries, dispatched by whether `spec` is passed:

| Entry | Required | Use |
| --- | --- | --- |
| Composition DSL | `data` (raw array) + `children` | Declare marks / guides in JSX (the common case) |
| Spec entry | `spec` (built PlotSpec) + `data` (dataset table) | Feed a persisted / AI-generated spec directly |

## API

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `data` | `Array<Row>` | — | Raw data rows (composition DSL entry); data never enters the IR |
| `children` | `ReactNode` | — | Mark / guide children (composition DSL entry) |
| `model` | `DataModel` | — | Data model (field names + types): enables strict validation and type-driven scale / guide derivation; see [Data Model](/plot/data/model) |
| `fieldMap` | `Record<string, string>` | — | Logical field name → actual path in the data (requires `model`); renames only, no value transform; see [Processing](/plot/data/processing) |
| `coordinate` | see [Coordinates](/plot/grammar/coordinate) | cartesian2D | Coordinate system: `"polar2D"` / `"cartesian1D"` / `"polar1D"` / `"ternary2D"` shorthand or the matching object; cartesian2D by default |
| `scaleX` | `'linear' \| 'time' \| 'point' \| 'log' \| 'sqrt'` | `'linear'` | Continuous x scale type; forced to band with `<BarMark>`, ignored under polar. `log` / `sqrt` are point / line only (interval / area baselines include 0 and will error) |
| `scaleY` | `'linear' \| 'log' \| 'sqrt'` | `'linear'` | Continuous y (value-axis) scale type; ignored under polar. `log` / `sqrt` are point / line only (interval / area will error). `log` requires positive data, `sqrt` requires non-negative |
| `bare` | `boolean?` | `false` | Draw only the data, no axes / grid; ignores `<Axis>` |
| `width` | `number` | — | Outer width (including axes and labels) |
| `height` | `number` | — | Outer height (including axes and labels) |
| `fontSize` | `number?` | — | Tick label font size |
| `margin` | `{ top?; right?; bottom?; left? }` | — | Override the surrounding padding |
| `spec` | `PlotSpec` | — | Spec entry: a prebuilt Plot IR root (mutually exclusive with `children`) |

In addition, `renderer` / `className` / `style` pass through to the underlying `<Layout>` — picking the render backend, an extra class name, and inline styles respectively.
