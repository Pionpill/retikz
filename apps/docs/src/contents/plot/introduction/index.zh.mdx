title: 简介
description: retikz 的图表层——声明式 Plot IR 描述「画什么」，数据单独喂入，由 lowerPlots 下沉成 core 图元渲染。
---

`@retikz/plot` 是建在 core 之上的图表层。你用一份 **Plot IR** 声明「这是一张什么图」——坐标系、比例尺、把哪个字段画成点还是线——数据则单独提供，最后由一个函数把两者合成可渲染的图。

它刻意把一张图拆成三块，各管一件事：

| 部分 | 是什么 | 职责 |
| --- | --- | --- |
| **Plot IR** | 配置对象（JSON 可序列化） | 决定图长什么样：图元、比例尺、坐标系、字段绑定 |
| **数据** | 外部数据集，任意 JS | 提供具体数值；**不进 IR**，避免 IR 随数据量膨胀 |
| **lowerPlots** | 写死的下沉函数 | 把 IR + 数据投影成 core 的点 / 线，交给现有 renderer |

`@retikz/plot-react` 把这三块包成一个 `<Plot>` 组件。下面这张图把 5 个月的营收同时画成折线和散点——同一份数据、同一套坐标，叠两层 mark：

用组合 DSL 写最直观：只声明「画什么」（哪个字段画成线、哪个画成点），比例尺与坐标系由 `<Plot>` 自动推断，数据是一组裸数据行单独传入：

```tsx
import { LineMark, Plot, PointMark } from '@retikz/plot-react';

const sales = [
  { month: 0, revenue: 10 },
  { month: 1, revenue: 14 },
  // ...
];

<Plot data={sales} width={360} height={200}>
  <LineMark x="month" y="revenue" order="month" />
  <PointMark x="month" y="revenue" />
</Plot>;
```

JSX 只是书写面：`<Plot>` 同步读取子图层，装配成一份规范化的 **Plot IR**——你也可以直接手写或由程序 / LLM 生成这份 IR，再用 `<Plot spec>` 入口渲染：

```tsx
const spec = {
  namespace: 'plot',
  type: 'plot',
  data: { ref: 'sales' }, // 只引用名字，不含数据
  scales: [
    { type: 'linear', name: '__x' },
    { type: 'linear', name: '__y' },
  ],
  coordinate: { type: 'cartesian2D', x: '__x', y: '__y' },
  marks: [
    { type: 'line', order: 'month', encoding: { x: { field: 'month' }, y: { field: 'revenue' } } },
    { type: 'point', encoding: { x: { field: 'month' }, y: { field: 'revenue' } } },
  ],
};

// 已有完整 IR 时：用 spec 入口 + 具名数据集
<Plot spec={spec} data={{ sales }} width={360} height={200} />;
```

IR 里只写了 `data: { ref: 'sales' }`（一个名字），真正的数据在渲染时单独传入——同一份 `spec` 换一份字段相符的数据集就能复用，这正是「IR 与数据解耦」的意义：配置描述形态，数据只填内容。

服务端 / 构建期可用 `@retikz/plot-vanilla` 的 `renderPlot` 出 SVG 字符串（零 DOM）：

```ts
import { renderPlot } from '@retikz/plot-vanilla';

const svg = renderPlot(spec, { sales }, { width: 360, height: 200 }); // SSR：SVG 字符串
```
