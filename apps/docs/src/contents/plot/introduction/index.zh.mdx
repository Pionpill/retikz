`@retikz/plot` 是建在 core 之上的图表层。你用一份 **Plot IR** 声明「这是一张什么图」——坐标系、比例尺、把哪个字段画成点还是线——数据则单独提供，最后由一个函数把两者合成可渲染的图。

它刻意把一张图拆成三块，各管一件事：

| 部分 | 是什么 | 职责 |
| --- | --- | --- |
| **Plot IR** | 配置对象（JSON 可序列化） | 决定图长什么样：图元、比例尺、坐标系、字段绑定 |
| **数据** | 外部数据集，任意 JS | 提供具体数值；**不进 IR**，避免 IR 随数据量膨胀 |
| **lowerPlots** | 写死的下沉函数 | 把 IR + 数据投影成 core 的点 / 线，交给现有 renderer |

`@retikz/plot-react` 把这三块包成一个 `<Plot>` 组件。下面这张图把 5 个月的营收同时画成折线和散点，并自动配上坐标轴和网格——同一份数据、同一套坐标，叠两层 mark：

用**组合 DSL** 写最直观：只声明「画什么」（哪个字段画成线、哪个画成点），比例尺与坐标系由 `<Plot>` 自动推断；数据是一组裸数据行，单独抽在 `line-scatter.data.ts` 里传入。完整写法见上图下方的「代码」面板——React 源码、装配出的 **Plot IR**、等价的 **Vanilla** builder 代码、以及数据文件都在那里。

JSX 只是书写面：`<Plot>` 同步读取子图层、装配成一份规范化的 **Plot IR**（即代码面板的 **IR** 一栏）。你也可以反过来——直接手写或由程序 / LLM 生成这份 IR，再用 `<Plot spec={spec} data={{ sales }}>` 入口渲染。无论走哪条路，IR 里关于数据都只写一个引用名（`data: { ref: 'sales' }`）：真正的数据在渲染时单独传入，同一份 spec 换一份字段相符的数据集就能复用——这正是「IR 与数据解耦」的意义，配置描述形态、数据只填内容。

服务端 / 构建期则用 `@retikz/plot-vanilla` 的 `renderPlot(spec, data, options)` 直接出 SVG 字符串（零 DOM），对应代码面板的 **Vanilla** 一栏。

## 坐标轴和网格默认就画

上图的刻度、轴线、网格都是自动出的——你只写了 mark，`<Plot>` 默认就配上 x / y 轴和一层水平网格。要定制就加 `<Axis>` 子组件；一旦写了任意 `<Axis>`，就**完全按你列的来**，不再自动补默认：

```tsx
<Plot data={sales} width={360} height={200}>
  <LineMark x="month" y="revenue" order="month" />
  <Axis dimension="x" tickCount={5} />
  <Axis dimension="y" grid />
</Plot>
```

`<Axis>` 的开关：`tickCount`（目标刻度数）、`grid`（这根轴是否画对齐刻度的网格线）、`tickLabels={false}`（只留轴线和刻度、不写数字）。网格是**轴的属性**、不是独立组件——x 轴的网格是竖线、y 轴的是横线，两者叠加才成「网格」。

只想要纯绘图、连轴都不要时，给 `<Plot>` 加 `bare`：

```tsx
<Plot data={sales} width={360} height={200} bare>
  <LineMark x="month" y="revenue" order="month" />
</Plot>
```

`width × height` 是含坐标轴和刻度标签在内的**外框**：轴和标签占掉边缘后，剩下的矩形才是真正画数据的绘图区。标签字号、四周留白可分别用 `fontSize`、`margin` 调。

## 数据来自哪里都一样

上面的数据是写死的，真实场景里数据通常来自别处——后端接口、公开 API、CSV。因为数据**不进 IR**、只在渲染时作为裸数据行传入，所以从外部 `fetch` 一份大数据集再作图，写法和本地数据完全一致：先把数据取回来，拿到后照样丢给 `<Plot data>`。

取数逻辑（`fetch` + 状态）和造的数据一样**抽到同级 `.data` 文件**、主 demo 只消费保持简单——React 端封装成一个 hook（`line-scatter-fetch.data.ts` 的 `useCarPerformance()`，见代码面板里的数据文件）。下面这张图实时从 CDN 拉取 [vega-datasets](https://github.com/vega/vega-datasets) 的经典「汽车性能」数据集（约 400 条），把马力与油耗画成散点；作图那几行和本地数据完全一样，数据来自哪里对 `<Plot>` 透明：
