[数据模型](/plot/data/model)声明了字段**是什么**；这页讲怎么把真实世界的数据**接进来**——换源、解析、校验。三件事都以声明了 model 为前提。

## 换数据源

数据不进配置（IR），所以同一份 `<Plot>` 配置能换着数据源用。前提是声明了 model——model 就是「可移植」的开关。换数据源有三种对接方式：

### 同名同类型

字段名 / 类型一致时，直接换 `data` 就能重新出图，配置一行不用动。

### 字段名不同：`fieldMap`

数据里字段名和配置里的逻辑名对不上时，给一份 `fieldMap`（逻辑名 → 数据里的实际路径），配置不动：

```tsx
// 数据里叫 period / amount，配置里仍用 month / revenue
<Plot data={rows} model={[{ name: 'month', type: 'temporal' }, { name: 'revenue', type: 'continuous' }]}
      fieldMap={{ month: 'period', revenue: 'amount' }}>
  <LineMark x="month" y="revenue" order="month" />
</Plot>
```

`fieldMap` 只做改名（值不变换，计算交给 transform）；它必须配合 model 使用——没有 model 就没有「逻辑字段」可映射。

### 存储格式不同但类型相同

同一个 `temporal` 字段，数据里是 `Date` 对象、ISO 字符串还是毫秒时间戳，都能正确读；`continuous` 字段是数字还是规范的数字串（`"120"`）也都认。无需任何额外配置。非默认的存储格式（如 `'2024/01/01'`、epoch 秒、`'50%'`）走下面的解析。

## 解析非默认格式

内置解析只认固定形态：`temporal` 收 `Date` / 毫秒 / 严格 ISO，`continuous` 收数字 / 严格数字串。词表外的格式需要你点名怎么解析——常见格式用声明式 `format`，真正任意的逻辑用 `resolveField` 兜底。

### 声明式格式：`format`

碰上 `'2024/01/01'`、epoch 秒、`'50%'` 这类常见的非默认格式，不必动用函数——给字段一个声明式 `format`，它**进 IR、可序列化、LLM 也能生成**：

```tsx
<Plot data={rows} model={[
  { name: 'createdAt', type: 'temporal',  format: 'slashDate' },    // '2024/01/01' → 时间
  { name: 'ts',        type: 'temporal',  format: 'epochSeconds' }, // 1700000000 → 毫秒
  { name: 'ratio',     type: 'continuous', format: 'percent' },     // '50%' → 0.5
]}>{/* … */}</Plot>
```

可选的 `format`（每个绑定一种类型）：

| `format` | 适用类型 | 解析 |
| --- | --- | --- |
| `iso` | temporal | 严格 ISO（等价不写） |
| `epochSeconds` | temporal | epoch 秒 → 毫秒 |
| `epochMillis` | temporal | epoch 毫秒 |
| `slashDate` | temporal | 严格 `YYYY/MM/DD`（按 UTC 零点） |
| `numberString` | continuous | 宽松数字串（容空白 / 千分位逗号） |
| `percent` | continuous | 百分比串 `'50%'` → 0.5 |

规则：

- **format 蕴含 type**：写了 `format` 就可省略 `type`——`{ name: 'ratio', format: 'percent' }` 自动当 continuous，不会被推断成分类。
- **必须与 type 兼容**：format 绑定的类型与你写的 `type` 冲突（如 `{ type: 'continuous', format: 'slashDate' }`）会**报清晰错误**，不静默。
- `slashDate` 只认严格 `YYYY/MM/DD`；`DD/MM/YYYY`、`MM/DD/YYYY` 这类有地区歧义的布局不收（避免同一串在不同环境解析成不同日期）。
- 解析优先级：`resolveField` 的 `parse` > 声明式 `format` > 内置默认。

### 自定义解析：`resolveField`

碰上词表外的**任意格式**（如 `'2024Q1'` 季度串、自定义分隔符）或想**强制某字段当某类型**（数值枚举当 `categorical`）时，给 `resolveField` 一个函数——它**不进 IR**（是运行时选项，和 `fieldMap` 同层），按字段名返回「类型覆盖 + 可选自定义解析器」：

```tsx
<Plot data={rows} model={[{ name: 'quarter', type: 'temporal' }]}
  resolveField={(field) => {
    // 词表外的格式：类型 + 解析一起给（'2024Q1' → 季度首日）
    if (field === 'quarter') return { type: 'temporal', parse: raw => {
      const [year, q] = String(raw).split('Q');
      return Date.UTC(Number(year), (Number(q) - 1) * 3, 1);
    } };
    if (field === 'statusCode') return { type: 'categorical' }; // 数值枚举当类别（值原样）
    return undefined;                                            // 其余走 model/推断 + 内置解析
  }}>
  {/* … */}
</Plot>
```

规则：

- 返回 `{ type }` 覆盖该字段类型（优先级高于 model 与推断）；返回 `{ parse }` 用自定义解析器把原始值转成规范值（`temporal`/`continuous` 返数字、`categorical` 返字符串/数字），返 `undefined` 跳过该值。
- **只给 `parse` 不给 `type`** 时，类型沿用 model 声明——所以 `parse` 单独出现**必须有 model**（无 model 时类型来源不清，会报错）。
- `resolveField` **不绕过 strict**：声明了 model，引用未列出的字段照样报错——它只覆盖类型/解析，不能把拼错字段洗成合法字段。
- 因为是函数、不可序列化，`resolveField` 适合手写代码的逃生舱；要 spec 完全可移植 / LLM 友好，优先用 model 的声明式 `type` / `format`。

## 校验脏数据

文档型 / 异构数据常有缺字段、类型不一的行。retikz 的处理是**跳过、不伪造、不静默画错**：

| 情形 | 行为 |
| --- | --- |
| 某行缺这个字段 | 跳过该数据点（异构数据常态，不报错） |
| 叶子是对象 / 数组（非标量） | 跳过该数据点（绝不把对象塞进图） |
| 同类型不同存储格式 | 自动转换（见上「换数据源」） |
| 字段是 `bigint`（如数据库 int64） | 自动转 number（仅安全整数；超出范围按非法处理） |
| 路径中途是数组（需展开） | 暂不支持自动展开，请先在数据层 flatten |

默认「跳过」可能让你对着一张空图摸不着头脑。打开 `validateData` 让 retikz **抽样**校验、有问题直接报错，并指出**哪个字段、多少非法 / 缺失**（如 `field "x": 3/100 invalid`），把「为什么图是空的」变成明确提示：

```tsx
<Plot data={rows} model={[...]} validateData>
  {/* 字段缺失 / 全部无法转换 → 编译期报错，而非空图 */}
</Plot>
```

要更严的**全量**校验，给 `invalid="error"`——图里实际用到的字段里任一非法值（含缺字段、无法转换、超范围 bigint）都立刻报错并指明字段，而不是抽样：

```tsx
<Plot data={rows} model={[...]} invalid="error">
  {/* 任一参与字段有非法值 → 报错并指明字段 */}
</Plot>
```

`invalid` 默认是 `'skip'`（上表的「跳过」行为）：非法点跳过、但**整行仍保留**，照常参与 stack / 排序等数据变换；`'error'` 只检查图里实际引用的字段，不波及其它脏列。
